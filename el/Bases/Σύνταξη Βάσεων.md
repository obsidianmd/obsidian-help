Όταν [[Δημιουργία βάσης|δημιουργείτε μια βάση]] στο Obsidian, αποθηκεύεται ως αρχείο `.base`. Οι βάσεις συνήθως επεξεργάζονται μέσω της διεπαφής της εφαρμογής, αλλά η σύνταξη μπορεί επίσης να επεξεργαστεί χειροκίνητα και να ενσωματωθεί σε ένα μπλοκ κώδικα.

Η σύνταξη [[Εισαγωγή στις Βάσεις|Βάσεων]] ορίζει [[Προβολές]], φίλτρα και [[Τύποι|τύπους]]. Οι βάσεις πρέπει να είναι έγκυρο YAML σύμφωνα με το σχήμα που ορίζεται παρακάτω.

## Παράδειγμα

Ακολουθεί ένα παράδειγμα αρχείου βάσης. Θα αναλύσουμε κάθε ενότητα λεπτομερώς.

```yaml
filters:
  or:
    - file.hasTag("tag")
    - and:
        - file.hasTag("book")
        - file.hasLink("Textbook")
    - not:
        - file.hasTag("book")
        - file.inFolder("Required Reading")
formulas:
  formatted_price: 'if(price, price.toFixed(2) + " dollars")'
  ppu: "(price / age).toFixed(2)"
properties:
  status:
    displayName: Status
  formula.formatted_price:
    displayName: "Price"
  file.ext:
    displayName: Extension
summaries:
  customAverage: 'values.mean().round(3)'
views:
  - type: table
    name: "My table"
    limit: 10
    groupBy:
      property: note.age
      direction: DESC
    filters:
      and:
        - 'status != "done"'
        - or:
            - "formula.ppu > 5"
            - "price > 2.1"
    order:
      - file.name
      - file.ext
      - note.age
      - formula.ppu
      - formula.formatted_price
    summaries:
      formula.ppu: Average
```

### Φίλτρα

Από προεπιλογή, μια βάση περιλαμβάνει κάθε αρχείο στο θησαυροφυλάκιο. Δεν υπάρχει `from` ή `source` όπως στην SQL ή το Dataview. Η ενότητα `filters` σας επιτρέπει να ορίσετε συνθήκες για τον περιορισμό του συνόλου δεδομένων.

```yaml
# Απλό φίλτρο:
filters:
  and:
    - file.hasTag("tag")

# Σύνθετο φίλτρο:
filters:
  or:
    - file.hasTag("tag")
    - and:
        - file.hasTag("book")
        - file.hasLink("Textbook")
    - not:
        - file.hasTag("book")
        - file.inFolder("Required Reading")
```

Υπάρχουν δύο σημεία εφαρμογής φίλτρων:

1. Στο καθολικό επίπεδο `filters` (που φαίνεται παραπάνω) όπου εφαρμόζονται σε όλες τις προβολές της βάσης.
2. Στο επίπεδο `view` όπου εφαρμόζονται μόνο σε μια συγκεκριμένη προβολή.

Αυτές οι δύο ενότητες είναι λειτουργικά ισοδύναμες και κατά την αξιολόγηση για μια προβολή συνενώνονται με `AND`.

Η ενότητα `filters` περιέχει είτε μια μεμονωμένη δήλωση φίλτρου ως συμβολοσειρά, είτε ένα αναδρομικά ορισμένο αντικείμενο φίλτρου. Τα αντικείμενα φίλτρου μπορούν να περιέχουν ένα από τα `and`, `or` ή `not`. Αυτά τα κλειδιά είναι μια ετερογενής λίστα από άλλα αντικείμενα φίλτρου ή δηλώσεις φίλτρου σε συμβολοσειρές. Μια δήλωση φίλτρου είναι μια γραμμή που αξιολογείται ως αληθής ή ψευδής όταν εφαρμόζεται σε μια σημείωση. Μπορεί να είναι ένα από τα ακόλουθα:

- Μια βασική σύγκριση χρησιμοποιώντας τυπικούς αριθμητικούς τελεστές.
- Μια συνάρτηση. Μια ποικιλία [[Συναρτήσεις|συναρτήσεων]] είναι ενσωματωμένες, και τα πρόσθετα μπορούν να προσθέσουν επιπλέον συναρτήσεις.

Η σύνταξη και οι διαθέσιμες συναρτήσεις για φίλτρα και τύπους είναι ίδιες.

### Τύποι

Η ενότητα `formulas` ορίζει [[Τύποι|ιδιότητες τύπων]] που μπορούν να εμφανίζονται σε όλες τις προβολές του αρχείου βάσης.

```yaml
formulas:
  formatted_price: 'if(price, price.toFixed(2) + " dollars")'
  ppu: "(price / age).toFixed(2)"
```

Οι ιδιότητες τύπων υποστηρίζουν βασικούς αριθμητικούς τελεστές και μια ποικιλία ενσωματωμένων [[Συναρτήσεις|συναρτήσεων]]. Στο μέλλον, τα πρόσθετα θα μπορούν να προσθέτουν συναρτήσεις για χρήση σε τύπους.

Μπορείτε να αναφέρεστε σε ιδιότητες με διαφορετικούς τρόπους ανάλογα με τον τύπο τους:

- Οι **ιδιότητες σημείωσης** είναι ιδιότητες που ορίζονται στα μεταδεδομένα αρχής της σημείωσης. Για παράδειγμα `note.price` ή `note["price"]`.  
  Αν δεν καθοριστεί πρόθεμα, η ιδιότητα θεωρείται ότι είναι ιδιότητα `note`.
- Οι **ιδιότητες αρχείου** περιγράφουν το ίδιο το αρχείο.  
  Για παράδειγμα `file.size` ή `file.ext`. Μπορείτε επίσης να αναφέρεστε απευθείας στο αντικείμενο αρχείου, π.χ. `file.hasLink()`.
- Οι **ιδιότητες τύπων** είναι άλλοι τύποι στη βάση.  
  Παράδειγμα `formula.formatted_price`.

Ένας τύπος μπορεί να χρησιμοποιεί τιμές από άλλες ιδιότητες τύπων, αρκεί να μην υπάρχει κυκλική αναφορά.  

Οι ιδιότητες τύπων αποθηκεύονται πάντα ως συμβολοσειρές σε YAML, αλλά ο πραγματικός **τύπος δεδομένων εξόδου** τους καθορίζεται από τον τύπο των υποκείμενων δεδομένων και την τιμή επιστροφής οποιωνδήποτε συναρτήσεων χρησιμοποιούνται.

Σημειώστε ότι η χρήση εμφωλευμένων εισαγωγικών είναι απαραίτητη για να συμπεριληφθούν κυριολεκτικά κειμένου στο πεδίο YAML. Τα κυριολεκτικά κειμένου πρέπει να περικλείονται σε μονά ή διπλά εισαγωγικά.

### Ιδιότητες

Η ενότητα `properties` επιτρέπει την αποθήκευση πληροφοριών ρύθμισης για κάθε ιδιότητα. Εξαρτάται από την κάθε προβολή πώς θα χρησιμοποιήσει αυτές τις τιμές ρύθμισης. Για παράδειγμα, στους πίνακες το εμφανιζόμενο όνομα χρησιμοποιείται για τις επικεφαλίδες στηλών.

```yaml
properties:
  status:
    displayName: Status
  formula.formatted_price:
    displayName: "Price"
  file.ext:
    displayName: Extension
```

Τα εμφανιζόμενα ονόματα δεν χρησιμοποιούνται σε φίλτρα ή τύπους.

### Συνόψεις

Η ενότητα `summaries` μπορεί να χρησιμοποιηθεί για τον ορισμό προσαρμοσμένων τύπων σύνοψης. Εκτός από τον ορισμό τύπων σύνοψης εδώ, υπάρχουν αρκετοί προεπιλεγμένοι τύποι σύνοψης διαθέσιμοι.

```yaml
summaries:
  customAverage: 'values.mean().round(3)'
```

Σε αυτό το παράδειγμα, ο τύπος `customAverage` είναι ίδιος με τον προεπιλεγμένο `Average`, εκτός από το ότι η τιμή στρογγυλοποιείται σε διαφορετικό αριθμό δεκαδικών ψηφίων. Στους τύπους σύνοψης, η λέξη-κλειδί `values` είναι μια λίστα που περιέχει όλες τις τιμές για αυτήν την ιδιότητα σε κάθε σημείωση στο σύνολο αποτελεσμάτων. Ο τύπος σύνοψης πρέπει να επιστρέφει μια μεμονωμένη `Value`.

Σημειώστε ότι αυτή η ενότητα `summaries` είναι διαφορετική από την ενότητα `summaries` στη ρύθμιση προβολής (εξηγείται παρακάτω) όπου οι τύποι σύνοψης αντιστοιχίζονται σε συγκεκριμένες ιδιότητες.

#### Προεπιλεγμένοι τύποι σύνοψης

| Όνομα     | Τύπος εισόδου | Περιγραφή                                                     |
| --------- | -------------- | ------------------------------------------------------------- |
| Average   | Αριθμός        | Ο μαθηματικός μέσος όρος όλων των αριθμών από τις τιμές εισόδου. |
| Min       | Αριθμός        | Ο μικρότερος αριθμός από τις τιμές εισόδου.                   |
| Max       | Αριθμός        | Ο μεγαλύτερος αριθμός από τις τιμές εισόδου.                  |
| Sum       | Αριθμός        | Το άθροισμα όλων των αριθμών στην είσοδο.                     |
| Range     | Αριθμός        | Η διαφορά μεταξύ `Max` και `Min`.                             |
| Median    | Αριθμός        | Η μαθηματική διάμεσος όλων των αριθμών από τις τιμές εισόδου. |
| Stddev    | Αριθμός        | Η τυπική απόκλιση όλων των αριθμών από τις τιμές εισόδου.     |
| Earliest  | Ημερομηνία     | Η παλαιότερη ημερομηνία από τις τιμές εισόδου.                |
| Latest    | Ημερομηνία     | Η πιο πρόσφατη ημερομηνία από τις τιμές εισόδου.              |
| Range     | Ημερομηνία     | Η διαφορά μεταξύ `Latest` και `Earliest`.                     |
| Checked   | Boolean        | Ο αριθμός των τιμών `true`.                                   |
| Unchecked | Boolean        | Ο αριθμός των τιμών `false`.                                  |
| Empty     | Οποιοσδήποτε   | Ο αριθμός τιμών στην είσοδο που είναι κενές.                  |
| Filled    | Οποιοσδήποτε   | Ο αριθμός τιμών στην είσοδο που δεν είναι κενές.              |
| Unique    | Οποιοσδήποτε   | Ο αριθμός μοναδικών τιμών στην είσοδο.                        |

### Προβολές

Η ενότητα `views` ορίζει πώς μπορούν να αποδοθούν τα δεδομένα. Κάθε καταχώρηση στη λίστα `views` ορίζει μια ξεχωριστή προβολή των ίδιων δεδομένων, και μπορούν να υπάρχουν όσες διαφορετικές προβολές χρειάζονται.

```yaml
views:
  - type: table
    name: "My table"
    limit: 10
    groupBy:
      property: note.age
      direction: DESC
    filters:
      and:
        - 'status != "done"'
        - or:
            - "formula.ppu > 5"
            - "price > 2.1"
    order:
      - file.name
      - file.ext
      - note.age
      - formula.ppu
      - formula.formatted_price
    summaries:
      formula.ppu: Average
```

- Το `type` επιλέγει από τους ενσωματωμένους και τους τύπους προβολής που προστίθενται από πρόσθετα.
- Το `name` είναι το εμφανιζόμενο όνομα και μπορεί να χρησιμοποιηθεί για τον ορισμό της προεπιλεγμένης προβολής.
- Τα `filters` είναι ακριβώς τα ίδια όπως περιγράφηκαν παραπάνω, αλλά εφαρμόζονται μόνο στην προβολή.
- Το `groupBy` καθορίζει μια ιδιότητα και κατεύθυνση ταξινόμησης. Η τιμή της καθορισμένης ιδιότητας για κάθε γραμμή χρησιμοποιείται για την τοποθέτηση της γραμμής σε ομάδες.
- Τα `summaries` αντιστοιχίζουν ονόματα ιδιοτήτων σε μια ονομασμένη σύνοψη. Οι συνόψεις εκτελούν μια συνάθροιση στην ιδιότητα σε όλες τις γραμμές.

Οι [[Προβολές]] μπορούν να προσθέσουν επιπλέον δεδομένα για την αποθήκευση οποιασδήποτε πληροφορίας χρειάζεται για τη διατήρηση κατάστασης ή τη σωστή απόδοση, ωστόσο οι δημιουργοί πρόσθετων θα πρέπει να προσέχουν να μην χρησιμοποιούν κλειδιά που χρησιμοποιούνται ήδη από το βασικό πρόσθετο Βάσεων. Ως παράδειγμα, μια προβολή πίνακα μπορεί να το χρησιμοποιήσει αυτό για τον περιορισμό του αριθμού γραμμών ή για την επιλογή ποια στήλη χρησιμοποιείται για ταξινόμηση γραμμών και σε ποια κατεύθυνση. Ένας διαφορετικός τύπος προβολής όπως ο χάρτης θα μπορούσε να χρησιμοποιήσει αυτό για την αντιστοίχιση ποια ιδιότητα στη σημείωση αντιστοιχεί στο γεωγραφικό πλάτος και μήκος και ποια ιδιότητα θα πρέπει να εμφανίζεται ως τίτλος της καρφίτσας.

Στο μέλλον, το API θα επιτρέπει στις προβολές να διαβάζουν και να γράφουν αυτές τις τιμές, επιτρέποντας στην προβολή να δημιουργήσει τη δική της διεπαφή για ρύθμιση.

## Ιδιότητες

Υπάρχουν τρία είδη ιδιοτήτων που χρησιμοποιούνται στις βάσεις:

1. **Ιδιότητες σημείωσης**, αποθηκευμένες στα μεταδεδομένα αρχής αρχείων Markdown.
2. **Ιδιότητες αρχείου**, προσβάσιμες για όλους τους τύπους αρχείων.
3. **Ιδιότητες τύπων**, ορισμένες στο ίδιο το αρχείο `.base` (βλ. παραπάνω).

### Ιδιότητες σημείωσης

Οι [[Ιδιότητες|ιδιότητες σημείωσης]] είναι διαθέσιμες μόνο για αρχεία Markdown και αποθηκεύονται στα μεταδεδομένα αρχής YAML κάθε σημείωσης. Αυτές οι ιδιότητες μπορούν να προσπελαστούν χρησιμοποιώντας τη μορφή `note.author` ή απλά `author` ως συντομογραφία.

### Ιδιότητες αρχείου

Οι ιδιότητες αρχείου αναφέρονται στο αρχείο που ελέγχεται ή αξιολογείται αυτή τη στιγμή. Οι ιδιότητες αρχείου είναι διαθέσιμες για όλους τους [[Αποδεκτές μορφές αρχείων|τύπους αρχείων]], συμπεριλαμβανομένων των συνημμένων.

Για παράδειγμα, ένα φίλτρο `file.ext == "md"` θα είναι αληθές για όλα τα αρχεία Markdown και ψευδές σε κάθε άλλη περίπτωση.

| Ιδιότητα       | Τύπος  | Περιγραφή                                                     |
| --------------- | ------ | ------------------------------------------------------------- |
| `file.backlinks`  | Λίστα  | Λίστα αρχείων οπισθοσυνδέσμων. Σημείωση: Αυτή η ιδιότητα είναι βαριά σε απόδοση. Όταν είναι δυνατόν, αντιστρέψτε την αναζήτηση και χρησιμοποιήστε `file.links`. Δεν ανανεώνει αυτόματα τα αποτελέσματα όταν αλλάζει το θησαυροφυλάκιο. |
| `file.ctime`  | Ημερομηνία | Χρόνος δημιουργίας                                           |
| `file.embeds` | Λίστα  | Λίστα όλων των ενσωματώσεων στη σημείωση                      |
| `file.ext`    | Συμβολοσειρά | Επέκταση αρχείου                                           |
| `file.file`   | Αρχείο | Αντικείμενο αρχείου, χρησιμοποιήσιμο μόνο σε συγκεκριμένες συναρτήσεις |
| `file.folder` | Συμβολοσειρά | Διαδρομή του φακέλου του αρχείου                           |
| `file.links`  | Λίστα  | Λίστα όλων των εσωτερικών συνδέσμων στη σημείωση, συμπεριλαμβανομένων των μεταδεδομένων αρχής |
| `file.mtime`  | Ημερομηνία | Χρόνος τροποποίησης                                         |
| `file.name`   | Συμβολοσειρά | Όνομα αρχείου                                              |
| `file.path`   | Συμβολοσειρά | Διαδρομή του αρχείου                                       |
| `file.properties`   | Αντικείμενο | Όλες οι ιδιότητες στο αρχείο. Σημείωση: Δεν ανανεώνει αυτόματα τα αποτελέσματα όταν αλλάζει το θησαυροφυλάκιο. |
| `file.size`   | Αριθμός | Μέγεθος αρχείου                                              |
| `file.tags`   | Λίστα  | Λίστα όλων των ετικετών στο περιεχόμενο αρχείου και τα μεταδεδομένα αρχής |

### Πρόσβαση σε ιδιότητες με `this`

Χρησιμοποιήστε το αντικείμενο `this` για πρόσβαση στις ιδιότητες αρχείου. Σε τι αναφέρεται το `this` εξαρτάται από το πού εμφανίζεται η βάση.

Όταν η βάση ανοίγεται στην κύρια περιοχή περιεχομένου, το `this` δείχνει στις ιδιότητες του ίδιου του αρχείου βάσης. Για παράδειγμα, η χρήση `this.file.folder` επιστρέφει τη διαδρομή φακέλου όπου βρίσκεται η βάση.

Όταν η βάση είναι ενσωματωμένη σε άλλο αρχείο, το `this` δείχνει στις ιδιότητες του αρχείου _ενσωμάτωσης_ (τη σημείωση ή τον Καμβά που περιέχει τη βάση). Για παράδειγμα, η χρήση `this.file.name` επιστρέφει το όνομα του αρχείου ενσωμάτωσης, όχι της βάσης.

Όταν η βάση βρίσκεται σε μια πλαϊνή μπάρα, το `this` αναφέρεται στο ενεργό αρχείο στην κύρια περιοχή περιεχομένου. Αυτό σας επιτρέπει να δημιουργείτε ερωτήματα βασισμένα στο ενεργό αρχείο. Για παράδειγμα, μπορείτε να χρησιμοποιήσετε `file.hasLink(this.file)` για να αναπαράγετε το πάνελ οπισθοσυνδέσμων.

## Τελεστές

### Αριθμητικοί τελεστές

Οι αριθμητικοί τελεστές εκτελούν αριθμητικές πράξεις σε αριθμούς. Για παράδειγμα, `radius * (2 * 3.14)`.

| Τελεστής | Περιγραφή     |
| -------- | ------------- |
| `+`      | πρόσθεση      |
| `-`      | αφαίρεση      |
| `*`      | πολλαπλασιασμός |
| `/`      | διαίρεση      |
| `%`      | υπόλοιπο      |
| `( )`    | παρένθεση     |

### Αριθμητική ημερομηνιών

Οι ημερομηνίες μπορούν να τροποποιηθούν με πρόσθεση και αφαίρεση χρονικών διαρκειών. Οι μονάδες διάρκειας δέχονται πολλαπλές μορφές:

| Μονάδα                   | Διάρκεια  |
| ------------------------ | --------- |
| `y`, `year`, `years`     | έτος      |
| `M`, `month`, `months`   | μήνας     |
| `d`, `day`, `days`       | ημέρα     |
| `w`, `week`, `weeks`     | εβδομάδα  |
| `h`, `hour`, `hours`     | ώρα       |
| `m`, `minute`, `minutes` | λεπτό     |
| `s`, `second`, `seconds` | δευτερόλεπτο |

Για να τροποποιήσετε ή να μετατοπίσετε αντικείμενα Ημερομηνίας, χρησιμοποιήστε τον τελεστή `+` ή `-` με μια συμβολοσειρά διάρκειας. Για παράδειγμα, `date + "1M"` προσθέτει 1 μήνα στην ημερομηνία, ενώ `date - "2h"` αφαιρεί 2 ώρες από την ημερομηνία.

Η καθολική [[Συναρτήσεις|συνάρτηση]] `today()` μπορεί να χρησιμοποιηθεί για τη λήψη της τρέχουσας ημερομηνίας, και η `now()` μπορεί να χρησιμοποιηθεί για τη λήψη της τρέχουσας ημερομηνίας με ώρα.

- `now() + "1 day"` επιστρέφει ημερομηνία-ώρα ακριβώς 24 ώρες από τη στιγμή εκτέλεσης.
- `file.mtime > now() - "1 week"` επιστρέφει `true` αν το αρχείο τροποποιήθηκε μέσα στην τελευταία εβδομάδα.
- `date("2024-12-01") + "1M" + "4h" + "3m"` επιστρέφει ένα αντικείμενο Ημερομηνίας που αντιπροσωπεύει `2025-01-01 04:03:00`.
- Αφαιρέστε δύο ημερομηνίες για να λάβετε τη διαφορά σε χιλιοστά του δευτερολέπτου μεταξύ τους, για παράδειγμα, `now() - file.ctime`.
- Για να λάβετε το τμήμα ημερομηνίας μιας Ημερομηνίας με ώρα, χρησιμοποιήστε `datetime.date()`.
- Για να μορφοποιήσετε ένα αντικείμενο Ημερομηνίας, χρησιμοποιήστε τη συνάρτηση `format()`, για παράδειγμα `datetime.format("YYYY-MM-DD")`.

### Τελεστές σύγκρισης

Οι τελεστές σύγκρισης μπορούν να χρησιμοποιηθούν για σύγκριση αριθμών ή αντικειμένων Ημερομηνίας. Η ισότητα και η ανισότητα μπορούν να χρησιμοποιηθούν με οποιοδήποτε είδος τιμής, όχι μόνο αριθμούς και ημερομηνίες.

| Τελεστής | Περιγραφή                |
| -------- | ------------------------ |
| `==`     | ίσο                     |
| `!=`     | δεν είναι ίσο           |
| `>`      | μεγαλύτερο από          |
| `<`      | μικρότερο από           |
| `>=`     | μεγαλύτερο ή ίσο με     |
| `<=`     | μικρότερο ή ίσο με      |

### Λογικοί τελεστές

Οι λογικοί τελεστές μπορούν να χρησιμοποιηθούν για τη συνδυασμό ή αντιστροφή λογικών τιμών, με αποτέλεσμα μια τιμή αληθή ή ψευδή.

| Τελεστής | Περιγραφή   |
| -------- | ----------- |
| `!`      | λογικό NOT  |
| `&&`     | λογικό AND  |
| \|\|     | λογικό OR   |

## Συναρτήσεις

Δείτε τη [[Συναρτήσεις|λίστα συναρτήσεων]] που μπορούν να χρησιμοποιηθούν σε τύπους και [[Προβολές|φίλτρα]].

## Τύποι δεδομένων

Οι βάσεις διαθέτουν ένα σύστημα τύπων που χρησιμοποιείται από τους τύπους και τα φίλτρα για την εφαρμογή συναρτήσεων σε ιδιότητες.

### Συμβολοσειρές, αριθμοί και booleans

Οι συμβολοσειρές, οι αριθμοί και τα booleans είναι «πρωτόγονες» τιμές που δεν απαιτούν συνάρτηση για τη δημιουργία τους.

- Οι συμβολοσειρές περικλείονται σε μονά ή διπλά εισαγωγικά, για παράδειγμα `"message"`.
- Οι αριθμοί γράφονται ως ψηφία και μπορούν προαιρετικά να περικλείονται σε παρένθεση για σαφήνεια. Για παράδειγμα, `1` ή `(2.5)`.
- Τα booleans γράφονται ως `true` ή `false` χωρίς εισαγωγικά.

### Ημερομηνίες και χρονικές διάρκειες

Οι ημερομηνίες αντιπροσωπεύουν μια συγκεκριμένη ημερομηνία, ή μια ημερομηνία και ώρα ανάλογα με τη συνάρτηση που χρησιμοποιήθηκε για τη δημιουργία τους, ή τον τύπο που έχει αντιστοιχιστεί στην [[Ιδιότητες|ιδιότητα]].

- Για να δημιουργήσετε μια ημερομηνία, χρησιμοποιήστε τη συνάρτηση `date`, για παράδειγμα `date("2025-01-01 12:00:00")`
- Για να τροποποιήσετε μια ημερομηνία, προσθέστε ή αφαιρέστε μια χρονική διάρκεια, για παράδειγμα `now() + "1 hour"` ή `today() + "7d"`
- Συγκρίνετε ημερομηνίες χρησιμοποιώντας τελεστές σύγκρισης (π.χ. `>` ή `<`) και αριθμητικούς τελεστές (για παράδειγμα, `(now() + "1d") - now()` επιστρέφει `86400000` χιλιοστά του δευτερολέπτου.)
- Για να εξάγετε τμήματα μιας ημερομηνίας, χρησιμοποιήστε τα διαθέσιμα πεδία (`now().hour`), ή μια βοηθητική συνάρτηση (`now.time()`).
- Πολλά άλλα [[Συναρτήσεις|πεδία και συναρτήσεις]] είναι διαθέσιμα σε αντικείμενα ημερομηνίας.

### Αντικείμενα και λίστες

- Μετατρέψτε ένα μεμονωμένο στοιχείο σε λίστα χρησιμοποιώντας τη συνάρτηση `list()`. Αυτό είναι ιδιαίτερα χρήσιμο για ιδιότητες που μπορεί να περιέχουν μείγμα λιστών ή μεμονωμένων τιμών.
- Προσπελάστε στοιχεία λίστας χρησιμοποιώντας αγκύλες και δείκτη βάσει 0. Για παράδειγμα, `property[0]` επιστρέφει το πρώτο στοιχείο από τη λίστα.
- Προσπελάστε στοιχεία αντικειμένων χρησιμοποιώντας αγκύλες και το όνομα του στοιχείου ή σημειογραφία τελείας. Για παράδειγμα, `property.subprop` ή `property["subprop"]`.

### Αρχεία και σύνδεσμοι

Οι [[Σύνδεση σημειώσεων|σύνδεσμοι wiki]] στις [[Ιδιότητες|ιδιότητες μεταδεδομένων αρχής]] αναγνωρίζονται αυτόματα ως αντικείμενα Συνδέσμου. Οι σύνδεσμοι θα αποδίδονται ως κλικάρισμα σύνδεσμος στην [[Προβολές|προβολή]].

- Για να δημιουργήσετε έναν σύνδεσμο, χρησιμοποιήστε την καθολική [[Συναρτήσεις|συνάρτηση]] `link`, για παράδειγμα `link("filename")` ή `link("https://obsidian.md")`.
- Μπορείτε να δημιουργήσετε έναν σύνδεσμο από οποιαδήποτε συμβολοσειρά, για παράδειγμα, `link(file.ctime.date().toString())`.
- Για να ορίσετε το κείμενο εμφάνισης, περάστε μια προαιρετική συμβολοσειρά ή εικονίδιο ως δεύτερη παράμετρο, για παράδειγμα `link("filename", "display")` ή `link("filename", icon("plus"))`.

Ένα αντικείμενο Αρχείου μπορεί να μετατραπεί σε σύνδεσμο χρησιμοποιώντας `file.asLink()` με προαιρετικό κείμενο εμφάνισης.

Οι σύνδεσμοι μπορούν να συγκριθούν με `==` και `!=`. Είναι ισοδύναμοι εφόσον δείχνουν στο ίδιο αρχείο, ή αν το αρχείο δεν υπάρχει κατά την αναζήτηση, το κείμενο συνδέσμου τους πρέπει να είναι πανομοιότυπο.

Οι σύνδεσμοι μπορούν να συγκριθούν με αρχεία όπως `file` ή `this`. Θα ισούνται αν ο σύνδεσμος επιλύεται στο αρχείο. Για παράδειγμα, `author == this`.

Οι σύνδεσμοι μπορούν επίσης να ελεγχθούν σε λίστα περιεχομένων, για παράδειγμα, `authors.contains(this)`.
