Το Obsidian υποστηρίζει HTML για να σας επιτρέπει να εμφανίζετε τις σημειώσεις σας όπως θέλετε, ή ακόμα και να [[Ενσωμάτωση ιστοσελίδων|ενσωματώσετε ιστοσελίδες]]. Η δυνατότητα χρήσης HTML μέσα στις σημειώσεις σας ενέχει κινδύνους. Για να αποτρέψει κακόβουλο κώδικα από το να προκαλέσει βλάβη, το Obsidian _εξυγιαίνει_ οποιοδήποτε HTML στις σημειώσεις σας.

> [!example]
> Το στοιχείο `<script>` κανονικά σας επιτρέπει να εκτελείτε JavaScript κάθε φορά που φορτώνεται. Εάν το Obsidian δεν εξυγίαινε το HTML, ένας επιτιθέμενος θα μπορούσε να σας πείσει να επικολλήσετε κείμενο που περιέχει JavaScript, το οποίο εξάγει ευαίσθητες πληροφορίες από τον υπολογιστή σας και τις στέλνει πίσω σε αυτόν.

Τούτου λεχθέντος, δεδομένου ότι η σύνταξη Markdown δεν υποστηρίζει όλες τις μορφές στυλ, η χρήση εξυγιασμένου HTML μπορεί να είναι ένας ακόμη τρόπος για τη βελτίωση της ποιότητας των σημειώσεών σας. Έχουμε συμπεριλάβει μερικές από τις πιο κοινές χρήσεις HTML.

## Περιορισμοί HTML

Το Obsidian έχει συγκεκριμένους περιορισμούς κατά τη χρήση HTML στις σημειώσεις σας:

### Χωρίς Markdown μέσα σε HTML

Το Obsidian δεν αποδίδει σύνταξη Markdown μέσα σε στοιχεία HTML. Αυτή είναι μια σκόπιμη σχεδιαστική επιλογή για βελτιστοποίηση απόδοσης και για τη διατήρηση χαμηλής πολυπλοκότητας του αναλυτή κατά τη διαχείριση μεγάλων εγγράφων.

Για παράδειγμα, αυτό δεν θα λειτουργήσει όπως αναμένεται:

```md
<div>
This **will not** be bold and this `will not` be code.
</div>
```

### Τα μπλοκ HTML πρέπει να είναι αυτοτελή

Τα μπλοκ HTML πρέπει να είναι πλήρη και δεν μπορούν να περιέχουν κενές γραμμές μέσα τους. Οι κενές γραμμές θα σπάσουν το μπλοκ HTML.

Αυτό θα λειτουργήσει:

```md
<table>
<tr>
<td>Content here</td>
</tr>
</table>
```

Αυτό δεν θα λειτουργήσει σωστά:

```md
<table>

<tr>

<td>Content here</td>

</tr>

</table>
```

### Όταν το Markdown φαίνεται να λειτουργεί μέσα σε HTML

Ορισμένες ενσωματωμένες ετικέτες HTML όπως `<span>` ή `<a>` έχουν περιορισμένη λειτουργικότητα και μπορεί να φαίνεται ότι αποδίδουν Markdown, αλλά αυτό δεν είναι στην πραγματικότητα αυτό που συμβαίνει. Το Markdown επεξεργάζεται εκτός του πλαισίου HTML.

Για περισσότερες λεπτομέρειες σχετικά με τον τρόπο που το Obsidian χειρίζεται το Markdown, δείτε [[Obsidian Flavored Markdown]].

## Κοινές χρήσεις HTML

> [!info] Περισσότερες λεπτομέρειες σχετικά με τη χρήση του `<iframe>` μπορείτε να βρείτε στο [[Ενσωμάτωση ιστοσελίδων]].

### Σχόλια

Τα [[Βασική σύνταξη μορφοποίησης#Σχόλια|σχόλια Markdown]] είναι ο προτιμώμενος τρόπος προσθήκης κρυφών σχολίων μέσα στις σημειώσεις σας. Ωστόσο, ορισμένες μέθοδοι μετατροπής σημειώσεων Markdown, όπως το [Pandoc](https://pandoc.org), έχουν περιορισμένη υποστήριξη σχολίων Markdown. Σε αυτές τις περιπτώσεις, μπορείτε να χρησιμοποιήσετε `<!-- HTML Comment -->` αντί αυτού!

### Υπογράμμιση

Εάν χρειάζεστε γρήγορα να υπογραμμίσετε ένα στοιχείο στις σημειώσεις σας, μπορείτε να χρησιμοποιήσετε `<u>Example</u>` για να δημιουργήσετε <u>το υπογραμμισμένο κείμενό σας</u>.

### Span/Div

Οι ετικέτες span και div μπορούν να χρησιμοποιηθούν για την εφαρμογή προσαρμοσμένων κλάσεων από ένα [[CSS αποσπάσματα|CSS απόσπασμα]], ή προσαρμοσμένων στυλ, σε μια επιλεγμένη περιοχή κειμένου. Για παράδειγμα, η χρήση `<span style="font-family: cursive">your text</span>` μπορεί να σας επιτρέψει γρήγορα να <span style="font-family: cursive">αλλάξετε τη γραμματοσειρά σας</span>.

## Διακριτή διαγραφή

Χρειάζεται να διαγράψετε <s>κάποιο κείμενο</s>; Χρησιμοποιήστε `<s>this</s>` για να το διαγράψετε.
