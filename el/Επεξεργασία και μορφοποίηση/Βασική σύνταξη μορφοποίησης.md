Μάθετε πώς να εφαρμόζετε βασική μορφοποίηση στις σημειώσεις σας, χρησιμοποιώντας [Markdown](https://daringfireball.net/projects/markdown/). Για πιο σύνθετη σύνταξη μορφοποίησης, ανατρέξτε στη [[Σύνθετη σύνταξη μορφοποίησης]].

## Παράγραφοι

Για να δημιουργήσετε παραγράφους σε Markdown, χρησιμοποιήστε μια **κενή γραμμή** για να διαχωρίσετε μπλοκ κειμένου. Κάθε μπλοκ κειμένου που διαχωρίζεται με κενή γραμμή αντιμετωπίζεται ως ξεχωριστή παράγραφος.

```md
Αυτή είναι μια παράγραφος.

Αυτή είναι μια άλλη παράγραφος.
```

Αυτή είναι μια παράγραφος.

Αυτή είναι μια άλλη παράγραφος.

Μια κενή γραμμή μεταξύ γραμμών κειμένου δημιουργεί ξεχωριστές παραγράφους. Αυτή είναι η προεπιλεγμένη συμπεριφορά στο Markdown.

> [!tip]- Πολλαπλά κενά διαστήματα
> Πολλαπλά διαδοχικά κενά διαστήματα εντός και μεταξύ παραγράφων συμπτύσσονται σε ένα μόνο κενό διάστημα όταν εμφανίζονται στην [[Προβολές και λειτουργία επεξεργασίας#Προβολή ανάγνωσης|προβολή ανάγνωσης]] ή σε ιστοσελίδες [[Εισαγωγή στο Obsidian Publish|Obsidian Publish]].
> 
> ```md
> Πολλαπλά          διαδοχικά          κενά
> 
> 
> 
> και πολλαπλές νέες γραμμές μεταξύ παραγράφων.
> ```
> 
> > Πολλαπλά          διαδοχικά          κενά
> > 
> > 
> > 
> > και πολλαπλές νέες γραμμές μεταξύ παραγράφων.
> 
> Εάν θέλετε να αποτρέψετε τη σύμπτυξη κενών ή να προσθέσετε πολλαπλά κενά διαστήματα, μπορείτε να χρησιμοποιήσετε τις ετικέτες HTML `&nbsp;` (μη διαχωριστικό κενό) ή `<br>` (αλλαγή γραμμής).

### Αλλαγές γραμμής

Από προεπιλογή στο Obsidian, πατώντας `Enter` μία φορά θα δημιουργηθεί μια νέα γραμμή στη σημείωσή σας, αλλά αυτή αντιμετωπίζεται ως *συνέχεια* της ίδιας παραγράφου στην εμφανιζόμενη έξοδο, ακολουθώντας την τυπική συμπεριφορά του Markdown. Για να εισαγάγετε μια αλλαγή γραμμής *εντός* μιας παραγράφου χωρίς να ξεκινήσετε νέα παράγραφο, μπορείτε είτε:

- Να προσθέσετε **δύο κενά** στο τέλος μιας γραμμής πριν πατήσετε `Enter`, ή
- Να χρησιμοποιήσετε τη συντόμευση `Shift+Enter` για να εισαγάγετε απευθείας μια αλλαγή γραμμής.

> [!question]- Γιατί πολλαπλά πατήματα `Enter` δεν δημιουργούν περισσότερες αλλαγές γραμμής στην προβολή ανάγνωσης;
> Στο Markdown, ένα μόνο `Enter` αγνοείται και πολλαπλά διαδοχικά πατήματα `Enter` έχουν ως αποτέλεσμα μόνο μία νέα παράγραφο. Αυτή η συμπεριφορά ευθυγραμμίζεται με τον κανόνα μαλακής αναδίπλωσης του Markdown, όπου οι επιπλέον κενές γραμμές δεν δημιουργούν πρόσθετες αλλαγές γραμμής ή παραγράφους — συμπτύσσονται σε ένα μόνο διαχωρισμό παραγράφων. Έτσι χειρίζεται το Markdown το κείμενο από προεπιλογή, διασφαλίζοντας ότι οι παράγραφοι ρέουν φυσικά χωρίς απροσδόκητες αλλαγές​.

Το Obsidian περιλαμβάνει μια ρύθμιση **[[Ρυθμίσεις#Αυστηρή αλλαγή γραμμής|Αυστηρή αλλαγή γραμμής]]**, η οποία κάνει το Obsidian να ακολουθεί την τυπική προδιαγραφή Markdown για τις αλλαγές γραμμής.

Για να ενεργοποιήσετε αυτή τη λειτουργία:

1. Ανοίξτε τις **[[Ρυθμίσεις]]**.
2. Μεταβείτε στην καρτέλα **Επεξεργαστής**.
3. Ενεργοποιήστε τη ρύθμιση **Αυστηρή αλλαγή γραμμής**.

Όταν η **Αυστηρή αλλαγή γραμμής** είναι ενεργοποιημένη στο Obsidian, οι αλλαγές γραμμής έχουν τρεις διαφορετικές συμπεριφορές ανάλογα με τον τρόπο διαχωρισμού των γραμμών:

**Μονό Enter χωρίς κενά**: Ένα μόνο `Enter` χωρίς κενά στο τέλος θα συνενώσει τις δύο ξεχωριστές γραμμές σε μία γραμμή κατά την απεικόνιση.

```md
γραμμή ένα
γραμμή δύο
```

Απεικονίζεται ως:

γραμμή ένα γραμμή δύο

**Μονό Enter με δύο ή περισσότερα κενά στο τέλος**: Εάν προσθέσετε δύο ή περισσότερα κενά στο τέλος της πρώτης γραμμής πριν πατήσετε `Enter`, οι δύο γραμμές παραμένουν μέρος της ίδιας παραγράφου, αλλά χωρίζονται με αλλαγή γραμμής (στοιχείο HTML `<br>`). Θα χρησιμοποιήσουμε δύο κάτω παύλες για να αντιπροσωπεύσουμε τα κενά σε αυτό το παράδειγμα.

```md
γραμμή τρία__  
γραμμή τέσσερα
```

Απεικονίζεται ως:

γραμμή τρία<br>
γραμμή τέσσερα

**Διπλό Enter (με ή χωρίς κενά στο τέλος)**: Πατώντας `Enter` δύο φορές (ή περισσότερες) διαχωρίζονται οι γραμμές σε δύο ξεχωριστές παραγράφους (στοιχεία HTML `<p>`), ανεξάρτητα από το αν προσθέσετε κενά στο τέλος της πρώτης γραμμής.

```md
γραμμή πέντε

γραμμή έξι
```

Απεικονίζεται ως:

<p>γραμμή πέντε</p>
<p>γραμμή έξι</p>

## Επικεφαλίδες

Για να δημιουργήσετε μια επικεφαλίδα, προσθέστε μέχρι έξι σύμβολα `#` πριν από το κείμενο της επικεφαλίδας. Ο αριθμός των συμβόλων `#` καθορίζει το επίπεδο της επικεφαλίδας (όπως φαίνεται στη [[Περίγραμμα]]).

```md
# Αυτή είναι επικεφαλίδα 1
## Αυτή είναι επικεφαλίδα 2
### Αυτή είναι επικεφαλίδα 3
#### Αυτή είναι επικεφαλίδα 4
##### Αυτή είναι επικεφαλίδα 5
###### Αυτή είναι επικεφαλίδα 6
```

%% These headings use HTML to avoid cluttering the Outline/Table of contents %%
<h1>Αυτή είναι επικεφαλίδα 1</h1>
<h2>Αυτή είναι επικεφαλίδα 2</h2>
<h3>Αυτή είναι επικεφαλίδα 3</h3>
<h4>Αυτή είναι επικεφαλίδα 4</h4>
<h5>Αυτή είναι επικεφαλίδα 5</h5>
<h6>Αυτή είναι επικεφαλίδα 6</h6>

## Έντονα, πλάγια, επισημάνσεις

Η μορφοποίηση κειμένου μπορεί επίσης να εφαρμοστεί χρησιμοποιώντας [[Συντομεύσεις επεξεργασίας]].

| Στυλ | Σύνταξη | Παράδειγμα | Έξοδος |
|-|-|-|-|
| Έντονα | `** **` ή `__ __` | `**Έντονο κείμενο**` | **Έντονο κείμενο** |
| Πλάγια | `* *` ή `_ _`  | `*Πλάγιο κείμενο*` | *Πλάγιο κείμενο* |
| Διακριτή διαγραφή | `~~ ~~` |  `~~Διεγραμμένο κείμενο~~` | ~~Διεγραμμένο κείμενο~~ |
| Επισήμανση | `== ==` |  `==Επισημασμένο κείμενο==` | ==Επισημασμένο κείμενο== |
| Έντονα και εμφωλευμένα πλάγια | `** **` και `_ _`  | `**Έντονο κείμενο και _εμφωλευμένο πλάγιο_ κείμενο**` | **Έντονο κείμενο και _εμφωλευμένο πλάγιο_ κείμενο** |
| Έντονα και πλάγια | `*** ***` ή `___ ___` |  `***Έντονο και πλάγιο κείμενο***` | ***Έντονο και πλάγιο κείμενο*** |

Η μορφοποίηση μπορεί να εξαναγκαστεί να εμφανιστεί ως απλό κείμενο προσθέτοντας μια ανάστροφη κάθετο `\` μπροστά της.

\*\*Αυτή η γραμμή δεν θα είναι έντονη\*\*

```markdown
\*\*Αυτή η γραμμή δεν θα είναι έντονη\*\*
```

\**Αυτή η γραμμή θα είναι πλάγια και θα εμφανίζει τους αστερίσκους*\*

```markdown
\**Αυτή η γραμμή θα είναι πλάγια και θα εμφανίζει τους αστερίσκους*\*
```

## Εσωτερικοί σύνδεσμοι

Το Obsidian υποστηρίζει δύο μορφές για [[Εσωτερικοί σύνδεσμοι|εσωτερικούς συνδέσμους]] μεταξύ σημειώσεων:

- Σύνδεσμος wiki: `[[Τρεις νόμοι της κίνησης]]`
- Markdown: `[Τρεις νόμοι της κίνησης](Τρεις%20νόμοι%20της%20κίνησης.md)`

## Εξωτερικοί σύνδεσμοι

Εάν θέλετε να συνδέσετε μια εξωτερική διεύθυνση URL, μπορείτε να δημιουργήσετε έναν ενσωματωμένο σύνδεσμο περικλείοντας το κείμενο του συνδέσμου σε αγκύλες (`[ ]`) και στη συνέχεια τη διεύθυνση URL σε παρενθέσεις (`( )`).

```md
[Βοήθεια Obsidian](https://help.obsidian.md)
```

[Βοήθεια Obsidian](https://help.obsidian.md)

Μπορείτε επίσης να δημιουργήσετε εξωτερικούς συνδέσμους προς αρχεία σε άλλα θησαυροφυλάκια, συνδέοντας σε ένα [[Obsidian URI|Obsidian URI]].

```md
[Σημείωση](obsidian://open?vault=MainVault&file=Note.md)
```

### Διαφυγή κενών διαστημάτων σε συνδέσμους

Εάν η διεύθυνση URL σας περιέχει κενά διαστήματα, πρέπει να τα διαφύγετε αντικαθιστώντας τα με `%20`.

```md
[Η Σημείωσή μου](obsidian://open?vault=MainVault&file=My%20Note.md)
```

Μπορείτε επίσης να διαφύγετε τη διεύθυνση URL περικλείοντάς την με γωνιακές αγκύλες (`< >`).

```md
[Η Σημείωσή μου](<obsidian://open?vault=MainVault&file=My Note.md>)
```

## Εξωτερικές εικόνες

Μπορείτε να προσθέσετε εικόνες με εξωτερικές διευθύνσεις URL, προσθέτοντας ένα σύμβολο `!` πριν από έναν [[#Εξωτερικοί σύνδεσμοι|εξωτερικό σύνδεσμο]].

```md
![Engelbart](https://history-computer.com/ModernComputer/Basis/images/Engelbart.jpg)
```

![Engelbart](https://history-computer.com/ModernComputer/Basis/images/Engelbart.jpg)

Μπορείτε να αλλάξετε τις διαστάσεις της εικόνας, προσθέτοντας `|640x480` στον προορισμό του συνδέσμου, όπου 640 είναι το πλάτος και 480 είναι το ύψος.

```md
![Engelbart|100x145](https://history-computer.com/ModernComputer/Basis/images/Engelbart.jpg)
```

Εάν καθορίσετε μόνο το πλάτος, η εικόνα κλιμακώνεται σύμφωνα με τις αρχικές της αναλογίες. Για παράδειγμα:

```md
![Engelbart|100](https://history-computer.com/ModernComputer/Basis/images/Engelbart.jpg)
```

> [!tip]- Συμβουλή
> Εάν θέλετε να προσθέσετε μια εικόνα από το εσωτερικό του θησαυροφυλακίου σας, μπορείτε επίσης να [[Ενσωμάτωση αρχείων#Ενσωμάτωση εικόνας σε σημείωση|ενσωματώσετε μια εικόνα σε μια σημείωση]].

## Λίστες

Μπορείτε να δημιουργήσετε μια μη ταξινομημένη λίστα προσθέτοντας `-`, `*` ή `+` πριν από το κείμενο.

```md
- Πρώτο στοιχείο λίστας
- Δεύτερο στοιχείο λίστας
- Τρίτο στοιχείο λίστας
```

- Πρώτο στοιχείο λίστας
- Δεύτερο στοιχείο λίστας
- Τρίτο στοιχείο λίστας

Για να δημιουργήσετε μια αριθμημένη λίστα, ξεκινήστε κάθε γραμμή με έναν αριθμό ακολουθούμενο από `.` ή `)`.

```md
1. Πρώτο στοιχείο λίστας
2. Δεύτερο στοιχείο λίστας
3. Τρίτο στοιχείο λίστας
```

1. Πρώτο στοιχείο λίστας
2. Δεύτερο στοιχείο λίστας
3. Τρίτο στοιχείο λίστας

```md
1) Πρώτο στοιχείο λίστας
2) Δεύτερο στοιχείο λίστας
3) Τρίτο στοιχείο λίστας
```

1) Πρώτο στοιχείο λίστας
2) Δεύτερο στοιχείο λίστας
3) Τρίτο στοιχείο λίστας

Μπορείτε να χρησιμοποιήσετε `Shift+Enter` για να εισαγάγετε μια [[#Αλλαγές γραμμής|αλλαγή γραμμής]] μέσα σε μια αριθμημένη λίστα χωρίς να αλλοιώσετε την αρίθμηση.

```md
1. Πρώτο στοιχείο λίστας
   
2. Δεύτερο στοιχείο λίστας
3. Τρίτο στοιχείο λίστας
   
4. Τέταρτο στοιχείο λίστας
5. Πέμπτο στοιχείο λίστας
6. Έκτο στοιχείο λίστας
```

### Λίστες εργασιών

Για να δημιουργήσετε μια λίστα εργασιών, ξεκινήστε κάθε στοιχείο λίστας με μια παύλα και κενό ακολουθούμενο από `[ ]`.

```md
- [x] Αυτή είναι μια ολοκληρωμένη εργασία.
- [ ] Αυτή είναι μια μη ολοκληρωμένη εργασία.
```

- [x] Αυτή είναι μια ολοκληρωμένη εργασία.
- [ ] Αυτή είναι μια μη ολοκληρωμένη εργασία.

Μπορείτε να εναλλάξετε μια εργασία στην προβολή ανάγνωσης επιλέγοντας το πλαίσιο ελέγχου.

> [!tip]- Συμβουλή
> Μπορείτε να χρησιμοποιήσετε οποιονδήποτε χαρακτήρα μέσα στις αγκύλες για να τη σημειώσετε ως ολοκληρωμένη.
>
> ```md
> - [x] Γάλα
> - [?] Αυγά
> - [-] Αυγά
> ```
>
> - [x] Γάλα
> - [?] Αυγά
> - [-] Αυγά

### Εμφωλευμένες λίστες

Μπορείτε να εμφωλεύσετε οποιονδήποτε τύπο λίστας — αριθμημένη, μη ταξινομημένη ή λίστα εργασιών — κάτω από οποιονδήποτε άλλο τύπο λίστας.

Για να δημιουργήσετε μια εμφωλευμένη λίστα, κάντε εσοχή σε ένα ή περισσότερα στοιχεία λίστας. Μπορείτε να αναμείξετε τύπους λίστας μέσα σε μια εμφωλευμένη δομή:

```md
1. Πρώτο στοιχείο λίστας
   1. Εμφωλευμένο αριθμημένο στοιχείο
2. Δεύτερο στοιχείο λίστας
   - Εμφωλευμένο μη ταξινομημένο στοιχείο
```

1. Πρώτο στοιχείο λίστας
   1. Εμφωλευμένο αριθμημένο στοιχείο
2. Δεύτερο στοιχείο λίστας
   - Εμφωλευμένο μη ταξινομημένο στοιχείο

Ομοίως, μπορείτε να δημιουργήσετε μια εμφωλευμένη λίστα εργασιών κάνοντας εσοχή σε ένα ή περισσότερα στοιχεία λίστας:

```md
- [ ] Εργασία 1
	- [ ] Υπο-εργασία 1
- [ ] Εργασία 2
	- [ ] Υπο-εργασία 1
```

- [ ] Εργασία 1
	- [ ] Υπο-εργασία 1
- [ ] Εργασία 2
	- [ ] Υπο-εργασία 1

Χρησιμοποιήστε `Tab` ή `Shift+Tab` για να αυξήσετε ή να μειώσετε την εσοχή των επιλεγμένων στοιχείων λίστας και να τα οργανώσετε εύκολα.

## Οριζόντια γραμμή

Μπορείτε να χρησιμοποιήσετε τρία ή περισσότερα αστέρια `***`, παύλες `---` ή κάτω παύλες `___` σε μια δική τους γραμμή για να προσθέσετε μια οριζόντια γραμμή. Μπορείτε επίσης να διαχωρίσετε τα σύμβολα χρησιμοποιώντας κενά.

```md
***
****
* * *
---
----
- - -
___
____
_ _ _
```

***

## Κώδικας

Μπορείτε να μορφοποιήσετε κώδικα τόσο ενσωματωμένα μέσα σε μια πρόταση, όσο και σε ξεχωριστό μπλοκ.

### Ενσωματωμένος κώδικας

Μπορείτε να μορφοποιήσετε κώδικα μέσα σε μια πρόταση χρησιμοποιώντας μονά backticks.

```md
Κείμενο μέσα σε `backticks` σε μια γραμμή θα μορφοποιηθεί ως κώδικας.
```

Κείμενο μέσα σε `backticks` σε μια γραμμή θα μορφοποιηθεί ως κώδικας.

Εάν θέλετε να βάλετε backticks σε ένα ενσωματωμένο μπλοκ κώδικα, περικλείστε το με διπλά backticks ως εξής: ενσωματωμένος ``κώδικας με backtick ` μέσα``.

### Μπλοκ κώδικα

Για να μορφοποιήσετε κώδικα ως μπλοκ, περικλείστε τον με τρία ή περισσότερα backticks ή τρεις ή περισσότερες περισπωμένες.

~~~
`````
cd ~/Desktop
`````
~~~
`````
~~~
cd ~/Desktop
~~~
`````
`````md
cd ~/Desktop
`````

Μπορείτε επίσης να δημιουργήσετε ένα μπλοκ κώδικα κάνοντας εσοχή στο κείμενο χρησιμοποιώντας `Tab` ή 4 κενά διαστήματα.
`````md
    cd ~/Desktop
`````

Μπορείτε να προσθέσετε χρωματισμό σύνταξης σε ένα μπλοκ κώδικα, προσθέτοντας έναν κωδικό γλώσσας μετά το πρώτο σύνολο backticks.

~~~md
  }
}
`````
~~~

#### Εμφωλευμένα μπλοκ κώδικα

Όταν χρειάζεται να συμπεριλάβετε ένα μπλοκ κώδικα μέσα σε ένα άλλο μπλοκ κώδικα (για παράδειγμα, όταν τεκμηριώνετε πώς να χρησιμοποιείτε μπλοκ κώδικα), μπορείτε να χρησιμοποιήσετε περισσότερα από τρία backticks ή περισπωμένες για το εξωτερικό μπλοκ κώδικα.

Για να εμφωλεύσετε μπλοκ κώδικα, χρησιμοποιήστε τέσσερα ή περισσότερα backticks (ή περισπωμένες) για το εξωτερικό μπλοκ, ενώ το εσωτερικό μπλοκ χρησιμοποιεί τρία:
`````md
````md
Δείτε πώς δημιουργείτε ένα μπλοκ κώδικα:
```js
console.log("Hello world")
```
````
`````

Μπορείτε επίσης να αναμείξετε backticks και περισπωμένες. Αυτό είναι ιδιαίτερα χρήσιμο όταν εργάζεστε με κώδικα που δημιουργεί άλλα μπλοκ κώδικα:

Η βασική αρχή είναι ότι το εξωτερικό μπλοκ κώδικα πρέπει να χρησιμοποιεί **περισσότερους** χαρακτήρες περίφραξης (backticks ή περισπωμένες) από οποιοδήποτε εσωτερικό μπλοκ κώδικα, ή να χρησιμοποιεί διαφορετικό τύπο χαρακτήρα περίφραξης.

## Υποσημειώσεις

Μπορείτε να προσθέσετε υποσημειώσεις[^footnote] στις σημειώσεις σας χρησιμοποιώντας την ακόλουθη σύνταξη:

[^footnote]: Αυτή είναι μια υποσημείωση.

```md
Αυτή είναι μια απλή υποσημείωση[^1].

[^1]: Αυτό είναι το κείμενο αναφοράς.
[^2]: Προσθέστε 2 κενά στην αρχή κάθε νέας γραμμής.
  Αυτό σας επιτρέπει να γράφετε υποσημειώσεις που εκτείνονται σε πολλές γραμμές.
[^note]: Οι ονομαστικές υποσημειώσεις εξακολουθούν να εμφανίζονται ως αριθμοί, αλλά μπορούν να διευκολύνουν τον εντοπισμό και τη σύνδεση αναφορών.
```

Μπορείτε επίσης να ενσωματώσετε υποσημειώσεις μέσα σε μια πρόταση. Σημειώστε ότι το καρέ πηγαίνει έξω από τις αγκύλες.

```md
Μπορείτε επίσης να χρησιμοποιήσετε ενσωματωμένες υποσημειώσεις. ^[Αυτή είναι μια ενσωματωμένη υποσημείωση.]
```

> [!note] Σημείωση
> Οι ενσωματωμένες υποσημειώσεις λειτουργούν μόνο στην προβολή ανάγνωσης, όχι στη ζωντανή προεπισκόπηση.

Χρησιμοποιήστε την [[Προβολή υποσημειώσεων]] για να δείτε όλες τις υποσημειώσεις σε μια σημείωση.

## Σχόλια

Μπορείτε να προσθέσετε σχόλια περικλείοντας κείμενο με `%%`. Τα σχόλια είναι ορατά μόνο στη λειτουργία επεξεργασίας.

```md
Αυτό είναι ένα %%ενσωματωμένο%% σχόλιο.

%%
Αυτό είναι ένα σχόλιο σε μπλοκ.

Τα σχόλια σε μπλοκ μπορούν να εκτείνονται σε πολλές γραμμές.
%%
```

## Διαφυγή σύνταξης Markdown

Σε ορισμένες περιπτώσεις, μπορεί να χρειαστεί να εμφανίσετε ειδικούς χαρακτήρες στο Markdown, όπως `*`, `_` ή `#`, χωρίς να ενεργοποιήσετε τη μορφοποίησή τους. Για να εμφανίσετε αυτούς τους χαρακτήρες κυριολεκτικά, τοποθετήστε μια ανάστροφη κάθετο (`\`) πριν από αυτούς.

> [!example] Συνηθισμένοι χαρακτήρες προς διαφυγή
> 
> - Αστερίσκος: `\*`
> - Κάτω παύλα: `\_`
> - Δίεση: `\#`
> - Backtick: `` \` ``
> - Κάθετη γραμμή (χρησιμοποιείται σε πίνακες): `\|`
> - Περισπωμένη: `\~`

```md
\*Αυτό το κείμενο δεν θα γίνει πλάγιο\*.
```

\*Αυτό το κείμενο δεν θα γίνει πλάγιο\*.

Όταν εργάζεστε με αριθμημένες λίστες, μπορεί να χρειαστεί να κάνετε διαφυγή της τελείας μετά τον αριθμό για να αποτρέψετε την αυτόματη μορφοποίηση λίστας. Τοποθετήστε την ανάστροφη κάθετο (`\`) πριν από την τελεία, **όχι** πριν από τον αριθμό.

```md
1\. Αυτό δεν θα είναι στοιχείο λίστας.
```

1\. Αυτό δεν θα είναι στοιχείο λίστας.
