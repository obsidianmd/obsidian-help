Το [[Εισαγωγή στο Obsidian Web Clipper|Web Clipper]] υποστηρίζει λογική προτύπων για συνθήκες, βρόχους και ανάθεση μεταβλητών. Αυτή η σύνταξη είναι εμπνευσμένη από τις γλώσσες προτύπων [Twig](https://twig.symfony.com/) και [Liquid](https://shopify.github.io/liquid/).


> [!warning] Απαιτείται η τελευταία έκδοση
> Οι δυνατότητες λογικής απαιτούν το Obsidian Web Clipper 1.0.0, το οποίο δεν έχει εγκριθεί ακόμη σε όλα τα καταστήματα επεκτάσεων.

## Συνθήκες

Χρησιμοποιήστε `{% if %}` για να συμπεριλάβετε υπό όρους περιεχόμενο βασισμένο σε μεταβλητές ή εκφράσεις.

```twig
{% if author %}
Author: {{author}}
{% endif %}
```

Χρησιμοποιήστε `{% else %}` για να παρέχετε εναλλακτικό περιεχόμενο, και `{% elseif %}` για να αλυσιδώσετε πολλαπλές συνθήκες:

```twig
{% if status == "published" %}
Live article
{% elseif status == "draft" %}
Draft article
{% else %}
Unknown status
{% endif %}
```

### Τελεστές σύγκρισης

Υποστηρίζονται οι ακόλουθοι τελεστές σύγκρισης:

| Τελεστής | Περιγραφή |
|----------|-------------|
| `==` | Ίσο με |
| `!=` | Διάφορο από |
| `>` | Μεγαλύτερο από |
| `<` | Μικρότερο από |
| `>=` | Μεγαλύτερο ή ίσο από |
| `<=` | Μικρότερο ή ίσο από |
| `contains` | Έλεγχος αν μια συμβολοσειρά περιέχει υποσυμβολοσειρά, ή αν ένας πίνακας περιέχει τιμή |

Παραδείγματα:
- `{% if title == "Home" %}` — ισότητα συμβολοσειρών
- `{% if price >= 100 %}` — αριθμητική σύγκριση
- `{% if title contains "Review" %}` — έλεγχος υποσυμβολοσειράς
- `{% if tags contains "important" %}` — συμμετοχή σε πίνακα

### Λογικοί τελεστές

Συνδυάστε συνθήκες χρησιμοποιώντας λογικούς τελεστές:

| Τελεστής | Εναλλακτικός | Περιγραφή                         |
| -------- | ----------- | ----------------------------------- |
| `and`    | `&&`        | Και οι δύο συνθήκες πρέπει να είναι αληθείς        |
| `or`     | \|\|        | Τουλάχιστον μία συνθήκη πρέπει να είναι αληθής |
| `not`    | `!`         | Αντιστρέφει μια συνθήκη                 |

Παραδείγματα:
- `{% if author and published %}` — και οι δύο πρέπει να υπάρχουν
- `{% if draft or archived %}` — οποιαδήποτε συνθήκη
- `{% if not hidden %}` — άρνηση
- `{% if (premium or featured) and published %}` — ομαδοποιημένες συνθήκες

### Αλήθεια τιμών

Όταν μια μεταβλητή χρησιμοποιείται χωρίς τελεστή σύγκρισης, αξιολογείται ως προς την «αλήθεια» της:

- Τα `false`, `null`, `undefined`, η κενή συμβολοσειρά `""` και το `0` θεωρούνται **ψευδή**.
- Οι κενοί πίνακες `[]` θεωρούνται **ψευδείς**.
- Όλα τα υπόλοιπα θεωρούνται **αληθή**.

```twig
{% if content %}
Has content
{% endif %}
```

## Ανάθεση μεταβλητής

Χρησιμοποιήστε `{% set %}` για να δημιουργήσετε ή να τροποποιήσετε μεταβλητές μέσα στο πρότυπό σας:

```twig
{% set slug = title|lower|replace:" ":"-" %}
File: {{slug}}.md
```

Οι μεταβλητές μπορούν να οριστούν σε:
- Άλλες μεταβλητές: `{% set name = author %}`
- Κυριολεκτικές τιμές: `{% set count = 5 %}` ή `{% set label = "Draft" %}`
- Εκφράσεις με φίλτρα: `{% set excerpt = content|truncate:100 %}`
- Αποτελέσματα επιλογέα: `{% set comments = selector:.comment %}`

Οι μεταβλητές που ορίζονται με `{% set %}` μπορούν να χρησιμοποιηθούν σε μεταγενέστερη λογική προτύπων και στην έξοδο `{{variable}}`.

## Εναλλακτικές τιμές

Χρησιμοποιήστε τον τελεστή `??` για να παρέχετε εναλλακτικές τιμές όταν μια μεταβλητή είναι κενή ή μη ορισμένη:

```twig
{{title ?? "Untitled"}}
```

Αν η `title` είναι κενή, μη ορισμένη ή ψευδής, θα χρησιμοποιηθεί η εναλλακτική τιμή `"Untitled"`.

Αυτό είναι μια συντομογραφία για την ισοδύναμη δήλωση `if`:

```twig
{% if title %}{{title}}{% else %}Untitled{% endif %}
```

### Αλυσιδωτές εναλλακτικές τιμές

Μπορείτε να αλυσιδώσετε πολλαπλές εναλλακτικές τιμές:

```twig
{{title ?? headline ?? "No title"}}
```

Αυτό θα χρησιμοποιήσει τη `title` αν είναι διαθέσιμη, αλλιώς τη `headline`, αλλιώς τη συμβολοσειρά `"No title"`.

### Με φίλτρα

Τα φίλτρα δεσμεύονται πιο σφιχτά από το `??`, οπότε τα φίλτρα εφαρμόζονται πριν τον έλεγχο εναλλακτικής τιμής:

```twig
{{title|upper ?? "UNTITLED"}}
```

Αυτό εφαρμόζει πρώτα το `upper` στη `title` και μετά επιστρέφει στο `"UNTITLED"` αν το αποτέλεσμα είναι κενό. Για να εφαρμόσετε φίλτρα στην εναλλακτική τιμή, χρησιμοποιήστε παρενθέσεις ή ξεχωριστές εκφράσεις:

```twig
{{title ?? "Untitled"|lower}}
```

Αυτό θα χρησιμοποιήσει τη `title` αν είναι διαθέσιμη, αλλιώς θα εφαρμόσει το `lower` στην εναλλακτική τιμή, δίνοντας ως αποτέλεσμα `"untitled"`.

## Βρόχοι

Χρησιμοποιήστε `{% for %}` για να επαναλάβετε πάνω σε πίνακες:

```twig
{% for item in schema:author %}
- {{item.name}}
{% endfor %}
```

### Πηγές βρόχων

Μπορείτε να επαναλάβετε πάνω σε:
- Πίνακες σχήματος: `{% for item in schema:author %}`
- Αποτελέσματα επιλογέα: `{% for comment in selector:.comment %}`
- Μεταβλητές που ορίστηκαν νωρίτερα: `{% set items = selector:.item %}{% for item in items %}`

### Μεταβλητές βρόχου

Μέσα σε ένα βρόχο, έχετε πρόσβαση σε ένα αντικείμενο `loop` με τις ακόλουθες ιδιότητες:

| Μεταβλητή | Περιγραφή |
|----------|-------------|
| `loop.index` | Τρέχουσα επανάληψη (αρχίζει από 1) |
| `loop.index0` | Τρέχουσα επανάληψη (αρχίζει από 0) |
| `loop.first` | `true` αν είναι η πρώτη επανάληψη |
| `loop.last` | `true` αν είναι η τελευταία επανάληψη |
| `loop.length` | Συνολικός αριθμός στοιχείων |

```twig
{% for tag in tags %}
{{loop.index}}. {{tag}}
{% if loop.last %} (end of list){% endif %}
{% endfor %}
```

Για συμβατότητα προς τα πίσω, μπορείτε επίσης να χρησιμοποιήσετε `item_index` (όπου `item` είναι το όνομα της μεταβλητής επανάληψής σας) για να λάβετε τη θέση με δείκτη 0:

```twig
{% for tag in tags %}
{{tag_index}}. {{tag}}
{% endfor %}
```

### Πρόσβαση σε στοιχεία πίνακα κατά δείκτη

Χρησιμοποιήστε σημειογραφία αγκύλης για να αποκτήσετε πρόσβαση σε στοιχεία πίνακα κατά δείκτη:

```twig
{{items[0]}}
{{items[loop.index0]}}
```

Αυτό είναι χρήσιμο όταν χρειάζεται να αποκτήσετε πρόσβαση σε στοιχεία από πολλαπλούς πίνακες παράλληλα:

```twig
{% set transcripts = selector:.transcript-text %}
{% set timestamps = selector:.timestamp %}

{% for line in transcripts %}
{{timestamps[loop.index0]}} - {{line}}
{% endfor %}
```

Η σημειογραφία αγκύλης λειτουργεί επίσης με ιδιότητες αντικειμένων:

```twig
{{user["name"]}}
{{data["my-key"]}}
```

### Εμφωλευμένοι βρόχοι

Οι βρόχοι μπορούν να εμφωλευτούν για σύνθετες δομές δεδομένων:

```twig
{% for section in sections %}
## {{section.title}}
{% for item in section.items %}
- {{item}}
{% endfor %}
{% endfor %}
```

## Συνδυασμός λογικής

Οι συνθήκες και οι βρόχοι μπορούν να συνδυαστούν:

```twig
{% for item in items %}
{% if item.active %}
- {{item.name}}
{% endif %}
{% endfor %}
```

## Σειρά αξιολόγησης

Η λογική προτύπων επεξεργάζεται με την ακόλουθη σειρά:

1. **Λογική προτύπων** — Τα `{% if %}`, `{% for %}`, `{% set %}` και `{{variables}}` αξιολογούνται πρώτα
2. **Μεταβλητές ερωτήματος** — Οι [[Μεταβλητές#Μεταβλητές ερωτήματος|μεταβλητές ερωτήματος]] όπως `{{"summarize this"|prompt}}` αποστέλλονται στον Διερμηνέα αφού ολοκληρωθεί η λογική προτύπων

Αυτό σημαίνει ότι μπορείτε να χρησιμοποιήσετε τη λογική προτύπων για να κατασκευάσετε ερωτήματα δυναμικά, αλλά τα αποτελέσματα ερωτημάτων δεν είναι διαθέσιμα για χρήση σε συνθήκες ή βρόχους.
