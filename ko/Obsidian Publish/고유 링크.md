[[고유 링크]]를 사용하여 노트의 URL 이름을 바꿀 수 있어요.

예를 들어, 다음과 같은 URL을:

```
https://publish.obsidian.md/username/Company/About+us
```

이렇게 바꿀 수 있어요:

```
https://publish.obsidian.md/username/about
```

노트에 고유 링크를 만들려면 [[속성]]에 `permalink` 속성을 추가하세요.

```yaml
---
permalink: about
---
```

누군가 원래 URL을 사용하여 노트를 방문하면 자동으로 고유 링크로 리디렉션돼요.

## 이전 노트 리디렉션

노트의 이름을 바꾸고 삭제하는 것은 살아있는 보관함을 유지하는 자연스러운 과정이에요. Obsidian은 로컬 보관함 내에서 노트를 옮길 때 링크를 자동으로 업데이트하지만, 다른 웹사이트에서는 여전히 게재된 [[Obsidian Publish 소개|Obsidian Publish]] 사이트의 이전 노트로 링크하고 있을 수 있어요. 독자를 한 노트에서 다른 노트로 리디렉션할 수 있어요.

노트를 한 폴더에서 다른 폴더로 옮기고 싶다고 가정해 볼게요:

- **Guides**
  - ~~Making friends.md~~ (삭제됨)
- **Tutorials**
  - *How to make friends.md* (추가됨)

노트를 옮긴 후 Obsidian은 보관함 내의 모든 링크를 자동으로 업데이트해요. 그러나 변경 사항을 Publish 사이트에 게재하면 `/Guides/Making+friends`로의 링크는 404 오류가 발생해요.

독자를 `/Guides/Making+friends`에서 `/Tutorials/How+to+make+friends`로 리디렉션하려면 리디렉션 대상인 `How to make friends.md`에 [[별칭]]을 추가해야 해요.

```md
---
alias: Guides/Making friends
---

# How to make friends
```

> [!important] 중요
> 별칭에 이전 노트의 전체 경로를 포함해야 해요. 노트 이름만 별칭으로 사용하면 로컬 보관함에서는 작동하지만, Publish에서 리디렉션하려면 노트의 전체 경로가 필요해요.

각 노트에 별칭을 추가하여 여러 노트를 리디렉션할 수 있어요.

```md
---
aliases: 
  - Guides/Making friends
  - Developing friendships
---

# How to make friends
```
