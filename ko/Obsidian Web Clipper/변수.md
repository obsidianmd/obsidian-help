[[Obsidian Web Clipper/템플릿|Web Clipper 템플릿]]은 변수를 사용하여 페이지의 데이터를 템플릿에 자동으로 미리 채울 수 있어요. 변수는 **노트 이름**, **노트 위치**, **속성**, 및 **노트 내용**에서 사용할 수 있어요. 변수는 [[필터]]를 사용하여 수정할 수도 있어요.

[[Obsidian Web Clipper 소개|Web Clipper]] 확장 프로그램에서 `...` 아이콘을 사용하면 템플릿에 사용할 현재 페이지 변수에 접근할 수 있어요. 사용할 수 있는 변수는 다섯 가지 유형이 있어요:

- [[변수#프리셋 변수|프리셋 변수]]
- [[변수#프롬프트 변수|프롬프트 변수]]
- [[변수#메타 변수|메타 변수]]
- [[변수#선택자 변수|선택자 변수]]
- [[변수#Schema.org 변수|Schema.org 변수]]

## 프리셋 변수

프리셋 변수는 페이지 콘텐츠를 기반으로 자동 생성돼요. 이 변수들은 일반적으로 대부분의 웹사이트에서 작동해요.

주요 콘텐츠 변수는 `{{content}}`로, 기사 콘텐츠, [[하이라이터|하이라이트]], 또는 페이지에서 선택한 텍스트가 있는 경우 해당 선택 항목을 포함해요. `{{content}}`는 페이지의 주요 콘텐츠를 추출하려고 시도하지만, 항상 원하는 결과를 제공하지는 않을 수 있어요. 이 경우 다른 프리셋 변수나 선택자 변수를 사용하여 필요한 콘텐츠를 추출할 수 있어요.

| 변수                | 설명                                                                                    |
| ------------------- | -------------------------------------------------------------------------------------- |
| `{{author}}`        | 페이지 작성자                                                                            |
| `{{content}}`       | 기사 콘텐츠, [[하이라이터\|하이라이트]], 또는 선택 항목 (Markdown 형식)          |
| `{{contentHtml}}`   | 기사 콘텐츠, [[하이라이터\|하이라이트]], 또는 선택 항목 (HTML 형식)              |
| `{{date}}`          | 현재 날짜, `date` 필터를 사용하여 서식 지정 가능                                          |
| `{{description}}`   | 설명 또는 발췌                                                                           |
| `{{domain}}`        | 도메인                                                                                   |
| `{{favicon}}`       | 파비콘 URL                                                                               |
| `{{fullHtml}}`      | 전체 페이지 콘텐츠의 가공되지 않은 HTML                                                   |
| `{{highlights}}`    | [[하이라이터\|하이라이트]] (텍스트 및 타임스탬프 포함)                            |
| `{{image}}`         | 소셜 공유 이미지 URL                                                                     |
| `{{published}}`     | 게시 날짜, `date` 필터를 사용하여 서식 지정 가능                                          |
| `{{selection}}`     | 선택 항목 (Markdown 형식)                                                                |
| `{{selectionHtml}}` | 선택 항목 (HTML 형식)                                                                    |
| `{{site}}`          | 사이트 이름 또는 게시자                                                                   |
| `{{title}}`         | 페이지 제목                                                                              |
| `{{time}}`          | 현재 날짜 및 시간                                                                        |
| `{{url}}`           | 현재 URL                                                                                 |
| `{{words}}`         | 단어 개수                                                                                |

## 프롬프트 변수

프롬프트 변수는 언어 모델을 활용하여 자연어를 사용해 데이터를 추출하고 수정해요. 프롬프트 변수를 사용하려면 [[인터프리터|인터프리터]]가 활성화되고 설정되어 있어야 해요.

프롬프트 변수는 `{{"a summary of the page"}}` 구문을 사용해요. 프롬프트 주위의 큰따옴표가 중요하며, 프롬프트를 프리셋 변수와 구별해요. 프롬프트 응답은 [[필터]]로 후처리할 수 있어요. 예: `{{"a summary of the page"|blockquote}}`.

### 프롬프트 변수를 사용할 때

프롬프트 변수는 매우 유연하고 작성하기 쉽다는 장점이 있지만, 몇 가지 단점이 있어요: 실행 속도가 느리고, 선택한 [[인터프리터#모델|제공자]]에 따라 비용 및 개인정보 보호 문제가 있을 수 있어요.

다른 변수 유형과 달리, 프롬프트 변수는 외부 언어 모델에 의해 처리되어야 하므로, [[인터프리터|인터프리터]]가 실행된 후에만 대체돼요.

추출하려는 데이터가 다른 변수 유형으로 추출할 수 있는 일관된 형식인 경우 프롬프트 변수를 사용하지 *않는* 것이 좋아요.

반면, 추출하려는 데이터가 웹사이트마다 *일관되지 않은* 형식인 경우 프롬프트 변수가 유용할 수 있어요. 예를 들어, 책 사이트에 구애받지 않는 책 저장용 [[Obsidian Web Clipper/템플릿|템플릿]]을 만들 수 있어요. `{{"author of the book"}}`과 같은 프롬프트 변수는 어떤 책 사이트에서든 작동하지만, 선택자 변수는 일반적으로 하나의 사이트에서만 작동해요.

### 예시

프롬프트는 거의 모든 자연어 쿼리를 사용할 수 있어요. 사용하는 모델에 따라 프롬프트는 여러 언어로 데이터를 쿼리하거나 번역할 수 있어요.

- `{{"a three bullet point summary, translated to French"}}` - 페이지에 대한 글머리 기호 3개 요약을 추출하고 프랑스어로 번역해요.
- `{{"un resumé de la page en trois points"}}` - 프랑스어 프롬프트를 사용하여 3개의 글머리 기호를 추출해요.

프롬프트는 페이지 콘텐츠를 [[필터]]로 조작할 수 있는 JSON으로 변환할 수 있어요. 예시:

```
{{"return a JSON object for each tweet, that includes the author, tweet_text, date in YYYY-MM-DD format, and images array (if there are any)"|map:tweet => ({text: tweet.tweet_text, author: tweet.author, date: tweet.date})|template:"${text}\n— [[@${author}]], [[${date}]]\n"}}
```


## 메타 변수

메타 변수를 사용하면 페이지의 [meta 요소](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/meta)에서 데이터를 추출할 수 있으며, 소셜 공유 미리보기를 채우는 데 사용되는 [Open Graph](https://ogp.me/) 데이터도 포함돼요.

- `{{meta:name}}`은 지정된 이름의 meta name 태그 콘텐츠를 반환해요. 예: `{{meta:name:description}}`은 `description` 메타 태그를 반환해요.
- `{{meta:property}}`는 지정된 속성의 meta property 태그 콘텐츠를 반환해요. 예: `{{meta:property:og:title}}`은 `og:title` 메타 태그를 반환해요.

## 선택자 변수

선택자 변수를 사용하면 [CSS 선택자](https://developer.mozilla.org/en-US/docs/Web/CSS/CSS_selectors/Selectors_and_combinators)를 사용하여 페이지의 요소에서 텍스트 콘텐츠를 추출할 수 있어요.

구문은 `{{selector:cssSelector?attribute}}`이며, `?attribute`는 선택 사항이에요. 속성을 지정하지 않으면 요소의 텍스트 콘텐츠가 반환돼요. `{{selectorHtml:cssSelector}}`를 사용하여 요소의 HTML 콘텐츠를 가져올 수도 있어요. 선택자 변수는 일관된 HTML 구조를 가진 특정 웹사이트 또는 웹사이트 집합에서 가장 잘 작동해요.

- `{{selector:h1}}`은 페이지의 모든 `h1` 요소의 텍스트 콘텐츠를 반환해요.
- `{{selector:.author}}`는 페이지의 모든 `.author` 요소의 텍스트 콘텐츠를 반환해요.
- `{{selector:img.hero?src}}`는 `hero` 클래스를 가진 이미지의 `src` 속성을 반환해요.
- `{{selector:a.main-link?href}}`는 `main-link` 클래스를 가진 앵커 태그의 `href` 속성을 반환해요.
- `{{selectorHtml:body|markdown}}`는 `body` 요소의 전체 HTML을 `markdown` [[필터#HTML 처리|필터]]를 사용하여 Markdown으로 변환하여 반환해요.
- 더 높은 특정성이 필요한 경우 중첩된 CSS 선택자와 결합자가 지원돼요.
- 여러 요소가 선택자에 일치하면 배열이 반환되며, `join`이나 `map`과 같은 [[필터#배열 및 객체|배열 및 객체 필터]]로 처리할 수 있어요.

선택자 변수는 [[템플릿#템플릿 로직|템플릿 로직]]에서 직접 사용할 수도 있어요:

- 반복문: `{% for comment in selector:.comment %}...{% endfor %}`
- 조건문: `{% if selector:.premium-badge %}...{% endif %}`
- 변수 할당: `{% set items = selector:.list-item %}`

## Schema.org 변수

스키마 변수를 사용하면 페이지의 [schema.org](https://schema.org/) JSON-LD에서 데이터를 추출할 수 있어요. Schema.org 데이터는 자동으로 [[Obsidian Web Clipper/템플릿#Schema.org 매칭|템플릿을 트리거]]하는 데에도 사용할 수 있어요.

- `{{schema:@Type:key}}`는 스키마에서 키의 값을 반환해요.
- `{{schema:@Type:parent.child}}`는 중첩된 속성의 값을 반환해요.
- `{{schema:@Type:arrayKey}}`는 배열의 첫 번째 항목을 반환해요.
- `{{schema:@Type:arrayKey[index].property}}`는 배열에서 지정된 인덱스의 항목을 반환해요.
- `{{schema:@Type:arrayKey[*].property}}`는 배열의 모든 항목에서 특정 속성을 반환해요.

스키마 유형을 지정하지 않는 축약 표기법도 사용할 수 있어요:

- `{{schema:author}}`는 모든 스키마 유형에서 처음 발견되는 `author` 속성을 일치시켜요.
- `{{schema:name}}`은 모든 스키마 유형에서 처음 발견되는 `name` 속성을 일치시켜요.

이 축약 표기법은 특정 스키마 유형을 모르거나 상관없지만, 찾고 있는 속성 이름을 알고 있을 때 특히 유용해요.

중첩된 속성과 배열 접근은 스키마 `@Type` 지정 여부에 관계없이 모두 작동해요:

- `{{schema:author.name}}`은 처음 발견되는 `author` 속성을 찾은 다음 그 `name` 하위 속성에 접근해요.
- `{{schema:author[0].name}}`은 작성자 배열에서 첫 번째 작성자의 `name`에 접근해요.
- `{{schema:author[*].name}}`은 모든 작성자 이름의 배열을 반환해요.
