[[Obsidian Web Clipper 소개|Web Clipper]]를 사용하면 웹 페이지에서 메타데이터를 자동으로 캡처하고 정리하는 템플릿을 만들 수 있어요. 예시 템플릿은 [clipper-templates 저장소](https://github.com/kepano/clipper-templates)에서 확인할 수 있어요.

## 템플릿 생성 또는 편집

템플릿을 **생성**하려면 Web Clipper 설정으로 이동하여 사이드바에서 **새 템플릿** 버튼을 클릭하세요. 오른쪽 상단의 **더 보기** 작업 메뉴에서 템플릿을 **복제**할 수도 있어요.

템플릿을 **편집**하려면 사이드바에서 템플릿을 선택하세요. 변경 사항은 자동으로 저장돼요.

템플릿은 [[변수]] 및 [[필터]]를 활용하여 콘텐츠가 저장되는 방식을 맞춤 설정할 수 있어요.

## Web Clipper 템플릿 들여오기 및 내보내기

템플릿을 들여오려면:

1. 확장 프로그램을 열고 **[[설정]]** 톱니바퀴 아이콘을 클릭하세요.
2. 목록에서 아무 템플릿으로 이동하세요.
3. 오른쪽 상단의 **들여오기**를 클릭하거나 `.json` 템플릿 파일을 템플릿 영역 아무 곳에 드래그 앤 드롭하세요.

템플릿을 내보내려면 오른쪽 상단의 **내보내기**를 클릭하세요. 템플릿 `.json` 파일이 다운로드돼요. **더 보기** 메뉴를 통해 템플릿 데이터를 클립보드에 복사할 수도 있어요.

## 템플릿 설정

### 동작

Web Clipper의 콘텐츠가 Obsidian에 추가되는 방식을 정의해요:

- **새 노트 생성**
- **기존 노트에 추가**, 상단 또는 하단에
- **일일 노트에 추가**, 상단 또는 하단에 ([[일일 노트]] 플러그인이 활성화되어 있어야 해요)

### 자동으로 템플릿 트리거

템플릿 트리거를 사용하면 현재 페이지 URL이나 [schema.org](https://schema.org/) 데이터를 기반으로 자동으로 템플릿을 선택할 수 있어요. 각 템플릿에 대해 여러 규칙을 정의할 수 있으며, 새 줄로 구분해요.

템플릿 목록에서 첫 번째로 일치하는 항목이 사용할 템플릿을 결정해요. Web Clipper 설정에서 템플릿을 위아래로 드래그하여 템플릿이 일치되는 순서를 변경할 수 있어요.

#### 단순 URL 매칭

단순 매칭은 현재 페이지 URL이 지정된 패턴으로 *시작하는* 경우 템플릿을 트리거해요. 예시:

- `https://obsidian.md`는 이 텍스트로 시작하는 모든 URL과 일치해요.

#### 정규식 매칭

정규식을 사용하여 더 복잡한 URL 패턴을 기반으로 템플릿을 트리거할 수 있어요. 정규식 패턴을 슬래시(`/`)로 감싸세요. 정규식 패턴에서 특수 문자(`.`이나 `/` 등)는 백슬래시(`\`)로 이스케이프해야 해요. 예시:

- `/^https:\/\/www\.imdb\.com\/title\/tt\d+\/reference\/?$/`는 모든 IMDB 레퍼런스 페이지와 일치해요.

#### Schema.org 매칭

페이지에 존재하는 [schema.org](https://schema.org/) 데이터를 기반으로 템플릿을 트리거할 수 있어요. `schema:` 접두사 뒤에 일치시킬 스키마 키를 사용하세요. 선택적으로 예상 값을 지정할 수 있어요. 예시:

- `schema:@Recipe`는 스키마 유형이 "Recipe"인 페이지와 일치해요.
- `schema:@Recipe.name`은 `@Recipe.name`이 존재하는 페이지와 일치해요.
- `schema:@Recipe.name=Cookie`는 `@Recipe.name`이 "Cookie"인 페이지와 일치해요.

Schema.org 값은 [[변수#Schema.org 변수|템플릿에 데이터를 미리 채우는]] 데에도 사용할 수 있어요.

### 인터프리터 컨텍스트

[[인터프리터|인터프리터]]가 활성화되면 [[변수#프롬프트 변수|프롬프트 변수]]를 사용하여 자연어로 페이지 콘텐츠를 추출할 수 있어요. 각 템플릿에 대해 인터프리터가 접근할 수 있는 [[인터프리터#컨텍스트|컨텍스트]]를 정의할 수 있어요.
