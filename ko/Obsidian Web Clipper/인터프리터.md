인터프리터는 [[Obsidian Web Clipper 소개|Web Clipper]] 기능으로, 자연어를 사용하여 웹 페이지와 상호작용할 수 있게 해줘요. 인터프리터는 Obsidian에 저장하려는 데이터를 캡처하고 수정하는 데 도움을 줘요. 예를 들어:

- 특정 텍스트 조각을 추출해요.
- 정보를 요약하거나 설명해요.
- 텍스트를 한 형식에서 다른 형식으로 변환해요.
- 텍스트를 다른 언어로 번역해요.

인터프리터는 언어 모델을 활용하여 웹 페이지의 정보를 처리하고, [[변수]]를 사용하여 결과를 반환해요. 이 변수는 [[Obsidian Web Clipper/템플릿|Web Clipper 템플릿]]에 추가할 수 있어요.

![[web-clipper-interpreter-demo.mp4#interface]]

## 프롬프트 예시

프롬프트는 [[변수|변수]] 구문 `{{"your prompt"}}`를 사용해요. 이 구문은 자연어 검색어와 함께 사용할 수 있어요. 예를 들어:

- `{{"a summary of the page"}}` — 페이지의 요약을 추출해요.
- `{{"a three bullet point summary, translated to French"}}` — 페이지에 대한 글머리 기호 요약을 추출하고 프랑스어로 번역해요.
- `{{"un resumé de la page en trois points"}}` — 프랑스어 프롬프트를 사용하여 세 가지 글머리 기호를 추출해요.

프롬프트의 출력은 [[필터]]를 사용하여 추가로 조작할 수 있어요. 필터는 모델로부터 프롬프트 응답을 받은 후에 처리돼요. 예를 들어: `{{"a summary of the page"|blockquote}}`는 응답을 인용 블록으로 변환해요.

## 시작하기

인터프리터는 기기에서 비공개로 실행되는 옵션을 포함하여 거의 모든 언어 모델 제공자와 함께 작동해요. 인터프리터를 설정하려면:

1. Web Clipper 설정에서 **인터프리터** 섹션으로 이동해요.
2. **인터프리터 활성화**를 토글해요.
3. 제공자와 모델을 설정해요. 아래 [[웹 페이지 해석#모델|모델]] 섹션을 참조하세요.
4. [[변수|프롬프트 변수]]를 [[Obsidian Web Clipper/템플릿|템플릿]]에 추가해요.
5. 템플릿에 프롬프트 변수가 포함되어 있으면 [[웹 페이지 클리핑|페이지를 클리핑]]할 때 인터프리터 섹션이 표시돼요. **해석**을 클릭하여 프롬프트 변수를 처리해요.

## 작동 방식

인터프리터가 활성화되어 있고 *동시에* 템플릿에 [[변수#프롬프트 변수|프롬프트 변수]]가 포함되어 있으면, 확장 프로그램 창에서 **Obsidian에 추가하기** 버튼 위에 새로운 인터프리터 섹션이 표시돼요. 이 섹션에서 모델을 선택하고 현재 페이지에 대해 인터프리터를 실행할 수 있어요.

**해석**을 클릭하면, 인터프리터는 페이지 컨텍스트를 선택한 모델에 전송하며, 템플릿의 *모든* 프롬프트를 하나의 요청으로 함께 보내요. 선택한 모델 제공자에 따라 외부 호출이 될 수도 있고 기기 로컬에서 처리될 수도 있어요. 모델은 페이지 컨텍스트에 대해 프롬프트를 평가하고 응답을 반환해요. 그런 다음 인터프리터가 프롬프트 변수를 응답 데이터로 대체해요.

전체 과정은 사용하는 모델과 처리하는 데이터의 양에 따라 밀리초에서 30초 이상까지 걸릴 수 있어요.

## 컨텍스트

*컨텍스트*라는 용어는 인터프리터가 프롬프트를 처리하는 데 사용하는 페이지 데이터를 말해요. 컨텍스트가 작을수록 인터프리터가 더 빠르게 실행돼요.

기본적으로 인터프리터는 전체 페이지 HTML을 컨텍스트로 사용하지만, 이로 인해 프롬프트가 필요 이상으로 느려지고 비용이 많이 들 수 있어요.

인터프리터 **고급 설정**에서 기본 컨텍스트를 재정의하고 [[Obsidian Web Clipper/템플릿|템플릿]]별로 컨텍스트를 정의할 수 있어요.

보다 정확한 컨텍스트를 정의하려면 [[변수#선택자 변수|선택자 변수]](또는 다른 변수 유형)를 사용하여 페이지의 특정 섹션을 해석해요. 예를 들어, 템플릿의 인터프리터 컨텍스트에 다음 선택자 변수를 사용할 수 있어요:

```
{{selectorHtml:#main}}
```

이렇게 하면 웹 페이지의 `#main` 요소가 존재하는 경우 해당 요소에서만 인터프리터가 실행돼요. `remove_html`, `strip_tags`, `strip_attr`와 같은 [[필터#HTML 처리|HTML 처리 필터]]는 컨텍스트 길이를 더 줄이고 처리 속도를 높이는 데 유용해요.

## 모델

> [!warning] 개인정보 보호
> 타사 모델 제공자를 사용하면 해당 제공자의 약관 및 개인정보 보호 정책에 동의하게 돼요. 인터프리터 요청은 선택한 제공자에게 직접 전송돼요. Obsidian은 요청에 대한 어떠한 데이터도 수집하거나 저장하지 않아요.

### 모델 선택하기

일반적으로 Web Clipper에서는 소형 모델을 사용하는 것을 권장해요. 소형 모델이 더 빠르고 이 작업에 대해 상당히 정확한 성능을 보이기 때문이에요. 소형 모델의 예로는 **Anthropic의 Claude Haiku**, **Google Gemini Flash**, 3B 또는 8B 파라미터의 **Llama**, 또는 **OpenAI의 Mini** 시리즈 모델이 있어요.

### 커스텀 제공자 및 모델

커스텀 제공자 및/또는 모델을 추가하려면 Web Clipper **[[설정]]** → **인터프리터**로 이동해요:

- **제공자 추가**를 사용하여 프리셋 및 커스텀 제공자를 설정해요.
- **모델 추가**를 사용하여 프리셋 및 커스텀 모델을 설정해요.

커스텀 제공자를 추가할 때는 **기본 URL**에 chat completions 엔드포인트를 사용하는 것을 권장해요 — 일반적으로 `/chat/completions`으로 끝나요.

### 로컬 모델

인터프리터는 더 높은 개인정보 보호와 오프라인 호환성을 제공하는 로컬 모델을 사용할 수 있어요. 로컬 모델을 실행하기 위한 여러 옵션이 있어요. 설정이 가장 쉬운 것 중 하나는 Ollama예요.

#### Ollama

[Ollama](https://ollama.com/)를 사용하면 기기에서 언어 모델을 로컬로 비공개로 실행할 수 있어요.

Ollama를 다운로드하고 설치한 후, 인터프리터 설정에서 **제공자 추가**를 사용하여 Ollama를 추가해요. Ollama는 API 키가 필요하지 않아요. 그런 다음 [모델 목록](https://ollama.com/search)에서 모델을 선택해요. 예를 들어 [Llama 3.2](https://ollama.com/library/llama3.2)를 사용하려면 **모델 추가**를 클릭한 후:

- **제공자:** Ollama
- **표시 이름:** Llama 3.2, 이 값은 사용자 지정할 수 있어요.
- **모델 ID:** `llama3.2`, Ollama의 모델 ID와 정확히 일치해야 해요.

**Ollama 서버 시작하기**

브라우저 확장 프로그램이 Ollama와 상호작용할 수 있도록 하려면 서버를 실행할 때 [명시적으로 지시](https://github.com/ollama/ollama/issues/2308)해야 해요. 그렇지 않으면 `403` 오류가 표시돼요.

Ollama 앱을 닫고 터미널에서 다음 명령을 실행해요. Chrome이나 Firefox를 사용하지 않는 경우 프로토콜을 브라우저의 확장 프로그램 프로토콜로 변경해야 해요.

```
OLLAMA_ORIGINS=moz-extension://*,chrome-extension://*,safari-web-extension://* ollama serve
```

그런 다음 일반적인 방법으로 Ollama에서 모델을 실행해요. 예:

```
ollama run llama3.2
```

**컨텍스트 길이**

Ollama의 컨텍스트 창은 기본적으로 2048 토큰이에요. 이는 메시지와 응답의 최대 토큰 수예요. 긴 웹 페이지를 클리핑할 때 이 제한을 쉽게 초과할 수 있어요. Ollama는 조용히 실패하고 관련 없는 결과를 반환해요. 몇 가지 옵션:

- Ollama의 `num_ctx` 파라미터를 늘려요. 더 긴 컨텍스트는 더 많은 메모리를 필요로 한다는 점에 유의하세요.
- 템플릿의 [[웹 페이지 해석#컨텍스트|컨텍스트]] 필드를 사용하여 페이지의 보다 정확한 섹션을 제공하거나, [[필터|필터]]를 사용하여 컨텍스트를 줄여요. 예: `{{content|slice:0,1000}}`.
