[[Obsidian Web Clipper 소개|Web Clipper]]는 조건문, 반복문, 변수 할당을 위한 템플릿 로직을 지원해요. 이 구문은 [Twig](https://twig.symfony.com/)와 [Liquid](https://shopify.github.io/liquid/) 템플릿 언어에서 영감을 받았어요.

> [!warning] 최신 버전이 필요해요
> 로직 기능은 아직 모든 확장 프로그램 스토어에서 승인되지 않은 Obsidian Web Clipper 1.0.0이 필요해요.

## 조건문

`{% if %}`를 사용하여 변수나 표현식에 따라 콘텐츠를 조건부로 포함할 수 있어요.

```twig
{% if author %}
Author: {{author}}
{% endif %}
```

`{% else %}`를 사용하여 대체 콘텐츠를 제공하고, `{% elseif %}`를 사용하여 여러 조건을 연결할 수 있어요:

```twig
{% if status == "published" %}
Live article
{% elseif status == "draft" %}
Draft article
{% else %}
Unknown status
{% endif %}
```

### 비교 연산자

다음 비교 연산자가 지원돼요:

| 연산자 | 설명 |
|----------|-------------|
| `==` | 같음 |
| `!=` | 같지 않음 |
| `>` | 초과 |
| `<` | 미만 |
| `>=` | 이상 |
| `<=` | 이하 |
| `contains` | 문자열에 부분 문자열이 포함되어 있는지, 또는 배열에 값이 포함되어 있는지 확인 |

예시:
- `{% if title == "Home" %}` — 문자열 동등 비교
- `{% if price >= 100 %}` — 숫자 비교
- `{% if title contains "Review" %}` — 부분 문자열 확인
- `{% if tags contains "important" %}` — 배열 포함 여부

### 논리 연산자

논리 연산자를 사용하여 조건을 결합해요:

| 연산자 | 대안 | 설명                         |
| -------- | ----------- | ----------------------------------- |
| `and`    | `&&`        | 두 조건이 모두 참이어야 해요        |
| `or`     | \|\|        | 적어도 하나의 조건이 참이어야 해요 |
| `not`    | `!`         | 조건을 부정해요                 |

예시:
- `{% if author and published %}` — 둘 다 존재해야 해요
- `{% if draft or archived %}` — 둘 중 하나의 조건
- `{% if not hidden %}` — 부정
- `{% if (premium or featured) and published %}` — 그룹화된 조건

### 참 거짓 판별 (Truthiness)

변수가 비교 연산자 없이 사용되면 "참 거짓(truthiness)"으로 평가돼요:

- `false`, `null`, `undefined`, 빈 문자열 `""`, `0`은 **거짓(falsy)**으로 간주돼요.
- 빈 배열 `[]`은 **거짓(falsy)**으로 간주돼요.
- 그 외 모든 것은 **참(truthy)**이에요.

```twig
{% if content %}
Has content
{% endif %}
```

## 변수 할당

`{% set %}`을 사용하여 템플릿 내에서 변수를 생성하거나 수정할 수 있어요:

```twig
{% set slug = title|lower|replace:" ":"-" %}
File: {{slug}}.md
```

변수는 다음과 같이 설정할 수 있어요:
- 다른 변수: `{% set name = author %}`
- 리터럴: `{% set count = 5 %}` 또는 `{% set label = "Draft" %}`
- 필터가 포함된 표현식: `{% set excerpt = content|truncate:100 %}`
- 선택자 결과: `{% set comments = selector:.comment %}`

`{% set %}`으로 설정된 변수는 이후 템플릿 로직과 `{{variable}}` 출력에서 사용할 수 있어요.

## 대체값

`??` 연산자를 사용하여 변수가 비어 있거나 정의되지 않은 경우 대체값을 제공할 수 있어요:

```twig
{{title ?? "Untitled"}}
```

`title`이 비어 있거나, 정의되지 않았거나, 거짓(falsy)이면 대체값 `"Untitled"`이 대신 사용돼요.

이것은 동등한 `if` 문의 축약형이에요:

```twig
{% if title %}{{title}}{% else %}Untitled{% endif %}
```

### 대체값 체이닝

여러 대체값을 체이닝할 수 있어요:

```twig
{{title ?? headline ?? "No title"}}
```

이렇게 하면 `title`이 있으면 `title`을, 없으면 `headline`을, 그것도 없으면 문자열 `"No title"`을 사용해요.

### 필터와 함께 사용

필터는 `??`보다 더 강하게 바인딩되므로, 대체값 확인 전에 필터가 먼저 적용돼요:

```twig
{{title|upper ?? "UNTITLED"}}
```

이것은 먼저 `title`에 `upper`를 적용한 다음, 결과가 비어 있으면 `"UNTITLED"`로 대체해요. 대체값에 필터를 적용하려면 괄호나 별도의 표현식을 사용하세요:

```twig
{{title ?? "Untitled"|lower}}
```

이렇게 하면 `title`이 있으면 `title`을 사용하고, 없으면 대체값에 `lower`를 적용하여 `"untitled"`이 돼요.

## 반복문

`{% for %}`를 사용하여 배열을 반복할 수 있어요:

```twig
{% for item in schema:author %}
- {{item.name}}
{% endfor %}
```

### 반복 소스

다음을 반복할 수 있어요:
- 스키마 배열: `{% for item in schema:author %}`
- 선택자 결과: `{% for comment in selector:.comment %}`
- 이전에 설정한 변수: `{% set items = selector:.item %}{% for item in items %}`

### 반복 변수

반복문 내에서 다음 속성을 가진 `loop` 객체에 접근할 수 있어요:

| 변수 | 설명 |
|----------|-------------|
| `loop.index` | 현재 반복 횟수 (1부터 시작) |
| `loop.index0` | 현재 반복 횟수 (0부터 시작) |
| `loop.first` | 첫 번째 반복이면 `true` |
| `loop.last` | 마지막 반복이면 `true` |
| `loop.length` | 총 항목 수 |

```twig
{% for tag in tags %}
{{loop.index}}. {{tag}}
{% if loop.last %} (end of list){% endif %}
{% endfor %}
```

하위 호환성을 위해 `item_index`(여기서 `item`은 반복자 변수 이름)를 사용하여 0부터 시작하는 위치를 가져올 수도 있어요:

```twig
{% for tag in tags %}
{{tag_index}}. {{tag}}
{% endfor %}
```

### 인덱스로 배열 항목 접근

대괄호 표기법을 사용하여 인덱스로 배열 요소에 접근할 수 있어요:

```twig
{{items[0]}}
{{items[loop.index0]}}
```

이는 여러 배열의 항목을 병렬로 접근해야 할 때 유용해요:

```twig
{% set transcripts = selector:.transcript-text %}
{% set timestamps = selector:.timestamp %}

{% for line in transcripts %}
{{timestamps[loop.index0]}} - {{line}}
{% endfor %}
```

대괄호 표기법은 객체 속성에서도 작동해요:

```twig
{{user["name"]}}
{{data["my-key"]}}
```

### 중첩 반복문

복잡한 데이터 구조를 위해 반복문을 중첩할 수 있어요:

```twig
{% for section in sections %}
## {{section.title}}
{% for item in section.items %}
- {{item}}
{% endfor %}
{% endfor %}
```

## 로직 결합

조건문과 반복문을 결합할 수 있어요:

```twig
{% for item in items %}
{% if item.active %}
- {{item.name}}
{% endif %}
{% endfor %}
```

## 평가 순서

템플릿 로직은 다음 순서로 처리돼요:

1. **템플릿 로직** — `{% if %}`, `{% for %}`, `{% set %}`, `{{variables}}`가 먼저 평가돼요
2. **프롬프트 변수** — `{{"summarize this"|prompt}}`와 같은 [[변수#프롬프트 변수|프롬프트 변수]]는 템플릿 로직이 완료된 후 인터프리터로 전송돼요

이는 템플릿 로직을 사용하여 프롬프트를 동적으로 구성할 수 있지만, 프롬프트 결과는 조건문이나 반복문에서 사용할 수 없다는 것을 의미해요.
