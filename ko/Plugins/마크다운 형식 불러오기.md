마크다운 형식 불러오기는 다른 애플리케이션의 Markdown을 Obsidian 형식으로 변환할 수 있는 [[코어 플러그인|코어 플러그인]]이에요. 또한 특정 [[속성]]을 새로운 필수 형식으로 변환할 수도 있어요.

> [!warning] 경고
> 마크다운 형식 불러오기는 설정에 따라 전체 보관함을 변환해요. 변환을 수행하기 전에 [[Obsidian 파일 백업|Obsidian 파일을 백업]]하세요.

보관함의 모든 노트를 변환하려면:

1. [[명령어 팔레트]]에서 **마크다운 형식 불러오기 열기**를 선택해요. 이것은 [[리본]]에서 **마크다운 형식 불러오기 열기** ( ![[lucide-binary.svg#icon]] ) 아이콘으로도 찾을 수 있어요.
2. 변환하려는 서식을 활성화해요.
3. **변환 시작**을 클릭해요.

자세한 정보는 [[기본 서식 구문]]을 참조하세요.

## 지원되는 형식

### Roam Research

마크다운 형식 불러오기는 다음 Roam Research 구문을 변환할 수 있어요:

- **태그**: `#tag` 및 `#[[tag]]`를 `[[tag]]`로 변환
- **강조 표시**: `^^highlight^^`를 `==highlight==`로 변환
- **TODO 항목**: `{{[[TODO]]}}`를 `[ ]`로 변환

### Bear

마크다운 형식 불러오기는 다음 Bear 구문을 변환할 수 있어요:

- **강조 표시**: `::highlight::`를 `==highlight==`로 변환

### Zettelkasten

마크다운 형식 불러오기는 다음 Zettelkasten 구문을 변환할 수 있어요:

- **전체 링크**: `[[UID]]`를 `[[UID File Name]]`으로 변환
- **표시 링크**: `[[UID]]`를 `[[UID File Name|File Name]]`으로 변환

### [[속성]]

Obsidian `1.9.3`부터 마크다운 형식 불러오기는 [[속성#사용 중단된 속성|사용 중단된 속성]] 형식을 현재 형식으로 변환할 수 있어요:

**별칭**

```yaml
# 변환 전

alias: My Note Title

# 변환 후

aliases:
  - My Note Title
```

**태그**

```yaml
# 변환 전

tag: project, important

# 변환 후

tags:
  - project
  - important
```

**CSS 클래스**

```yaml
# 변환 전

cssclass: custom-style

# 변환 후

cssclasses:
  - custom-style
```
