검색은 검색어와 연산자를 사용하여 결과를 좁혀 Obsidian 보관함에서 데이터를 찾을 수 있도록 도와주는 [[코어 플러그인|코어 플러그인]]입니다.

기본적으로 왼쪽 사이드바( ![[lucide-search.svg#icon]] )에서 검색을 찾을 수 있습니다. `Ctrl+Shift+F`(Windows/Linux) 또는 `Command+Shift+F`(macOS)를 눌러 검색을 열 수도 있습니다.

- **선택한 텍스트 검색**: 편집기에서 텍스트를 선택하고 키보드 단축키로 검색을 열면, 선택한 텍스트에 대한 검색 결과가 표시됩니다.
- **최근 검색어 검색**: 빈 검색어로 검색을 열면 최근 검색어 목록이 표시됩니다. 아무 항목이나 클릭하면 해당 검색어를 다시 사용할 수 있습니다.

> [!info] 제외하는 파일
> [[설정#제외하는 파일|제외하는 파일]] 패턴과 일치하는 파일은 검색 결과에 나타나지 않습니다.

## 검색어

검색어는 검색 필드에 입력하는 단어 또는 구문입니다. 검색어를 효과적으로 작성하는 방법을 배우면 대규모 보관함에서도 원하는 것을 빠르게 찾을 수 있습니다. Obsidian은 노트와 캔버스의 내용만 검색합니다.

> [!tip]- 경로 및 파일 이름 검색
> 기본적으로 노트와 캔버스의 경로 및 파일 이름만 검색할 수 있습니다. 보관함 내 모든 파일의 경로 또는 파일 이름을 검색하려면 `path` 또는 `file` 연산자를 사용하세요.

검색어의 각 단어는 각 파일 내에서 독립적으로 매칭됩니다. 정확한 구문을 검색하려면 따옴표로 감싸세요. 예: `"star wars"`. 정확한 구문 내에서 따옴표가 포함된 텍스트를 검색하려면 따옴표 앞에 백슬래시(`\`)를 추가하여 _이스케이프_ 할 수 있습니다. 예: `"they said \"hello\" to each other"`.

검색어의 _모든_ 단어를 포함하는 파일을 반환할지, _어떤_ 단어든 포함하는 파일을 반환할지 제어할 수 있습니다:

- `meeting work`는 `meeting`과 `work`를 모두 포함하는 파일을 반환합니다.
- `meeting OR work`는 `meeting` 또는 `work`를 포함하는 파일을 반환합니다.

동일한 검색어에서 두 가지를 결합할 수도 있습니다.

- `meeting work OR meetup personal`은 업무 미팅과 개인 모임에 관한 파일을 반환합니다.

괄호를 사용하여 각 표현식의 우선순위를 제어할 수 있습니다.

- `meeting (work OR meetup) personal`은 `meeting`, `personal`, 그리고 `work` 또는 `meetup` 중 하나를 포함하는 파일을 반환합니다.

검색 결과에서 단어를 제외하거나 부정하려면 앞에 하이픈(`-`)을 추가하세요:

- `meeting -work`는 `meeting`을 포함하지만 `work`는 포함하지 않는 파일을 반환합니다.

여러 표현식을 제외할 수 있습니다:

- `meeting -work -meetup`은 `meeting`을 포함하지만 `work` 또는 `meetup`은 포함하지 않는 파일을 반환합니다.

괄호를 사용하여 표현식 조합을 제외할 수 있습니다:

- `meeting -(work meetup)`은 `meeting`을 포함하지만 `work`와 `meetup`을 _모두_ 포함하지는 않는 파일을 반환합니다.

보다 작음(`<`) 및 보다 큼(`>`) 연산자를 사용하여 결과를 필터링하려면 대괄호(`[]`) 또는 따옴표(`""`)로 감싸세요:

- `meeting [duration:<5]`는 meeting이 있고 duration이 5보다 작은 파일을 반환합니다.
- `meeting [duration:>5]`는 meeting이 있고 duration이 5보다 큰 파일을 반환합니다.

> [!tip]- 검색어 설명
> 복잡한 검색어의 문제를 해결해야 하는 경우, 검색에서 **검색어 설명**을 클릭하면 검색어에 대한 설명을 볼 수 있습니다.

## 검색 연산자

검색 연산자를 사용하면 더 세밀한 검색어로 결과를 더욱 필터링할 수 있습니다.

일부 연산자는 괄호 안에 중첩 검색어를 추가할 수도 있습니다. 예: `task:(call OR email)`.

| 검색 연산자 | 설명 |
| --------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `file:`         | 파일 이름에서 텍스트를 찾습니다. 보관함의 모든 파일과 매칭됩니다.<p/>예: `file:.jpg` 또는 `file:202209`.                                                                                                                                                                                                                    |
| `path:`         | 파일 경로에서 텍스트를 찾습니다. 보관함의 모든 파일과 매칭됩니다.<p/>예: `path:"Daily notes/2022-07"`.                                                                                                                                                                                                                   |
| `content:`      | 파일 내용에서 텍스트를 찾습니다.<p/>예: `content:"happy cat"`.                                                                                                                                                                                                                                                      |
| `match-case:`   | 대소문자를 구분하여 매칭합니다.<p/>예: `match-case:HappyCat`.                                                                                                                                                                                                                                                           |
| `ignore-case:`  | 대소문자를 구분하지 않고 매칭합니다.<p/>예: `ignore-case:ikea`.                                                                                                                                                                                                                                                            |
| `tag:`          | 파일에서 태그를 찾습니다.<p/>예: `tag:#work`.<p/>`tag:#work`를 검색하면 `#myjob/work`에 대한 결과는 반환되지 않는다는 점에 유의하세요.<br /><br />**참고**: `tag:`는 코드 블록 및 Markdown이 아닌 콘텐츠의 매칭을 무시하므로, `#work`에 대한 일반 전체 텍스트 검색보다 더 빠르고 정확한 경우가 많습니다. |
| `line:`         | 최소 한 줄이 `x`와 매칭되는 파일을 찾습니다.<p/>예: `line:(mix flour)`.<p/><br>**참고:** `-line`을 사용하면 검색을 부정하여, 어떤 줄도 `x`와 매칭되지 않는 파일을 찾습니다.                                                                                                                   |
| `block:`        | 같은 블록 내에서 매칭을 찾습니다.<p/>예: `block:(dog cat)`.<p/>**참고**: `block:`는 모든 파일의 Markdown 콘텐츠를 파싱해야 하므로, 검색어 처리에 더 오랜 시간이 걸릴 수 있습니다.                                                                                               |
| `section:`      | 같은 섹션(두 제목 사이의 텍스트) 내에서 매칭을 찾습니다.<p/>예: `section:(dog cat)`.                                                                                                                                                                                                                     |
| `task:`         | [[기본 서식 구문#작업 목록\|작업]]에서 블록 단위로 매칭을 찾습니다.<p/>예: `task:call`.                                                                                                                                                                                                 |
| `task-todo:`    | *미완료* [[기본 서식 구문#작업 목록\|작업]]에서 블록 단위로 매칭을 찾습니다.<p/>예: `task-todo:call`.                                                                                                                                                                             |
| `task-done:`    | *완료된* [[기본 서식 구문#작업 목록\|작업]]에서 블록 단위로 매칭을 찾습니다.<p/>예: `task-done:call`.                                                                                                                                                                                |

## 속성 검색

검색어에서 [[속성]]에 저장된 데이터를 사용할 수 있습니다.

속성 이름을 대괄호로 감싸서 `[property]` 해당 속성이 있는 파일을 반환합니다:

- `[aliases]`는 `aliases` 속성을 포함하는 파일을 반환합니다

대괄호와 콜론 `[property:value]`을 사용하여 해당 속성과 값이 있는 파일을 반환합니다:

- `[aliases:Name]`은 `aliases` 속성 값이 `Name`인 파일을 반환합니다

값으로 `null`을 사용하면 값이 없는 속성을 찾을 수 있습니다:

- `[aliases:null]`은 `aliases` 속성이 존재하지만 값이 없는 파일을 반환합니다

> [!info]+ 빈 값
> `null` 연산자는 속성이 비어 있을 때(예: `aliases: `) 작동하지만, 속성에 빈 따옴표(`""`)나 빈 대괄호(`[]`)가 포함된 경우에는 작동하지 않습니다.

속성과 값 모두 그룹화를 위한 괄호, `OR` 연산자, 정확한 매칭을 위한 큰따옴표, 정규식과 같은 하위 쿼리를 허용합니다.

- `[status:Draft OR Published]`는 `status` 속성 값이 `Draft` 또는 `Published`인 파일을 반환합니다

## 대소문자 구분 변경

기본적으로 검색어는 대소문자를 구분하지 않습니다. 검색어의 정확한 대소문자를 검색하려면 검색 바 내의 **대소문자 구분**( ![[obsidian-icon-upper-lowercase.svg#icon]] )을 선택하세요.

이 설정은 전환할 수 있습니다. **대소문자 구분** 아이콘이 강조 표시되어 있으면 현재 대소문자를 구분하여 검색하고 있다는 의미입니다.

## 결과 정렬 순서 변경

1. [[#검색어|검색어]]를 입력합니다.
2. 검색 필드 아래에서 오른쪽의 드롭다운을 선택합니다.
3. 원하는 정렬 순서를 선택합니다. 기본값은 "파일 이름 (알파벳순)"입니다.

다음 옵션을 사용할 수 있습니다:

- 파일 이름 (알파벳순)
- 파일 이름 (알파벳 역순)
- 업데이트 날짜 (최신순)
- 업데이트 날짜 (오래된 순)
- 생성일 (최신순)
- 생성일 (오래된 순)

## 정규식 사용

정규식은 텍스트 패턴을 설명하는 문자 집합입니다. 검색어에서 정규식을 사용하려면 표현식을 슬래시(`/`)로 감싸세요.

- `/\d{4}-\d{2}-\d{2}/`는 2022-01-01과 같은 ISO 8601 날짜와 매칭됩니다.

정규식을 검색 연산자와 결합할 수도 있습니다:

- `path:/\d{4}-\d{2}-\d{2}/`는 파일 경로에 날짜가 포함된 파일을 반환합니다.

정규식 작성 방법에 대한 자세한 내용은 FreeCodeCamp의 [실용 정규식 가이드](https://www.freecodecamp.org/news/practical-regex-guide-with-real-life-examples/) 또는 Mozilla의 [정규 표현식](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Guide/Regular_Expressions)을 참조하세요.

> [!info]+ JavaScript 기반 정규식
> 정규식에는 서로 다르게 보일 수 있는 다양한 종류가 있습니다. Obsidian은 JavaScript 기반 정규식을 사용합니다.

## 검색 설정 구성

검색을 설정하려면 검색 바 오른쪽의 **검색 설정**( ![[lucide-sliders-horizontal.svg#icon]] )을 선택하여 토글을 확인하세요.

| 설정                 | 설명                                                                 |
|-------------------------|-----------------------------------------------------------------------------|
| **검색어 설명** | 검색어를 분해하여 일반 텍스트로 설명합니다.                 |
| **결과 더 작게 표시**    | 검색 컨텍스트를 표시할지 여부를 전환합니다.                                 |
| **내용 더 길게 표시하기**   | 매칭 주변의 더 많은 텍스트를 표시하도록 검색 결과를 확장합니다.               |

## 노트에 검색 결과 임베드

노트에 검색 결과를 임베드하려면 `query` 코드 블록을 추가하세요:

````
```query
embed OR search
```
````

[[Obsidian Publish 소개|Obsidian Publish]]는 임베디드 [[Publish 제한 사항#검색|검색 결과]]를 지원하지 않습니다. 실시간 렌더링 예시를 보려면 보관함 내에서 위의 코드 블록을 사용하세요.
