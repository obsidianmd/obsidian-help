설정을 통해 Obsidian 경험을 사용자 지정할 수 있습니다. 일반 옵션, 편집기 환경설정, 외형, 키보드 단축키를 구성하고 코어 플러그인과 커뮤니티 플러그인을 관리할 수 있습니다.

## 설정 열기

### 데스크톱

[[사이드바#숨겨진 사이드바 열기|왼쪽 사이드바]]에서 **[[설정]]**( ![[lucide-cog.svg#icon]] )을 선택합니다. [[명령어 팔레트]]를 사용하여 설정을 열 수도 있습니다.

### 모바일

왼쪽 사이드바에서 **[[설정]]**( ![[lucide-cog.svg#icon]] )을 선택합니다.

## 설정 구성

설정은 다음 카테고리로 구성됩니다:

- **옵션** — [[#일반]], [[#편집기]], [[#파일 및 링크]], [[#외형]], [[#단축키]]를 포함한 핵심 앱 설정.
- **[[코어 플러그인]]** — Obsidian에 기본 포함된 플러그인.
- **[[커뮤니티 플러그인]]** — 커뮤니티에서 설치한 타사 플러그인.

## 일반

일반 페이지에는 버전 정보, 업데이트 설정, 언어 환경설정, 계정 관리가 포함되어 있습니다.

### 버전 및 업데이트

페이지 상단에서 현재 Obsidian 버전과 설치 프로그램 버전을 확인할 수 있습니다. **업데이트 확인**을 선택하여 수동으로 새 버전을 확인하거나, **[변경 로그 읽기](<https://obsidian.md/changelog/>)**를 선택하여 릴리즈 노트를 확인하세요.

**자동 업데이트**

활성화하면 Obsidian이 자동으로 새 버전을 확인하고 업데이트가 가능할 때 알려줍니다.

**얼리 액세스 버전 가져오기**

최신 [[얼리 액세스 버전]]으로 자동 업데이트합니다. 이 버전에는 새로운 기능이 포함되어 있지만 안정성이 떨어질 수 있습니다.

> [!info]+ Catalyst 라이선스 필요
> 이 옵션은 [[Catalyst 라이선스]]를 보유한 사용자만 사용할 수 있습니다.

### 언어

Obsidian 인터페이스에 사용할 언어를 선택합니다. **[[번역|Obsidian에 새 언어를 추가하는 방법 알아보기]]**를 선택하여 번역에 기여할 수 있습니다.

### 도움말

도움말 리소스(문서, 커뮤니티 포럼, 문제 해결 가이드 포함)에 접근하려면 **[[홈|열기]]**를 선택합니다.

### 계정

Obsidian 계정과 라이선스를 관리합니다.

**내 계정**

사용자 이름과 이메일을 포함한 계정 정보를 확인합니다. **[관리](https://obsidian.md/account)**를 선택하여 계정 설정에 접근하거나, **로그아웃**을 선택하여 앱에서 로그아웃합니다.

**Catalyst 라이선스**

Catalyst 라이선스 상태를 확인합니다. 활성 [[Catalyst 라이선스]]가 있는 경우 여기에 라이선스 등급이 표시됩니다.

**상용 라이선스**

[업무용 Obsidian](<https://obsidian.md/blog/free-for-work/>)을 지원하는 경우, **인증**을 선택하여 상용 라이선스 키를 입력하거나, **[구매](https://obsidian.md/account/commercial)**를 선택하여 라이선스를 구매하세요. 상용 라이선스에 대한 정보는 **[[상용 라이선스|자세히 알아보기]]**를 선택하세요.

### 고급 설정

#### 앱 시작이 예상보다 느려질 경우 알리기

활성화하면 시작이 비정상적으로 느린 경우 Obsidian이 알림을 표시합니다. 알림에는 지연 원인에 대한 진단 정보가 포함됩니다. 이 설정 옆의 타이머 아이콘( ![[lucide-timer.svg#icon]] )을 선택하여 [[도움말 및 지원#시작 시간 확인|시작 시간을 확인]]할 수 있습니다.

## 편집기

편집기 페이지에는 노트를 보고 편집하는 방법에 대한 설정이 포함되어 있습니다.

### 항상 새 탭으로 전환

링크를 새 탭에서 열 경우, 즉시 해당 탭으로 전환합니다.

### 새 탭에 사용하는 기본 화면

새 Markdown 파일 탭이 열릴 때 사용되는 기본 보기입니다. [[뷰와 편집 모드#읽기 보기|읽기 보기]] 또는 [[뷰와 편집 모드#편집 보기|편집기]] 중에서 선택합니다.

### 기본 편집 모드

새 패널이 시작되는 기본 편집 모드. [[뷰와 편집 모드#실시간 미리보기|실시간 미리보기]] 또는 [[뷰와 편집 모드#원본 모드|원본 모드]] 중에서 선택합니다.

### 편집기 상태 보기

[[상태 표시줄|상태 표시줄]]에 편집 모드 전환 버튼을 표시합니다.

### 표시

#### 읽기 쉬운 행 길이

최대 줄 길이를 제한합니다. 화면에 맞는 콘텐츠는 줄어들지만 긴 단락을 더 쉽게 읽을 수 있게 됩니다.

#### 엄격한 줄 바꿈

Markdown 사양은 읽기 보기에서 단일 [[기본 서식 구문#줄 바꿈|줄 바꿈]]을 무시합니다. 단일 줄 바꿈을 표시하려면 이 옵션을 끄세요.

#### 문서 내 속성

노트 상단에 [[속성|속성]]이 표시되는 방식을 선택합니다. **원본**을 선택하면 속성이 원시 YAML로 표시됩니다.

#### 제목 접기

제목 옆의 접기 아이콘을 선택하여 제목 아래의 모든 콘텐츠를 [[접기|접습니다]].

### 들여쓰기 접기

목록과 같은 들여쓰기 부분을 접기 아이콘을 선택하여 [[접기|접습니다]].

#### 줄 번호

여백에 행 번호를 표시합니다.

#### 들여쓰기 가이드

시각적 안내를 위해 글머리 기호 사이의 수직 관계선을 표시합니다.

#### 오른쪽-왼쪽 방향 (RTL)

텍스트를 오른쪽에서 왼쪽 방향으로 표시하도록 설정합니다.

#### 괄호 자동 닫기

괄호와 따옴표를 자동으로 닫습니다.

#### 마크다운 문법 자동 닫기

볼드체, 이탤릭체, 하이라이트, 취소선, 코드에 사용되는 영역을 자동으로 닫아줍니다.

#### 스마트 들여쓰기

들여쓰기와 목록들을 앱이 알아서 정리해줍니다.

#### 들여쓰기로 탭 사용

Tab 키를 눌러 탭으로 들여쓰기합니다. 이 옵션을 끄면 4개의 공백으로 들여쓰기합니다.

#### HTML 자동 변환

웹 페이지에서 붙여넣기 및 드래그 앤 드롭할 때 HTML을 자동으로 Markdown으로 변환합니다.

변환 없이 HTML을 붙여넣으려면 `Ctrl/Cmd+Shift+V`를 사용하세요.

### 동작

#### 맞춤법 검사

맞춤법 검사기를 사용하도록 설정합니다. 기어 아이콘을 선택하여 사용자 지정 사전에서 저장된 항목을 제거할 수 있습니다.

#### 맞춤법 검사 언어

**Windows 및 Linux:**

맞춤법 검사기에 사용할 언어를 선택하세요. 더하기(+) 아이콘을 선택하여 언어를 추가합니다. 언어를 제거하려면 옆의 X를 선택합니다.

**macOS:**

네이티브 맞춤법 검사기가 OS에서 사용하는 언어를 자동으로 감지합니다.

#### 들여쓰기 크기

탭을 나타내는 띄어쓰기 공백 수를 결정합니다.

### 고급 설정

#### Vim 키 바인딩 설정

편집 화면에서 Vim 키 바인딩을 사용할 수 있게 해줍니다.

## 파일 및 링크

파일 및 링크 페이지에는 파일 관리, 링크, 보관함 동작에 대한 설정이 포함되어 있습니다.

### 새 노트를 생성하는 위치

새로 생성하는 노트의 위치를 결정합니다. 옵션은 다음과 같습니다:

- **보관함 최상위 폴더** — 보관함의 루트에 노트가 생성됩니다.
- **현재 파일과 동일한 폴더** — 현재 활성 노트와 동일한 폴더에 노트가 생성됩니다.
- **아래에 지정된 폴더** — 선택한 특정 폴더에 노트가 생성됩니다.

### 새 첨부 파일을 생성할 위치

새로 추가된 [[첨부 파일|첨부 파일]]이 배치되는 위치입니다. 옵션은 다음과 같습니다:

- **보관함 최상위 폴더** — 첨부 파일이 보관함의 루트에 배치됩니다.
- **아래에 지정된 폴더** — 첨부 파일이 선택한 특정 폴더에 배치됩니다.
- **현재 파일과 동일한 폴더** — 첨부 파일이 현재 활성 노트와 동일한 폴더에 배치됩니다.
- **현재 폴더의 하위 폴더** — 첨부 파일이 현재 노트 폴더 내의 하위 폴더에 배치됩니다.

### 링크

#### 새로 생성하는 링크 형식

[[내부 링크|내부 링크]]를 자동 생성할 때 삽입되는 링크 형식입니다. 옵션은 다음과 같습니다:

- **가능한 최단 경로** — 링크된 파일로의 가장 짧은 고유 경로를 사용합니다.
- **파일에 대한 상대 경로** — 현재 파일에 대한 상대 경로를 사용합니다.
- **보관함 내 절대 경로** — 보관함 루트에서의 전체 경로를 사용합니다.

#### 항상 업데이트되는 내부 링크

활성화하면 파일 이름을 변경할 때 Obsidian이 자동으로 내부 링크를 업데이트합니다. 이름 변경 후 링크 업데이트 여부를 확인받으려면 이 옵션을 끄세요.

#### 위키 링크 사용

Markdown 링크와 이미지 대신 `[[링크]]` 및 `![[이미지]]`에 대해 위키 링크를 자동 생성합니다. 이 옵션을 비활성화하면 Markdown 링크가 생성됩니다.

#### 모든 파일 형식 표시

Obsidian이 기본적으로 열지 못하는 파일들이라도 파일 목록에 포함하게 합니다. 파일을 링크하고 파일 탐색기나 빠른 전환기로 확인할 때 유용합니다.

### 휴지통

#### 파일 삭제 확인

파일을 삭제하기 전에 확인합니다.

#### 삭제된 파일

삭제하는 파일들을 처리할 방법. 옵션은 다음과 같습니다:

- **시스템 휴지통으로 이동** — 파일이 운영체제의 휴지통으로 이동됩니다.
- **Obsidian 휴지통으로 이동** — 파일이 보관함의 `.trash` 폴더로 이동됩니다.
- **영구적으로 삭제** — 파일이 영구적으로 삭제되며 복원할 수 없습니다.

### 고급 설정

#### 제외하는 파일

제외된 파일은 [[검색]], [[그래프 뷰]], 링크되지 않은 호출([[백링크]] 및 [[나가는 링크]])에서 숨겨지며, [[빠른 전환기]]와 링크 제안에서 덜 눈에 띄게 됩니다. **관리**를 선택하여 제외할 파일을 설정합니다.

#### 설정 폴더 우선순위 전환

기본 [[설정 폴더|설정 폴더]] 대신 다른 폴더를 사용합니다. 반드시 점으로 시작해야 합니다.

#### URI 콜백 허용

[[Obsidian URI|Obsidian URI]]를 처리할 때 x-success 또는 x-error를 통한 x-callback-url 사용을 활성화합니다.

#### 보관함 캐시 다시 빌드하기

보관함의 [[Obsidian의 데이터 저장 방식#메타데이터 캐시|메타데이터 캐시]]를 다시 빌드합니다. 보관함의 크기에 따라 캐시를 다시 빌드하는 작업은 몇 초부터 몇 분까지 걸릴 수 있습니다. **다시 빌드하기**를 선택하여 프로세스를 시작합니다.

## 외형

외형 페이지에는 Obsidian의 모양을 사용자 지정하는 설정이 포함되어 있습니다.

### 기본 테마

Obsidian의 기본 테마를 선택합니다. 옵션은 다음과 같습니다:

- **시스템 테마 적용** — 운영체제의 라이트 또는 다크 모드와 일치시킵니다.
- **라이트** — 라이트 모드를 사용합니다.
- **다크** — 다크 모드를 사용합니다.

### 강조 색상

앱 전체에서 사용되는 강조 색상을 선택합니다. 색상 선택기 또는 초기화 아이콘을 선택하여 사용자 지정합니다.

### 테마

설치된 [[테마|테마]]를 관리하고 커뮤니티 테마를 탐색합니다. **관리**를 선택하여 테마를 보고 설치합니다. 드롭다운에 현재 활성화된 테마가 표시됩니다. 폴더 아이콘( ![[lucide-folder-open.svg#icon]] )을 선택하면 [[설정 폴더|설정 폴더]] 내의 테마 폴더가 열립니다.

### 현재 커뮤니티 테마

설치된 커뮤니티 테마 수를 표시합니다.

### 폰트

#### 인터페이스 폰트

Obsidian 전체에서 사용할 기본 폰트를 선택합니다. **관리**를 선택하여 커스텀 폰트를 선택하거나 폰트 이름을 입력합니다.

#### 텍스트 폰트

편집 화면 및 읽기 화면에서 사용할 폰트를 설정합니다. **관리**를 선택하여 커스텀 폰트를 선택하거나 폰트 이름을 입력합니다.

#### 모노스페이스 폰트

코드 블럭이나 프런트매터 등의 도구들을 사용할 때 적용되는 폰트를 설정합니다. **관리**를 선택하여 커스텀 폰트를 선택하거나 폰트 이름을 입력합니다.

#### 폰트 크기

픽셀 단위로 표기되는 폰트 크기는 편집 화면 및 읽기 화면에 영향을 줍니다. 슬라이더를 사용하여 조정합니다.

#### 빠른 폰트 크기 조정

`Ctrl+스크롤`(Windows/Linux) 또는 `Cmd+스크롤`(macOS), 또는 트랙패드 핀치 줌 제스처를 사용하여 폰트 크기를 조정합니다.

### 인터페이스

#### 인라인 제목

파일 내용과 함께 파일 이름을 편집 가능한 제목으로 인라인 표시합니다. 인라인 제목은 Obsidian 내에서 제목으로 작동하지만 노트에 제목을 추가하지는 않습니다.

#### 탭 타이틀바 표시

모든 탭 상단에 헤더를 표시합니다.

#### 리본 표시하기

창 측면에 세로로 긴 툴바를 표시합니다. [[리본|리본]]에 대해 자세히 알아보세요.

#### 리본 메뉴 설정

내비게이션바에 위치한 리본 메뉴에 어떤 명령어들을 표시할지 설정합니다. **관리**를 선택하여 사용자 지정합니다.

> [!info]+ 모바일 리본 설정
> 모바일에서는 리본 사용자 지정이 다르게 작동합니다. 자세한 내용은 [[리본#모바일|모바일에서의 리본 사용자 지정]]을 참조하세요.

### 고급 설정

#### 배율

앱의 전반적인 배율을 조정합니다. 슬라이더를 사용하여 조정합니다.

#### 네이티브 메뉴

앱 전체에서 사용되는 메뉴를 운영체제와 일치시킵니다. 일치된 메뉴는 테마의 영향을 받지 않습니다.

#### 창 프레임 스타일

Obsidian 창의 제목 표시줄 스타일을 결정합니다. 적용하려면 완전히 다시 시작해야 합니다. 옵션은 다음과 같습니다:

- **Obsidian 프레임** — Obsidian의 커스텀 창 프레임을 사용합니다.
- **네이티브 프레임** — 운영체제의 기본 창 프레임을 사용합니다.
- **숨김 프레임** — 제목 표시줄을 완전히 숨깁니다.

#### 커스텀 앱 아이콘

앱에 사용할 커스텀 아이콘을 설정합니다. **선택**을 선택하여 아이콘 파일(`.icns`, `.ico`, `.png` 또는 `.svg`)을 지정합니다. [[외형#커스텀 앱 아이콘|커스텀 앱 아이콘]]에 대해 자세히 알아보세요.

#### 반투명 창

반투명 효과를 사용하여 깊이감을 더해줍니다. 다크 모드에서 사용하기에 이상적입니다. Linux 환경에서는 지원되지 않습니다.

> [!info]+ macOS 전용
> 이 기능은 주로 macOS를 위해 설계되었습니다. [[외형#반투명|반투명]]에 대해 자세히 알아보세요.

#### 하드웨어 가속

하드웨어 가속을 활성화하고 Obsidian의 동작을 더욱 가볍게 하기 위해 GPU를 사용합니다. 이 옵션을 끄면 앱 성능이 심각하게 저하될 수 있지만 일부 드문 문제를 해결할 수 있습니다.

### CSS 스니펫

보관함에 저장된 [[CSS 스니펫|CSS 스니펫]]을 관리하고 확인합니다. CSS 스니펫은 `/vault/.obsidian/snippets/`에 저장됩니다. 새로고침 아이콘을 선택하여 스니펫을 다시 불러오거나, 폴더 아이콘( ![[lucide-folder-open.svg#icon]] )을 선택하여 스니펫 폴더를 엽니다.

## 단축키

[[단축키]] 페이지에서는 Obsidian의 명령에 대한 [[편집 단축키|키보드 단축키]]를 확인, 설정, 사용자 지정할 수 있습니다.

검색 바를 사용하여 명령을 필터링하고, 필터 아이콘을 선택하여 키보드 단축키가 할당된 명령만 표시합니다. 키보드 단축키를 추가하려면 명령 옆의 더하기(+) 아이콘을 선택합니다. 키보드 단축키를 제거하려면 키 조합 옆의 X 아이콘을 선택합니다.

## 코어 플러그인

[[코어 플러그인]]은 Obsidian에 기본 포함된 플러그인입니다. 각 플러그인은 앱에 특정 기능을 추가합니다.

코어 플러그인을 활성화하거나 비활성화하려면 플러그인 이름 옆의 토글을 선택합니다. 더하기( ![[lucide-plus-circle.svg#icon]] ) 아이콘을 선택하여 플러그인의 [[단축키]]를 확인합니다. 설정( ![[lucide-settings.svg#icon]] ) 아이콘을 선택하여 플러그인의 설정 페이지를 엽니다.

검색 바를 사용하여 이름으로 플러그인을 필터링합니다.

## 커뮤니티 플러그인

[[커뮤니티 플러그인]]은 Obsidian 커뮤니티에서 만든 타사 플러그인입니다. **탐색**을 선택하여 플러그인을 찾아보고 설치합니다.

### 제한 모드

[[플러그인 보안#제한 모드|제한 모드]]는 보안을 위해 모든 커뮤니티 플러그인을 비활성화합니다. **활성화**를 선택하여 제한 모드를 활성화합니다.

### 현재 설치된 플러그인

설치된 커뮤니티 플러그인 수를 표시합니다. **업데이트 확인**을 선택하여 플러그인 업데이트를 확인합니다.

### 설치된 플러그인

설치된 커뮤니티 플러그인을 확인하고 관리합니다. 검색 바를 사용하여 이름으로 플러그인을 필터링합니다. 각 플러그인에는 설정, 키보드 단축키, 후원, 삭제를 위한 아이콘과 활성화/비활성화 토글이 있습니다.
