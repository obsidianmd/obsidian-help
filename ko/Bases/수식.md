수식을 사용하면 [[베이스 소개|베이스]]에서 다른 [[속성]]의 데이터를 활용하여 계산된 속성을 만들 수 있어요. 계산 수행, 텍스트 조작, 날짜 작업 등이 가능해요.

## 수식으로 할 수 있는 것

수식은 다음과 같은 작업에 도움이 돼요:

- **값 계산**, 가격 합산, 총계 계산 또는 수학 연산 수행.
- **텍스트 조작**, 문자열 결합, 대소문자 변경 또는 부분 문자열 추출.
- **날짜 작업**, 시간 차이 계산, 날짜 서식 지정 또는 마감일 결정.
- **로직 적용**, 조건문을 사용하여 다른 값 표시.
- **목록 처리**, 목록 데이터 필터링, 정렬, 매핑 또는 집계.

## 수식 속성 생성

수식 속성을 생성하려면:

1. 베이스에서 툴바의 **속성**을 클릭해요.
2. 메뉴 하단의 **수식 추가하기**를 클릭해요.
3. 수식 속성의 이름을 입력해요.
4. **수식** 필드에 수식을 입력해요.
5. 대화 상자를 닫아요.

수식 편집기는 입력하는 동안 [[함수|함수]]와 속성 이름을 자동 완성하여 수식 구문을 검증해요. 수식이 유효하면 녹색 체크 표시가 나타나요.

생성된 수식 속성은 베이스의 다른 속성과 동일하게 사용할 수 있어요. [[보기|보기]]에 추가하거나, 필터에 사용하거나, 정렬 기준으로 사용하는 등 다양하게 활용할 수 있어요.

## 수식 작성

수식 편집기에서 속성, 연산자, 함수를 사용하여 표현식을 입력해요.

### 속성 참조

수식에서 다양한 유형의 속성을 참조할 수 있어요:

- **노트 속성** — 노트의 [[속성|프런트매터]]에 있는 속성.
- **파일 속성** — `file.name`, `file.size`, `file.mtime` 같은 내장 속성.
- **수식 속성** — `formula.formula_name`을 사용하는 다른 수식.

**예시:**

- `price * quantity` — 두 노트 속성의 곱셈
- `file.name + " - " + description` — 파일 이름과 노트 속성의 결합
- `formula.price_per_unit * 1.1` — 다른 수식 속성 사용

### 연산자 사용

**산술 연산자**는 숫자에 대해 수학 연산을 수행해요:

- `price + tax` — 덧셈
- `price - discount` — 뺄셈
- `price * quantity` — 곱셈
- `price / quantity` — 나눗셈
- `(part / whole) * 100` — 괄호를 사용한 연산 순서 지정

**비교 연산자**는 값을 비교해요:

- `price > 100` — 초과
- `age < 18` — 미만
- `status == "Done"` — 같음
- `status != "Done"` — 같지 않음
- `file.mtime > now() - '7d'` — 날짜 비교

**불리언 연산자**는 논리 조건을 결합해요:

- `!completed` — 부정
- `price > 0 && quantity > 0` — and
- `urgent || important` — or

[[베이스 구문#연산자|베이스 구문]]에서 자세히 알아보세요.

### 함수 사용

함수는 값에 대해 연산을 수행해요. 사용 가능한 함수는 작업하는 값의 유형에 따라 달라요. [[함수]]에서 전체 목록을 확인하세요.

**주요 함수 카테고리:**

- **전역 함수** — `if()`, `now()`, `date()`, `link()`, `max()`, `min()`
- **문자열 함수** — `contains()`, `replace()`, `split()`, `lower()`, `title()`
- **숫자 함수** — `round()`, `ceil()`, `floor()`, `abs()`, `toFixed()`
- **날짜 함수** — `format()`, `relative()`, `date()`, `time()`
- **목록 함수** — `filter()`, `map()`, `sort()`, `join()`, `unique()`

**예시:**

- `if(price, "$" + price.toFixed(2), "")` 조건문과 숫자 서식을 정의해요.
- `file.name.lower()` 소문자로 변환해요.
- `tags.contains("urgent")` 태그 목록에 특정 값이 포함되어 있는지 확인해요.
- `due_date.format("YYYY-MM-DD")` 날짜 서식을 지정해요.

## 수식 예시

### 마감일 계산

프로젝트의 마감일을 시작일로부터 2주 후로 설정해요:

```js
start_date + "2w"
```

### 기한 초과 상태 표시

마감일이 지났고 상태가 "Done"이 아닌 경우 "Overdue"를 표시해요:

```js
if(due_date < now() && status != "Done", "Overdue", "")
```

### 통화 서식 지정

소수점 이하 2자리와 통화 기호로 가격을 표시해요:

```js
if(price, "$" + price.toFixed(2), "")
```

### 목록 항목 수 세기

목록 속성의 항목 수를 세요:

```js
tasks.length
```

### 우선순위 점수 계산

여러 요소를 결합하여 우선순위 점수를 산출해요:

```js
(impact * urgency) / effort
```

### 텍스트 필드 결합

이름과 성을 결합하여 성명을 만들어요:

```js
first_name + " " + last_name
```

### 총 비용 계산

월간 비용에 소유 월수를 곱해요:

```js
monthlyUses * formula.Owned.round()
```

## 데이터 유형

수식은 다양한 유형의 데이터를 처리해요:

- **문자열** — 따옴표로 둘러싼 텍스트: `"hello"` 또는 `'world'`
- **숫자** — 숫자 값: `42`, `3.14`, `(2 + 2)`
- **불리언** — 참 또는 거짓: `true`, `false`
- **날짜** — `date()`, `today()`, 또는 `now()`로 생성
- **목록** — 값의 모음: `[1, 2, 3]`
- **객체** — 키-값 쌍: `{"name": "value"}`

수식의 출력 유형은 사용된 데이터와 함수에 의해 결정돼요.

## 다른 수식 참조

수식은 다른 수식을 참조하여 파생 계산을 만들 수 있어요. 예를 들어 `price_per_unit`이라는 수식이 있다면:

```js
price / quantity
```

다른 수식에서 이를 참조할 수 있어요:

```js
formula.price_per_unit * 1.1
```

> [!warning] 순환 참조 방지
> 수식은 직접적으로 또는 다른 수식을 통해 간접적으로 자기 자신을 참조할 수 없어요.
