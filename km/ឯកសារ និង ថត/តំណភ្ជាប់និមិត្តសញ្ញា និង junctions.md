អ្នកអាចប្រើ [តំណភ្ជាប់និមិត្តសញ្ញា](https://en.wikipedia.org/wiki/Symbolic_link) (symlinks) និង [junctions](https://learn.microsoft.com/en-us/windows/win32/fileio/hard-links-and-junctions#junctions) នៅក្នុង vault របស់អ្នក ដើម្បីរក្សាទុកឯកសារនៅក្រៅ vault និង [[របៀបដែល Obsidian រក្សាទុកទិន្នន័យ#Global settings|ថតប្រព័ន្ធ]]។

> [!danger] ប្រើប្រាស់ដោយទទួលខុសត្រូវផ្ទាល់ខ្លួន
> យើងណែនាំយ៉ាងខ្លាំងប្រឆាំងនឹងការប្រើ symbolic links។ ការប្រើ symbolic links និង junctions នៅក្នុង vault របស់អ្នក អាចបណ្ដាលឱ្យបាត់បង់ ឬខូចទិន្នន័យ ឬ Obsidian បញ្ឈប់ការដំណើរការ។ ត្រូវប្រាកដថាអ្នកធ្វើការបម្រុងទុក vault និងការកំណត់ជាប្រចាំ។

ខាងក្រោមនេះជាដែនកំណត់ ឬបញ្ហាដែលយើងបានដឹង ដែលអ្នកប្រហែលជាចង់ចងចាំ៖

- Symlink loops មិនត្រូវបានអនុញ្ញាតទេ ដើម្បីការពារ Obsidian ពីការគាំងដោយសារ infinite loop។
- គោលដៅ Symlink ត្រូវតែបំបែកទាំងស្រុងចេញពីថតឫសដើម vault ឬគោលដៅ symlink ផ្សេងទៀត។ ការបំបែក មានន័យថា ថតមួយមិនមានថតមួយទៀត ឬផ្ទុយមកវិញ។ Obsidian មិនអើពើ symlink ទៅកាន់ថតខាងលើ vault ឬពីថតមួយក្នុង vault ទៅថតផ្សេងទៀតក្នុង vault តែមួយ។ វាជាវិធានការការពារ ដើម្បីធានាថាអ្នកមិនមានឯកសារស្ទួននៅក្នុង vault ដែលអាចបណ្ដាលឱ្យតំណភ្ជាប់មានភាពមិនច្បាស់លាស់។
- Symlinks អាចមិនសម្របសម្រួលល្អជាមួយ Obsidian Sync ឬ _សមកាលកម្មប្រភេទផ្សេងទៀត_។ ប្រសិនបើគោលដៅ symlink ខ្លួនឯងជាថតដែលត្រូវបានធ្វើសមកាលកម្មដោយ vault Obsidian ផ្សេង អ្នកអាច (ប្រហែលជា) ជួបប្រទះជម្លោះសមកាលកម្ម ឬការបាត់បង់ទិន្នន័យ។ ឧបករណ៍ sync មួយចំនួន ដូចជា Git មិនដើរតាម symlinks ទេ ប៉ុន្តែ sync _ផ្លូវ_ ដែល symlink ចង្អុលទៅ ដែលអាចបណ្ដាលឱ្យមានលទ្ធផលមិនល្អ ប្រសិនបើអ្នកចែករំលែក vault របស់អ្នកជាមួយអ្នកដទៃតាមវិធីនោះ។
- កម្មវិធីគ្រប់គ្រងឯកសាររបស់ Obsidian មិនអាចផ្លាស់ទីឯកសារឆ្លងកាត់ព្រំដែនឧបករណ៍បានទេ ដូច្នេះប្រសិនបើអ្នក symlink ទៅថតដែលនៅក្នុង drive ផ្សេងពី vault អ្នកនឹងមិនអាចអូសឯកសាររវាងថតនោះ និងថតផ្សេងទៀតដោយប្រើ [[កម្មវិធីរុករកឯកសារ]] របស់ Obsidian។ (អ្នកត្រូវប្រើ explorer របស់ OS របស់អ្នកសម្រាប់ការផ្លាស់ប្ដូរបែបនេះ ហើយ Obsidian នឹងឃើញការផ្លាស់ទីជាការលុប និងការបង្កើតឯកសារថ្មី។ វា _នឹងមិន_ អាប់ដេត តំណភ្ជាប់ណាមួយដែលពឹងផ្អែកលើផ្លូវនៃឯកសារនោះបានទេ។)
- File symlinks (ផ្ទុយពី folder symlinks) _អាច_ ដំណើរការបាន ប៉ុន្តែមិនទទួលបានការគាំទ្រជាផ្លូវការនៅពេលនេះទេ។ ការផ្លាស់ប្ដូរដែលធ្វើក្រៅ Obsidian មិនត្រូវបានតាមដានទេ ដូច្នេះប្រសិនបើអ្នកផ្លាស់ប្ដូរឯកសារដោយផ្ទាល់ Obsidian នឹងមិនរកឃើញការផ្លាស់ប្ដូរ អាប់ដេតលិបិក្រម search ជាដើម។
- ការ Symlink រឿងនានានៅក្រោម `.obsidian/` folder ដើម្បីចែករំលែកវារវាង vaults **មានឱកាសខ្ពស់ក្នុងការខូចការកំណត់របស់អ្នក** លុះត្រាតែអ្នក _ពិតជា_ ដឹងច្បាស់អំពីអ្វីដែលអ្នកកំពុងធ្វើ។ ប្រសិនបើអ្នកសម្រេចចិត្តទៅតាមផ្លូវនេះ យ៉ាងហោចណាស់ត្រូវមានការបម្រុងទុក។
