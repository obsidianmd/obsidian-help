Obsidian រក្សាទុកកំណត់ត្រាក្នុងឧបករណ៍របស់អ្នក ដើម្បីឱ្យអ្នកអាចចូលប្រើបានទាំងពេលគ្មានអ៊ីនធឺណិត។ ដើម្បីចូលប្រើកំណត់ត្រារបស់អ្នកលើឧបករណ៍ជាច្រើន អ្នកត្រូវវិធីសមកាល។

ការណែនាំនេះគ្របដណ្ដប់វិធីសមកាលទូទៅ រួមទាំងគន្លឹះដើម្បីជៀសវាងការបាត់ទិន្នន័យ និងធានាបទពិសោធន៍រលូន។

យើងក៏ណែនាំឱ្យអានការណែនាំ [[បម្រុងទុកឯកសារ Obsidian របស់អ្នក|បម្រុងទុក]] ដើម្បីការពារទិន្នន័យរបស់អ្នក។

## វិធីសមកាល

ជាមួយ Obsidian ទិន្នន័យរបស់អ្នកត្រូវបានរក្សាទុកជាឯកសារក្នុងថតដែលហៅថា [[Vault មូលដ្ឋាន និង Vault ពីចម្ងាយ|vault]]។ ដូច្នេះមានវិធីជាច្រើនដើម្បីធ្វើសមកាលទិន្នន័យ។

នេះជាវិធីសមកាលទូទៅដែលត្រូវបានរាយការណ៍ដោយសមាជិក [សហគមន៍ Obsidian](https://obsidian.md/community):

1. **សមកាលដំបូងភាគី**: [[#Obsidian Sync]]
2. **សមកាល cloud ភាគីទីបី**: [[#iCloud]], [[#OneDrive]], និង [[#Google Drive]]
3. **សមកាលក្នុងតំបន់**: [[#Syncthing]]
4. **ការគ្រប់គ្រងកំណែ**: [[#Git]] និង [[#Working Copy]]

## Obsidian Sync

**ប្រព័ន្ធដែលណែនាំ**: `Windows`, `macOS`, `Linux`, `iOS`, `Android`

វិធីសមកាលដ៏ត្រង់ និងមានការគាំទ្រផ្លូវការបំផុតគឺដំណោះស្រាយដំបូងភាគីរបស់យើង: [[ការណែនាំអំពី Obsidian Sync|Obsidian Sync]]។

Obsidian Sync ធ្វើឱ្យ vault របស់អ្នកធ្វើសមកាលឆ្លងឧបករណ៍ទាំងអស់ ដោយប្រើ vault ពីចម្ងាយដើម្បីចម្លងទិន្នន័យ។ ច្បាប់ចម្លងក្នុងតំបន់នៅតែស្ថិតនៅលើឧបករណ៍របស់អ្នកជានិច្ច។

អនុវត្តតាម [[ដំឡើង Obsidian Sync|ការណែនាំដំឡើង]] ដើម្បីកំណត់ Obsidian Sync។

## iCloud

**ប្រព័ន្ធដែលណែនាំ**: `macOS`, `iOS`, `iPadOS`

iCloud អាចប្រើសម្រាប់ធ្វើសមកាល vault រវាង iOS និង macOS។ ប៉ុន្តែ **iCloud Drive លើ Windows** អាចនាំឱ្យមានការស្ទួន ឬខូចឯកសារ។

**របៀបបង្កើតនិងទុក vault ក្នុង iCloud Drive**:

- **បើកដំណើរការ iCloud Drive**:
    - លើ macOS: ចូល **System Preferences → Apple ID → iCloud → iCloud Drive**
    - លើ iOS: ចូល **Settings → [ឈ្មោះរបស់អ្នក] → iCloud → iCloud Drive**
- **បង្កើត vault ថ្មីក្នុង iCloud**:
    - លើ macOS:
        1. បើក **Obsidian** ហើយជ្រើស **Create new vault**
        2. ក្នុង file picker រុករកទៅ **iCloud Drive → Obsidian**
        3. បង្កើតថតសម្រាប់ vault ហើយដំឡើងឈ្មោះ
        4. ជ្រើស **Create** ដើម្បីបញ្ចប់
    - លើ iOS:
        1. បើក **Obsidian** ហើយចុច **Create new vault**
        2. បញ្ចូលឈ្មោះ vault
        3. ចុចបើក **Store in iCloud**
        4. ចុច **Create**
- **បើក vault លើឧបករណ៍ Apple ផ្សេង**:
    - លើឧបករណ៍ macOS ឬ iOS ផ្សេង បើក **Obsidian** ចូល [[គ្រប់គ្រង​វ៉ាល់|ប្តូរ vault]] ហើយជ្រើស **Open folder as vault**។ រុករកទៅ **iCloud Drive → Obsidian**

> [!warning]+ ទីតាំងថតលើ iOS និង iPadOS
> នៅពេលប្រើ iCloud លើឧបករណ៍ mobile ប្រាកដថា vault ស្ថិតនៅទីតាំងត្រឹមត្រូវ: `iCloud Drive/Obsidian/[ឈ្មោះ Vault របស់អ្នក]`។
>
> Vault គួរស្ថិតក្នុងថត **Obsidian** ក្នុង iCloud Drive។ ថតខាងស្ដាំដែលមានរូបតំណាង Obsidian គឺត្រឹមត្រូវ។ កុំប្រើថតធម្មតាដែលគ្មានរូបសញ្ញាកម្មវិធី។
>
> ![[iCloud-folder-location.png#interface]]
>
> ដើម្បីផ្ទៀងផ្ទាត់ទីតាំង vault បើក app **Files** ចុច **Browse** ជ្រើស **iCloud Drive** ក្នុង **Locations** ហើយបញ្ជាក់ថា vault ស្ថិតក្នុងថត **Obsidian**។ ប្រសិនបើ vault ស្ថិតនៅទីតាំងផ្សេង អ្នកអាចជួបបញ្ហាក្នុងការធ្វើសមកាល។

> [!tip] ការអនុវត្តល្អ
> - សម្រាប់ **macOS 14 (Sonoma) និងចាស់ជាង**: បិទ **Optimize Mac Storage** ក្នុងការកំណត់ iCloud ដើម្បីការពារការ offload ឯកសារ។
> - សម្រាប់ **macOS 15 (Sequoia)**: ចុចប្ដូររូបថតលើថត **Obsidian** ក្នុង iCloud Drive ហើយជ្រើស **Keep Downloaded**។

## OneDrive

**ប្រព័ន្ធដែលណែនាំ**: `Windows`, `macOS` (មានមុខងារមានកំណត់លើ Android)

[OneDrive](https://support.microsoft.com/en-us/office/Sync-with-OneDrive-bb89981b-e382-4969-b8fd-d413a90b6db3#ID0EAABAAA=Set_up) គឺជាជម្រើស cloud storage ដ៏ពេញនិយមសម្រាប់អ្នកប្រើ Windows និង macOS។ ប៉ុន្តែវាមានដែនកំណត់លើ Android ហើយមិនមានការគាំទ្រផ្លូវការសម្រាប់ iOS។

> [!info] ទុកឯកសារអោយប្រើបានក្រៅបណ្ដាញ
> មុននឹងប្រើ OneDrive ប្រាកដថាថត vault ត្រូវបានសម្គាល់ **Always keep on this device**។ ការនេះការពារ OneDrive ពីការ offload ឯកសារ ដែលធ្វើឱ្យ Obsidian គិតថាឯកសារបាត់។

**របៀបបង្កើតនិងទុក vault ក្នុង OneDrive**:

1. **ដំឡើង OneDrive**:
   - លើ Windows: ចូលតាម OneDrive app ឬគណនី Microsoft
   - លើ macOS: ទាញយក OneDrive app ហើយចូល
2. **បង្កើត vault ថ្មីក្នុង OneDrive**:
   - លើ Windows/macOS:
     1. បើក **File Explorer** (Windows) ឬ **Finder** (macOS) ហើយរុករកទៅ **OneDrive → Documents**
     2. បង្កើតថតថ្មី (ឧ. "Obsidian Vault")
     3. បើក **Obsidian** ជ្រើស **Create new vault** ហើយជ្រើសថត OneDrive
3. **បើក vault លើឧបករណ៍ផ្សេង**:
   - លើឧបករណ៍ផ្សេង បើក **Obsidian** ចូល [[គ្រប់គ្រង​វ៉ាល់|ប្តូរ vault]] ហើយជ្រើស **Open folder as vault**។ រុករកទៅ **OneDrive → Documents**

> [!info] ការធ្វើសមកាល Android
> OneDrive ប្រហែលជាមិនដំណើរការល្អសម្រាប់ Android ។ ពិចារណាប្រើ app ដូចជា [Dropsync](https://play.google.com/store/apps/details?id=com.ttxapps.dropsync) ឬ [FolderSync](https://play.google.com/store/apps/details?id=dk.tacit.android.foldersync.lite)។

> [!tip] ការអនុវត្តល្អ
> - រក្សាឯកសារ vault ឱ្យ **Available Offline** ជានិច្ច ដោយចុចប្ដូររូបថតលើថត ហើយជ្រើស **Always keep on this device**
> - ជៀសវាង **Files On-Demand** របស់ OneDrive សម្រាប់ vault ដើម្បីការពារបញ្ហាសមកាល

## Google Drive

**ប្រព័ន្ធដែលណែនាំ**: `Windows`, `macOS`, `Android` (មានមុខងារមានកំណត់លើ iOS)

[Google Drive](https://support.google.com/drive/answer/10838124?hl=en) គឺជាដំណោះស្រាយ cloud storage ដ៏ពេញនិយមមួយទៀត។ ទោះបីជាវាមិនមានការគាំទ្រផ្លូវការ អ្នកអាចប្រើ app ភាគីទីបី ឬ plugin ដើម្បីធ្វើសមកាល។

> [!info] ការគាំទ្រ iOS
> Google Drive មិនមានការគាំទ្រផ្លូវការសម្រាប់ iOS។ ពិចារណាប្រើដំណោះស្រាយ ឬ plugin ភាគីទីបី។

**របៀបបង្កើតនិងទុក vault ក្នុង Google Drive**:

1. **ដំឡើង Google Drive**:
    - លើ Windows ឬ macOS: ទាញយក Google Drive app ហើយចូល
    - លើ Android: ប្រាកដថា Google Drive ត្រូវបានបើកដំណើរការ
2. **បង្កើត vault ថ្មីក្នុង Google Drive**:
    - លើ Windows/macOS:
        1. បើក **File Explorer** (Windows) ឬ **Finder** (macOS) ហើយរុករកទៅ **Google Drive**
        2. បង្កើតថតថ្មី (ឧ. "Obsidian Vault")
        3. បើក **Obsidian** ជ្រើស **Create new vault** ហើយជ្រើសថត Google Drive
3. **បើក vault លើឧបករណ៍ផ្សេង**:
    - លើឧបករណ៍ផ្សេង បើក **Obsidian** ចូល [[គ្រប់គ្រង​វ៉ាល់|ប្តូរ vault]] ហើយជ្រើស **Open folder as vault**។ រុករកទៅថត Google Drive

> [!tip] ការអនុវត្តល្អ
> - កំណត់ឯកសារ vault ឱ្យ **Available Offline** ក្នុង Google Drive ដើម្បីជៀសវាងបញ្ហា
> - សម្រាប់ iOS ពិចារណាវិធីជំនួស ដូចជា [[ការណែនាំអំពី Obsidian Sync|Obsidian Sync]], [[#iCloud]] ឬប្រើ plugin **Remotely Save**

## Syncthing

**ប្រព័ន្ធដែលណែនាំ**: `Windows`, `macOS`, `Linux`

Syncthing គឺជាឧបករណ៍សមកាលឯកសារដែលមិនអាស្រ័យ cloud storage។ វាធ្វើសមកាល vault ដោយផ្ទាល់រវាងឧបករណ៍តាមបណ្ដាញ។

> [!info]+ ការគាំទ្រ Android
> Syncthing app ផ្លូវការ Android មិនមែនមានការថែទាំទៀតហើយ។ ទោះបីជាយ៉ាងណា ការ fork សហគមន៍ [Syncthing-Fork](https://github.com/Catfriend1/syncthing-android) នៅតែបន្តការអភិវឌ្ឍ ហើយអាចប្រើលើ Android បាន។

**របៀបបង្កើតនិងទុក vault ដោយប្រើ Syncthing**:

1. **ដំឡើង Syncthing**:
   - ដំឡើង Syncthing លើឧបករណ៍នីមួយៗ។ សូមមើល [គេហទំព័រ Syncthing](https://syncthing.net/) សម្រាប់ការណែនាំ
   - លើ Android ដំឡើង [Syncthing-Fork](https://github.com/Catfriend1/syncthing-android)
2. **បង្កើតនិងកំណត់ shared folder**:
   - លើឧបករណ៍ទាំងអស់:
     1. បើក Syncthing ហើយបង្កើត shared folder។ កំណត់ folder path ទៅ vault Obsidian
     2. ប្រាកដថា folder ដូចគ្នាត្រូវបានជ្រើសលើឧបករណ៍ទាំងអស់
     3. កំណត់ folder syncing preferences (ឧ. **Send & Receive** សម្រាប់ bidirectional)
3. **បើក vault ក្នុង Obsidian**:
   - នៅពេល folder ត្រូវបានធ្វើសមកាលឆ្លងឧបករណ៍ បើក **Obsidian** ចូល [[គ្រប់គ្រង​វ៉ាល់|ប្តូរ vault]] ហើយជ្រើស **Open folder as vault**

> [!info] ភាពអាចប្រើបានរបស់ឧបករណ៍
> Syncthing ដំណើរការល្អបំផុតនៅពេលឧបករណ៍យ៉ាងហោចណាស់មួយ តែងតែបើក ដើម្បីធានាការធ្វើសមកាលដែលមិនរំខាន។

> [!tip] ការអនុវត្តល្អ
> - សម្រាប់ local sync ប្រាកដថាឧបករណ៍ទាំងអស់ភ្ជាប់ទៅបណ្ដាញតែមួយ
> - ដក `.obsidian` ពីការធ្វើសមកាល ប្រសិនបើអ្នកចង់បានការកំណត់ដាច់ដោយឡែក
> - ប្រើ ignore patterns ដើម្បីជៀសវាងការសមកាលឯកសារបណ្ដោះអាសន្ន ឬ backup

## Git

**ប្រព័ន្ធដែលណែនាំ**: `Windows`, `macOS`, `Linux`

**Git** គឺជាប្រព័ន្ធគ្រប់គ្រងកំណែ ដែលអនុញ្ញាតឱ្យអ្នកតាមដានការផ្លាស់ប្ដូរ ធ្វើការសហការ និងធ្វើសមកាល vault តាមរយៈ repository ដូចជា GitHub, GitLab ឬ server ផ្ទាល់ខ្លួន។

**របៀបធ្វើសមកាល vault ដោយប្រើ Git**:

1. **ដំឡើង remote repository**:
    - បង្កើត repository លើ Git hosting platform (ឧ. GitHub, GitLab ឬ server ផ្ទាល់ខ្លួន)
2. **ធ្វើសមកាល vault**:
    1. បើក terminal ឬ Git GUI (ឧ. GitKraken, Sourcetree)
    2. Initialize Git repository ក្នុងថត vault ដោយប្រើ `git init`
    3. បន្ថែម remote repository: `git remote add origin [URL]`
    4. Commit ការផ្លាស់ប្ដូរ: `git add .` និង `git commit -m "សារ"`
    5. Push ការផ្លាស់ប្ដូរ: `git push origin main`
3. **Pull ការផ្លាស់ប្ដូរលើឧបករណ៍ផ្សេង**:
    - Clone repository លើឧបករណ៍ផ្សេង ហើយ pull ការផ្លាស់ប្ដូរដោយប្រើ `git pull origin main`

> [!info] ត្រូវធ្វើសមកាលដោយខ្លួនឯង
> Git ផ្ដល់ការគ្រប់គ្រងកំណែដ៏ខ្លាំង ប៉ុន្តែការសមកាលមិនស្វ័យប្រវត្តិ។ អ្នកត្រូវ push និង pull ការផ្លាស់ប្ដូរដោយដៃ។

### Working Copy

**ប្រព័ន្ធដែលណែនាំ**: `iOS`
**ត្រូវការ**: [[#Git]]

**Working Copy** គឺជា Git client សម្រាប់ iOS ដែលអនុញ្ញាតឱ្យ clone, commit និង push ការផ្លាស់ប្ដូរទៅ Git repository។ វាដំណើរការល្អសម្រាប់ Git vault sync ទោះបីជាមុខងារខ្លះត្រូវការការទូទាត់ក្នុង app។

**របៀបធ្វើសមកាល vault ដោយប្រើ Working Copy**:

1. **ដំឡើង Working Copy**:
    - ទាញយក app **[Working Copy](https://apps.apple.com/us/app/working-copy-git-client/id896694807)** លើ iPhone ឬ iPad
2. **Clone Git repository**:
    - បើក Working Copy ចុច **Add Repository** ហើយបញ្ចូល URL repository (ឧ. GitHub, GitLab)
3. **ភ្ជាប់ repository ទៅ Obsidian**:
    - ភ្ជាប់ repository folder ដែល clone ទៅ vault ទទេក្នុង **Obsidian**
4. **Commit និង push ការផ្លាស់ប្ដូរ**:
    - បន្ទាប់ពីកែប្រែកំណត់ត្រាក្នុង Obsidian ប្រើ Working Copy ដើម្បី **Commit** និង **Push**
    - លើឧបករណ៍ផ្សេង pull ការផ្លាស់ប្ដូរដោយប្រើ Git ដើម្បីធ្វើសមកាល vault

> [!info] ការប្រើប្រាស់ក្នុងសហគមន៍
> ទោះបីជា Working Copy មិនមានការគាំទ្រផ្លូវការ អ្នកប្រើប្រាស់ជាច្រើនបានប្រើវាដោយជោគជ័យ។

## សំណួរដែលសួរញឹកញាប់

**ហេតុអ្វីបានជា sync service ដែលខ្ញុំចូលចិត្តមិនមានការគាំទ្រផ្លូវការ?**

មិនដូច app កំណត់ត្រាខ្លះ Obsidian ត្រូវការចូលប្រើ vault ទាំងមូលសម្រាប់មុខងាររបស់ខ្លួន (ឧ. ធ្វើបច្ចុប្បន្នភាព link នៅពេលប្ដូរឈ្មោះឯកសារ)។ ដូច្នេះ service ខ្លះដំណើរការមិនល្អជាមួយ Obsidian។

**ហេតុអ្វីខ្ញុំត្រូវទុក "Available Offline"?**

ប្រសិនបើ service ដូចជា OneDrive ឬ iCloud offload ឯកសារ Obsidian មិនអាចចូលប្រើ ដែលបណ្ដាលឱ្យមានបញ្ហា។ សម្គាល់ vault folder ជា **Always keep on this device** (OneDrive) ឬប្រាកដថា **Keep Downloaded** ត្រូវបានបើក (iCloud)។

**តើខ្ញុំគ្រប់គ្រងការកំណត់ខុសៗគ្នាសម្រាប់ vault ដោយរបៀបណា?**

Obsidian អនុញ្ញាតឱ្យអ្នកប្ដូរ configuration folder ក្នុងឧបករណ៍នីមួយៗ ដោយប្រើមុខងារ [[ថតកំណត់រចនាសម្ព័ន្ធ|Config folders]]។
