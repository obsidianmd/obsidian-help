អ្នកអាចរៀបចំដូមែនផ្ទាល់ខ្លួន ឬដូមែនរង សម្រាប់គេហទំព័រ [[ការណែនាំអំពី Obsidian Publish|Obsidian Publish]] របស់អ្នក។

> [!warning] ការព្រមាន
> នៅពេលនេះ យើងគាំទ្រតែការកំណត់រចនាសម្ព័ន្ធដូមែនផ្ទាល់ខ្លួនដោយប្រើវិធីសាស្ត្រខាងក្រោមប៉ុណ្ណោះ៖
>
> - [[#រៀបចំដោយប្រើ CloudFlare]] ដោយប្រើ [Full mode](https://developers.cloudflare.com/ssl/origin-configuration/ssl-modes/full/)។
> - [[#រៀបចំដោយប្រើ proxy]]
> 
> យើងមិនទាន់មានវិធីផ្តល់វិញ្ញាបនបត្រ SSL ក្នុងនាមអ្នកនៅឡើយទេ។

## រៀបចំដោយប្រើ CloudFlare

វិធីងាយស្រួលបំផុតក្នុងការរៀបចំដូមែនផ្ទាល់ខ្លួន ឬដូមែនរង គឺបង្កើតគណនី [CloudFlare](https://cloudflare.com) និងអនុញ្ញាតឱ្យ CloudFlare គ្រប់គ្រង DNS នៃដូមែនរបស់អ្នក។

ជំហានខាងក្រោមប្រើ CloudFlare ដើម្បីកំណត់រចនាសម្ព័ន្ធដូមែនផ្ទាល់ខ្លួនសម្រាប់គេហទំព័រ Obsidian Publish របស់អ្នក ទោះជាប្រើដូមែនឫស (`mysite.com`) ឬដូមែនរង (`notes.mysite.com`)។

> [!important] សំខាន់
> CloudFlare គឺជា **អ្នកផ្តល់សេវាដែលគាំទ្រជាផ្លូវការតែមួយគត់** សម្រាប់ការរៀបចំដូមែនផ្ទាល់ខ្លួន។ ការប្រើការណែនាំខាងក្រោមជាមួយអ្នកផ្តល់សេវាផ org ទៃមិនទំនងថានឹងដំណើរការឡើយ។

**CloudFlare:**

1. បើក Cloudflare ទៅដូមែនដែលអ្នកចង់បង្ហោះគេហទំព័រ Publish របស់អ្នក ដូចជា `mysite.com` ទោះបីជាអ្នកចង់បានដូមែនរងដូចជា `notes.mysite.com` ក៏ដោយ។
2. ទៅ **DNS** ហើយចុច **Add Record**។
3. ជ្រើសរើស **CNAME**។
4. ក្នុង **name** បញ្ចូលដូមែន ឬដូមែនរងរបស់អ្នក ឧទាហរណ៍ `notes.mysite.com`។
5. ក្នុង **target** បញ្ចូល `publish-main.obsidian.md`។ កុំបញ្ចូល sub-URL ផ្ទាល់ខ្លួនរបស់អ្នកក្នុងតម្លៃនេះ។ Obsidian Publish គ្រប់គ្រងរឿងនេះពីការកំណត់រចនាសម្ព័ន្ធរបស់អ្នក។
6. ត្រូវប្រាកដថា **proxy status** ត្រូវបានបើក។ វាគួរតែត្រូវបានបើកតាមលំនាំដើម។
7. ទៅ **SSL/TLS** ហើយកំណត់របៀបអ៊ិនគ្រីប SSL/TLS ទៅ "Full" ដើម្បីកំណត់រចនាសម្ព័ន្ធវិញ្ញាបនបត្រ SSL/TLS ដោយស្វ័យប្រវត្តិ។

> [!note] ចំណាំ
> ដើម្បីបញ្ជូនបន្តទាំង `mysite.com` និង `www.mysite.com` ទៅ Obsidian Publish អ្នកត្រូវបង្កើត [Page Rule](https://support.cloudflare.com/hc/en-us/articles/200172336-Creating-Page-Rules) ជាមួយការកំណត់ខាងក្រោម៖
>
> - URL match: `www.mysite.com/*`
> - Forward URL - 301 Permanent Redirect
> - Redirect URL: `https://mysite.com/$1`
>
> បន្ទាប់ពីអ្នកបានបង្កើត page rule សូមបង្កើត CNAME record សម្រាប់ `www.mysite.com` ដូចដែលអ្នកបានធ្វើសម្រាប់ `mysite.com`។

**Obsidian:**

1. បើក Obsidian នៅលើកុំព្យូទ័ររបស់អ្នក។
2. នៅក្នុង [[រីបិន]] នៅខាងឆ្វេង ចុច **Publish changes** ![[lucide-send.svg#icon]]។
3. នៅក្រោម **Publish changes** ជ្រើសរើស **Change site options** ![[lucide-cog.svg#icon]]។
4. នៅជាប់ **Custom domain** ជ្រើសរើស **Configure**។
5. ក្នុង **Custom URL** បញ្ចូល URL ទៅដូមែន ឬដូមែនរងរបស់អ្នក។ ត្រូវប្រាកដថាកុំដាក់ `www.` នៅក្នុងប្រអប់ custom URL។

> [!note] ចំណាំ
> ប្រសិនបើការរៀបចំដូមែនផ្ទាល់ខ្លួនរបស់អ្នកបញ្ចប់នៅក្នុងរង្វិលបញ្ជូនបន្ត វាទំនងជាដោយសារតែរបៀបអ៊ិនគ្រីបក្នុង CloudFlare ត្រូវបានកំណត់ទៅ "Flexible" ជំនួសឱ្យ "Full"។

## រៀបចំដោយប្រើ proxy

អ្នកក៏អាចរៀបចំ SSL/TLS សម្រាប់ដូមែនផ្ទាល់ខ្លួនរបស់អ្នកដោយប្រើម៉ាស៊ីនបម្រើវែបផ្ទាល់ខ្លួនរបស់អ្នកផងដែរ។

ប្រសិនបើអ្នកកំពុងបង្ហោះគេហទំព័រនៅក្រោមដូមែន ឬដូមែនរងរបស់អ្នករួចហើយ អ្នកក៏អាចប្រើជម្រើសនេះ និងរៀបចំគេហទំព័ររបស់អ្នកដើម្បីផ្ទុកគេហទំព័រ Obsidian Publish នៅក្រោមផ្លូវ URL ជាក់លាក់មួយ ជំនួសឱ្យការបង្ហោះគេហទំព័រពេញលេញ។

បញ្ជូន (proxy) សំណើទាំងអស់នៅក្រោមផ្លូវ URL នោះទៅ `https://publish.obsidian.md/serve?url=mysite.com/my-notes/...` ហើយកំណត់រចនាសម្ព័ន្ធជម្រើសគេហទំព័រនៅក្នុង Obsidian ទៅផ្លូវ URL ដូចគ្នា ដោយកំណត់ **Custom URL** ទៅ `mysite.com/my-notes`។

អ្នកក៏អាចរៀបចំ Obsidian Publish ជា sub-URL នៃគេហទំព័រដែលអ្នកជាម្ចាស់ផងដែរ។ ឧទាហរណ៍ `https://mysite.com/my-notes/`។ ដើម្បីសម្រេចបាន អ្នកត្រូវបង្ហោះម៉ាស៊ីនបម្រើផ្ទាល់ខ្លួនរបស់អ្នក ហើយបញ្ជូន (proxy) សំណើទាំងអស់ទៅម៉ាស៊ីនបម្រើរបស់យើងនៅ `https://publish.obsidian.md/`។

ឧទាហរណ៍ការរៀបចំ proxy ខាងក្រោមមិនមែនជាទាំងអស់នោះទេ ប៉ុន្តែផ្តល់វិធីសាស្ត្រទូទៅសម្រាប់ការអនុវត្តនេះ។

### NGINX

នៅក្នុងការកំណត់រចនាសម្ព័ន្ធ NGINX របស់អ្នក បន្ថែមខាងក្រោម៖

```nginx
location /my-notes {
  proxy_pass https://publish.obsidian.md/serve?url=mysite.com/my-notes/;
  proxy_ssl_server_name on;
  proxy_set_header Host publish.obsidian.md;
}
```


អ្នកប្រើប្រាស់ខ្លះបានរាយការណ៍ថាការបន្ថែម `$request_uri` ទៅ proxy pass អាចត្រូវការ៖

```nginx
location /my-notes {
  proxy_pass https://publish.obsidian.md/serve?url=mysite.com/my-notes$request_uri;
  proxy_ssl_server_name on;
  proxy_set_header Host publish.obsidian.md;
}
```

### Apache

នៅក្នុង `.htaccess` បន្ថែមខាងក្រោម៖

```htaccess
RewriteEngine  on
RewriteRule    "^my-notes/(.*)$"  "https://publish.obsidian.md/serve?url=mysite.com/my-notes/$1"  [L,P]
```

> [!note] ចំណាំ
> `mod_rewrite` ត្រូវតែបើក ហើយអ្នកក៏អាចត្រូវកំណត់រចនាសម្ព័ន្ធ [SSLProxyEngine](https://stackoverflow.com/questions/40938148/reverse-proxy-for-external-url-apache) ផងដែរ

### Netlify

នៅក្នុង `netlify.toml` [កំណត់រចនាសម្ព័ន្ធការបញ្ជូនបន្ត](https://docs.netlify.com/routing/redirects/#syntax-for-the-netlify-configuration-file)៖

```plain
[[redirects]]
  from = "https://mysite.com/my-notes/*"
  to = "https://publish.obsidian.md/serve?url=mysite.com/my-notes/:splat"
  status = 200
  force = true
```

### Vercel

នៅក្នុង `vercel.json` [កំណត់រចនាសម្ព័ន្ធ rewrites](https://vercel.com/docs/configuration#project/rewrites)៖

```json
{
  ...

  "rewrites": [
    {
      "source": "/my-notes/",
      "destination": "https://publish.obsidian.md/serve?url=mysite.com/my-notes"
    },
    {
      "source": "/my-notes/:path*",
      "destination": "https://publish.obsidian.md/serve?url=mysite.com/my-notes/:path*"
    }
  ]
}
```

### Caddy

```plain
mysite.com {
  encode zstd gzip
  handle /my-notes* {
    reverse_proxy https://publish.obsidian.md {
      header_up Host {upstream_hostport}
    }
    rewrite * /serve?url=mysite.com{path}
  }
}
```

### Traefik

ការដកស្រង់ការកំណត់រចនាសម្ព័ន្ធអប្បបរមានេះបញ្ជូនបន្ត `mysite.com` ទៅ Obsidian Publish។
សូមមើល [ឯកសារ Traefik](https://doc.traefik.io/traefik/routing/overview/)
សម្រាប់ឧទាហរណ៍ពេញលេញ។

```yaml
http:
  routers:
    mysite:
      rule: Host(`mysite.com`)
      service: obsidian-publish
      middlewares:
        - "publish-headers"
  services:
    obsidian-publish:
      loadBalancer:
        servers:
          - url: https://publish.obsidian.md
  middlewares:
    publish-headers:
      headers:
        customRequestHeaders:
          Host: "publish.obsidian.md"
          x-obsidian-custom-domain: "mysite.com"
```

### HTTP X-Headers ដែលគាំទ្រ

ប្រសិនបើសេវាកម្ម proxy របស់អ្នកមិនអនុញ្ញាតផ្លូវសំណួរ (query paths) អ្នកអាចប្រើ `https://publish.obsidian.md/` ជាមួយ header ផ្ទាល់ខ្លួន `x-obsidian-custom-domain` ដែលកំណត់ទៅ URL គេហទំព័ររបស់អ្នក `mysite.com/my-subpath`។

## បញ្ជូនបន្តគេហទំព័រចាស់ទៅដូមែនផ្ទាល់ខ្លួន

ប្រសិនបើអ្នកចង់បញ្ជូនបន្តអ្នកទស org្សនារបស់អ្នកពីគេហទំព័រ `publish.obsidian.md` ចាស់ទៅដូមែនផ្ទាល់ខ្លួនថ្មីរបស់អ្នក សូមបើកជម្រើស **Redirect to your custom domain** នៅពេលកំណត់រចនាសម org្ព័ន្ធដូមែនផ្ទាល់ខ្លួនរបស់អ្នក។
