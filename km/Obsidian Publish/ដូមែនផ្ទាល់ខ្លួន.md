អ្នកអាចដំឡើងដូមែន ឬដូមែនរងផ្ទាល់ខ្លួនសម្រាប់គេហទំព័រ [[ការណែនាំអំពី Obsidian Publish|Obsidian Publish]] របស់អ្នក។

> [!warning] ព្រមាន
> នៅពេលនេះ យើងគាំទ្រតែការកំណត់រចនាសម្ព័ន្ធដូមែនផ្ទាល់ខ្លួនដោយប្រើវិធីសាស្ត្រខាងក្រោម៖
>
> - [[#ដំឡើងដោយប្រើ CloudFlare]] ដោយប្រើ [Full mode](https://developers.cloudflare.com/ssl/origin-configuration/ssl-modes/full/)។
> - [[#ដំឡើងដោយប្រើ proxy]]
>
> យើងមិនទាន់មានវិធីដើម្បីផ្ដល់វិញ្ញាបនបត្រ SSL ក្នុងនាមអ្នកនៅឡើយទេ។

## ដំឡើងដោយប្រើ CloudFlare

វិធីងាយបំផុតដើម្បីដំឡើងដូមែន ឬដូមែនរងផ្ទាល់ខ្លួន គឺការបង្កើតគណនី [CloudFlare](https://cloudflare.com) ហើយឱ្យ CloudFlare គ្រប់គ្រង DNS នៃដូមែនរបស់អ្នក។

ជំហានខាងក្រោមប្រើ CloudFlare ដើម្បីកំណត់រចនាសម្ព័ន្ធដូមែនផ្ទាល់ខ្លួនសម្រាប់គេហទំព័រ Obsidian Publish របស់អ្នក ទាំងការប្រើ root domain (`mysite.com`) ឬ subdomain (`notes.mysite.com`)។

> [!important] សំខាន់
> CloudFlare គឺជា **អ្នកផ្ដល់សេវាដែលទទួលស្គាល់ជាផ្លូវការតែមួយគត់** សម្រាប់ការដំឡើងដូមែនផ្ទាល់ខ្លួន។ ការប្រើការណែនាំខាងក្រោមជាមួយអ្នកផ្ដល់សេវាផ្សេងទៀតប្រហែលជានឹងមិនដំណើរការ។

**CloudFlare:**

1. បើក Cloudflare ទៅកាន់ដូមែនដែលអ្នកចង់ host គេហទំព័រ Publish របស់អ្នក ដូចជា `mysite.com` សូម្បីតែអ្នកចង់ប្រើ subdomain ដូចជា `notes.mysite.com`។
2. ទៅ **DNS** ហើយចុច **Add Record**។
3. ជ្រើស **CNAME**។
4. នៅ **name** បញ្ចូលដូមែន ឬ subdomain របស់អ្នក ឧទាហរណ៍ `notes.mysite.com`។
5. នៅ **target** បញ្ចូល `publish-main.obsidian.md`។ កុំបញ្ចូល sub-URL ផ្ទាល់ខ្លួនរបស់អ្នកក្នុងតម្លៃនេះ។ Obsidian Publish ដោះស្រាយការនេះពីការកំណត់រចនាសម្ព័ន្ធរបស់អ្នក។
6. ត្រូវប្រាកដថា **proxy status** ត្រូវបានបើក។ វាគួរតែបើកដោយស្វ័យប្រវត្តិ។
7. ទៅ **SSL/TLS** ហើយកំណត់ SSL/TLS encryption mode ទៅ "Full" ដើម្បីកំណត់រចនាសម្ព័ន្ធវិញ្ញាបនបត្រ SSL/TLS ដោយស្វ័យប្រវត្តិ។

> [!note] កំណត់សម្គាល់
> ដើម្បីបញ្ជូនបន្ត `mysite.com` និង `www.mysite.com` ទាំងពីរទៅ Obsidian Publish អ្នកត្រូវបង្កើត [Page Rule](https://support.cloudflare.com/hc/en-us/articles/200172336-Creating-Page-Rules) ដោយការកំណត់ខាងក្រោម៖
>
> - URL match: `www.mysite.com/*`
> - Forward URL - 301 Permanent Redirect
> - Redirect URL: `https://mysite.com/$1`
>
> បន្ទាប់ពីអ្នកបង្កើត page rule រួចហើយ សូមបង្កើតកំណត់ត្រា CNAME សម្រាប់ `www.mysite.com` ដូចដែលអ្នកបានធ្វើសម្រាប់ `mysite.com`។

**Obsidian:**

1. បើក Obsidian នៅលើកុំព្យូទ័ររបស់អ្នក។
2. នៅ [[រីបិន]] ខាងឆ្វេង ចុច **Publish changes** ![[lucide-send.svg#icon]]។
3. នៅក្រោម **Publish changes** ជ្រើស **Change site options** ![[lucide-cog.svg#icon]]។
4. នៅជាប់ **Custom domain** ជ្រើស **Configure**។
5. នៅ **Custom URL** បញ្ចូល URL ទៅដូមែន ឬ subdomain របស់អ្នក។ ត្រូវប្រាកដថាមិនដាក់ `www.` ក្នុងប្រអប់ custom URL។

> [!note] កំណត់សម្គាល់
> បើការដំឡើងដូមែនផ្ទាល់ខ្លួនរបស់អ្នកបណ្ដាលឱ្យមានការ redirect loop វាប្រហែលជាដោយសារ encryption mode នៅ CloudFlare ត្រូវបានកំណត់ទៅ "Flexible" ជំនួសឱ្យ "Full"។

## ដំឡើងដោយប្រើ proxy

អ្នកក៏អាចដំឡើង SSL/TLS សម្រាប់ដូមែនផ្ទាល់ខ្លួនរបស់អ្នកដោយប្រើ web server ផ្ទាល់ខ្លួនរបស់អ្នកផងដែរ។

ប្រសិនបើអ្នកបានhost គេហទំព័រមួយនៅក្រោមដូមែន ឬ subdomain របស់អ្នករួចហើយ អ្នកក៏អាចប្រើជម្រើសនេះ ហើយដំឡើងគេហទំព័ររបស់អ្នកដើម្បីផ្ទុកគេហទំព័រ Obsidian Publish របស់អ្នកក្រោម URL path ជាក់លាក់ ជំនួសឱ្យការ host គេហទំព័រពេញ។

Proxy សំណើទាំងអស់នៅក្រោម URL path នោះទៅ `https://publish.obsidian.md/serve?url=mysite.com/my-notes/...` ហើយកំណត់រចនាសម្ព័ន្ធ site options នៅ Obsidian ទៅ URL path ដូចគ្នា ដោយការកំណត់ **Custom URL** ទៅ `mysite.com/my-notes`។

អ្នកក៏អាចដំឡើង Obsidian Publish ជា sub-URL នៃគេហទំព័រដែលអ្នកមាន។ ឧទាហរណ៍ `https://mysite.com/my-notes/`។ ដើម្បីសម្រេចរឿងនេះ អ្នកត្រូវ host server ផ្ទាល់ខ្លួន ហើយ proxy សំណើទាំងអស់ទៅ server របស់យើងនៅ `https://publish.obsidian.md/`។

ឧទាហរណ៍ការដំឡើង proxy ខាងក្រោមមិនមែនពេញលេញទេ ប៉ុន្តែផ្ដល់វិធីសាស្ត្រទូទៅសម្រាប់ការអនុវត្តនេះ។

### NGINX

នៅក្នុងការកំណត់រចនាសម្ព័ន្ធ NGINX របស់អ្នក បន្ថែមខាងក្រោម៖

```nginx
location /my-notes {
  proxy_pass https://publish.obsidian.md/serve?url=mysite.com/my-notes/;
  proxy_ssl_server_name on;
  proxy_set_header Host publish.obsidian.md;
}
```

អ្នកប្រើប្រាស់ខ្លះបានរាយការណ៍ថាការបន្ថែម `$request_uri` ទៅ proxy pass អាចតម្រូវ៖

```nginx
location /my-notes {
  proxy_pass https://publish.obsidian.md/serve?url=mysite.com/my-notes$request_uri;
  proxy_ssl_server_name on;
  proxy_set_header Host publish.obsidian.md;
}
```

### Apache

នៅ `.htaccess` បន្ថែមខាងក្រោម៖

```htaccess
RewriteEngine  on
RewriteRule    "^my-notes/(.*)$"  "https://publish.obsidian.md/serve?url=mysite.com/my-notes/$1"  [L,P]
```

> [!note] កំណត់សម្គាល់
> `mod_rewrite` ត្រូវតែបើក ហើយអ្នកប្រហែលជាត្រូវកំណត់រចនាសម្ព័ន្ធ [SSLProxyEngine](https://stackoverflow.com/questions/40938148/reverse-proxy-for-external-url-apache) ផងដែរ

### Netlify

នៅ `netlify.toml` [កំណត់រចនាសម្ព័ន្ធ redirects](https://docs.netlify.com/routing/redirects/#syntax-for-the-netlify-configuration-file)៖

```plain
[[redirects]]
  from = "https://mysite.com/my-notes/*"
  to = "https://publish.obsidian.md/serve?url=mysite.com/my-notes/:splat"
  status = 200
  force = true
```

### Vercel

នៅ `vercel.json` [កំណត់រចនាសម្ព័ន្ធ rewrites](https://vercel.com/docs/configuration#project/rewrites)៖

```json
{
  ...

  "rewrites": [
    {
      "source": "/my-notes/",
      "destination": "https://publish.obsidian.md/serve?url=mysite.com/my-notes"
    },
    {
      "source": "/my-notes/:path*",
      "destination": "https://publish.obsidian.md/serve?url=mysite.com/my-notes/:path*"
    }
  ]
}
```

### Caddy

```plain
mysite.com {
  encode zstd gzip
  handle /my-notes* {
    reverse_proxy https://publish.obsidian.md {
      header_up Host {upstream_hostport}
    }
    rewrite * /serve?url=mysite.com{path}
  }
}
```

### Traefik

ការដកស្រង់ការកំណត់រចនាសម្ព័ន្ធអប្បបរមានេះបញ្ជូនបន្ត `mysite.com` ទៅ Obsidian Publish។
មើល [ឯកសារ Traefik](https://doc.traefik.io/traefik/routing/overview/)
សម្រាប់ឧទាហរណ៍ពេញលេញ។

```yaml
http:
  routers:
    mysite:
      rule: Host(`mysite.com`)
      service: obsidian-publish
      middlewares:
        - "publish-headers"
  services:
    obsidian-publish:
      loadBalancer:
        servers:
          - url: https://publish.obsidian.md
  middlewares:
    publish-headers:
      headers:
        customRequestHeaders:
          Host: "publish.obsidian.md"
          x-obsidian-custom-domain: "mysite.com"
```

### HTTP X-Headers ដែលគាំទ្រ

ប្រសិនបើសេវា proxy របស់អ្នកមិនអនុញ្ញាតឱ្យប្រើ query paths អ្នកអាចប្រើ `https://publish.obsidian.md/` ជាមួយ header ផ្ទាល់ខ្លួន `x-obsidian-custom-domain` ដែលកំណត់ទៅ URL គេហទំព័ររបស់អ្នក `mysite.com/my-subpath`។

## បញ្ជូនបន្តគេហទំព័រចាស់ទៅដូមែនផ្ទាល់ខ្លួន

ប្រសិនបើអ្នកចង់បញ្ជូនបន្តអ្នកទស្សនាពីគេហទំព័រ `publish.obsidian.md` ចាស់ទៅដូមែនផ្ទាល់ខ្លួនថ្មីរបស់អ្នក សូមបើកជម្រើស **Redirect to your custom domain** នៅពេលកំណត់រចនាសម្ព័ន្ធដូមែនផ្ទាល់ខ្លួនរបស់អ្នក។

## ដោះស្រាយបញ្ហា

នៅពេលអ្នកដំឡើងដូមែនផ្ទាល់ខ្លួនរបស់អ្នករួចហើយ ប្រសិនបើអ្នកបានចូលទៅគេហទំព័ររបស់អ្នកពី link `https://publish.obsidian.md/slug` ពីមុន អ្នកអាចត្រូវលុបឃ្លាំងសម្ងាត់ browser សម្រាប់មុខងារខ្លះ (ដូចជា fonts ក្រាហ្វ ឬការចូលប្រើដោយពាក្យសម្ងាត់) ឱ្យដំណើរការបានត្រឹមត្រូវ។ ការនេះដោយសារតែការដាក់កំហិតសុវត្ថិភាព cross-domain ដែលដាក់ចេញដោយ browser សម័យទំនើប។ ដំណឹងល្អគឺអ្នកអានគេហទំព័ររបស់អ្នកមិនគួរជួបបញ្ហានេះទេ ប្រសិនបើអ្នកអនុញ្ញាតឱ្យអ្នកទស្សនាប្រើដូមែនផ្ទាល់ខ្លួនរបស់អ្នកប៉ុណ្ណោះ។
