## អថេរកំណត់ជាមុន

អថេរកំណត់ជាមុនត្រូវបានបង្កើតដោយស្វ័យប្រវត្តិដោយផ្អែកលើមាតិកាទំព័រ។ ជាទូទៅវាដំណើរការសម្រាប់គេហទំព័រភាគច្រើន។

អថេរមាតិកាចម្បងគឺ `{{content}}` ដែលមានមាតិកាអត្ថបទ ឬ [[ពណ៌ចំណាំទំព័របណ្តាញ|ការពណ៌ចំណាំ]] ឬការជ្រើសរើសប្រសិនបើមានអត្ថបទដែលបានជ្រើសរើសនៅលើទំព័រ។ សូមចំណាំថា `{{content}}` ព្យាយាមស្រង់មាតិកាចម្បងរបស់ទំព័រ ដែលអាចមិនមែនជាអ្វីដែលអ្នកចង់បានជានិច្ច។ ក្នុងករណីនោះ អ្នកអាចប្រើអថេរកំណត់ជាមុនផ្សេងទៀត ឬអថេរ selector ដើម្បីស្រង់មាតិកាដែលអ្នកត្រូវការ។

## អថេរសំណួរ

អថេរសំណួរប្រើម៉ូដែលភាសាដើម្បីស្រង់ និងកែប្រែទិន្នន័យដោយប្រើភាសាធម្មជាតិ។ អថេរសំណួរតម្រូវឱ្យ [[បកស្រាយទំព័របណ្តាញ|កម្មវិធីបកស្រាយ]] ត្រូវបានបើក និងកំណត់រចនាសម្ព័ន្ធ។

អថេរសំណួរប្រើវាក្យសម្ពន្ធ `{{"សង្ខេបនៃទំព័រ"}}`។ សញ្ញាសម្រង់ពីរជុំវិញសំណួរមានសារៈសំខាន់ និងបែងចែកសំណួរពីអថេរកំណត់ជាមុន។ ការឆ្លើយតបសំណួរអាចត្រូវបានដំណើរការបន្ថែមជាមួយ [[តម្រង]] ឧ. `{{"សង្ខេបនៃទំព័រ"|blockquote}}`។

### ពេលណាដែលគួរប្រើអថេរសំណួរ

អថេរសំណួរមានអត្ថប្រយោជន៍ក្នុងការមានភាពបត់បែនខ្ពស់ និងងាយស្រួលសរសេរ ប៉ុន្តែវាមានការផ្សំគ្នាជាច្រើន: វាដំណើរការយឺតជាង ហើយអាចមានការពិចារណាអំពីថ្លៃដើម និងឯកជនភាពអាស្រ័យលើ [[បកស្រាយទំព័របណ្តាញ#គំរូ|អ្នកផ្តល់សេវា]] ដែលអ្នកជ្រើសរើស។

មិនដូចប្រភេទអថេរផ្សេងទៀតទេ អថេរសំណួរត្រូវការដំណើរការដោយម៉ូដែលភាសាខាងក្រៅ ដូច្នេះពួកវាត្រូវបានជំនួសតែមួយដងប៉ុណ្ណោះនៅពេល [[បកស្រាយទំព័របណ្តាញ|កម្មវិធីបកស្រាយ]] បានដំណើរការ។

ល្អបំផុតគឺ *កុំ* ប្រើអថេរសំណួរប្រសិនបើទិន្នន័យដែលអ្នកចង់ស្រង់មានទម្រង់ស្របគ្នាដែលអាចស្រង់បានជាមួយប្រភេទអថេរផ្សេងទៀត។

ម្យ៉ាងវិញទៀត អថេរសំណួរអាចមានប្រយោជន៍ប្រសិនបើទិន្នន័យដែលអ្នកចង់ស្រង់មានទម្រង់ *មិនស្របគ្នា* នៅលើគេហទំព័រផ្សេងៗ។ ឧទាហរណ៍ អ្នកអាចបង្កើត [[ពុម្ព|ពុម្ព]] ដើម្បីរក្សាទុកសៀវភៅដែលមិនអាស្រ័យលើគេហទំព័រសៀវភៅជាក់លាក់។ អថេរសំណួរដូចជា `{{"អ្នកនិពន្ធសៀវភៅ"}}` នឹងដំណើរការលើគេហទំព័រសៀវភៅណាមួយ ចំណែកអថេរ selector ជាទូទៅដំណើរការសម្រាប់គេហទំព័រមួយប៉ុណ្ណោះ។

### ឧទាហរណ៍

សំណួរអាចប្រើការសួរជាភាសាធម្មជាតិស្ទើរតែគ្រប់ប្រភេទ។ អាស្រ័យលើម៉ូដែលដែលអ្នកប្រើ សំណួរអាចសួរ ឬបកប្រែទិន្នន័យឆ្លងភាសា។

- `{{"សង្ខេបជាចំណុចបីចំណុច បកប្រែទៅភាសាបារាំង"}}` ដើម្បីស្រង់ចំណុចសំខាន់អំពីទំព័រ ហើយបកប្រែទៅភាសាបារាំង។
- `{{"un resumé de la page en trois points"}}` ដើម្បីស្រង់ចំណុចបីចំណុចដោយប្រើសំណួរជាភាសាបារាំង។

សំណួរអាចបម្លែងមាតិកាទំព័រទៅជា JSON ដែលអាចរៀបចំបានជាមួយ [[តម្រង|តម្រង]]។ ឧទាហរណ៍:

```
{{"return a JSON object for each tweet, that includes the author, tweet_text, date in YYYY-MM-DD format, and images array (if there are any)"|map:tweet => ({text: tweet.tweet_text, author: tweet.author, date: tweet.date})|template:"${text}\n— [[@${author}]], [[${date}]]\n"}}
```


## អថេរ Meta

អថេរ Meta អនុញ្ញាតឱ្យអ្នកស្រង់ទិន្នន័យពី [ធាតុ meta](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/meta) នៅក្នុងទំព័រ រួមទាំងទិន្នន័យ [Open Graph](https://ogp.me/) ដែលប្រើដើម្បីបំពេញការមើលជាមុននៃការចែករំលែកបណ្ដាញសង្គម។

- `{{meta:name}}` ត្រឡប់មាតិកានៃ meta name tag ជាមួយឈ្មោះដែលបានផ្តល់ ឧ. `{{meta:name:description}}` សម្រាប់ meta tag `description`។
- `{{meta:property}}` ត្រឡប់មាតិកានៃ meta property tag ជាមួយលក្ខណៈសម្បត្តិដែលបានផ្តល់ ឧ. `{{meta:property:og:title}}` សម្រាប់ meta tag `og:title`។

## អថេរ Selector

អថេរ Selector អនុញ្ញាតឱ្យអ្នកស្រង់មាតិកាអត្ថបទពីធាតុនៅលើទំព័រដោយប្រើ [CSS selectors](https://developer.mozilla.org/en-US/docs/Web/CSS/CSS_selectors/Selectors_and_combinators)។

វាក្យសម្ពន្ធគឺ `{{selector:cssSelector?attribute}}` ដែល `?attribute` ជាជម្រើស។ ប្រសិនបើគ្មាន attribute ត្រូវបានបញ្ជាក់ទេ មាតិកាអត្ថបទរបស់ធាតុត្រូវបានត្រឡប់។ អ្នកក៏អាចប្រើ `{{selectorHtml:cssSelector}}` ដើម្បីទទួលមាតិកា HTML របស់ធាតុ។ អថេរ Selector មាននិន្នាការដំណើរការល្អបំផុតនៅលើគេហទំព័រជាក់លាក់ ឬសំណុំគេហទំព័រដែលមានរចនាសម្ព័ន្ធ HTML ស្របគ្នា។

- `{{selector:h1}}` ត្រឡប់មាតិកាអត្ថបទនៃធាតុ `h1` ណាមួយនៅលើទំព័រ។
- `{{selector:.author}}` ត្រឡប់មាតិកាអត្ថបទនៃធាតុ `.author` ណាមួយនៅលើទំព័រ។
- `{{selector:img.hero?src}}` ត្រឡប់ attribute `src` នៃរូបភាពជាមួយ class `hero`។
- `{{selector:a.main-link?href}}` ត្រឡប់ attribute `href` នៃ anchor tag ជាមួយ class `main-link`។
- `{{selectorHtml:body|markdown}}` ត្រឡប់ HTML ទាំងមូលរបស់ធាតុ `body` ដែលបម្លែងទៅ Markdown ដោយប្រើ [[តម្រង#ដំណើរការ HTML|តម្រង]] `markdown`។
- CSS selectors ជ្រុល និង combinators ត្រូវបានគាំទ្រប្រសិនបើអ្នកត្រូវការភាពជាក់លាក់បន្ថែម។
- ប្រសិនបើធាតុច្រើនផ្គូផ្គងជាមួយ selector នោះ array ត្រូវបានត្រឡប់ ដែលអ្នកអាចដំណើរការជាមួយ [[តម្រង#អារេ និងវត្ថុ|តម្រង array និងវត្ថុ]] ដូចជា `join` ឬ `map`។

អថេរ Selector ក៏អាចប្រើដោយផ្ទាល់នៅក្នុង [[តក្កវិទ្យា|តក្កវិទ្យាពុម្ព]] ផងដែរ:

- នៅក្នុង loops: `{% for comment in selector:.comment %}...{% endfor %}`
- នៅក្នុង conditionals: `{% if selector:.premium-badge %}...{% endif %}`
- នៅក្នុងការកំណត់អថេរ: `{% set items = selector:.list-item %}`

## អថេរ Schema.org

អថេរ Schema អនុញ្ញាតឱ្យអ្នកស្រង់ទិន្នន័យពី [schema.org](https://schema.org/) JSON-LD នៅលើទំព័រ។ ទិន្នន័យ Schema.org ក៏អាចប្រើដើម្បី [[ពុម្ព#ការផ្គូផ្គង Schema.org|បើកដំណើរការពុម្ព]] ដោយស្វ័យប្រវត្តិផងដែរ។

- `{{schema:@Type:key}}` ត្រឡប់តម្លៃនៃ key ពី schema។
- `{{schema:@Type:parent.child}}` ត្រឡប់តម្លៃនៃលក្ខណៈសម្បត្តិជ្រុល។
- `{{schema:@Type:arrayKey}}` ត្រឡប់ធាតុដំបូងនៅក្នុង array។
- `{{schema:@Type:arrayKey[index].property}}` ត្រឡប់ធាតុនៅ index ដែលបានបញ្ជាក់នៅក្នុង array។
- `{{schema:@Type:arrayKey[*].property}}` ត្រឡប់លក្ខណៈសម្បត្តិជាក់លាក់ពីធាតុទាំងអស់នៅក្នុង array។

អ្នកក៏អាចប្រើសញ្ញាណសង្ខេបដោយមិនបញ្ជាក់ប្រភេទ schema ផងដែរ:

- `{{schema:author}}` នឹងផ្គូផ្គងលក្ខណៈសម្បត្តិ `author` ដំបូងដែលរកឃើញនៅក្នុងប្រភេទ schema ណាមួយ។
- `{{schema:name}}` នឹងផ្គូផ្គងលក្ខណៈសម្បត្តិ `name` ដំបូងដែលរកឃើញនៅក្នុងប្រភេទ schema ណាមួយ។

សញ្ញាណសង្ខេបនេះមានប្រយោជន៍ជាពិសេសនៅពេលដែលអ្នកមិនដឹង ឬមិនខ្វល់អំពីប្រភេទ schema ជាក់លាក់ ប៉ុន្តែអ្នកដឹងឈ្មោះលក្ខណៈសម្បត្តិដែលអ្នកកំពុងស្វែងរក។

លក្ខណៈសម្បត្តិជ្រុល និងការចូលប្រើ array ដំណើរការផងដែរ ទាំងជាមួយ និងដោយមិនបញ្ជាក់ `@Type` schema:

- `{{schema:author.name}}` នឹងរកលក្ខណៈសម្បត្តិ `author` ដំបូង ហើយបន្ទាប់មកចូលប្រើលក្ខណៈសម្បត្តិរង `name` របស់វា។
- `{{schema:author[0].name}}` នឹងចូលប្រើ `name` នៃអ្នកនិពន្ធដំបូងនៅក្នុង array នៃអ្នកនិពន្ធ។
- `{{schema:author[*].name}}` នឹងត្រឡប់ array នៃឈ្មោះអ្នកនិពន្ធទាំងអស់។
