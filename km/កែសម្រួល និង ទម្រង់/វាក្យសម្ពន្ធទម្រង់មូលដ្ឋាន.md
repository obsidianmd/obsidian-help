រៀនពីរបៀបអនុវត្តការធ្វើទ្រង់ទ្រាយមូលដ្ឋានចំពោះកំណត់ត្រារបស់អ្នក ដោយប្រើ [Markdown](https://daringfireball.net/projects/markdown/)។ សម្រាប់វាក្យសម្ពន្ធទម្រង់កម្រិតខ្ពស់ សូមមើល [[វាក្យសម្ពន្ធទម្រង់កម្រិតខ្ពស់]]។

## កថាខណ្ឌ

ដើម្បីបង្កើតកថាខណ្ឌក្នុង Markdown ប្រើ **បន្ទាត់ទទេ** ដើម្បីញែកប្លុកអត្ថបទ។ ប្លុកអត្ថបទនីមួយៗដែលញែកដោយបន្ទាត់ទទេ ត្រូវបានចាត់ទុកជាកថាខណ្ឌដាច់ដោយឡែក។

```md
This is a paragraph.

This is another paragraph.
```

This is a paragraph.

This is another paragraph.

បន្ទាត់ទទេរវាងបន្ទាត់អត្ថបទបង្កើតកថាខណ្ឌដាច់ដោយឡែក។ នេះជាឥរិយាបថលំនាំដើមក្នុង Markdown។

> [!tip]- ចន្លោះទទេច្រើន

>

>

> ប្រសិនបើអ្នកចង់ការពារចន្លោះពីការរួបរួម ឬបន្ថែមចន្លោះទទេច្រើន អ្នកអាចប្រើស្លាក HTML `&nbsp;` (ចន្លោះមិនបំបែក) ឬ `<br>` (បំបែកបន្ទាត់)។

### បំបែកបន្ទាត់

តាមលំនាំដើមក្នុង Obsidian ការចុច `Enter` មួយដងនឹងបង្កើតបន្ទាត់ថ្មីក្នុងកំណត់ត្រារបស់អ្នក ប៉ុន្តែវាត្រូវបានចាត់ទុកជា *ការបន្ត* នៃកថាខណ្ឌដដែលក្នុងលទ្ធផលបង្ហាញ តាមឥរិយាបថ Markdown ធម្មតា។ ដើម្បីបញ្ចូលបំបែកបន្ទាត់ *ក្នុង* កថាខណ្ឌដោយមិនចាប់ផ្ដើមកថាខណ្ឌថ្មី អ្នកអាច៖

- បន្ថែម **ចន្លោះពីរ** នៅចុងបន្ទាត់មុនចុច `Enter` ឬ
- ប្រើផ្លូវកាត់ `Shift+Enter` ដើម្បីបញ្ចូលបំបែកបន្ទាត់ដោយផ្ទាល់។

> [!question]- ហេតុអ្វីការចុច `Enter` ច្រើនដងមិនបង្កើតបំបែកបន្ទាត់ច្រើននៅក្នុង reading view?
> ក្នុង Markdown ការចុច `Enter` តែមួយត្រូវបានមិនអើពើ ហើយការចុច `Enter` ច្រើនដងជាប់គ្នាបណ្ដាលឱ្យមានកថាខណ្ឌថ្មីតែមួយប៉ុណ្ណោះ។ ឥរិយាបថនេះស្របនឹងច្បាប់ soft wrap របស់ Markdown ដែលបន្ទាត់ទទេបន្ថែមមិនបង្កើតបំបែកបន្ទាត់ ឬកថាខណ្ឌបន្ថែម ទេ ពួកវាត្រូវបានរួបរួមជាការបំបែកកថាខណ្ឌតែមួយ។ នេះជារបៀបដែល Markdown ដោះស្រាយអត្ថបទតាមលំនាំដើម ដោយធានាថាកថាខណ្ឌហូរទៅដោយធម្មជាតិដោយគ្មានការបំបែកដែលមិននឹកស្មាន។

Obsidian មានការកំណត់ **[[ការកំណត់#Strict line breaks|Strict line breaks]]** ដែលធ្វើឱ្យ Obsidian ធ្វើតាមលក្ខណៈបច្ចេកទេស Markdown ស្ដង់ដារសម្រាប់បំបែកបន្ទាត់។

ដើម្បីបើកមុខងារនេះ៖

1. បើក **[[ការកំណត់]]**។
2. ទៅផ្ទាំង **Editor**។
3. បើក **Strict Line Breaks**។

នៅពេល **Strict Line Breaks** ត្រូវបានបើកក្នុង Obsidian ការបំបែកបន្ទាត់មានឥរិយាបថបីយ៉ាងខុសគ្នា អាស្រ័យលើរបៀបញែកបន្ទាត់៖

**ចុច Enter តែម្ដងដោយគ្មានចន្លោះ**៖ ការចុច `Enter` តែម្ដងដោយគ្មានចន្លោះបន្ទាប់នឹងបញ្ចូលរួមបន្ទាត់ដាច់ដោយឡែកពីរជាបន្ទាត់តែមួយ នៅពេលបង្ហាញ។

```md
line one
line two
```

បង្ហាញជា៖

line one line two

**ចុច Enter តែម្ដងដោយមានចន្លោះពីរ ឬច្រើននៅខាងចុង**៖ ប្រសិនបើអ្នកបន្ថែមចន្លោះពីរ ឬច្រើននៅចុងបន្ទាត់ទីមួយ មុនចុច `Enter` បន្ទាត់ទាំងពីររក្សាជាកថាខណ្ឌតែមួយ ប៉ុន្តែបំបែកដោយបំបែកបន្ទាត់ (ធាតុ HTML `<br>`)។ យើងនឹងប្រើ underscore ពីរ ជំនួសចន្លោះក្នុងឧទាហរណ៍នេះ។

```md
line three__  
line four
```

បង្ហាញជា៖

line three<br>
line four

**ចុច Enter ពីដង (ជាមួយ ឬគ្មានចន្លោះខាងចុង)**៖ ការចុច `Enter` ពីដង (ឬច្រើន) ញែកបន្ទាត់ជាកថាខណ្ឌដាច់ដោយឡែកពីរ (ធាតុ HTML `<p>`) ដោយមិនអើពើថាតើអ្នកបន្ថែមចន្លោះនៅចុងបន្ទាត់ទីមួយឬអត់។

```md
line five

line six
```

បង្ហាញជា៖

<p>line five</p>
<p>line six</p>

## ចំណងជើង

ដើម្បីបង្កើតចំណងជើង ដាក់និមិត្តសញ្ញា `#` រហូតដល់ប្រាំមួយ មុនអត្ថបទចំណងជើងរបស់អ្នក។ ចំនួននិមិត្តសញ្ញា `#` កំណត់កម្រិតចំណងជើង (ដូចបង្ហាញក្នុង [[គ្រោង]])។

```md
# This is a heading 1
## This is a heading 2
### This is a heading 3
#### This is a heading 4
##### This is a heading 5
###### This is a heading 6
```

%% These headings use HTML to avoid cluttering the Outline/Table of contents %%
<h1>This is a heading 1</h1>
<h2>This is a heading 2</h2>
<h3>This is a heading 3</h3>
<h4>This is a heading 4</h4>
<h5>This is a heading 5</h5>
<h6>This is a heading 6</h6>

## ដិត អក្សរទ្រេត ពណ៌ចំណាំ

ការធ្វើទ្រង់ទ្រាយអត្ថបទក៏អាចអនុវត្តបានដោយប្រើ [[ផ្លូវកាត់កែសម្រួល]]។

| រចនាប័ទ្ម | វាក្យសម្ពន្ធ | ឧទាហរណ៍ | លទ្ធផល |
|-|-|-|-|
| ដិត | `** **` ឬ `__ __` | `**Bold text**` | **Bold text** |
| អក្សរទ្រេត | `* *` ឬ `_ _` | `*Italic text*` | *Italic text* |
| គូសសម្គាល់ | `~~ ~~` | `~~Striked out text~~` | ~~Striked out text~~ |
| ពណ៌ចំណាំ | `== ==` | `==Highlighted text==` | ==Highlighted text== |
| ដិត និងអក្សរទ្រេតក្នុង | `** **` និង `_ _` | `**Bold text and _nested italic_ text**` | **Bold text and _nested italic_ text** |
| ដិត និងអក្សរទ្រេត | `*** ***` ឬ `___ ___` | `***Bold and italic text***` | ***Bold and italic text*** |

ការធ្វើទ្រង់ទ្រាយអាចបង្ខំឱ្យបង្ហាញជាអត្ថបទធម្មតា ដោយបន្ថែម backslash `\` នៅខាងមុខ។

\*\*This line will not be bold\*\*

```markdown
\*\*This line will not be bold\*\*
```

\**This line will be italic and show the asterisks*\*

```markdown
\**This line will be italic and show the asterisks*\*
```

## តំណភ្ជាប់ផ្ទៃក្នុង

Obsidian គាំទ្រទ្រង់ទ្រាយពីរសម្រាប់ [[តំណភ្ជាប់ផ្ទៃក្នុង]] រវាងកំណត់ត្រា៖

- Wikilink: `[[Three laws of motion]]`
- Markdown: `[Three laws of motion](Three%20laws%20of%20motion.md)`

## តំណភ្ជាប់ខាងក្រៅ

ប្រសិនបើអ្នកចង់តំណភ្ជាប់ទៅ URL ខាងក្រៅ អ្នកអាចបង្កើតតំណភ្ជាប់ inline ដោយរុំអត្ថបទតំណភ្ជាប់ក្នុងតង្កៀប (`[ ]`) ហើយ URL ក្នុងវង់ក្រចក (`( )`)។

```md
[Obsidian Help](https://help.obsidian.md)
```

[Obsidian Help](https://help.obsidian.md)

អ្នកក៏អាចបង្កើតតំណភ្ជាប់ខាងក្រៅទៅឯកសារក្នុង vault ផ្សេង ដោយតំណភ្ជាប់ទៅ [[Obsidian URI|Obsidian URI]]។

```md
[Note](obsidian://open?vault=MainVault&file=Note.md)
```

### គេចចន្លោះទទេក្នុងតំណភ្ជាប់

```md

```

អ្នកក៏អាចគេច URL ដោយរុំវាដោយតង្កៀបមុំ (`< >`)។

## កំណត់ចុងក្រោយ

អ្នកអាចបន្ថែមកំណត់ចុងក្រោយ[^footnote] ទៅកំណត់ត្រារបស់អ្នក ដោយប្រើវាក្យសម្ពន្ធខាងក្រោម៖

[^footnote]: This is a footnote.

```md
This is a simple footnote[^1].

[^1]: This is the referenced text.
[^2]: Add 2 spaces at the start of each new line.
  This lets you write footnotes that span multiple lines.
[^note]: Named footnotes still appear as numbers, but can make it easier to identify and link references.
```

## ការគេចវាក្យសម្ពន្ធ Markdown

ក្នុងករណីខ្លះ អ្នកអាចត្រូវការបង្ហាញតួអក្សរពិសេសក្នុង Markdown ដូចជា `*`, `_`, ឬ `#` ដោយមិនដំណើរការការធ្វើទ្រង់ទ្រាយរបស់ពួកវា។ ដើម្បីបង្ហាញតួអក្សរទាំងនេះដោយផ្ទាល់ ដាក់ backslash (`\`) មុនពួកវា។

> [!example] តួអក្សរទូទៅដែលត្រូវគេច

> - Asterisk: `\*`
> - Underscore: `\_`
> - Hashtag: `\#`
> - Backtick: `` \` ``
> - Pipe (ប្រើក្នុងតារាង): `\|`
> - Tilde: `\~`

