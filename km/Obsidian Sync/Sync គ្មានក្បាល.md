[[ការណែនាំអំពី Obsidian Sync|Obsidian Sync]] ផ្តល់ម៉ាស៊ីនភ្ញៀវ headless សម្រាប់សមកាលកម្ម vault ដោយមិនប្រើកម្មវិធីកុំព្យូទ័រ។ មានប្រយោជន៍សម្រាប់ CI pipelines, agents, និង workflow ស្វ័យប្រវត្តិ។ សមកាលកម្មការផ្លាស់ប្តូរចុងក្រោយ ឬរក្សាឯកសារឱ្យទាន់សម័យជាបន្តបន្ទាប់។

ដំឡើង [[Obsidian Headless]] **(open beta)** ដើម្បីធ្វើអន្តរកម្មជាមួយ [[ការណែនាំអំពី Obsidian Sync|Obsidian Sync]] ពីបន្ទាត់ពាក្យបញ្ជាដោយមិនប្រើកម្មវិធី Obsidian សម្រាប់កុំព្យូទ័រ។ Headless Sync ប្រើ[[សុវត្ថិភាព និងឯកជនភាព|ការការពារការអ៊ិនគ្រីប និងឯកជនភាព]]ដូចគ្នាទៅនឹងកម្មវិធីកុំព្យូទ័រ រួមទាំងការអ៊ិនគ្រីបពីចុងដល់ចុង។

## ចាប់ផ្តើមរហ័ស

> [!error] បម្រុងទុកទិន្នន័យរបស់អ្នកមុនពេលចាប់ផ្តើម
> 1. បម្រុងទុកទិន្នន័យរបស់អ្នកជានិច្ចមុនពេលចាប់ផ្តើម ក្នុងករណីមានអ្វីមិនរំពឹង។
> 2. កុំប្រើ Sync នៃកម្មវិធីកុំព្យូទ័រ *និង* Headless Sync ទាំងពីរលើឧបករណ៍តែមួយ ព្រោះវាអាចបណ្តាលឱ្យមានជម្លោះទិន្នន័យ។ ប្រើវិធីសមកាលកម្មតែមួយក្នុងមួយឧបករណ៍។

ដំឡើង [[Obsidian Headless|Obsidian Headless]] **(open beta)**:

```shell
npm install -g obsidian-headless
```

អ្នកត្រូវមាន[[គម្រោង និងដែនកំណត់ទំហំផ្ទុក|ការជាវ Obsidian Sync]]ដែលសកម្ម។

```shell
# ចូលគណនី
ob login

# បង្ហាញ vault ពីចម្ងាយរបស់អ្នក
ob sync-list-remote

# រៀបចំ vault សម្រាប់សមកាលកម្ម
cd ~/vaults/my-vault
ob sync-setup --vault "My Vault"

# ដំណើរការសមកាលកម្មម្តង
ob sync

# ដំណើរការសមកាលកម្មជាបន្តបន្ទាប់ (តាមដានការផ្លាស់ប្តូរ)
ob sync --continuous
```

## ពាក្យបញ្ជា

### `ob sync-list-remote`

បង្ហាញ vault ពីចម្ងាយទាំងអស់ដែលមានក្នុងគណនីរបស់អ្នក រួមទាំង vault ដែលចែករំលែក។

### `ob sync-list-local`

បង្ហាញ vault ដែលបានកំណត់នៅលើម៉ាស៊ីន និងផ្លូវរបស់វា។

### `ob sync-create-remote`

បង្កើត vault ពីចម្ងាយថ្មី។

```
ob sync-create-remote --name "Vault Name" [--encryption <standard|e2ee>] [--password <password>] [--region <region>]
```

| ជម្រើស | ការពិពណ៌នា |
| --- | --- |
| `--name` | ឈ្មោះ vault (ចាំបាច់) |
| `--encryption` | `standard` សម្រាប់ការអ៊ិនគ្រីបដែលគ្រប់គ្រង, `e2ee` សម្រាប់ការអ៊ិនគ្រីបពីចុងដល់ចុង |
| `--password` | ពាក្យសម្ងាត់ការអ៊ិនគ្រីបពីចុងដល់ចុង (នឹងសួរប្រសិនបើមិនបានដាក់) |
| `--region` | [[តំបន់ Sync\|តំបន់]]ម៉ាស៊ីនមេ (ស្វ័យប្រវត្តិប្រសិនបើមិនបានដាក់) |

### `ob sync-setup`

រៀបចំសមកាលកម្មរវាង vault មូលដ្ឋាន និង vault ពីចម្ងាយ។

```
ob sync-setup --vault <id-or-name> [--path <local-path>] [--password <password>] [--device-name <name>] [--config-dir <name>]
```

| ជម្រើស | ការពិពណ៌នា |
| --- | --- |
| `--vault` | ID ឬឈ្មោះ vault ពីចម្ងាយ (ចាំបាច់) |
| `--path` | ថតមូលដ្ឋាន (លំនាំដើម: ថតបច្ចុប្បន្ន) |
| `--password` | ពាក្យសម្ងាត់ការអ៊ិនគ្រីប E2E (នឹងសួរប្រសិនបើមិនបានដាក់) |
| `--device-name` | ឈ្មោះឧបករណ៍ដែលបង្ហាញក្នុង[[ប្រវត្តិកំណែ\|ប្រវត្តិកំណែសមកាលកម្ម]] |
| `--config-dir` | ឈ្មោះ[[ថតកំណត់រចនាសម្ព័ន្ធ\|ថតកំណត់រចនាសម្ព័ន្ធ]] (លំនាំដើម: `.obsidian`) |

### `ob sync`

ដំណើរការសមកាលកម្មសម្រាប់ vault ដែលបានកំណត់រចនាសម្ព័ន្ធ។

```
ob sync [--path <local-path>] [--continuous]
```

| ជម្រើស | ការពិពណ៌នា |
| --- | --- |
| `--path` | ផ្លូវ vault មូលដ្ឋាន (លំនាំដើម: ថតបច្ចុប្បន្ន) |
| `--continuous` | ដំណើរការជាបន្តបន្ទាប់ តាមដានការផ្លាស់ប្តូរ |

### `ob sync-config`

មើល ឬផ្លាស់ប្តូរ[[ការកំណត់ Sync និងជ្រើសរើorg សមកាលកម្ម|ការកំណត់សមកាលកម្ម]]សម្រាប់ vault។ ដំណើរការដោយគ្មានជម្រើសដើម្បីបង្ហាញការកំណត់រចនាសម្ព័ន្ធបច្ចុប្បន្ន។

```
ob sync-config [--path <local-path>] [options]
```

| ជម្រើស                | ការពិពណ៌នា                                                                                                                                                                                                    |
| --------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `--path`              | ផ្លូវ vault មូលដ្ឋាន (លំនាំដើម: ថតបច្ចុប្បន្ន)                                                                                                                                                                  |
| `--mode`              | របៀបសមកាលកម្ម: `bidirectional` (លំនាំដើម), `pull-only` (ទាញយកតែប៉ុណ្ណោះ មិនអើពើការផ្លាស់ប្តូរមូលដ្ឋាន), ឬ `mirror-remote` (ទាញយកតែប៉ុណ្ណោះ ត្រឡប់ការផ្លាស់ប្តូរមូលដ្ឋានវិញ)                                                              |
| `--conflict-strategy` | `merge` ឬ `conflict`                                                                                                                                                                                          |
| `--file-types`        | ប្រភេទឯកសារភ្ជាប់ដែលត្រូវសមកាលកម្ម: `image`, `audio`, `video`, `pdf`, `unsupported` (បំបែកដោយសញ្ញាក្បៀស, ទទេដើម្បីលុប)                                                                                                    |
| `--configs`           | ប្រភេទកំណត់រចនាសម្ព័ន្ធដែលត្រូវសមកាលកម្ម: `app`, `appearance`, `appearance-data`, `hotkey`, `core-plugin`, `core-plugin-data`, `community-plugin`, `community-plugin-data` (បំបែកដោយសញ្ញាក្បៀស, ទទេដើម្បីបិទសមកាលកម្មកំណត់រចនាសម្ព័ន្ធ) |
| `--excluded-folders`  | ថតដែលត្រូវដកចេញ (បំបែកដោយសញ្ញាក្បៀស, ទទេដើម្បីលុប)                                                                                                                                                           |
| `--device-name`       | ឈ្មោះឧបករណ៍ដើម្បីកំណត់អត្តសញ្ញាណម៉ាស៊ីនភ្ញៀវនេះក្នុងប្រវត្តិកំណែសមកាលកម្ម                                                                                                                                                |
| `--config-dir`        | ឈ្មោះថតកំណត់រចនាសម្ព័ន្ធ (លំនាំដើម: `.obsidian`)                                                                                                                                                                   |

### `ob sync-status`

បង្ហាញស្ថានភាពសមកាលកម្ម និងការកំណត់រចនាសម្ព័ន្ធសម្រាប់ vault។

```
ob sync-status [--path <local-path>]
```

### `ob sync-unlink`

ផ្តាច់ vault ពីសមកាលកម្ម និងលុបព័ត៌មានសម្ងាត់ដែលបានរក្សាទុក។

```
ob sync-unlink [--path <local-path>]
```

## ម៉ូឌុលដើម

Obsidian Headless រួមបញ្ចូល addon ដើមដែលបានបង្កើតរួចសម្រាប់កំណត់ពេលវេលាបង្កើតឯកសារ (birthtime) នៅលើ Windows និង macOS។ នេះរក្សាត្រាពេលវេលាបង្កើតដើមនៅពេលទាញយកឯកសារពីម៉ាស៊ីនមេ។

addon នេះកំណត់គោលដៅ N-API កំណែ 3 ដូច្នេះ binary ដែលបានចងក្រងគឺមានស្ថិរភាព ABI និងដំណើរការឆ្លងកំណែ Node.js ដោយមិនចាំបាច់ចងក្រងឡើងវិញ។

នៅលើ Linux, birthtime មិនត្រូវបានគាំទ្រ — addon មិនត្រូវបានរួមបញ្ចូល ហើយសមកាលកម្មដំណើរការធម្មតាដោយគ្មានវា។

Binary ដែលបានបង្កើតរួចត្រូវបានរួមបញ្ចូលសម្រាប់:

- `win32-x64`
- `win32-arm64`
- `win32-ia32`
- `darwin-x64`
- `darwin-arm64`
