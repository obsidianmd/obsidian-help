[[Obsidian Web Clipper/תבניות|תבניות Web Clipper]] יכולות להשתמש במשתנים כדי לאכלס אוטומטית נתונים מהדף בתוך תבנית. ניתן להשתמש במשתנים ב**שם הפתק**, **מיקום ההערה**, **מאפיינים** ו**תוכן הפתק**. ניתן גם לשנות משתנים באמצעות [[מסננים|מסננים]].

השתמשו בסמל ה-`...` בתוסף [[מבוא ל-Obsidian Web Clipper|Web Clipper]] כדי לגשת למשתני העמוד הנוכחי לשימוש בתבניות. ישנם חמישה סוגי משתנים שניתן להשתמש בהם:

## משתנים מוכנים מראש

משתנים מוכנים מראש נוצרים אוטומטית על בסיס תוכן הדף. אלו בדרך כלל עובדים עבור רוב האתרים.

משתנה התוכן העיקרי הוא `{{content}}`, המכיל את תוכן המאמר, או את [[מרקר|ההדגשות]], או את הבחירה אם יש טקסט נבחר בדף. שימו לב ש-`{{content}}` מנסה לחלץ את התוכן העיקרי של הדף, מה שלא תמיד עשוי להיות מה שאתם רוצים. במקרה כזה, ניתן להשתמש במשתנים מוכנים מראש אחרים או במשתני סלקטור כדי לחלץ את התוכן הדרוש.

| משתנה            | תיאור                                                                            |
| ------------------- | -------------------------------------------------------------------------------------- |
| `{{author}}`        | מחבר הדף                                                                     |
| `{{content}}`       | תוכן המאמר, [[מרקר\|הדגשות]], או בחירה, בפורמט Markdown |
| `{{contentHtml}}`   | תוכן המאמר, [[מרקר\|הדגשות]], או בחירה, בפורמט HTML     |
| `{{date}}`          | תאריך נוכחי, ניתן לעצב באמצעות מסנן `date`                                 |
| `{{description}}`   | תיאור או תקציר                                                                 |
| `{{domain}}`        | דומיין                                                                                 |
| `{{favicon}}`       | כתובת URL של Favicon                                                            |
| `{{fullHtml}}`      | HTML לא מעובד של תוכן הדף המלא                                             |
| `{{highlights}}`    | [[מרקר\|הדגשות]] עם טקסט וחותמות זמן                           |
| `{{image}}`         | כתובת URL של תמונת שיתוף חברתי                                                 |
| `{{published}}`     | תאריך פרסום, ניתן לעצב באמצעות מסנן `date`                               |
| `{{selection}}`     | בחירה בפורמט Markdown                                                           |
| `{{selectionHtml}}` | בחירה בפורמט HTML                                                               |
| `{{site}}`          | שם האתר או המפרסם                                                 |
| `{{title}}`         | כותרת הדף                                                                      |
| `{{time}}`          | תאריך ושעה נוכחיים                                                                  |
| `{{url}}`           | כתובת URL נוכחית                                                            |
| `{{words}}`         | ספירת מילים                                                                             |

## משתני פרומפט

משתני פרומפט ממנפים מודלים לשוניים כדי לחלץ ולשנות נתונים באמצעות שפה טבעית. משתני פרומפט דורשים ש[[מתרגם|מתרגם]] יהיה מופעל ומוגדר.

משתני פרומפט משתמשים בתחביר `{{"סיכום של הדף"}}`. המירכאות הכפולות סביב הפרומפט חשובות ומבדילות בין פרומפטים למשתנים מוכנים מראש. תגובות פרומפט ניתנות לעיבוד נוסף עם [[מסננים|מסננים]], לדוגמה `{{"סיכום של הדף"|blockquote}}`.

### מתי להשתמש במשתני פרומפט

למשתני פרומפט יש את היתרון של גמישות קיצונית וקלות כתיבה, אולם הם מגיעים עם מספר פשרות: הם איטיים יותר להרצה, ועשויים להיות בעלי שיקולי עלות ופרטיות בהתאם ל[[מתרגם#דגמים|ספק]] שתבחרו.

בניגוד לסוגי משתנים אחרים, משתני פרומפט צריכים לעבור עיבוד על ידי מודל שפה חיצוני, ולכן הם מוחלפים רק לאחר ש[[מתרגם|מתרגם]] הורץ.

מומלץ *לא* להשתמש במשתני פרומפט אם הנתונים שאתם רוצים לחלץ הם בפורמט עקבי שניתן לחלצו באמצעות סוגי משתנים אחרים.

מצד שני, משתני פרומפט יכולים להיות שימושיים אם הנתונים שאתם רוצים לחלץ הם בפורמט *לא עקבי* בין אתרים שונים. לדוגמה, ניתן ליצור [[Obsidian Web Clipper/תבניות|תבנית]] לשמירת ספרים שאינה תלויה באתר הספרים. משתני פרומפט כמו `{{"מחבר הספר"}}` יעבדו בכל אתר ספרים, בעוד שמשתני סלקטור בדרך כלל עובדים רק עבור אתר אחד.

### דוגמאות

פרומפטים יכולים להשתמש כמעט בכל שאילתת שפה טבעית. בהתאם למודל שאתם משתמשים בו, פרומפטים יכולים לשאול או לתרגם נתונים בין שפות.

- `{{"סיכום בשלוש נקודות, מתורגם לצרפתית"}}` כדי לחלץ נקודות תבליט על הדף ולתרגם אותן לצרפתית.
- `{{"un resumé de la page en trois points"}}` כדי לחלץ שלוש נקודות תבליט באמצעות פרומפט בצרפתית.

פרומפטים יכולים להפוך תוכן דף ל-JSON שניתן לעבד עם [[מסננים|מסננים]]. לדוגמה:

```
{{"return a JSON object for each tweet, that includes the author, tweet_text, date in YYYY-MM-DD format, and images array (if there are any)"|map:tweet => ({text: tweet.tweet_text, author: tweet.author, date: tweet.date})|template:"${text}\n— [[@${author}]], [[${date}]]\n"}}
```


## משתני Meta

משתני Meta מאפשרים לכם לחלץ נתונים מ[אלמנטי meta](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/meta) בדף, כולל נתוני [Open Graph](https://ogp.me/) המשמשים לאכלוס תצוגות מקדימות של שיתוף חברתי.

- `{{meta:name}}` מחזיר את התוכן של תגית meta name עם השם הנתון, לדוגמה `{{meta:name:description}}` עבור תגית ה-meta של `description`.
- `{{meta:property}}` מחזיר את התוכן של תגית meta property עם המאפיין הנתון, לדוגמה `{{meta:property:og:title}}` עבור תגית ה-meta של `og:title`.

## משתני סלקטור

משתני סלקטור מאפשרים לכם לחלץ תוכן טקסט מאלמנטים בדף באמצעות [סלקטורי CSS](https://developer.mozilla.org/en-US/docs/Web/CSS/CSS_selectors/Selectors_and_combinators).

התחביר הוא `{{selector:cssSelector?attribute}}`, כאשר `?attribute` הוא אופציונלי. אם לא צוין מאפיין, תוכן הטקסט של האלמנט מוחזר. ניתן גם להשתמש ב-`{{selectorHtml:cssSelector}}` כדי לקבל את תוכן ה-HTML של האלמנט. משתני סלקטור נוטים לעבוד הכי טוב על אתר ספציפי או קבוצת אתרים בעלי מבנה HTML עקבי.

- `{{selector:h1}}` מחזיר תוכן טקסט של כל אלמנטי `h1` בדף.
- `{{selector:.author}}` מחזיר תוכן טקסט של כל אלמנטי `.author` בדף.
- `{{selector:img.hero?src}}` מחזיר את מאפיין `src` של התמונה עם מחלקת `hero`.
- `{{selector:a.main-link?href}}` מחזיר את מאפיין `href` של תגית העוגן עם מחלקת `main-link`.
- `{{selectorHtml:body|markdown}}` מחזיר את כל ה-HTML של אלמנט `body`, מומר ל-Markdown באמצעות [[מסננים#עיבוד HTML|מסנן]] `markdown`.
- סלקטורי CSS מקוננים וקומבינטורים נתמכים אם אתם צריכים יותר ספציפיות.
- אם מספר אלמנטים תואמים את הסלקטור, מוחזר מערך, שניתן לעבד עם [[מסננים#מערכים ואובייקטים|מסנני מערכים ואובייקטים]] כמו `join` או `map`.

משתני סלקטור יכולים גם לשמש ישירות ב[[לוגיקה|לוגיקת תבנית]]:

- בלולאות: `{% for comment in selector:.comment %}...{% endfor %}`
- בתנאים: `{% if selector:.premium-badge %}...{% endif %}`
- בהקצאת משתנים: `{% set items = selector:.list-item %}`

## משתני Schema.org

משתני Schema מאפשרים לכם לחלץ נתונים מ-[schema.org](https://schema.org/) JSON-LD בדף. נתוני Schema.org יכולים גם לשמש כדי [[Obsidian Web Clipper/תבניות#Schema.org matching|להפעיל תבנית]] אוטומטית.

- `{{schema:@Type:key}}` מחזיר את הערך של המפתח מה-schema.
- `{{schema:@Type:parent.child}}` מחזיר את הערך של מאפיין מקונן.
- `{{schema:@Type:arrayKey}}` מחזיר את הפריט הראשון במערך.
- `{{schema:@Type:arrayKey[index].property}}` מחזיר את הפריט באינדקס המצוין במערך.
- `{{schema:@Type:arrayKey[*].property}}` מחזיר מאפיין ספציפי מכל הפריטים במערך.

ניתן גם להשתמש בתחביר מקוצר ללא ציון סוג ה-schema:

- `{{schema:author}}` יתאים למאפיין `author` הראשון שנמצא בכל סוג schema.
- `{{schema:name}}` יתאים למאפיין `name` הראשון שנמצא בכל סוג schema.

תחביר מקוצר זה שימושי במיוחד כאשר אתם לא יודעים או לא מעוניינים בסוג ה-schema הספציפי, אבל אתם יודעים את שם המאפיין שאתם מחפשים.

מאפיינים מקוננים וגישה למערכים עובדים גם כן, גם עם וגם בלי ציון `@Type` של ה-schema:

- `{{schema:author.name}}` ימצא את מאפיין `author` הראשון ואז יגש למאפיין המשנה `name` שלו.
- `{{schema:author[0].name}}` יגש ל-`name` של המחבר הראשון במערך של מחברים.
- `{{schema:author[*].name}}` יחזיר מערך של כל שמות המחברים.
