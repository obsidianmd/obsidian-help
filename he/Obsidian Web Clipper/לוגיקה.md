[[מבוא ל-Obsidian Web Clipper|Web Clipper]] תומך בלוגיקת תבניות עבור תנאים, לולאות והקצאת משתנים. תחביר זה מושפע משפות התבניות [Twig](https://twig.symfony.com/) ו-[Liquid](https://shopify.github.io/liquid/).


> [!warning] דרושה הגרסה האחרונה
> תכונות לוגיקה דורשות Obsidian Web Clipper 1.0.0 שטרם אושר בכל חנויות ההרחבות.

## תנאים

השתמשו ב-`{% if %}` כדי לכלול תוכן באופן מותנה בהתבסס על משתנים או ביטויים.

```twig
{% if author %}
Author: {{author}}
{% endif %}
```

השתמשו ב-`{% else %}` כדי לספק תוכן חלופי, וב-`{% elseif %}` כדי לשרשר מספר תנאים:

```twig
{% if status == "published" %}
Live article
{% elseif status == "draft" %}
Draft article
{% else %}
Unknown status
{% endif %}
```

### אופרטורי השוואה

אופרטורי ההשוואה הבאים נתמכים:

| אופרטור | תיאור |
|----------|-------------|
| `==` | שווה ל |
| `!=` | לא שווה ל |
| `>` | גדול מ |
| `<` | קטן מ |
| `>=` | גדול מ או שווה ל |
| `<=` | קטן מ או שווה ל |
| `contains` | בדיקה אם מחרוזת מכילה תת-מחרוזת, או מערך מכיל ערך |

דוגמאות:
- `{% if title == "Home" %}` — שוויון מחרוזות
- `{% if price >= 100 %}` — השוואה מספרית
- `{% if title contains "Review" %}` — בדיקת תת-מחרוזת
- `{% if tags contains "important" %}` — חברות במערך

### אופרטורים לוגיים

שלבו תנאים באמצעות אופרטורים לוגיים:

| אופרטור | חלופה | תיאור                         |
| -------- | ----------- | ----------------------------------- |
| `and`    | `&&`        | שני התנאים חייבים להיות אמת        |
| `or`     | \|\|        | לפחות תנאי אחד חייב להיות אמת |
| `not`    | `!`         | שולל תנאי                 |

דוגמאות:
- `{% if author and published %}` — שניהם חייבים להתקיים
- `{% if draft or archived %}` — אחד מהתנאים
- `{% if not hidden %}` — שלילה
- `{% if (premium or featured) and published %}` — תנאים מקובצים

### ערכי אמת

כאשר משתנה נמצא בשימוש ללא אופרטור השוואה, הוא מוערך לפי "ערכי אמת":

- `false`, `null`, `undefined`, מחרוזת ריקה `""`, ו-`0` נחשבים **כוזבים (falsy)**.
- מערכים ריקים `[]` נחשבים **כוזבים (falsy)**.
- כל השאר נחשב **אמיתי (truthy)**.

```twig
{% if content %}
Has content
{% endif %}
```

## הקצאת משתנה

השתמשו ב-`{% set %}` כדי ליצור או לשנות משתנים בתוך התבנית שלכם:

```twig
{% set slug = title|lower|replace:" ":"-" %}
File: {{slug}}.md
```

ניתן להגדיר משתנים ל:
- משתנים אחרים: `{% set name = author %}`
- ליטרלים: `{% set count = 5 %}` או `{% set label = "Draft" %}`
- ביטויים עם מסננים: `{% set excerpt = content|truncate:100 %}`
- תוצאות סלקטור: `{% set comments = selector:.comment %}`

משתנים שהוגדרו עם `{% set %}` ניתנים לשימוש בלוגיקת תבנית עוקבת ובפלט `{{variable}}`.

## ערכים חלופיים

השתמשו באופרטור `??` כדי לספק ערכים חלופיים כאשר משתנה ריק או לא מוגדר:

```twig
{{title ?? "Untitled"}}
```

אם `title` ריק, לא מוגדר, או כוזב, הערך החלופי `"Untitled"` ישמש במקומו.

זוהי קיצור עבור הוראת ה-`if` המקבילה:

```twig
{% if title %}{{title}}{% else %}Untitled{% endif %}
```

### שרשור ערכים חלופיים

ניתן לשרשר מספר ערכים חלופיים:

```twig
{{title ?? headline ?? "No title"}}
```

זה ישתמש ב-`title` אם זמין, אחרת ב-`headline`, אחרת במחרוזת `"No title"`.

### עם מסננים

מסננים נקשרים חזק יותר מ-`??`, כך שמסננים מיושמים לפני בדיקת הערך החלופי:

```twig
{{title|upper ?? "UNTITLED"}}
```

זה מיישם `upper` על `title` תחילה, ואז חוזר ל-`"UNTITLED"` אם התוצאה ריקה. כדי ליישם מסננים על הערך החלופי, השתמשו בסוגריים או בביטויים נפרדים:

```twig
{{title ?? "Untitled"|lower}}
```

זה ישתמש ב-`title` אם זמין, אחרת ייישם `lower` על הערך החלופי, ויתקבל `"untitled"`.

## לולאות

השתמשו ב-`{% for %}` כדי לעבור על מערכים:

```twig
{% for item in schema:author %}
- {{item.name}}
{% endfor %}
```

### מקורות לולאה

ניתן לעבור בלולאה על:
- מערכי schema: `{% for item in schema:author %}`
- תוצאות סלקטור: `{% for comment in selector:.comment %}`
- משתנים שהוגדרו קודם: `{% set items = selector:.item %}{% for item in items %}`

### משתני לולאה

בתוך לולאה, יש לכם גישה לאובייקט `loop` עם המאפיינים הבאים:

| משתנה | תיאור |
|----------|-------------|
| `loop.index` | איטרציה נוכחית (מאונדקס 1) |
| `loop.index0` | איטרציה נוכחית (מאונדקס 0) |
| `loop.first` | `true` אם זו האיטרציה הראשונה |
| `loop.last` | `true` אם זו האיטרציה האחרונה |
| `loop.length` | מספר הפריטים הכולל |

```twig
{% for tag in tags %}
{{loop.index}}. {{tag}}
{% if loop.last %} (end of list){% endif %}
{% endfor %}
```

לצורך תאימות לאחור, ניתן גם להשתמש ב-`item_index` (כאשר `item` הוא שם משתנה האיטרטור שלכם) כדי לקבל את המיקום באינדקס 0:

```twig
{% for tag in tags %}
{{tag_index}}. {{tag}}
{% endfor %}
```

### גישה לפריטי מערך לפי אינדקס

השתמשו בסימון סוגריים מרובעים כדי לגשת לאלמנטי מערך לפי אינדקס:

```twig
{{items[0]}}
{{items[loop.index0]}}
```

זה שימושי כאשר צריך לגשת לפריטים ממספר מערכים במקביל:

```twig
{% set transcripts = selector:.transcript-text %}
{% set timestamps = selector:.timestamp %}

{% for line in transcripts %}
{{timestamps[loop.index0]}} - {{line}}
{% endfor %}
```

סימון סוגריים מרובעים עובד גם עם מאפייני אובייקט:

```twig
{{user["name"]}}
{{data["my-key"]}}
```

### לולאות מקוננות

ניתן לקנן לולאות עבור מבני נתונים מורכבים:

```twig
{% for section in sections %}
## {{section.title}}
{% for item in section.items %}
- {{item}}
{% endfor %}
{% endfor %}
```

## שילוב לוגיקה

ניתן לשלב תנאים ולולאות:

```twig
{% for item in items %}
{% if item.active %}
- {{item.name}}
{% endif %}
{% endfor %}
```

## סדר הערכה

לוגיקת תבנית מעובדת בסדר הבא:

1. **לוגיקת תבנית** — `{% if %}`, `{% for %}`, `{% set %}`, ו-`{{variables}}` מוערכים ראשונים
2. **משתני פרומפט** — [[משתנים#משתני פרומפט|משתני פרומפט]] כמו `{{"summarize this"|prompt}}` נשלחים למתרגם לאחר שלוגיקת התבנית הושלמה

משמעות הדבר היא שניתן להשתמש בלוגיקת תבנית כדי לבנות פרומפטים באופן דינמי, אך תוצאות פרומפט אינן זמינות לשימוש בתנאים או בלולאות.
