מתרגם הוא תכונה של [[מבוא ל-Obsidian Web Clipper|Web Clipper]] שמאפשרת לך לקיים אינטראקציה עם דפי אינטרנט באמצעות שפה טבעית. המתרגם עוזר לך ללכוד ולשנות נתונים שברצונך לשמור ב-Obsidian. לדוגמה:

- חילוץ קטעי טקסט ספציפיים.
- סיכום או הסבר של מידע.
- המרת טקסט מפורמט אחד לאחר.
- תרגום טקסט לשפה אחרת.

המתרגם ממנף מודלים של שפה לעיבוד מידע בדף אינטרנט, ומחזיר תוצאות באמצעות [[משתנים]] שניתן להוסיף ל[[Obsidian Web Clipper/תבניות|תבניות Web Clipper]].

## דוגמאות לפרומפטים

פרומפטים משתמשים בתחביר [[משתנים|משתנה]] `{{"your prompt"}}`. ניתן להשתמש בתחביר זה עם כל שאילתא בשפה טבעית, לדוגמה:

- `{{"a summary of the page"}}` לחילוץ סיכום של הדף.
- `{{"a three bullet point summary, translated to French"}}` לחילוץ תבליטים על הדף ותרגומם לצרפתית.
- `{{"un resumé de la page en trois points"}}` לחילוץ שלושה תבליטים באמצעות פרומפט בצרפתית.

ניתן לעבד עוד את הפלט של הפרומפטים שלך באמצעות [[מסננים]]. מסננים מעובדים לאחר קבלת תגובת הפרומפט מהמודל. לדוגמה: `{{"a summary of the page"|blockquote}}` יהפוך את התגובה לציטוט בבלוק.

## תחילת העבודה

המתרגם עובד עם כמעט כל ספק מודל שפה, כולל אפשרויות שרצות באופן פרטי על המכשיר שלך. להגדרת המתרגם:

1. עבור לסעיף **מתרגם** בהגדרות Web Clipper.
2. הפעל את **הפעל מתרגם**.
3. הגדר את הספק והמודל שלך, ראה סעיף [[פרש דפי אינטרנט#דגמים|דגמים]] להלן.
4. הוסף [[משתנים|משתני פרומפט]] ל[[Obsidian Web Clipper/תבניות|תבניות]] שלך.
5. אם התבנית שלך כוללת משתני פרומפט, סעיף המתרגם יהיה גלוי כאשר תבצע [[גזירת דפי אינטרנט|גזירת דף]]. לחץ על **פרש** כדי לעבד את משתני הפרומפט.

## כיצד זה עובד

כאשר המתרגם מופעל *וגם* התבנית שלך מכילה [[משתנים#משתני פרומפט|משתני פרומפט]], סעיף מתרגם חדש מוצג בחלון התוסף, מעל כפתור **הוסף ל-Obsidian**. סעיף זה מאפשר לך לבחור מודל ולהפעיל את המתרגם עבור הדף הנוכחי.

כאשר אתה לוחץ על **פרש**, המתרגם שולח את הקשר הדף למודל שבחרת, יחד עם *כל* הפרומפטים בתבנית שלך בבקשה אחת. בהתאם לספק המודל שתבחר, זו יכולה להיות קריאה חיצונית או מקומית למכשיר שלך. המודל מעריך את הפרומפטים שלך מול הקשר הדף ומחזיר את תגובותיו. לאחר מכן המתרגם מחליף את משתני הפרומפט בנתוני התגובה.

כל התהליך יכול לקחת אלפיות שנייה או יותר מ-30 שניות בהתאם למודל שאתה משתמש בו ולכמות הנתונים שאתה מעבד.

## הקשר

המונח *הקשר* מתייחס לנתוני הדף שהמתרגם משתמש בהם לעיבוד פרומפטים. ככל שההקשר קטן יותר, המתרגם רץ מהר יותר.

כברירת מחדל, המתרגם משתמש בכל ה-HTML של הדף כהקשר שלו, אולם זה עלול להפוך פרומפטים לאיטיים ויקרים מהנדרש.

ניתן לעקוף את הקשר ברירת המחדל ב**הגדרות מתקדמות** של המתרגם ולהגדיר הקשר לכל [[Obsidian Web Clipper/תבניות|תבנית]].

כדי להגדיר הקשר ממוקד יותר, השתמש ב[[משתנים#משתני בורר|משתני בורר]] (או סוגי משתנים אחרים) כדי לפרש חלק מהדף. לדוגמה, ניתן להשתמש במשתנה הבורר הבא בהקשר המתרגם של התבנית שלך:

```
{{selectorHtml:#main}}
```

 זה יפעיל את המתרגם רק על אלמנט `#main` בדף האינטרנט, אם הוא קיים. [[מסננים#עיבוד HTML|מסנני עיבוד HTML]] כמו `remove_html`, `strip_tags` ו-`strip_attr` יכולים להיות שימושיים כדי לצמצם עוד את אורך ההקשר ולהאיץ את העיבוד.

### בחירת מודל

באופן כללי אנו ממליצים להשתמש במודלים קטנים עם Web Clipper כי הם מהירים יותר ומבצעים בצורה מדויקת למדי למשימה זו. דוגמאות למודלים קטנים יותר כוללות **Claude Haiku של Anthropic**, **Google Gemini Flash**, **Llama** עם 3B או 8B פרמטרים, או סדרת המודלים **Mini של OpenAI**.

### ספקים ומודלים מותאמים אישית

כדי להוסיף ספק ו/או מודל מותאם אישית, עבור ל**[[הגדרות]]** של Web Clipper → **מתרגם**:

- **הוסף ספק** כדי להגדיר ספקים מוגדרים מראש ומותאמים אישית.
- **הוסף מודל** כדי להגדיר מודלים מוגדרים מראש ומותאמים אישית.

כאשר מוסיפים ספק מותאם אישית, אנו ממליצים להשתמש בנקודת הקצה של השלמות צ'אט שלהם עבור **URL בסיסי** — היא בדרך כלל מסתיימת ב-`/chat/completions`.

### מודלים מקומיים

המתרגם יכול להשתמש במודלים מקומיים המציעים פרטיות גבוהה יותר ותאימות לא מקוונת. קיימות מספר אפשרויות להפעלת מודלים מקומיים. אחת הקלות ביותר להגדרה היא Ollama.

#### Ollama

[Ollama](https://ollama.com/) מאפשר לך להריץ מודלי שפה באופן מקומי ופרטי על המכשיר שלך.

לאחר שהורדת והתקנת את Ollama, הוסף את Ollama באמצעות **הוסף ספק** בהגדרות המתרגם. Ollama אינו דורש מפתח API. לאחר מכן בחר מודל מ[רשימת המודלים](https://ollama.com/search). לדוגמה, אם ברצונך להשתמש ב-[Llama 3.2](https://ollama.com/library/llama3.2), לחץ על **הוסף מודל**, ואז:

- **ספק:** Ollama
- **שם תצוגה:** Llama 3.2, ערך זה ניתן להתאמה אישית.
- **מזהה דגם:** `llama3.2`, זה חייב להתאים בדיוק למזהה הדגם מ-Ollama.

**הפעלת שרת Ollama**

כדי לאפשר לתוסף דפדפן לתקשר עם Ollama עליך [לתת לו הוראה מפורשת](https://github.com/ollama/ollama/issues/2308) בעת הפעלת השרת, אחרת תראה שגיאת `403`.

סגור את אפליקציית Ollama, והרץ את הפקודה הבאה בטרמינל שלך. יש לשנות את הפרוטוקול לפרוטוקול התוסף של הדפדפן שלך אם אינך משתמש ב-Chrome או Firefox.

```
OLLAMA_ORIGINS=moz-extension://*,chrome-extension://*,safari-web-extension://* ollama serve
```

לאחר מכן הרץ את המודל שלך עם Ollama בדרך הרגילה, לדוגמה:

```
ollama run llama3.2
```

**אורך הקשר**

חלון ההקשר של Ollama מוגדר כברירת מחדל ל-2048 טוקנים. זהו המספר המקסימלי של טוקנים עבור ההודעה והתגובה. בעת גזירת דף אינטרנט ארוך ניתן לחרוג בקלות ממגבלה זו. Ollama ייכשל בשקט ויחזיר תוצאות לא רלוונטיות. מספר אפשרויות:

- הגדל את פרמטר `num_ctx` של Ollama. שים לב שהקשר ארוך יותר דורש יותר זיכרון.
- השתמש בשדה [[#הקשר]] בתבנית שלך כדי לספק חלק ממוקד יותר של הדף, או קצץ את ההקשר באמצעות [[מסננים|מסנן]] לדוגמה `{{content|slice:0,1000}}`.
