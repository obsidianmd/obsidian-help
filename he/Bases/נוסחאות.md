נוסחאות מאפשרות לך ליצור מאפיינים מחושבים ב-[[מבוא ל-Bases|Bases]] באמצעות נתונים מ[[מאפיינים]] אחרים. ניתן לבצע חישובים, לעבד טקסט, לעבוד עם תאריכים ועוד.

## מה נוסחאות יכולות לעשות

נוסחאות יכולות לעזור לך:

- **לחשב ערכים**, לסכם מחירים, לחשב סכומים כוללים או לבצע פעולות מתמטיות.
- **לעבד טקסט**, לשלב מחרוזות, לשנות רישיות או לחלץ תת-מחרוזות.
- **לעבוד עם תאריכים**, לחשב הפרשי זמן, לעצב תאריכים או לקבוע מועדים אחרונים.
- **להפעיל לוגיקה**, להשתמש בתנאים כדי להציג ערכים שונים.
- **לעבד רשימות**, לסנן, למיין, למפות או לצבור נתוני רשימות.

## יצירת מאפיין נוסחה

כדי ליצור מאפיין נוסחה:

1. ב-base שלך, לחץ על **מאפיינים** בסרגל הכלים.
2. לחץ על **הוסף נוסחה** בתחתית התפריט.
3. הזן שם למאפיין הנוסחה.
4. הקלד את הנוסחה בשדה **נוסחה**.
5. סגור את תיבת הדו-שיח.

עורך הנוסחאות ישלים אוטומטית שמות [[פונקציות|פונקציות]] ומאפיינים תוך כדי הקלדה כדי לאמת את תחביר הנוסחה שלך. סימן וי ירוק מופיע כאשר הנוסחה תקינה.

לאחר יצירתו, ניתן להשתמש במאפיין נוסחה כמו כל מאפיין אחר ב-base שלך. הוסף אותו ל[[תצוגות]], השתמש בו במסננים, מיין לפיו ועוד.

## כתיבת נוסחה

בעורך הנוסחאות, הקלד ביטוי באמצעות מאפיינים, אופרטורים ופונקציות.

### הפניה למאפיינים

ניתן להפנות לסוגים שונים של מאפיינים בנוסחאות שלך:

- **מאפייני הערה** — מאפיינים מה[[מאפיינים|מטא-נתונים בתחילת הקובץ]] של הערה.
- **מאפייני קובץ** — מאפיינים מובנים כמו `file.name`, `file.size` או `file.mtime`.
- **מאפייני נוסחה** — נוסחאות אחרות באמצעות `formula.formula_name`.

**דוגמאות:**

- `price * quantity` — הכפלת שני מאפייני הערה
- `file.name + " - " + description` — שילוב שם קובץ עם מאפיין הערה
- `formula.price_per_unit * 1.1` — שימוש במאפיין נוסחה אחר

### שימוש באופרטורים

**אופרטורים אריתמטיים** מבצעים חישובים על מספרים:

- `price + tax` — חיבור
- `price - discount` — חיסור
- `price * quantity` — כפל
- `price / quantity` — חילוק
- `(part / whole) * 100` — שימוש בסוגריים לסדר פעולות

**אופרטורי השוואה** משווים ערכים:

- `price > 100` — גדול מ
- `age < 18` — קטן מ
- `status == "Done"` — שווה ל
- `status != "Done"` — לא שווה ל
- `file.mtime > now() - '7d'` — השוואת תאריכים

**אופרטורים בוליאניים** משלבים תנאים לוגיים:

- `!completed` — לא
- `price > 0 && quantity > 0` — וגם
- `urgent || important` — או

למד עוד ב-[[תחביר Bases#אופרטורים|תחביר Bases]].

### שימוש בפונקציות

פונקציות מבצעות פעולות על ערכים. הפונקציות הזמינות תלויות בסוג הערך שאיתו אתה עובד. ראה את הרשימה המלאה של [[פונקציות]].

**קטגוריות פונקציות נפוצות:**

- **פונקציות גלובליות** — `if()`, `now()`, `date()`, `link()`, `max()`, `min()`
- **פונקציות מחרוזת** — `contains()`, `replace()`, `split()`, `lower()`, `title()`
- **פונקציות מספר** — `round()`, `ceil()`, `floor()`, `abs()`, `toFixed()`
- **פונקציות תאריך** — `format()`, `relative()`, `date()`, `time()`
- **פונקציות רשימה** — `filter()`, `map()`, `sort()`, `join()`, `unique()`

**דוגמאות:**

- `if(price, "$" + price.toFixed(2), "")` להגדרת תנאי עם עיצוב מספרים.
- `file.name.lower()` להמרה לאותיות קטנות.
- `tags.contains("urgent")` לבדיקה אם רשימת התגים מכילה ערך.
- `due_date.format("YYYY-MM-DD")` לעיצוב תאריך.

## דוגמאות נוסחאות

### חישוב מועד אחרון

הגדרת מועד אחרון לפרויקט כשבועיים לאחר תאריך ההתחלה:

```js
start_date + "2w"
```

### הצגת סטטוס איחור

הצגת "Overdue" אם המועד האחרון עבר והסטטוס אינו "Done":

```js
if(due_date < now() && status != "Done", "Overdue", "")
```

### עיצוב מטבע

הצגת מחיר עם 2 ספרות עשרוניות וסמל מטבע:

```js
if(price, "$" + price.toFixed(2), "")
```

### ספירת פריטי רשימה

ספירת מספר הפריטים במאפיין רשימה:

```js
tasks.length
```

### חישוב ציון עדיפות

שילוב מספר גורמים לציון עדיפות:

```js
(impact * urgency) / effort
```

### שילוב שדות טקסט

יצירת שם מלא משם פרטי ושם משפחה:

```js
first_name + " " + last_name
```

### חישוב עלות כוללת

הכפלת עלות חודשית במספר חודשי הבעלות:

```js
monthlyUses * formula.Owned.round()
```

## סוגי נתונים

נוסחאות עובדות עם סוגי נתונים שונים:

- **מחרוזות** — טקסט בתוך מרכאות: `"hello"` או `'world'`
- **מספרים** — ערכים מספריים: `42`, `3.14`, `(2 + 2)`
- **בוליאניים** — אמת או שקר: `true`, `false`
- **תאריכים** — נוצרים עם `date()`, `today()` או `now()`
- **רשימות** — אוספי ערכים: `[1, 2, 3]`
- **אובייקטים** — זוגות מפתח-ערך: `{"name": "value"}`

סוג הפלט של נוסחה נקבע על ידי הנתונים והפונקציות שבשימוש.

## הפניה לנוסחאות אחרות

נוסחאות יכולות להפנות לנוסחאות אחרות, ולייצר חישובים נגזרים. לדוגמה, אם יש לך נוסחה בשם `price_per_unit`:

```js
price / quantity
```

ניתן להפנות אליה בנוסחה אחרת:

```js
formula.price_per_unit * 1.1
```

> [!warning] הימנע מהפניות מעגליות
> נוסחה לא יכולה להפנות לעצמה באופן ישיר או עקיף דרך נוסחאות אחרות.
