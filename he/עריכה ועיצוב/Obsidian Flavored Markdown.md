Obsidian שואפת ליכולת מרבית מבלי לשבור פורמטים קיימים. כתוצאה מכך, אנו משתמשים בשילוב של טעמים שונים של [[תחביר עיצוב בסיסי|Markdown]].

Obsidian תומכת ב-[CommonMark](https://commonmark.org/), ב-[GitHub Flavored Markdown](https://github.github.com/gfm/), וב-[LaTeX](https://www.latex-project.org/).

> [!tip]- Markdown בתוך HTML
> Obsidian אינה מעבדת תחביר Markdown בתוך אלמנטי HTML. זוהי החלטת עיצוב מכוונת לאופטימיזציית ביצועים ולשמירה על מורכבות מנתח נמוכה בעת ניהול מסמכים גדולים.
>
> לדוגמה, עיצוב Markdown כמו `**bold**` או `` `code` `` לא יעובד בתוך `<div>`, `<span>`, `<table>`, או כל תגית HTML אחרת.
>
> ```md
> <div>
> This **will not** be bold.
> </div>
> ```

### הרחבות Markdown נתמכות

| תחביר          | תיאור                                                           |
| --------------- | --------------------------------------------------------------------- |
| `[[Link]]`      | [[קישורים פנימיים]]                                                    |
| `![[Link]]`     | [[הטמעת קבצים]]                                                       |
| `![[Link#^id]]` | [[קישורים פנימיים#קישור לבלוק בהערה\|הפניות לבלוק]]        |
| `^id`           | [[קישורים פנימיים#קישור לבלוק בהערה\|הגדרת בלוק]]        |
| `[^id]`         | [[תחביר עיצוב בסיסי#הערות שוליים\|הערות שוליים]]                      |
| `%%Text%%`      | [[תחביר עיצוב בסיסי#הערות\|הערות]]                        |
| `~~Text~~`      | [[תחביר עיצוב בסיסי#מודגש, נטוי, הדגשות\|קו חוצה]] |
| `==Text==`      | [[תחביר עיצוב בסיסי#מודגש, נטוי, הדגשות\|הדגשות]]     |
| `` ``` ``       | [[תחביר עיצוב בסיסי#בלוקי קוד\|בלוקי קוד]]                  |
| `- [ ]`         | [[תחביר עיצוב בסיסי#רשימות מטלות\|מטלה לא הושלמה]]               |
| `- [x]`         | [[תחביר עיצוב בסיסי#רשימות מטלות\|מטלה שהושלמה]]                |
| `> [!note]`     | [[ציטוטים]]                                                          |
| (ראו קישור)      | [[תחביר עיצוב מתקדם#טבלאות\|טבלאות]]                         |
