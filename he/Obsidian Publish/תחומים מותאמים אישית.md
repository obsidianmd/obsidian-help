ניתן להגדיר תחום מותאם אישית או תת-תחום עבור אתר [[מבוא ל-Obsidian Publish|Obsidian Publish]] שלך.

> [!warning] אזהרה
> כרגע, אנחנו תומכים רק בהגדרת תחומים מותאמים אישית באמצעות השיטות הבאות:
>
> - [[#הגדרה באמצעות CloudFlare]] בשימוש ב-[מצב Full](https://developers.cloudflare.com/ssl/origin-configuration/ssl-modes/full/).
> - [[#הגדרה באמצעות פרוקסי]]
> 
> אין לנו עדיין דרך להנפיק תעודת SSL בשמכם.

## הגדרה באמצעות CloudFlare

הדרך הקלה ביותר להגדיר תחום מותאם אישית או תת-תחום היא ליצור חשבון [CloudFlare](https://cloudflare.com) ולאפשר ל-CloudFlare לנהל את ה-DNS של התחום שלכם.

השלבים הבאים משתמשים ב-CloudFlare כדי להגדיר תחום מותאם אישית עבור אתר Obsidian Publish שלכם, בין אם באמצעות תחום שורש (`mysite.com`) או תת-תחום (`notes.mysite.com`).

> [!important] חשוב
> CloudFlare הוא **הספק היחיד הנתמך רשמית** להגדרת תחומים מותאמים אישית. שימוש בהוראות הבאות עם ספקים אחרים ככל הנראה לא יעבוד.

**CloudFlare:**

1. פתחו את Cloudflare בתחום שבו אתם רוצים לארח את אתר ה-Publish שלכם, כגון `mysite.com`, גם אם אתם רוצים תת-תחום כמו `notes.mysite.com`.
2. עברו ל-**DNS** ולחצו על **Add Record**.
3. בחרו **CNAME**.
4. ב-**name**, הזינו את התחום או תת-התחום שלכם, לדוגמה `notes.mysite.com`.
5. ב-**target**, הזינו `publish-main.obsidian.md`. אל תכללו את כתובת ה-sub-URL האישית שלכם בערך זה. Obsidian Publish מטפל בזה מההגדרות שלכם.
6. ודאו ש-**proxy status** מופעל. הוא אמור להיות מופעל כברירת מחדל.
7. עברו ל-**SSL/TLS** והגדירו את מצב הצפנת SSL/TLS ל-"Full" כדי להגדיר את תעודת SSL/TLS באופן אוטומטי.

> [!note] הערה
> כדי להפנות גם את `mysite.com` וגם את `www.mysite.com` ל-Obsidian Publish, עליכם ליצור [Page Rule](https://support.cloudflare.com/hc/en-us/articles/200172336-Creating-Page-Rules) עם ההגדרות הבאות:
>
> - URL match: `www.mysite.com/*`
> - Forward URL - 301 Permanent Redirect
> - Redirect URL: `https://mysite.com/$1`
>
> לאחר שיצרתם את כלל הדף, צרו רשומת CNAME עבור `www.mysite.com` בדיוק כפי שעשיתם עבור `mysite.com`.

**Obsidian:**

1. פתחו את Obsidian במחשב שלכם.
2. ב[[סרגל כלים]] בצד שמאל, לחצו על **פרסם שינויים** ![[lucide-send.svg#icon]].
3. תחת **פרסם שינויים**, בחרו **שנה אפשרויות אתר** ![[lucide-cog.svg#icon]].
4. ליד **Custom domain**, בחרו **Configure**.
5. ב-**Custom URL**, הזינו את הכתובת של התחום או תת-התחום שלכם. ודאו שלא לשים `www.` בתיבת ה-Custom URL.

> [!note] הערה
> אם הגדרת התחום המותאם אישית שלכם מסתיימת בלולאת הפניה, סביר להניח שמצב ההצפנה ב-CloudFlare הוגדר ל-"Flexible" במקום ל-"Full".

## הגדרה באמצעות פרוקסי

ניתן גם להגדיר SSL/TLS עבור התחום המותאם אישית שלכם באמצעות שרת אינטרנט משלכם.

אם אתם כבר מארחים אתר תחת התחום או תת-התחום שלכם, תוכלו גם להשתמש באפשרות זו ולהגדיר את האתר שלכם לטעון את אתר Obsidian Publish שלכם תחת נתיב URL ספציפי, במקום לארח את האתר המלא.

העבירו את כל הבקשות תחת נתיב ה-URL הזה אל `https://publish.obsidian.md/serve?url=mysite.com/my-notes/...` והגדירו את אפשרויות האתר ב-Obsidian לאותו נתיב URL, על ידי הגדרת **Custom URL** ל-`mysite.com/my-notes`.

ניתן גם להגדיר את Obsidian Publish כ-sub-URL של אתר שבבעלותכם. לדוגמה, `https://mysite.com/my-notes/`. כדי להשיג זאת, עליכם לארח שרת משלכם ולהעביר את כל הבקשות לשרת שלנו בכתובת `https://publish.obsidian.md/`.

דוגמאות הגדרת הפרוקסי הבאות אינן ממצות, אך מספקות שיטות נפוצות ליישום זה.

### NGINX

בהגדרות ה-NGINX שלכם, הוסיפו את הבא:

```nginx
location /my-notes {
  proxy_pass https://publish.obsidian.md/serve?url=mysite.com/my-notes/;
  proxy_ssl_server_name on;
  proxy_set_header Host publish.obsidian.md;
}
```


חלק מהמשתמשים דיווחו שהוספת `$request_uri` ל-proxy pass עשויה להיות נדרשת:

```nginx
location /my-notes {
  proxy_pass https://publish.obsidian.md/serve?url=mysite.com/my-notes$request_uri;
  proxy_ssl_server_name on;
  proxy_set_header Host publish.obsidian.md;
}
```

### Apache

ב-`.htaccess`, הוסיפו את הבא:

```htaccess
RewriteEngine  on
RewriteRule    "^my-notes/(.*)$"  "https://publish.obsidian.md/serve?url=mysite.com/my-notes/$1"  [L,P]
```

> [!note] הערה
> `mod_rewrite` חייב להיות מופעל, וייתכן שתצטרכו גם להגדיר [SSLProxyEngine](https://stackoverflow.com/questions/40938148/reverse-proxy-for-external-url-apache)

### Netlify

ב-`netlify.toml`, [הגדירו הפניות](https://docs.netlify.com/routing/redirects/#syntax-for-the-netlify-configuration-file):

```plain
[[redirects]]
  from = "https://mysite.com/my-notes/*"
  to = "https://publish.obsidian.md/serve?url=mysite.com/my-notes/:splat"
  status = 200
  force = true
```

### Vercel

ב-`vercel.json`, [הגדירו rewrites](https://vercel.com/docs/configuration#project/rewrites):

```json
{
  ...

  "rewrites": [
    {
      "source": "/my-notes/",
      "destination": "https://publish.obsidian.md/serve?url=mysite.com/my-notes"
    },
    {
      "source": "/my-notes/:path*",
      "destination": "https://publish.obsidian.md/serve?url=mysite.com/my-notes/:path*"
    }
  ]
}
```

### Caddy

```plain
mysite.com {
  encode zstd gzip
  handle /my-notes* {
    reverse_proxy https://publish.obsidian.md {
      header_up Host {upstream_hostport}
    }
    rewrite * /serve?url=mysite.com{path}
  }
}
```

### Traefik

תצורה מינימלית זו מפנה את `mysite.com` ל-Obsidian Publish.
ראו את [תיעוד Traefik](https://doc.traefik.io/traefik/routing/overview/)
לדוגמה מלאה.

```yaml
http:
  routers:
    mysite:
      rule: Host(`mysite.com`)
      service: obsidian-publish
      middlewares:
        - "publish-headers"
  services:
    obsidian-publish:
      loadBalancer:
        servers:
          - url: https://publish.obsidian.md
  middlewares:
    publish-headers:
      headers:
        customRequestHeaders:
          Host: "publish.obsidian.md"
          x-obsidian-custom-domain: "mysite.com"
```

### כותרות HTTP X נתמכות

אם שירות הפרוקסי שלכם לא מאפשר נתיבי שאילתא, תוכלו להשתמש ב-`https://publish.obsidian.md/` עם כותרת מותאמת אישית `x-obsidian-custom-domain` המוגדרת לכתובת האתר שלכם `mysite.com/my-subpath`.

## הפניית האתר הישן לתחום מותאם אישית

אם ברצונכם להפנות את המבקרים שלכם מאתר ה-`publish.obsidian.md` הישן לתחום המותאם אישית החדש שלכם, הפעילו את האפשרות **הפנה מחדש לדומיין המותאם אישית שלך** בעת הגדרת התחום המותאם אישית.

## פתרון בעיות

לאחר שהגדרתם את התחום המותאם אישית שלכם, אם ביקרתם באתר שלכם מהקישור הקודם `https://publish.obsidian.md/slug`, ייתכן שתצטרכו לנקות את מטמון הדפדפן כדי שדברים מסוימים (כמו גופנים, גרפים, או גישה מוגנת בסיסמה) יעבדו כראוי. זה נובע ממגבלות האבטחה חוצות-תחומים שנאכפות על ידי דפדפנים מודרניים. החדשות הטובות הן שקוראי האתר שלכם לעולם לא אמורים להיתקל בבעיה זו אם אתם מאפשרים למבקרים להשתמש רק בתחום המותאם אישית שלכם.
