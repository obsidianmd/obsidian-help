當你在 Obsidian 中[[建立資料庫|建立資料庫]]時，它會被儲存為 `.base` 檔案。資料庫通常透過應用程式介面進行編輯，但其語法也可以手動編輯，並嵌入到程式碼區塊中。

[[資料庫介紹|資料庫]]語法定義了[[檢視]]、過濾設定和[[公式]]。資料庫必須是符合以下定義的結構描述的有效 YAML。

## 範例

以下是一個資料庫檔案的範例。我們將逐一詳細說明每個部分。

```yaml
filters:
  or:
    - file.hasTag("tag")
    - and:
        - file.hasTag("book")
        - file.hasLink("Textbook")
    - not:
        - file.hasTag("book")
        - file.inFolder("Required Reading")
formulas:
  formatted_price: 'if(price, price.toFixed(2) + " dollars")'
  ppu: "(price / age).toFixed(2)"
properties:
  status:
    displayName: Status
  formula.formatted_price:
    displayName: "Price"
  file.ext:
    displayName: Extension
summaries:
  customAverage: 'values.mean().round(3)'
views:
  - type: table
    name: "My table"
    limit: 10
    groupBy:
      property: note.age
      direction: DESC
    filters:
      and:
        - 'status != "done"'
        - or:
            - "formula.ppu > 5"
            - "price > 2.1"
    order:
      - file.name
      - file.ext
      - note.age
      - formula.ppu
      - formula.formatted_price
    summaries:
      formula.ppu: Average
```

### 過濾設定

預設情況下，資料庫包含保管庫中的每個檔案。這裡沒有像 SQL 或 Dataview 中的 `from` 或 `source`。`filters` 部分讓你定義條件來縮小資料集範圍。

```yaml
# 簡易過濾：
filters:
  and:
    - file.hasTag("tag")

# 複雜過濾：
filters:
  or:
    - file.hasTag("tag")
    - and:
        - file.hasTag("book")
        - file.hasLink("Textbook")
    - not:
        - file.hasTag("book")
        - file.inFolder("Required Reading")
```

有兩個套用過濾設定的機會：

1. 在全域 `filters` 層級（如上所示），它們適用於資料庫中的所有檢視。
2. 在 `view` 層級，僅適用於特定檢視。

這兩個部分在功能上是等效的，在評估檢視時，它們會以 `AND` 串接。

`filters` 部分包含一個字串形式的單一過濾陳述式，或一個遞迴定義的過濾物件。過濾物件可以包含 `and`、`or` 或 `not` 之一。這些鍵是一個異質清單，包含其他過濾物件或字串形式的過濾陳述式。過濾陳述式是一行在套用到筆記時會評估為真值或假值的語句。它可以是以下之一：

- 使用標準算術運算子的基本比較。
- 函式。內建了多種[[函式]]，外掛也可以新增額外的函式。

過濾設定和公式的語法及可用函式相同。

### 公式

`formulas` 部分定義了可以在資料庫檔案中所有檢視中顯示的[[公式|公式屬性]]。

```yaml
formulas:
  formatted_price: 'if(price, price.toFixed(2) + " dollars")'
  ppu: "(price / age).toFixed(2)"
```

公式屬性支援基本算術運算子和多種內建[[函式]]。未來外掛將能夠新增可在公式中使用的函式。

你可以根據屬性類型以不同方式引用它們：

- **筆記屬性**是定義在筆記前置中繼資料中的屬性。例如 `note.price` 或 `note["price"]`。
  如果未指定前綴，則假設該屬性為 `note` 屬性。
- **檔案屬性**描述檔案本身。
  例如 `file.size` 或 `file.ext`。你也可以直接引用檔案物件，例如 `file.hasLink()`。
- **公式屬性**是資料庫中的其他公式。
  例如 `formula.formatted_price`。

公式可以使用其他公式屬性中的值，只要沒有循環引用即可。

公式屬性在 YAML 中始終以字串形式儲存，但它們的實際**輸出資料類型**由底層資料的類型和所使用函式的回傳值決定。

請注意，要在 YAML 欄位中包含文字常量，必須使用巢狀引號。文字常量必須用單引號或雙引號括起來。

### 屬性

`properties` 部分允許儲存每個屬性的設定資訊。個別檢視可以自行決定如何使用這些設定值。例如，在表格中，顯示名稱用於直欄標題。

```yaml
properties:
  status:
    displayName: Status
  formula.formatted_price:
    displayName: "Price"
  file.ext:
    displayName: Extension
```

顯示名稱不會在過濾設定或公式中使用。

### 摘要

`summaries` 部分可用於定義自訂摘要公式。除了在此定義摘要公式外，還有幾個可用的預設摘要公式。

```yaml
summaries:
  customAverage: 'values.mean().round(3)'
```

在此範例中，`customAverage` 公式與預設的 `Average` 相同，只是值被四捨五入到不同的小數位數。在摘要公式中，`values` 關鍵字是一個清單，包含結果集中每筆筆記的該屬性所有值。摘要公式應回傳單一 `Value`。

請注意，此 `summaries` 部分與檢視設定中的 `summaries` 部分不同（如下所述），後者是將摘要公式指定給特定屬性。

#### 預設摘要公式

| 名稱      | 輸入類型 | 描述                                                   |
| --------- | ---------- | ------------------------------------------------------------- |
| Average   | 數字     | 輸入值中所有數字的算術平均數。   |
| Min       | 數字     | 輸入值中的最小數字。                    |
| Max       | 數字     | 輸入值中的最大數字。                     |
| Sum       | 數字     | 輸入值中所有數字的總和。                          |
| Range     | 數字     | `Max` 和 `Min` 之間的差值。                       |
| Median    | 數字     | 輸入值中所有數字的中位數。 |
| Stddev    | 數字     | 輸入值中所有數字的標準差。  |
| Earliest  | 日期       | 輸入值中最早的日期。                      |
| Latest    | 日期       | 輸入值中最晚的日期。                        |
| Range     | 日期       | `Latest` 和 `Earliest` 之間的差值。               |
| Checked   | 布林值    | `true` 值的數量。                                  |
| Unchecked | 布林值    | `false` 值的數量。                                 |
| Empty     | 任意        | 輸入值中空值的數量。             |
| Filled    | 任意        | 輸入值中非空值的數量。         |
| Unique    | 任意        | 輸入值中唯一值的數量。                     |

### 檢視

`views` 部分定義了資料的呈現方式。`views` 清單中的每個項目定義了相同資料的不同檢視，可以建立任意數量的不同檢視。

```yaml
views:
  - type: table
    name: "My table"
    limit: 10
    groupBy:
      property: note.age
      direction: DESC
    filters:
      and:
        - 'status != "done"'
        - or:
            - "formula.ppu > 5"
            - "price > 2.1"
    order:
      - file.name
      - file.ext
      - note.age
      - formula.ppu
      - formula.formatted_price
    summaries:
      formula.ppu: Average
```

- `type` 從內建和外掛新增的檢視類型中選取。
- `name` 是顯示名稱，可用於定義預設檢視。
- `filters` 與上述描述完全相同，但僅適用於該檢視。
- `groupBy` 指定一個屬性和排序方向。每橫列中指定屬性的值用於將該橫列放入群組中。
- `summaries` 將屬性名稱對應到具名摘要。摘要對所有橫列的屬性執行聚合運算。

[[檢視]]可以新增額外資料來儲存維持狀態或正確呈現所需的任何資訊，但外掛作者應注意不要使用核心資料庫外掛已使用的鍵。例如，表格檢視可以用此來限制橫列數量或選取用於排序橫列的直欄及其方向。不同的檢視類型（如地圖）可以用此來對應筆記中哪個屬性對應到經度和緯度，以及哪個屬性應顯示為圖釘標題。

未來，API 將允許檢視讀取和寫入這些值，使檢視能夠建立自己的設定介面。

## 屬性

資料庫中使用三種屬性：

1. **筆記屬性**，儲存在 Markdown 檔案的前置中繼資料中。
2. **檔案屬性**，適用於所有檔案類型。
3. **公式屬性**，定義在 `.base` 檔案本身中（見上文）。

### 筆記屬性

[[屬性|筆記屬性]]僅適用於 Markdown 檔案，儲存在每則筆記的 YAML 前置中繼資料中。這些屬性可以使用 `note.author` 格式存取，或簡寫為 `author`。

### 檔案屬性

檔案屬性指的是目前正在測試或評估的檔案。檔案屬性適用於所有[[支援的檔案格式|檔案類型]]，包括附件。

例如，過濾設定 `file.ext == "md"` 對所有 Markdown 檔案為真，否則為假。

| 屬性      | 類型   | 描述                                                   |
| ------------- | ------ | ------------------------------------------------------------- |
| `file.backlinks`  | 清單   | 反向連結檔案清單。注意：此屬性效能消耗較大。如果可能，請反轉查詢並使用 `file.links`。當保管庫發生變更時不會自動重新整理結果。 |
| `file.ctime`  | 日期   | 建立時間                                                  |
| `file.embeds` | 清單   | 筆記中所有嵌入的清單                                |
| `file.ext`    | 字串 | 副檔名                                                |
| `file.file`   | 檔案   | 檔案物件，僅可在特定函式中使用                |
| `file.folder` | 字串 | 檔案資料夾的路徑                                       |
| `file.links`  | 清單   | 筆記中所有內部連結的清單，包括前置中繼資料 |
| `file.mtime`  | 日期   | 修改時間                                                 |
| `file.name`   | 字串 | 檔案名稱                                                     |
| `file.path`   | 字串 | 檔案路徑                                              |
| `file.properties`   | 物件 | 檔案上的所有屬性。注意：當保管庫發生變更時不會自動重新整理結果。 |
| `file.size`   | 數字 | 檔案大小                                                     |
| `file.tags`   | 清單   | 檔案內容和前置中繼資料中所有標籤的清單          |

### 使用 `this` 存取屬性

使用 `this` 物件來存取檔案屬性。`this` 指向的內容取決於資料庫顯示的位置。

當資料庫在主要內容區域開啟時，`this` 指向資料庫檔案本身的屬性。例如，使用 `this.file.folder` 會回傳資料庫所在的資料夾路徑。

當資料庫嵌入在另一個檔案中時，`this` 指向_嵌入檔案_（包含資料庫的筆記或 Canvas）的屬性。例如，使用 `this.file.name` 會回傳嵌入檔案的名稱，而非資料庫的名稱。

當資料庫在側邊欄中時，`this` 指向主要內容區域中的作用中檔案。這讓你可以根據作用中檔案建立查詢。例如，你可以使用 `file.hasLink(this.file)` 來複製反向連結面板的功能。

## 運算子

### 算術運算子

算術運算子對數字執行算術運算。例如，`radius * (2 * 3.14)`。

| 運算子 | 描述 |
| -------- | ----------- |
| `+`      | 加        |
| `-`      | 減       |
| `*`      | 乘    |
| `/`      | 除      |
| `%`      | 取餘數      |
| `( )`    | 括號 |

### 日期算術

日期可以透過加減持續時間來修改。持續時間單位接受多種格式：

| 單位                     | 持續時間 |
| ------------------------ | -------- |
| `y`、`year`、`years`     | 年     |
| `M`、`month`、`months`   | 月    |
| `d`、`day`、`days`       | 日      |
| `w`、`week`、`weeks`     | 週     |
| `h`、`hour`、`hours`     | 小時     |
| `m`、`minute`、`minutes` | 分鐘   |
| `s`、`second`、`seconds` | 秒   |

要修改或偏移日期物件，請使用 `+` 或 `-` 運算子搭配持續時間字串。例如，`date + "1M"` 會將日期加上 1 個月，而 `date - "2h"` 會從日期減去 2 小時。

全域[[函式|函式]] `today()` 可用於取得目前日期，`now()` 可用於取得目前的日期與時間。

- `now() + "1 day"` 回傳從執行時間起恰好 24 小時後的日期時間。
- `file.mtime > now() - "1 week"` 如果檔案在過去一週內被修改則回傳 `true`。
- `date("2024-12-01") + "1M" + "4h" + "3m"` 回傳代表 `2025-01-01 04:03:00` 的日期物件。
- 將兩個日期相減可取得兩者之間的毫秒差值，例如 `now() - file.ctime`。
- 要取得帶時間的日期的日期部分，請使用 `datetime.date()`。
- 要格式化日期物件，請使用 `format()` 函式，例如 `datetime.format("YYYY-MM-DD")`。

### 比較運算子

比較運算子可用於比較數字或日期物件。等於和不等於可以用於任何類型的值，不僅限於數字和日期。

| 運算子 | 描述              |
| -------- | ------------------------ |
| `==`     | 等於                   |
| `!=`     | 不等於                |
| `>`      | 大於             |
| `<`      | 小於                |
| `>=`     | 大於或等於 |
| `<=`     | 小於或等於    |

### 布林運算子

布林運算子可用於組合或反轉邏輯值，產生真或假的結果。

| 運算子 | 描述 |
| -------- | ----------- |
| `!`      | 邏輯非 |
| `&&`     | 邏輯且 |
| \|\|     | 邏輯或  |

請參閱可在公式和[[檢視|過濾設定]]中使用的[[函式|函式清單]]。

## 類型

資料庫具有類型系統，公式和過濾設定使用它來將函式套用到屬性上。

### 字串、數字和布林值

字串、數字和布林值是「原始」值，不需要函式來建立。

- 字串用單引號或雙引號括起來，例如 `"message"`。
- 數字以數字書寫，可以選擇性地用括號括起來以提高清晰度。例如 `1` 或 `(2.5)`。
- 布林值寫為不加引號的 `true` 或 `false`。

### 日期和持續時間

日期代表特定日期，或日期和時間，取決於用來建立它們的函式，或指定給[[屬性|屬性]]的類型。

- 要建構日期，請使用 `date` 函式，例如 `date("2025-01-01 12:00:00")`
- 要修改日期，請加上或減去持續時間，例如 `now() + "1 hour"` 或 `today() + "7d"`
- 使用比較運算子（例如 `>` 或 `<`）和算術運算子比較日期（例如，`(now() + "1d") - now()` 回傳 `86400000` 毫秒。）
- 要提取日期的部分，請使用可用的欄位（`now().hour`），或便利函式（`now.time()`）。
- 日期物件上還有許多其他可用的[[函式|欄位和函式]]。

### 物件和清單

- 使用 `list()` 函式將單一元素轉換為清單。這對於可能包含清單或單一值混合的屬性特別有用。
- 使用方括號和從 0 開始的索引存取清單元素。例如，`property[0]` 回傳清單中的第一個元素。
- 使用方括號搭配元素名稱或點記法存取物件元素。例如，`property.subprop` 或 `property["subprop"]`。

### 檔案和連結

[[連結筆記|Wiki 連結]]在[[屬性|前置中繼資料屬性]]中會自動被識別為連結物件。連結將在[[檢視|檢視]]中呈現為可點擊的連結。

- 要建構連結，請使用全域 `link` [[函式|函式]]，例如 `link("filename")` 或 `link("https://obsidian.md")`。
- 你可以從任何字串建立連結，例如 `link(file.ctime.date().toString())`。
- 要設定顯示文字，請傳入可選的字串或圖示作為第二個參數，例如 `link("filename", "display")` 或 `link("filename", icon("plus"))`。

檔案物件可以使用 `file.asLink()` 搭配可選的顯示文字轉換為連結。

連結可以使用 `==` 和 `!=` 進行比較。只要它們指向相同的檔案就是等價的，或者如果查詢時檔案不存在，則它們的連結文字必須完全相同。

連結可以與 `file` 或 `this` 等檔案進行比較。如果連結解析到該檔案，它們就相等。例如，`author == this`。

連結也可以在清單包含中檢查，例如 `authors.contains(this)`。
