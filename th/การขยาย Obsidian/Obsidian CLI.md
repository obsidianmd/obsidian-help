Obsidian CLI เป็น command line interface ที่ช่วยให้คุณควบคุม Obsidian จากเทอร์มินัลสำหรับการเขียนสคริปต์ การทำงานอัตโนมัติ และการเชื่อมต่อกับเครื่องมือภายนอก

ทุกอย่างที่คุณทำได้ใน Obsidian คุณสามารถทำได้จาก command line Obsidian CLI ยังมี[[#คำสั่งสำหรับนักพัฒนา|คำสั่งสำหรับนักพัฒนา]]เพื่อเข้าถึงเครื่องมือนักพัฒนา ตรวจสอบ element ถ่ายภาพหน้าจอ รีโหลดปลั๊กอิน และอื่นๆ อีกมาก

![[obsidian-cli.mp4#interface]]

> [!warning] ต้องใช้ตัวติดตั้ง Obsidian 1.12
> การใช้ CLI ต้องใช้ตัวติดตั้ง Obsidian 1.12 ดู[[อัพเดต Obsidian#การอัปเดตตัวติดตั้ง|คู่มืออัปเดตเวอร์ชันตัวติดตั้ง]]

## ติดตั้ง Obsidian CLI

อัปเกรดไปยัง[[อัพเดต Obsidian|เวอร์ชันตัวติดตั้ง Obsidian]] ล่าสุด (1.11.7) และ[[เวอร์ชั่นเข้าถึงก่อน|เวอร์ชันเข้าถึงก่อน]]ล่าสุด (1.12.x)

เปิดใช้งาน Obsidian CLI ใน Obsidian:

1. ไปที่ **การตั้งค่า** → **ทั่วไป**
2. เปิดใช้งาน **Command line interface**
3. ทำตามคำแนะนำเพื่อลงทะเบียน Obsidian CLI

หากคุณพบปัญหาในการติดตั้ง Obsidian CLI ดู [[#การแก้ไขปัญหา]]

## เริ่มต้นใช้งาน

Obsidian CLI รองรับทั้งคำสั่งเดี่ยวและ terminal user interface (TUI) พร้อมวิธีใช้แบบโต้ตอบและออโต้คอมพลีท

> [!info] แอป Obsidian ต้องทำงานอยู่
> Obsidian CLI ต้องการให้แอป Obsidian ทำงานอยู่ หาก Obsidian ไม่ได้ทำงานอยู่ คำสั่งแรกที่คุณรันจะเปิด Obsidian ขึ้นมา
>
> ต้องการซิงค์โดยไม่ต้องใช้แอปเดสก์ท็อป? ดู [[Obsidian Headless|Obsidian Headless]]

### รันคำสั่ง

รันคำสั่งเดี่ยวโดยไม่ต้องเปิด TUI:

```shell
# รันคำสั่ง help
obsidian help
```

### ใช้ terminal interface

ใช้ TUI โดยพิมพ์ `obsidian` คำสั่งถัดไปสามารถพิมพ์ได้โดยไม่ต้องใส่ `obsidian`

```shell
# เปิด TUI จากนั้นรัน help
obsidian
help
```

TUI รองรับออโต้คอมพลีท ประวัติคำสั่ง และการค้นหาย้อนกลับ ใช้ `Ctrl+R` เพื่อค้นหาประวัติคำสั่ง ดู [[#ทางลัดแป้นพิมพ์]] สำหรับทางลัดทั้งหมดที่มี

## ตัวอย่าง

นี่คือตัวอย่างบางส่วนของสิ่งที่ Obsidian CLI ทำได้

### การใช้งานทั่วไป

```shell
# เปิดโน้ตรายวันของวันนี้
obsidian daily

# เพิ่มงานในโน้ตรายวัน
obsidian daily:append content="- [ ] Buy groceries"

# ค้นหาในห้องนิรภัย
obsidian search query="meeting notes"

# อ่านไฟล์ที่ใช้งานอยู่
obsidian read

# แสดงรายการงานทั้งหมดจากโน้ตรายวัน
obsidian tasks daily

# สร้างโน้ตใหม่จากเทมเพลต
obsidian create name="Trip to Paris" template=Travel

# แสดงรายการแท็กทั้งหมดในห้องนิรภัยพร้อมจำนวน
obsidian tags counts

# เปรียบเทียบสองเวอร์ชันของไฟล์
obsidian diff file=README from=1 to=3
```

### สำหรับนักพัฒนา

[[#คำสั่งสำหรับนักพัฒนา|คำสั่งสำหรับนักพัฒนา]]จำนวนมากมีให้สำหรับการพัฒนาปลั๊กอินและธีม คำสั่งเหล่านี้ช่วยให้เครื่องมือ agentic coding สามารถทดสอบและดีบักโดยอัตโนมัติ

```shell
# เปิดเครื่องมือนักพัฒนา
obsidian devtools

# รีโหลดปลั๊กอินชุมชนที่คุณกำลังพัฒนา
obsidian plugin:reload id=my-plugin

# ถ่ายภาพหน้าจอของแอป
obsidian dev:screenshot path=screenshot.png

# รัน JavaScript ในคอนโซลของแอป
obsidian eval code="app.vault.getFiles().length"
```

## วิธีการ

### ใช้พารามิเตอร์และแฟล็ก

คำสั่งสามารถใช้**พารามิเตอร์**และ**แฟล็ก** คำสั่งส่วนใหญ่ไม่ต้องการพารามิเตอร์หรือแฟล็กใดๆ พารามิเตอร์ที่จำเป็นจะถูกระบุว่า `required` ตัวอย่างเช่น:

```shell
# สร้างโน้ตใหม่โดยใช้ชื่อค่าเริ่มต้น "Untitled"
obsidian create
```

**พารามิเตอร์**รับค่า เขียนในรูปแบบ `parameter=value` หากค่ามีช่องว่าง ให้ครอบด้วยเครื่องหมายคำพูด:

```shell
# สร้างโน้ตใหม่ชื่อ "Note" พร้อมเนื้อหา "Hello world"
obsidian create name=Note content="Hello world"
```

**แฟล็ก**คือสวิตช์บูลีนที่ไม่มีค่า ใส่เข้าไปเพื่อเปิด ตัวอย่างเช่น `open` และ `overwrite` เป็นแฟล็ก:

```shell
# สร้างโน้ตและเปิดมัน
obsidian create name=Note content="Hello" open overwrite
```

สำหรับเนื้อหาหลายบรรทัด ใช้ `\n` สำหรับบรรทัดใหม่ ใช้ `\t` สำหรับแท็บ

```bash
obsidian create name=Note content="# Title\n\nBody text"
```

### ระบุห้องนิรภัย

หากไดเรกทอรีปัจจุบันของเทอร์มินัลเป็นโฟลเดอร์ห้องนิรภัย ห้องนิรภัยนั้นจะถูกใช้เป็นค่าเริ่มต้น มิฉะนั้น ห้องนิรภัยที่ใช้งานอยู่ในปัจจุบันจะถูกใช้

ใช้ `vault=<name>` หรือ `vault=<id>` เพื่อระบุห้องนิรภัยเฉพาะ ต้องเป็นพารามิเตอร์แรกก่อนคำสั่งของคุณ:

```shell
obsidian vault=Notes daily
obsidian vault="My Vault" search query="test"
```

ใน TUI ใช้ `vault:open <name>` หรือ `<id>` เพื่อสลับไปยังห้องนิรภัยอื่น

### ระบุไฟล์

คำสั่งจำนวนมากรับพารามิเตอร์ `file` และ `path` เพื่อระบุไฟล์เฉพาะ หากไม่ได้ระบุทั้งสองอย่าง คำสั่งจะใช้ไฟล์ที่ใช้งานอยู่เป็นค่าเริ่มต้น

- `file=<name>` แก้ไขชื่อไฟล์โดยใช้การแก้ไขลิงก์แบบเดียวกับ[[ลิงค์ภายใน|ลิงก์วิกิ]] จับคู่ด้วยชื่อไฟล์โดยไม่ต้องใช้เส้นทางเต็มหรือนามสกุล
- `path=<path>` ต้องใช้เส้นทางที่แน่นอนจากรากห้องนิรภัย เช่น `folder/note.md`

```shell
# สองคำสั่งนี้เทียบเท่ากันหาก "Recipe.md" เป็นไฟล์เดียวที่มีชื่อนี้
obsidian read file=Recipe
obsidian read path="Templates/Recipe.md"
```

### คัดลอกผลลัพธ์

เพิ่ม `--copy` ในคำสั่งใดก็ได้เพื่อคัดลอกผลลัพธ์ไปยังคลิปบอร์ด:

```shell
read --copy
search query="TODO" --copy
```

### `version`

แสดงเวอร์ชัน Obsidian

### `reload`

โหลดหน้าต่างแอปใหม่

### `restart`

## ฐาน

คำสั่งสำหรับ[[แนะนำฐาน|ฐาน]]

### `bases`

แสดงรายการไฟล์ `.base` ทั้งหมดในห้องนิรภัย

### `base:views`

แสดงรายการมุมมองในไฟล์ฐานปัจจุบัน

### `base:create`

สร้างรายการใหม่ในฐาน ใช้มุมมองฐานที่ใช้งานอยู่เป็นค่าเริ่มต้นหากไม่ได้ระบุไฟล์

```bash
file=<name>        # ชื่อไฟล์ฐาน
path=<path>        # เส้นทางไฟล์ฐาน
view=<name>        # ชื่อมุมมอง
name=<name>        # ชื่อไฟล์ใหม่
content=<text>     # เนื้อหาเริ่มต้น

open               # เปิดไฟล์หลังสร้าง
newtab             # เปิดในแท็บใหม่
```

### `base:query`

คิวรีฐานและส่งคืนผลลัพธ์

```bash
file=<name>                    # ชื่อไฟล์ฐาน
path=<path>                    # เส้นทางไฟล์ฐาน
view=<name>                    # ชื่อมุมมองที่จะคิวรี
format=json|csv|tsv|md|paths   # รูปแบบผลลัพธ์ (ค่าเริ่มต้น: json)
```

## บุ๊กมาร์ก

คำสั่งสำหรับ[[บุ๊กมาร์ก]]

### `bookmarks`

แสดงรายการบุ๊กมาร์ก

```bash
total              # ส่งคืนจำนวนบุ๊กมาร์ก
verbose            # รวมประเภทบุ๊กมาร์ก
format=json|tsv|csv  # รูปแบบผลลัพธ์ (ค่าเริ่มต้น: tsv)
```

### `bookmark`

เพิ่มบุ๊กมาร์ก

```bash
file=<path>        # ไฟล์ที่จะบุ๊กมาร์ก
subpath=<subpath>  # เส้นทางย่อย (หัวเรื่องหรือบล็อก) ภายในไฟล์
folder=<path>      # โฟลเดอร์ที่จะบุ๊กมาร์ก
search=<query>     # คิวรีค้นหาที่จะบุ๊กมาร์ก
url=<url>          # URL ที่จะบุ๊กมาร์ก
title=<title>      # ชื่อบุ๊กมาร์ก
```

## กระดานคำสั่ง

คำสั่งสำหรับ[[กระดานคำสั่ง]]และ[[ฮ็อตคีย์]] ซึ่งรวมถึงคำสั่งทั้งหมดที่ลงทะเบียนโดยปลั๊กอิน

### `commands`

แสดงรายการ command ID ที่มี

```bash
filter=<prefix>    # กรองตามคำนำหน้า ID
```

### `command`

ดำเนินการคำสั่ง Obsidian

```bash
id=<command-id>    # (required) command ID ที่จะดำเนินการ
```

### `hotkeys`

แสดงรายการฮ็อตคีย์สำหรับคำสั่งทั้งหมด

```bash
total              # ส่งคืนจำนวนฮ็อตคีย์
verbose            # แสดงว่าฮ็อตคีย์เป็นแบบกำหนดเองหรือไม่
format=json|tsv|csv  # รูปแบบผลลัพธ์ (ค่าเริ่มต้น: tsv)
```

### `hotkey`

ดูฮ็อตคีย์ของคำสั่ง

```bash
id=<command-id>    # (required) command ID

verbose            # แสดงว่าเป็นแบบกำหนดเองหรือค่าเริ่มต้น
```

## โน้ตประจำวัน

คำสั่งสำหรับ[[โน้ตประจำวัน]]

### `daily`

เปิดโน้ตรายวัน

```bash
paneType=tab|split|window    # ประเภทหน้าต่างที่จะเปิด
```

### `daily:path`

ดูเส้นทางโน้ตรายวัน ส่งคืนเส้นทางที่คาดไว้แม้ไฟล์ยังไม่ถูกสร้าง

### `daily:read`

อ่านเนื้อหาโน้ตรายวัน

### `daily:append`

ต่อเนื้อหาท้ายโน้ตรายวัน

```bash
content=<text>     # (required) เนื้อหาที่จะต่อท้าย
paneType=tab|split|window    # ประเภทหน้าต่างที่จะเปิด

inline             # ต่อท้ายโดยไม่มีบรรทัดใหม่
open               # เปิดไฟล์หลังเพิ่ม
```

### `daily:prepend`

แทรกเนื้อหาก่อนโน้ตรายวัน

```bash
content=<text>     # (required) เนื้อหาที่จะแทรก
paneType=tab|split|window    # ประเภทหน้าต่างที่จะเปิด

inline             # แทรกโดยไม่มีบรรทัดใหม่
open               # เปิดไฟล์หลังเพิ่ม
```

## ประวัติไฟล์

### `diff`

แสดงรายการหรือเปรียบเทียบเวอร์ชันจาก[[การกู้คืนข้อมูล]]ในเครื่องและ[[แนะนำ Obsidian Sync|Sync]] เวอร์ชันจะถูกเรียงลำดับจากใหม่สุดไปเก่าสุด

```bash
file=<name>          # ชื่อไฟล์
path=<path>          # เส้นทางไฟล์
from=<n>             # หมายเลขเวอร์ชันที่จะ diff จาก
to=<n>               # หมายเลขเวอร์ชันที่จะ diff ถึง
filter=local|sync    # กรองตามแหล่งที่มาของเวอร์ชัน
```

**ตัวอย่าง:**

```shell
# แสดงรายการเวอร์ชันทั้งหมดของไฟล์ที่ใช้งานอยู่
diff

# แสดงรายการเวอร์ชันทั้งหมดของไฟล์เฉพาะ
diff file=Recipe

# เปรียบเทียบเวอร์ชันล่าสุดกับไฟล์ปัจจุบัน
diff file=Recipe from=1

# เปรียบเทียบสองเวอร์ชัน
diff file=Recipe from=2 to=1

# แสดงเฉพาะเวอร์ชัน Sync
diff filter=sync
```

### `history`

แสดงรายการเวอร์ชันจาก[[การกู้คืนข้อมูล]]เท่านั้น ดู [[#Sync|sync:history]] สำหรับคำสั่ง Sync ที่เทียบเท่า

```bash
file=<name>        # ชื่อไฟล์
path=<path>        # เส้นทางไฟล์
```

### `history:list`

แสดงรายการไฟล์ทั้งหมดที่มีประวัติในเครื่อง

### `history:read`

อ่านเวอร์ชันประวัติในเครื่อง

```bash
file=<name>        # ชื่อไฟล์
path=<path>        # เส้นทางไฟล์
version=<n>        # หมายเลขเวอร์ชัน (ค่าเริ่มต้น: 1)
```

### `history:restore`

คืนค่าเวอร์ชันประวัติในเครื่อง

```bash
file=<name>        # ชื่อไฟล์
path=<path>        # เส้นทางไฟล์
version=<n>        # (required) หมายเลขเวอร์ชัน
```

### `history:open`

เปิดการกู้คืนข้อมูล

```bash
file=<name>        # ชื่อไฟล์
path=<path>        # เส้นทางไฟล์
```

## ไฟล์และโฟลเดอร์

### `file`

แสดงข้อมูลไฟล์ (ค่าเริ่มต้น: ไฟล์ที่ใช้งานอยู่)

```bash
file=<name>        # ชื่อไฟล์
path=<path>        # เส้นทางไฟล์
```

ตัวอย่าง:

```
path       Notes/Recipe.md
name       Recipe
extension  md
size       1024
created    1700000000000
modified   1700001000000
```

### `files`

แสดงรายการไฟล์ในห้องนิรภัย

```bash
folder=<path>      # กรองตามโฟลเดอร์
ext=<extension>    # กรองตามนามสกุล

total              # ส่งคืนจำนวนไฟล์
```

### `folder`

แสดงข้อมูลโฟลเดอร์

```bash
path=<path>              # (required) เส้นทางโฟลเดอร์
info=files|folders|size  # ส่งคืนข้อมูลเฉพาะเจาะจง
```

### `folders`

แสดงรายการโฟลเดอร์ในห้องนิรภัย

```bash
folder=<path>      # กรองตามโฟลเดอร์แม่

total              # ส่งคืนจำนวนโฟลเดอร์
```

### `open`

เปิดไฟล์

```bash
file=<name>        # ชื่อไฟล์
path=<path>        # เส้นทางไฟล์

newtab             # เปิดในแท็บใหม่
```

```bash
file=<name>        # ชื่อไฟล์
```

```bash
file=<name>        # ชื่อไฟล์
```

### `delete`

```bash
file=<name>        # แสดงคุณสมบัติของไฟล์
path=<path>        # แสดงคุณสมบัติของเส้นทาง
name=<name>        # ดูจำนวนคุณสมบัติเฉพาะ
sort=count         # จัดเรียงตามจำนวน (ค่าเริ่มต้น: ชื่อ)
format=yaml|json|tsv  # รูปแบบผลลัพธ์ (ค่าเริ่มต้น: yaml)

total              # ส่งคืนจำนวนคุณสมบัติ
counts             # รวมจำนวนการเกิดขึ้น
active             # แสดงคุณสมบัติของไฟล์ที่ใช้งานอยู่
```

```bash
name=<name>                                    # (required) ชื่อคุณสมบัติ

file=<name>                                    # ชื่อไฟล์
path=<path>                                    # เส้นทางไฟล์
```

### `property:remove`

```bash
name=<name>        # (required) ชื่อคุณสมบัติ
file=<name>        # ชื่อไฟล์
path=<path>        # เส้นทางไฟล์
```

### `property:read`

อ่านค่าคุณสมบัติจากไฟล์ (ค่าเริ่มต้น: ไฟล์ที่ใช้งานอยู่)

```bash
done               # แสดงงานที่เสร็จแล้ว
todo               # แสดงงานที่ยังไม่เสร็จ
verbose            # จัดกลุ่มตามไฟล์พร้อมหมายเลขบรรทัด
format=json|tsv|csv  # รูปแบบผลลัพธ์ (ค่าเริ่มต้น: text)
active             # แสดงงานของไฟล์ที่ใช้งานอยู่
daily              # แสดงงานจากโน้ตรายวัน
```

**ตัวอย่าง:**

```bash
# แสดงรายการงานทั้งหมดในห้องนิรภัย
tasks

# แสดงรายการงานที่ยังไม่เสร็จในห้องนิรภัย
tasks todo

# แสดงรายการงานที่เสร็จแล้วจากไฟล์เฉพาะ
tasks file=Recipe done

# แสดงรายการงานจากโน้ตรายวันของวันนี้
tasks daily

# นับงานในโน้ตรายวัน
tasks daily total

# แสดงรายการงานพร้อมเส้นทางไฟล์และหมายเลขบรรทัด
tasks verbose

# กรองตามสถานะที่กำหนดเอง (ใส่เครื่องหมายคำพูดอักขระพิเศษ)
tasks 'status=?'
```

### `task`

แสดงหรืออัปเดตงาน

```bash
ref=<path:line>    # การอ้างอิงงาน (path:line)
file=<name>        # ชื่อไฟล์
path=<path>        # เส้นทางไฟล์
line=<n>           # หมายเลขบรรทัด
status="<char>"    # ตั้งค่าอักขระสถานะ

toggle             # สลับสถานะงาน
daily              # โน้ตรายวัน
done               # ทำเครื่องหมายว่าเสร็จ
todo               # ทำเครื่องหมายว่ายังไม่เสร็จ
```

**ตัวอย่าง:**

```bash
# แสดงข้อมูลงาน
task file=Recipe line=8
task ref="Recipe.md:8"

# สลับสถานะงานที่เสร็จ
task ref="Recipe.md:8" toggle

# สลับสถานะงานในโน้ตรายวัน
task daily line=3 toggle

# ตั้งค่าสถานะงาน
task file=Recipe line=8 done      # → [x]
task file=Recipe line=8 todo      # → [ ]
task file=Recipe line=8 status=-  # → [-]
task daily line=3 done            # ทำเครื่องหมายงานในโน้ตรายวันว่าเสร็จ
```


## เทมเพลต

คำสั่งสำหรับ[[เทมเพลต|เทมเพลต]]

### `templates`

แสดงรายการเทมเพลต

```bash
total              # ส่งคืนจำนวนเทมเพลต
```

### `template:read`

อ่านเนื้อหาเทมเพลต

```bash
name=<template>    # (required) ชื่อเทมเพลต
title=<title>      # ชื่อสำหรับการแก้ไขตัวแปร

resolve            # แก้ไขตัวแปรเทมเพลต
```

### `template:insert`

แทรกเทมเพลตในไฟล์ที่ใช้งานอยู่

```bash
name=<template>    # (required) ชื่อเทมเพลต
```

**หมายเหตุ:**
- ตัวเลือก `resolve` จะประมวลผลตัวแปร `{{date}}`, `{{time}}`, `{{title}}`
- ใช้ `create path=<path> template=<name>` เพื่อสร้างไฟล์พร้อมเทมเพลต
