Obsidian URI เป็นโปรโตคอล URI แบบกำหนดเองที่ Obsidian รองรับ ซึ่งช่วยให้คุณเรียกใช้การกระทำต่างๆ เช่น เปิดโน้ตหรือสร้างโน้ต Obsidian URI ช่วยให้สามารถทำงานอัตโนมัติและเวิร์กโฟลว์ข้ามแอปได้

## รูปแบบ URI

Obsidian URI ใช้รูปแบบต่อไปนี้:

```
obsidian://action?param1=value&param2=value
```

พารามิเตอร์ `action` คือการกระทำที่คุณต้องการดำเนินการ การกระทำที่ใช้ได้มีดังนี้:

- `open` เพื่อเปิดโน้ต
- `new` เพื่อสร้างหรือเพิ่มเนื้อหาลงในโน้ตที่มีอยู่
- `daily` เพื่อสร้างหรือเปิดโน้ตรายวันของคุณ
- `unique` เพื่อสร้างโน้ตใหม่ที่ไม่ซ้ำ
- `search` เพื่อเปิดการค้นหา
- `choose-vault` เพื่อเปิดตัวจัดการห้องนิรภัย

> [!warning] การเข้ารหัส
> ตรวจสอบให้แน่ใจว่าค่าของคุณถูกเข้ารหัส URI อย่างถูกต้อง ตัวอย่างเช่น เครื่องหมายทับ `/` ต้องถูกเข้ารหัสเป็น `%2F` และอักขระเว้นวรรคต้องถูกเข้ารหัสเป็น `%20`
> 
 สิ่งนี้สำคัญอย่างยิ่งเนื่องจากอักขระ "สงวน" ที่เข้ารหัสไม่ถูกต้องอาจทำให้การตีความ URI ผิดพลาดได้ [ดูรายละเอียดที่นี่](https://en.wikipedia.org/wiki/Percent-encoding)

## เปิดโน้ต

การกระทำ `open` จะเปิดห้องนิรภัย Obsidian หรือเปิดไฟล์ภายในห้องนิรภัยนั้น

### ตัวอย่าง

- `obsidian://open?vault=my%20vault`
  จะเปิดห้องนิรภัย `my vault` หากห้องนิรภัยเปิดอยู่แล้ว จะโฟกัสที่หน้าต่างนั้น
- `obsidian://open?vault=ef6ca3e3b524d22f`
  จะเปิดห้องนิรภัยที่ระบุด้วย ID `ef6ca3e3b524d22f`
- `obsidian://open?vault=my%20vault&file=my%20note`
  จะเปิดโน้ต `my note.md` ในห้องนิรภัย `my vault` โดยสมมติว่าไฟล์มีอยู่
- `obsidian://open?path=%2Fhome%2Fuser%2Fmy%20vault%2Fpath%2Fto%2Fmy%20note`
  จะค้นหาห้องนิรภัยใดก็ได้ที่มีเส้นทาง `/home/user/my vault/path/to/my note` จากนั้นเส้นทางที่เหลือจะถูกส่งไปยังพารามิเตอร์ `file` ตัวอย่างเช่น หากมีห้องนิรภัยอยู่ที่ `/home/user/my vault` จะเทียบเท่ากับพารามิเตอร์ `file` ที่ตั้งเป็น `path/to/my note`


> [!tip] เปิดหัวเรื่องหรือบล็อก
> ด้วยการเข้ารหัส URI ที่ถูกต้อง คุณสามารถนำทางไปยังหัวเรื่องหรือบล็อกภายในโน้ตได้ `Note%23Heading` จะนำทางไปยังหัวเรื่องชื่อ "Heading" ในขณะที่ `Note%23%5EBlock` จะนำทางไปยังบล็อกชื่อ "Block"

### พารามิเตอร์

- `vault` สามารถเป็นชื่อห้องนิรภัยหรือ ID ของห้องนิรภัย[^1]
- `file` สามารถเป็นชื่อไฟล์ หรือเส้นทางจากรากของห้องนิรภัยไปยังไฟล์ที่ระบุ หากนามสกุลไฟล์เป็น `md` สามารถละนามสกุลได้
- `path` เส้นทางระบบไฟล์แบบสัมบูรณ์ไปยังไฟล์
  - การใช้พารามิเตอร์นี้จะแทนที่ทั้ง `vault` และ `file`
  - จะทำให้แอปค้นหาห้องนิรภัยที่ตรงที่สุดซึ่งมีเส้นทางไฟล์ที่ระบุ
  - จากนั้นเส้นทางที่เหลือจะแทนที่พารามิเตอร์ `file`
- `prepend` จะเพิ่มที่ด้านบนของไฟล์และพยายามผสานคุณสมบัติ
- `append` จะเพิ่มที่ด้านล่างของไฟล์และพยายามผสานคุณสมบัติเช่นกัน
- `paneType` (ไม่บังคับ) กำหนดว่าโน้ตจะถูกเปิดที่ไหนใน UI
  - หากไม่ระบุ แท็บที่ใช้งานอยู่ล่าสุดจะถูกแทนที่
  - `paneType=tab` เปิดในแท็บใหม่
  - `paneType=split` เปิดในกลุ่มแท็บใหม่
  - `paneType=window` เปิดในหน้าต่างแยก (เฉพาะเดสก์ท็อป)

## สร้างโน้ต

การกระทำ `new` จะสร้างโน้ตใหม่ในห้องนิรภัย โดยเลือกที่จะมีเนื้อหาบางส่วนได้

### ตัวอย่าง

- `obsidian://new?vault=my%20vault&name=my%20note`
  จะเปิดห้องนิรภัย `my vault` และสร้างโน้ตใหม่ชื่อ `my note`
- `obsidian://new?vault=my%20vault&file=path%2Fto%2Fmy%20note`
  จะเปิดห้องนิรภัย `my vault` และสร้างโน้ตใหม่ที่ `path/to/my note`

### พารามิเตอร์

- `vault` สามารถเป็นชื่อห้องนิรภัยหรือ ID ของห้องนิรภัย[^1] เหมือนกับการกระทำ `open`
- `name` ชื่อไฟล์ที่จะสร้าง หากระบุ ตำแหน่งไฟล์จะถูกเลือกตามการตั้งค่า "ตำแหน่งเริ่มต้นโน้ตใหม่" ของคุณ
- `file` เส้นทางสัมบูรณ์ภายในห้องนิรภัย รวมถึงชื่อ จะแทนที่ `name` หากระบุ
- `path` เส้นทางสัมบูรณ์ทั่วไป ทำงานคล้ายกับตัวเลือก `path` ในการกระทำ `open` ซึ่งจะแทนที่ทั้ง `vault` และ `file`
- `paneType` (ไม่บังคับ) กำหนดว่าโน้ตจะถูกเปิดที่ไหนใน UI เหมือนกับการกระทำ `open`
- `content` (ไม่บังคับ) เนื้อหาของโน้ต
- `clipboard` (ไม่บังคับ) ใช้เนื้อหาจากคลิปบอร์ดแทนการระบุ `content`
- `silent` (ไม่บังคับ) รวมพารามิเตอร์นี้หากคุณไม่ต้องการเปิดโน้ตใหม่
- `append` (ไม่บังคับ) รวมพารามิเตอร์นี้เพื่อต่อท้ายไฟล์ที่มีอยู่หากมี
- `overwrite` (ไม่บังคับ) เขียนทับไฟล์ที่มีอยู่หากมี แต่เฉพาะเมื่อไม่ได้ตั้ง `append`
- `x-success` (ไม่บังคับ) ดู [[#ใช้พารามิเตอร์ x-callback-url]]

## สร้างหรือเปิดโน้ตรายวัน

การกระทำ `daily` จะสร้างหรือเปิดโน้ตรายวันของคุณ ต้องเปิดใช้งานปลั๊กอิน [[โน้ตประจำวัน]] ก่อน

### ตัวอย่าง

- `obsidian://daily?vault=my%20vault`
  จะเปิดห้องนิรภัย `my vault` และสร้างหรือเปิดโน้ตรายวัน

### พารามิเตอร์

การกระทำ `daily` ยอมรับพารามิเตอร์เดียวกับการกระทำ `new`

## โน้ตที่ไม่ซ้ำ

การกระทำ `unique` จะสร้างโน้ตใหม่ที่ไม่ซ้ำในห้องนิรภัย ต้องเปิดใช้งานปลั๊กอิน [[ปลั๊กอิน/ตัวสร้างโน้ตแบบไม่ซ้ำ|ตัวสร้างโน้ตแบบไม่ซ้ำ]] ก่อน

### ตัวอย่าง

- `obsidian://unique?vault=my%20vault`
  จะเปิดห้องนิรภัย `my vault` และสร้างโน้ตใหม่ที่ไม่ซ้ำ
- - `obsidian://unique?vault=my%20vault&content=Hello%20World`
  จะเปิดห้องนิรภัย `my vault` และสร้างโน้ตใหม่ที่ไม่ซ้ำพร้อมเนื้อหา `Hello World`

### พารามิเตอร์

- `vault` สามารถเป็นชื่อห้องนิรภัยหรือ ID ของห้องนิรภัย[^1] เหมือนกับการกระทำ `open`
- `paneType` (ไม่บังคับ) กำหนดว่าโน้ตจะถูกเปิดที่ไหนใน UI เหมือนกับการกระทำ `open`
- `content` (ไม่บังคับ) เนื้อหาของโน้ต
- `clipboard` (ไม่บังคับ) ใช้เนื้อหาจากคลิปบอร์ดแทนการระบุ `content`
- `x-success` (ไม่บังคับ) ดู [[#ใช้พารามิเตอร์ x-callback-url]]

## เปิดการค้นหา

การกระทำ `search` จะเปิด [[ค้นหา]] ในห้องนิรภัยที่ระบุ และเลือกที่จะดำเนินการค้นหาได้

### ตัวอย่าง

- `obsidian://search?vault=my%20vault`
  จะเปิดห้องนิรภัย `my vault` และเปิด [[ค้นหา]]
- `obsidian://search?vault=my%20vault&query=Obsidian`
  จะเปิดห้องนิรภัย `my vault` เปิด [[ค้นหา]] และดำเนินการค้นหาคำว่า `Obsidian`

### พารามิเตอร์

- `vault` สามารถเป็นชื่อห้องนิรภัยหรือ ID ของห้องนิรภัย[^1] เหมือนกับการกระทำ `open`
- `query` (ไม่บังคับ) คำค้นหาที่จะดำเนินการ

## เปิดตัวจัดการห้องนิรภัย

การกระทำ `choose-vault` จะเปิด [[จัดการห้องนิรภัย|ตัวจัดการห้องนิรภัย]]

### ตัวอย่าง

- `obsidian://choose-vault`

## เชื่อมต่อกับ Hook

การกระทำ Obsidian URI นี้ใช้สำหรับ [Hook](https://hookproductivity.com/)

### ตัวอย่าง

`obsidian://hook-get-address`

### พารามิเตอร์

- `vault` (ไม่บังคับ) สามารถเป็นชื่อห้องนิรภัยหรือ ID ของห้องนิรภัย[^1] หากไม่ระบุ จะใช้ห้องนิรภัยปัจจุบันหรือที่โฟกัสล่าสุด
- `x-success` (ไม่บังคับ) ดู [[#ใช้พารามิเตอร์ x-callback-url]]
- `x-error` (ไม่บังคับ) ดู [[#ใช้พารามิเตอร์ x-callback-url]]

หากกำหนด `x-success` API นี้จะใช้เป็น x-callback-url มิฉะนั้น จะคัดลอกลิงก์ Markdown ของโน้ตที่โฟกัสอยู่ปัจจุบันไปยังคลิปบอร์ด เป็น URL `obsidian://open`

## ใช้พารามิเตอร์ x-callback-url

จุดปลายบางส่วนจะยอมรับพารามิเตอร์ x-callback-url `x-success` และ `x-error` เมื่อระบุ Obsidian จะส่งข้อมูลต่อไปนี้ไปยัง callback `x-success`:

- `name` ชื่อของไฟล์ โดยไม่มีนามสกุลไฟล์
- `url` URI `obsidian://` สำหรับไฟล์นี้
- `file` (เฉพาะเดสก์ท็อป) URL `file://` สำหรับไฟล์นี้

ตัวอย่างเช่น หาก Obsidian ได้รับ
`obsidian://.....x-success=myapp://x-callback-url` การตอบกลับจะเป็น `myapp://x-callback-url?name=...&url=obsidian%3A%2F%2Fopen...&file=file%3A%2F%2F...`

## รูปแบบย่อ

นอกจากรูปแบบด้านบนแล้ว ยังมีรูปแบบ "ย่อ" อีกสองแบบที่สามารถใช้เปิดห้องนิรภัยและไฟล์:

1. `obsidian://vault/my vault/my note` เทียบเท่ากับ `obsidian://open?vault=my%20vault&file=my%20note`
2. `obsidian:///absolute/path/to/my note` เทียบเท่ากับ `obsidian://open?path=%2Fabsolute%2Fpath%2Fto%2Fmy%20note`

## การแก้ไขปัญหา

### ลงทะเบียน Obsidian URI

บน Windows และ macOS การเรียกใช้แอปครั้งเดียวน่าจะเพียงพอสำหรับการลงทะเบียนโปรโตคอล Obsidian URI บนคอมพิวเตอร์ของคุณ

บน Linux เป็นกระบวนการที่ซับซ้อนกว่ามาก:

1. ตรวจสอบให้แน่ใจว่าคุณสร้างไฟล์ `obsidian.desktop` [ดูรายละเอียดที่นี่](https://developer.gnome.org/documentation/guidelines/maintainer/integrating.html#desktop-files)
2. ตรวจสอบให้แน่ใจว่าไฟล์ desktop ของคุณระบุฟิลด์ `Exec` เป็น `Exec=executable %u` โดย `%u` ใช้สำหรับส่ง URI `obsidian://` ไปยังแอป
3. หากคุณใช้ตัวติดตั้ง AppImage คุณอาจต้องแตกไฟล์โดยใช้ `Obsidian-x.y.z.AppImage --appimage-extract` จากนั้นตรวจสอบให้แน่ใจว่าคำสั่ง `Exec` ชี้ไปยังไฟล์ที่แตกออกมา


[^1]: ID ของห้องนิรภัยคือรหัส 16 อักขระแบบสุ่มที่กำหนดให้กับห้องนิรภัย เช่น `ef6ca3e3b524d22f` ID นี้ไม่ซ้ำกันสำหรับแต่ละโฟลเดอร์บนคอมพิวเตอร์ของคุณ สามารถค้นหา ID ได้โดยเปิดตัวสลับห้องนิรภัยแล้วคลิก "คัดลอก ID ห้องนิรภัย" ในเมนูบริบทของห้องนิรภัยที่ต้องการ
