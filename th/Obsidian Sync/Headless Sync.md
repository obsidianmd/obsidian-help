[[แนะนำ Obsidian Sync|Obsidian Sync]] มีไคลเอนต์แบบ headless สำหรับซิงค์ห้องนิรภัยโดยไม่ต้องใช้แอปเดสก์ท็อป เหมาะสำหรับ CI pipelines, agents และเวิร์กโฟลว์อัตโนมัติ ซิงค์การเปลี่ยนแปลงล่าสุดหรือคอยอัปเดตไฟล์อย่างต่อเนื่อง

ติดตั้ง [[Obsidian Headless]] **(open beta)** เพื่อโต้ตอบกับ [[แนะนำ Obsidian Sync|Obsidian Sync]] จากบรรทัดคำสั่งโดยไม่ต้องใช้แอปเดสก์ท็อป Obsidian Headless Sync ใช้[[ความปลอดภัยและความเป็นส่วนตัว|การเข้ารหัสและการปกป้องความเป็นส่วนตัว]]เดียวกันกับแอปเดสก์ท็อป รวมถึงการเข้ารหัสแบบต้นทางถึงปลายทาง

## เริ่มอย่างรวดเร็ว

> [!error] สำรองข้อมูลของคุณก่อนเริ่มต้น
> 1. สำรองข้อมูลของคุณก่อนเริ่มต้นเสมอ ในกรณีที่เกิดสิ่งไม่คาดคิด
> 2. อย่าใช้ Sync ของแอปเดสก์ท็อป *และ* Headless Sync บนอุปกรณ์เดียวกัน เพราะอาจทำให้เกิดข้อขัดแย้งของข้อมูล ใช้วิธีซิงค์เพียงวิธีเดียวต่ออุปกรณ์

ติดตั้ง [[Obsidian Headless|Obsidian Headless]] **(open beta)**:

```shell
npm install -g obsidian-headless
```

คุณต้องมี[[แผนและขีดจำกัดพื้นที่จัดเก็บ|การสมัครสมาชิก Obsidian Sync]] ที่ใช้งานอยู่

```shell
# เข้าสู่ระบบ
ob login

# แสดงรายการห้องนิรภัยบนคลาวด์ของคุณ
ob sync-list-remote

# ตั้งค่าห้องนิรภัยสำหรับการซิงค์
cd ~/vaults/my-vault
ob sync-setup --vault "My Vault"

# ซิงค์ครั้งเดียว
ob sync

# ซิงค์ต่อเนื่อง (เฝ้าดูการเปลี่ยนแปลง)
ob sync --continuous
```

## คำสั่ง

### `ob sync-list-remote`

แสดงรายการห้องนิรภัยบนคลาวด์ทั้งหมดที่สามารถเข้าถึงได้ในบัญชีของคุณ รวมถึงห้องนิรภัยที่แชร์

### `ob sync-list-local`

แสดงรายการห้องนิรภัยที่ตั้งค่าในเครื่องและเส้นทางของไฟล์

### `ob sync-create-remote`

สร้างห้องนิรภัยบนคลาวด์ใหม่

```
ob sync-create-remote --name "Vault Name" [--encryption <standard|e2ee>] [--password <password>] [--region <region>]
```

| ตัวเลือก | คำอธิบาย |
| --- | --- |
| `--name` | ชื่อห้องนิรภัย (จำเป็น) |
| `--encryption` | `standard` สำหรับการเข้ารหัสแบบจัดการ, `e2ee` สำหรับการเข้ารหัสแบบต้นทางถึงปลายทาง |
| `--password` | รหัสผ่านการเข้ารหัสแบบต้นทางถึงปลายทาง (จะถามหากไม่ได้ระบุ) |
| `--region` | [[ภูมิภาคของ Sync|ภูมิภาค]]ของเซิร์ฟเวอร์ (อัตโนมัติหากไม่ได้ระบุ) |

### `ob sync-setup`

ตั้งค่าการซิงค์ระหว่างห้องนิรภัยในเครื่องและห้องนิรภัยบนคลาวด์

```
ob sync-setup --vault <id-or-name> [--path <local-path>] [--password <password>] [--device-name <name>] [--config-dir <name>]
```

| ตัวเลือก | คำอธิบาย |
| --- | --- |
| `--vault` | ID หรือชื่อของห้องนิรภัยบนคลาวด์ (จำเป็น) |
| `--path` | ไดเรกทอรีในเครื่อง (ค่าเริ่มต้น: ไดเรกทอรีปัจจุบัน) |
| `--password` | รหัสผ่านการเข้ารหัสแบบ E2E (จะถามหากไม่ได้ระบุ) |
| `--device-name` | ชื่ออุปกรณ์ที่แสดงใน[[ประวัติเวอร์ชัน|ประวัติเวอร์ชันการซิงค์]] |
| `--config-dir` | ชื่อ[[โฟลเดอร์การตั้งค่า|ไดเรกทอรีการตั้งค่า]] (ค่าเริ่มต้น: `.obsidian`) |

### `ob sync`

เรียกใช้การซิงค์สำหรับห้องนิรภัยที่ตั้งค่าแล้ว

```
ob sync [--path <local-path>] [--continuous]
```

| ตัวเลือก | คำอธิบาย |
| --- | --- |
| `--path` | เส้นทางห้องนิรภัยในเครื่อง (ค่าเริ่มต้น: ไดเรกทอรีปัจจุบัน) |
| `--continuous` | ทำงานต่อเนื่อง เฝ้าดูการเปลี่ยนแปลง |

### `ob sync-config`

ดูหรือเปลี่ยน[[การตั้งค่า Sync และเลือกซิงค์บางไฟล์|การตั้งค่า sync]] สำหรับห้องนิรภัย เรียกใช้โดยไม่มีตัวเลือกเพื่อแสดงการตั้งค่าปัจจุบัน

```
ob sync-config [--path <local-path>] [options]
```

| ตัวเลือก                | คำอธิบาย                                                                                                                                                                                                        |
| --------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `--path`              | เส้นทางห้องนิรภัยในเครื่อง (ค่าเริ่มต้น: ไดเรกทอรีปัจจุบัน)                                                                                                                                                                  |
| `--mode`              | โหมดซิงค์: `bidirectional` (ค่าเริ่มต้น), `pull-only` (ดาวน์โหลดเท่านั้น ไม่สนใจการเปลี่ยนแปลงในเครื่อง) หรือ `mirror-remote` (ดาวน์โหลดเท่านั้น ย้อนกลับการเปลี่ยนแปลงในเครื่อง)                                                              |
| `--conflict-strategy` | `merge` หรือ `conflict`                                                                                                                                                                                          |
| `--file-types`        | ประเภทไฟล์แนบที่จะซิงค์: `image`, `audio`, `video`, `pdf`, `unsupported` (คั่นด้วยจุลภาค, เว้นว่างเพื่อล้าง)                                                                                                    |
| `--configs`           | หมวดหมู่การตั้งค่าที่จะซิงค์: `app`, `appearance`, `appearance-data`, `hotkey`, `core-plugin`, `core-plugin-data`, `community-plugin`, `community-plugin-data` (คั่นด้วยจุลภาค, เว้นว่างเพื่อปิดการซิงค์การตั้งค่า) |
| `--excluded-folders`  | โฟลเดอร์ที่ต้องการยกเว้น (คั่นด้วยจุลภาค, เว้นว่างเพื่อล้าง)                                                                                                                                                           |
| `--device-name`       | ชื่ออุปกรณ์เพื่อระบุตัวไคลเอนต์นี้ในประวัติเวอร์ชันการซิงค์                                                                                                                                                |
| `--config-dir`        | ชื่อไดเรกทอรีการตั้งค่า (ค่าเริ่มต้น: `.obsidian`)                                                                                                                                                                   |

### `ob sync-status`

แสดงสถานะการซิงค์และการตั้งค่าสำหรับห้องนิรภัย

```
ob sync-status [--path <local-path>]
```

### `ob sync-unlink`

หยุดการเชื่อมต่อห้องนิรภัยจากการซิงค์และลบข้อมูลรับรองที่จัดเก็บไว้

```
ob sync-unlink [--path <local-path>]
```

## โมดูลเนทีฟ

Obsidian Headless รวม addon เนทีฟที่คอมไพล์แล้วสำหรับตั้งค่าเวลาสร้างไฟล์ (birthtime) บน Windows และ macOS ซึ่งจะรักษาเวลาสร้างต้นฉบับเมื่อดาวน์โหลดไฟล์จากเซิร์ฟเวอร์

addon นี้ใช้ N-API เวอร์ชั่น 3 ดังนั้นไบนารีที่คอมไพล์แล้วจึงมี ABI เสถียรและทำงานข้าม Node.js เวอร์ชั่นต่าง ๆ ได้โดยไม่ต้องคอมไพล์ใหม่

บน Linux ไม่รองรับ birthtime — addon จะไม่ถูกรวมมาและการซิงค์จะทำงานปกติโดยไม่ต้องใช้มัน

ไบนารีที่คอมไพล์แล้วรวมอยู่สำหรับ:

- `win32-x64`
- `win32-arm64`
- `win32-ia32`
- `darwin-x64`
- `darwin-arm64`
