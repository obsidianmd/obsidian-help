แผนที่เป็นประเภทหนึ่งของ[[มุมมอง|มุมมอง]]ที่คุณสามารถใช้ใน[[แนะนำฐาน|ฐาน]] ต้องติดตั้ง[ปลั๊กอิน Maps](obsidian://show-plugin?id=maps) ก่อน

เลือก ![[lucide-map.svg#icon]]  **Map** จากเมนูมุมมองเพื่อแสดงไฟล์เป็นแผนที่แบบโต้ตอบพร้อมสัญลักษณ์แสดงหัวข้อย่อยสำหรับแต่ละไฟล์ และพรีวิวที่แสดงคุณสมบัติของไฟล์นั้น

![[bases-map-places.png#interface]]

## ติดตั้งปลั๊กอิน Maps

มุมมองแผนที่ต้องการ Obsidian 1.10 [ปลั๊กอิน Maps](obsidian://show-plugin?id=maps) เป็น[[ปลั๊กอินโดยชุมชน|ปลั๊กอินชุมชน]]อย่างเป็นทางการที่คุณสามารถดาวน์โหลดแยกต่างหาก

1. ทำตามคำแนะนำใน[[ปลั๊กอินโดยชุมชน#ติดตั้งปลั๊กอินชุมชน]]
2. ดาวน์โหลดและเปิดใช้งาน [Maps](obsidian://show-plugin?id=maps) จากรายการ

## ตัวอย่าง

เริ่มต้นโดยลองสร้างโน้ตชื่อ **Eiffel Tower** และคัดลอกคุณสมบัติต่อไปนี้ลงไป:

```yaml
---
coordinates:
  - "48.85837"
  - "2.294481"
icon: "landmark"
color: "red"
tags:
  - places
---
```

นี่คือความหมายของโค้ดด้านบน:

| คุณสมบัติ      | ค่า                      |                                                                                                                                                      |
| ------------- | ------------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------- |
| `coordinates` | `48.85837`<br>`2.294481` | พิกัดจัดเก็บเป็น `latitude, longitude` คุณสามารถรับพิกัดโดยคลิกขวาที่ตำแหน่งบนแผนที่แล้วเลือก **Copy coordinates** |
| `icon`        | `landmark`               | ชื่อของไอคอนจาก [ไลบรารี Lucide](https://lucide.dev/)                                                                                  |
| `color`       | `red`                    | ค่า CSS ที่ถูกต้อง: hex, RGB, ชื่อสี เป็นต้น                                                                                                       |
| `tags`        | `places`                 | แท็กที่เราจะใช้ค้นหาสัญลักษณ์แสดงหัวข้อย่อยบนแผนที่ในฐานของเรา                                                                                                   |
ตอนนี้สร้างมุมมองแผนที่พร้อมตัวกรองสำหรับแท็ก `places` และตั้งค่าพิกัดสัญลักษณ์แสดงหัวข้อย่อย ไอคอน และสีโดยใช้คุณสมบัติที่ระบุด้านบน

คุณยังสามารถเปิด[ไฟล์ตัวอย่าง](https://github.com/obsidianmd/obsidian-maps/tree/master/examples)เหล่านี้ใน Obsidian เพื่อดูมุมมองแผนที่ที่ใช้งานได้พร้อมสัญลักษณ์แสดงหัวข้อย่อย ไอคอน และสีที่ตั้งค่าไว้แล้ว

## การตั้งค่า

การตั้งค่ามุมมองแผนที่สามารถตั้งค่าได้ใน[[มุมมอง#การตั้งค่ามุมมอง|การตั้งค่ามุมมอง]]

- ความสูงแบบฝัง
- พิกัดจุดศูนย์กลาง
- ข้อจำกัดการซูม
- พิกัด สี และไอคอนของสัญลักษณ์แสดงหัวข้อย่อย
- พื้นหลัง

### สัญลักษณ์แสดงหัวข้อย่อย

#### พิกัด

เพื่อแสดงหมุดบนแผนที่ ไปที่[[มุมมอง#การตั้งค่ามุมมอง|การตั้งค่ามุมมอง]]แล้วเลือกคุณสมบัติ **marker coordinates** คุณสมบัตินั้นต้องมีพิกัด latitude และ longitude รูปแบบต่อไปนี้ได้รับการยอมรับ:

```yaml
# คุณสมบัติข้อความ
coordinates: "lat, lng"

# คุณสมบัติรายการ
coordinates:
  - "lat"
  - "lng"
```

หากคุณจัดเก็บพิกัดเป็นคุณสมบัติ `latitude` และ `longitude` แยกกัน คุณสามารถรวมมันด้วยคุณสมบัติ[[ไวยากรณ์ฐาน#สูตร|สูตร]]โดยกำหนดเป็นอาร์เรย์ของพิกัดโดยใช้สูตรต่อไปนี้: `[latitude, longitude]`

#### ไอคอน

เพิ่มไอคอนให้สัญลักษณ์แสดงหัวข้อย่อยโดยกำหนดคุณสมบัติ **marker icons** ตัวอย่างเช่น คุณสามารถเพิ่มคุณสมบัติชื่อ `icon` ให้โน้ตของคุณแล้วกำหนดค่าเช่น `landmark` หรือ `utensils` จาก[ไลบรารี Lucide](https://lucide.dev/icons/) ที่มีมาพร้อมกับ Obsidian

##### ใช้สูตรเพื่อกำหนดไอคอน

สมมติว่าคุณต้องการให้ร้านอาหารทุกร้านมีไอคอนเดียวกันบนแผนที่:

1. สร้างโน้ตชื่อ **Restaurants** และเพิ่มคุณสมบัติชื่อ `icon` ที่มีค่า `utensils`
2. ให้โน้ตร้านอาหารมีคุณสมบัติชื่อ `type` ที่ลิงก์ไปยังโน้ต `[[Restaurants]]`
3. เพิ่มคุณสมบัติสูตรชื่อ `Type icon` ลงในฐานของคุณด้วยโค้ดต่อไปนี้:
	```js
	list(type)[0].asFile().properties.icon
	```
4. เลือก `Type icon` เป็นไอคอนสัญลักษณ์แสดงหัวข้อย่อยในการตั้งค่ามุมมอง

เรียบร้อย! ตอนนี้แผนที่ของคุณจะแสดงไอคอนจาก *ประเภท* ของสถานที่ ไม่ใช่จากตัวสถานที่เอง

#### สี

ตั้งค่าสีของสัญลักษณ์แสดงหัวข้อย่อย รับค่าเป็น RGB `rgb(0,0,0)`, HEX `#000` หรือตัวแปร CSS เช่น `var(--color-blue)` เช่นเดียวกับตัวอย่างไอคอนด้านบน คุณสามารถใช้คุณสมบัติสูตรเพื่อกำหนดสีแบบไดนามิก

### พื้นหลัง

#### ไทล์แผนที่

ไทล์แผนที่เป็นวิธีมาตรฐานในการแสดงแผนที่ดิจิทัล มีบริการหลายตัวที่คุณสามารถใช้เพื่อปรับแต่งแผนที่ด้วยสไตล์ สี และฟอนต์ที่ไม่ซ้ำ Maps รองรับทั้งไทล์แบบ raster และ vector และรับ URL ไทล์ส่วนใหญ่รวมถึง URL แบบ TileJSON

[OpenFreeMap](https://openfreemap.org/) เสนอสไตล์บางส่วนที่คุณสามารถใช้ได้ฟรี ลองใช้ URL ต่อไปนี้ในการตั้งค่า **Map tiles**:

| ชื่อ     | URL                                              |
| -------- | ------------------------------------------------ |
| Dark     | `https://tiles.openfreemap.org/styles/dark`      |
| Positron | `https://tiles.openfreemap.org/styles/positron`  |
| Liberty  | ``https://tiles.openfreemap.org/styles/liberty`` |
#### ลิงก์ที่มีประโยชน์

- [Maputnik](https://maputnik.github.io/) สำหรับปรับแต่งไทล์แผนที่
- [Protomaps](https://protomaps.com/) สำหรับโฮสต์ไทล์แผนที่ด้วยตนเอง
- บริการโฮสต์อื่นๆ ที่มีระดับฟรีรวมถึง [MapTiler](https://www.maptiler.com/) และ [Mapbox](https://www.mapbox.com/)


## เคล็ดลับ

คุณสามารถลิงก์ไปยังบริการแผนที่ยอดนิยมโดยใช้[[สูตร]] ตัวอย่างเช่น หมุดของคุณสามารถแสดงลิงก์ไปยัง Google Maps โดยใช้สูตรต่อไปนี้:

```js
link("https://www.google.com/maps/search/" + file.name.replace(" ","+"),"Google Maps")
```

หากแผนที่แสดงว่างเปล่าเมื่อคุณโหลดปลั๊กอิน Maps ครั้งแรก ลอง[[อัพเดต Obsidian|อัปเดตเวอร์ชั่นตัวติดตั้ง Obsidian]]

[ปลั๊กอิน Maps](https://github.com/obsidianmd/obsidian-maps) เป็นโอเพนซอร์ส คุณสามารถช่วยได้โดยการส่งรายงานบัก คำขอฟีเจอร์ และ pull request
