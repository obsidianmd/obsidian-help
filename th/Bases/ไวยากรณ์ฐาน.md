เมื่อคุณ[[สร้างฐาน|สร้างฐาน]]ใน Obsidian ไฟล์จะถูกบันทึกเป็นไฟล์ `.base` โดยทั่วไปฐานจะถูกแก้ไขผ่านอินเทอร์เฟซของแอป แต่ไวยากรณ์สามารถแก้ไขด้วยตนเองได้เช่นกัน และสามารถฝังในบล็อกโค้ดได้

ไวยากรณ์[[แนะนำฐาน|ฐาน]]กำหนด[[มุมมอง]] ตัวกรอง และ[[สูตร]] ฐานต้องเป็น YAML ที่ถูกต้องตามสคีมาที่กำหนดไว้ด้านล่าง

## ตัวอย่าง

นี่คือตัวอย่างของไฟล์ฐาน เราจะอธิบายแต่ละส่วนอย่างละเอียด

```yaml
filters:
  or:
    - file.hasTag("tag")
    - and:
        - file.hasTag("book")
        - file.hasLink("Textbook")
    - not:
        - file.hasTag("book")
        - file.inFolder("Required Reading")
formulas:
  formatted_price: 'if(price, price.toFixed(2) + " dollars")'
  ppu: "(price / age).toFixed(2)"
properties:
  status:
    displayName: Status
  formula.formatted_price:
    displayName: "Price"
  file.ext:
    displayName: Extension
summaries:
  customAverage: 'values.mean().round(3)'
views:
  - type: table
    name: "My table"
    limit: 10
    groupBy:
      property: note.age
      direction: DESC
    filters:
      and:
        - 'status != "done"'
        - or:
            - "formula.ppu > 5"
            - "price > 2.1"
    order:
      - file.name
      - file.ext
      - note.age
      - formula.ppu
      - formula.formatted_price
    summaries:
      formula.ppu: Average
```

### ตัวกรอง

โดยค่าเริ่มต้น ฐานจะรวมทุกไฟล์ในห้องนิรภัย ไม่มี `from` หรือ `source` เหมือนใน SQL หรือ Dataview ส่วน `filters` ช่วยให้คุณกำหนดเงื่อนไขเพื่อจำกัดชุดข้อมูลให้แคบลง

```yaml
# ตัวกรองแบบง่าย:
filters:
  and:
    - file.hasTag("tag")

# ตัวกรองแบบซับซ้อน:
filters:
  or:
    - file.hasTag("tag")
    - and:
        - file.hasTag("book")
        - file.hasLink("Textbook")
    - not:
        - file.hasTag("book")
        - file.inFolder("Required Reading")
```

มีสองจุดที่สามารถใช้ตัวกรองได้:

1. ที่ระดับ `filters` แบบทั่วไป (แสดงด้านบน) ซึ่งจะใช้กับทุกมุมมองในฐาน
2. ที่ระดับ `view` ซึ่งใช้เฉพาะกับมุมมองที่ระบุเท่านั้น

สองส่วนนี้ทำงานเหมือนกันในเชิงฟังก์ชัน และเมื่อประเมินสำหรับมุมมอง จะถูกเชื่อมต่อกันด้วย `AND`

ส่วน `filters` ประกอบด้วยคำสั่งตัวกรองเดียวในรูปแบบสตริง หรืออ็อบเจกต์ตัวกรองที่กำหนดแบบเรียกซ้ำ อ็อบเจกต์ตัวกรองอาจมี `and`, `or` หรือ `not` อย่างใดอย่างหนึ่ง คีย์เหล่านี้เป็นรายการที่มีทั้งอ็อบเจกต์ตัวกรองอื่นหรือคำสั่งตัวกรองในรูปแบบสตริง คำสั่งตัวกรองคือบรรทัดที่ประเมินเป็นค่าจริงหรือเท็จเมื่อใช้กับโน้ต ซึ่งอาจเป็นอย่างใดอย่างหนึ่งต่อไปนี้:

- การเปรียบเทียบพื้นฐานโดยใช้ตัวดำเนินการทางคณิตศาสตร์มาตรฐาน
- ฟังก์ชัน มี[[ฟังก์ชัน]]ในตัวหลายตัว และปลั๊กอินสามารถเพิ่มฟังก์ชันเพิ่มเติมได้

ไวยากรณ์และฟังก์ชันที่ใช้ได้สำหรับตัวกรองและสูตรเหมือนกัน

### สูตร

ส่วน `formulas` กำหนด[[สูตร|คุณสมบัติสูตร]]ที่สามารถแสดงได้ในทุกมุมมองในไฟล์ฐาน

```yaml
formulas:
  formatted_price: 'if(price, price.toFixed(2) + " dollars")'
  ppu: "(price / age).toFixed(2)"
```

คุณสมบัติสูตรรองรับตัวดำเนินการทางคณิตศาสตร์พื้นฐานและ[[ฟังก์ชัน]]ในตัวหลายตัว ในอนาคต ปลั๊กอินจะสามารถเพิ่มฟังก์ชันสำหรับใช้ในสูตรได้

คุณสามารถอ้างอิงคุณสมบัติได้หลายวิธีขึ้นอยู่กับประเภท:

- **คุณสมบัติของโน้ต** คือคุณสมบัติที่กำหนดในเมทาดาทาตอนต้นของโน้ต เช่น `note.price` หรือ `note["price"]`
  ถ้าไม่ระบุคำนำหน้า จะถือว่าคุณสมบัตินั้นเป็นคุณสมบัติ `note`
- **คุณสมบัติของไฟล์** อธิบายตัวไฟล์เอง
  เช่น `file.size` หรือ `file.ext` คุณยังสามารถอ้างอิงอ็อบเจกต์ไฟล์โดยตรงได้ เช่น `file.hasLink()`
- **คุณสมบัติสูตร** คือสูตรอื่นในฐาน
  เช่น `formula.formatted_price`

สูตรสามารถใช้ค่าจากคุณสมบัติสูตรอื่นได้ ตราบใดที่ไม่มีการอ้างอิงแบบวนซ้ำ

คุณสมบัติสูตรจะถูกเก็บเป็นสตริงใน YAML เสมอ แต่**ประเภทข้อมูลผลลัพธ์**จริงจะถูกกำหนดโดยประเภทของข้อมูลพื้นฐานและค่าที่ส่งกลับจากฟังก์ชันที่ใช้

โปรดทราบว่าการใช้เครื่องหมายคำพูดซ้อนกันเป็นสิ่งจำเป็นเพื่อรวมข้อความตรงตัวในฟิลด์ YAML ข้อความตรงตัวต้องอยู่ในเครื่องหมายคำพูดเดี่ยวหรือคำพูดคู่

### คุณสมบัติ

ส่วน `properties` อนุญาตให้เก็บข้อมูลการตั้งค่าเกี่ยวกับแต่ละคุณสมบัติ ขึ้นอยู่กับมุมมองแต่ละตัวว่าจะใช้ค่าการตั้งค่าเหล่านี้อย่างไร เช่น ในตาราง ชื่อที่แสดงจะถูกใช้เป็นหัวคอลัมน์

```yaml
properties:
  status:
    displayName: Status
  formula.formatted_price:
    displayName: "Price"
  file.ext:
    displayName: Extension
```

ชื่อที่แสดงจะไม่ถูกใช้ในตัวกรองหรือสูตร

### การสรุป

ส่วน `summaries` สามารถใช้กำหนดสูตรสรุปแบบกำหนดเองได้ นอกจากการกำหนดสูตรสรุปที่นี่แล้ว ยังมีสูตรสรุปค่าเริ่มต้นหลายตัวที่ใช้ได้

```yaml
summaries:
  customAverage: 'values.mean().round(3)'
```

ในตัวอย่างนี้ สูตร `customAverage` เหมือนกับ `Average` ค่าเริ่มต้น ยกเว้นว่าค่าจะถูกปัดเศษเป็นจำนวนตำแหน่งที่แตกต่างกัน ในสูตรสรุป คีย์เวิร์ด `values` เป็นรายการที่มีค่าทั้งหมดของคุณสมบัตินั้นจากทุกโน้ตในชุดผลลัพธ์ สูตรสรุปควรส่งกลับ `Value` เดียว

โปรดทราบว่าส่วน `summaries` นี้แตกต่างจากส่วน `summaries` ในการตั้งค่ามุมมอง (อธิบายด้านล่าง) ที่สูตรสรุปถูกกำหนดให้กับคุณสมบัติเฉพาะ

#### สูตรสรุปค่าเริ่มต้น

| ชื่อ      | ประเภทอินพุต | คำอธิบาย                                                   |
| --------- | ---------- | ------------------------------------------------------------- |
| Average   | Number     | ค่าเฉลี่ยทางคณิตศาสตร์ของตัวเลขทั้งหมดจากค่าอินพุต   |
| Min       | Number     | ตัวเลขที่เล็กที่สุดจากค่าอินพุต                    |
| Max       | Number     | ตัวเลขที่ใหญ่ที่สุดจากค่าอินพุต                     |
| Sum       | Number     | ผลรวมของตัวเลขทั้งหมดในอินพุต                          |
| Range     | Number     | ผลต่างระหว่าง `Max` และ `Min`                       |
| Median    | Number     | ค่ามัธยฐานทางคณิตศาสตร์ของตัวเลขทั้งหมดจากค่าอินพุต |
| Stddev    | Number     | ค่าเบี่ยงเบนมาตรฐานของตัวเลขทั้งหมดจากค่าอินพุต  |
| Earliest  | Date       | วันที่เก่าที่สุดจากค่าอินพุต                      |
| Latest    | Date       | วันที่ล่าสุดจากค่าอินพุต                        |
| Range     | Date       | ผลต่างระหว่าง `Latest` และ `Earliest`               |
| Checked   | Boolean    | จำนวนค่า `true`                                  |
| Unchecked | Boolean    | จำนวนค่า `false`                                 |
| Empty     | Any        | จำนวนค่าในอินพุตที่ว่างเปล่า             |
| Filled    | Any        | จำนวนค่าในอินพุตที่ไม่ว่างเปล่า         |
| Unique    | Any        | จำนวนค่าที่ไม่ซ้ำกันในอินพุต                     |

### มุมมอง

ส่วน `views` กำหนดวิธีที่ข้อมูลสามารถแสดงผลได้ แต่ละรายการในรายการ `views` กำหนดมุมมองแยกต่างหากของข้อมูลเดียวกัน และสามารถมีมุมมองต่างๆ ได้มากเท่าที่ต้องการ

```yaml
views:
  - type: table
    name: "My table"
    limit: 10
    groupBy:
      property: note.age
      direction: DESC
    filters:
      and:
        - 'status != "done"'
        - or:
            - "formula.ppu > 5"
            - "price > 2.1"
    order:
      - file.name
      - file.ext
      - note.age
      - formula.ppu
      - formula.formatted_price
    summaries:
      formula.ppu: Average
```

- `type` เลือกจากประเภทมุมมองในตัวและที่เพิ่มโดยปลั๊กอิน
- `name` คือชื่อที่แสดง และสามารถใช้เพื่อกำหนดมุมมองเริ่มต้น
- `filters` เหมือนกับที่อธิบายข้างต้นทุกประการ แต่ใช้กับมุมมองเท่านั้น
- `groupBy` ระบุคุณสมบัติและทิศทางการจัดเรียง ค่าของคุณสมบัติที่ระบุสำหรับแต่ละแถวจะถูกใช้เพื่อจัดแถวเข้ากลุ่ม
- `summaries` จับคู่ชื่อคุณสมบัติกับการสรุปที่มีชื่อ การสรุปจะทำการรวมค่าของคุณสมบัตินั้นจากทุกแถว

[[มุมมอง]]สามารถเพิ่มข้อมูลเพิ่มเติมเพื่อเก็บข้อมูลที่จำเป็นในการรักษาสถานะหรือแสดงผลอย่างถูกต้อง อย่างไรก็ตาม ผู้พัฒนาปลั๊กอินควรระวังไม่ใช้คีย์ที่ปลั๊กอินฐานหลักใช้อยู่แล้ว เป็นตัวอย่าง มุมมองตารางอาจใช้สิ่งนี้เพื่อจำกัดจำนวนแถวหรือเลือกคอลัมน์ที่ใช้จัดเรียงแถวและทิศทาง ประเภทมุมมองอื่น เช่น แผนที่ อาจใช้สิ่งนี้เพื่อจับคู่คุณสมบัติในโน้ตที่สอดคล้องกับละติจูดและลองจิจูด และคุณสมบัติใดที่ควรแสดงเป็นชื่อพิน

ในอนาคต API จะอนุญาตให้มุมมองอ่านและเขียนค่าเหล่านี้ ทำให้มุมมองสามารถสร้างอินเทอร์เฟซสำหรับการตั้งค่าของตัวเองได้

## คุณสมบัติ

มีคุณสมบัติสามชนิดที่ใช้ในฐาน:

1. **คุณสมบัติของโน้ต** เก็บในเมทาดาทาตอนต้นของไฟล์ Markdown
2. **คุณสมบัติของไฟล์** เข้าถึงได้สำหรับไฟล์ทุกประเภท
3. **คุณสมบัติสูตร** กำหนดในไฟล์ `.base` เอง (ดูด้านบน)

### คุณสมบัติของโน้ต

[[คุณสมบัติ|คุณสมบัติของโน้ต]]ใช้ได้เฉพาะไฟล์ Markdown และเก็บในเมทาดาทาตอนต้น YAML ของแต่ละโน้ต คุณสมบัติเหล่านี้สามารถเข้าถึงได้โดยใช้รูปแบบ `note.author` หรือเพียงแค่ `author` เป็นตัวย่อ

### คุณสมบัติของไฟล์

คุณสมบัติของไฟล์อ้างอิงถึงไฟล์ที่กำลังถูกทดสอบหรือประเมินอยู่ คุณสมบัติของไฟล์ใช้ได้สำหรับ[[รูปแบบไฟล์ที่รองรับ|ไฟล์ทุกประเภท]] รวมถึงไฟล์แนบ

เช่น ตัวกรอง `file.ext == "md"` จะเป็นจริงสำหรับไฟล์ Markdown ทั้งหมดและเป็นเท็จในกรณีอื่น

| คุณสมบัติ      | ประเภท   | คำอธิบาย                                                   |
| ------------- | ------ | ------------------------------------------------------------- |
| `file.backlinks`  | List   | รายการไฟล์แบ็คลิงค์ หมายเหตุ: คุณสมบัตินี้ใช้ทรัพยากรมาก หากเป็นไปได้ ให้กลับทิศทางการค้นหาและใช้ `file.links` ผลลัพธ์จะไม่รีเฟรชอัตโนมัติเมื่อห้องนิรภัยมีการเปลี่ยนแปลง |
| `file.ctime`  | Date   | เวลาที่สร้าง                                                  |
| `file.embeds` | List   | รายการไฟล์ที่ฝังทั้งหมดในโน้ต                                |
| `file.ext`    | String | นามสกุลไฟล์                                                |
| `file.file`   | File   | อ็อบเจกต์ไฟล์ ใช้ได้เฉพาะในฟังก์ชันเฉพาะ                |
| `file.folder` | String | เส้นทางของโฟลเดอร์ไฟล์                                       |
| `file.links`  | List   | รายการลิงก์ภายในทั้งหมดในโน้ต รวมถึงเมทาดาทาตอนต้น |
| `file.mtime`  | Date   | เวลาที่แก้ไข                                                 |
| `file.name`   | String | ชื่อไฟล์                                                     |
| `file.path`   | String | เส้นทางของไฟล์                                              |
| `file.properties`   | Object | คุณสมบัติทั้งหมดบนไฟล์ หมายเหตุ: ผลลัพธ์จะไม่รีเฟรชอัตโนมัติเมื่อห้องนิรภัยมีการเปลี่ยนแปลง |
| `file.size`   | Number | ขนาดไฟล์                                                     |
| `file.tags`   | List   | รายการแท็กทั้งหมดในเนื้อหาไฟล์และเมทาดาทาตอนต้น          |

### เข้าถึงคุณสมบัติด้วย `this`

ใช้อ็อบเจกต์ `this` เพื่อเข้าถึงคุณสมบัติของไฟล์ สิ่งที่ `this` อ้างอิงถึงจะขึ้นอยู่กับที่ที่ฐานถูกแสดง

เมื่อเปิดฐานในพื้นที่เนื้อหาหลัก `this` จะชี้ไปยังคุณสมบัติของไฟล์ฐานเอง เช่น การใช้ `this.file.folder` จะส่งกลับเส้นทางโฟลเดอร์ที่ฐานตั้งอยู่

เมื่อฐานถูกฝังในไฟล์อื่น `this` จะชี้ไปยังคุณสมบัติของ _ไฟล์ที่ฝัง_ (โน้ตหรือ Canvas ที่มีฐานอยู่) เช่น การใช้ `this.file.name` จะส่งกลับชื่อของไฟล์ที่ฝัง ไม่ใช่ฐาน

เมื่อฐานอยู่ในแถบด้านข้าง `this` จะอ้างอิงถึงไฟล์ที่ใช้งานอยู่ในพื้นที่เนื้อหาหลัก ซึ่งช่วยให้คุณสร้างคิวรีตามไฟล์ที่ใช้งานอยู่ เช่น คุณสามารถใช้ `file.hasLink(this.file)` เพื่อจำลองหน้าต่างแบ็คลิงค์

## ตัวดำเนินการ

### ตัวดำเนินการทางคณิตศาสตร์

ตัวดำเนินการทางคณิตศาสตร์ทำการคำนวณกับตัวเลข เช่น `radius * (2 * 3.14)`

| ตัวดำเนินการ | คำอธิบาย |
| -------- | ----------- |
| `+`      | บวก        |
| `-`      | ลบ       |
| `*`      | คูณ    |
| `/`      | หาร      |
| `%`      | มอดูโล      |
| `( )`    | วงเล็บ |

### การคำนวณวันที่

วันที่สามารถแก้ไขได้โดยการบวกและลบช่วงเวลา หน่วยช่วงเวลายอมรับหลายรูปแบบ:

| หน่วย                     | ช่วงเวลา |
| ------------------------ | -------- |
| `y`, `year`, `years`     | ปี     |
| `M`, `month`, `months`   | เดือน    |
| `d`, `day`, `days`       | วัน      |
| `w`, `week`, `weeks`     | สัปดาห์   |
| `h`, `hour`, `hours`     | ชั่วโมง    |
| `m`, `minute`, `minutes` | นาที   |
| `s`, `second`, `seconds` | วินาที   |

เพื่อแก้ไขหรือเลื่อนอ็อบเจกต์ Date ให้ใช้ตัวดำเนินการ `+` หรือ `-` กับสตริงช่วงเวลา เช่น `date + "1M"` เพิ่ม 1 เดือนให้กับวันที่ ในขณะที่ `date - "2h"` ลบ 2 ชั่วโมงจากวันที่

[[ฟังก์ชัน|ฟังก์ชัน]]แบบทั่วไป `today()` สามารถใช้เพื่อรับวันที่ปัจจุบัน และ `now()` สามารถใช้เพื่อรับวันที่พร้อมเวลาปัจจุบัน

- `now() + "1 day"` ส่งกลับวันที่และเวลาที่ห่างจากเวลาดำเนินการ 24 ชั่วโมงพอดี
- `file.mtime > now() - "1 week"` ส่งกลับ `true` ถ้าไฟล์ถูกแก้ไขภายในสัปดาห์ที่ผ่านมา
- `date("2024-12-01") + "1M" + "4h" + "3m"` ส่งกลับอ็อบเจกต์ Date ที่แทน `2025-01-01 04:03:00`
- ลบวันที่สองตัวเพื่อรับผลต่างเป็นมิลลิวินาทีระหว่างสองค่า เช่น `now() - file.ctime`
- เพื่อรับส่วนวันที่ของ Date ที่มีเวลา ใช้ `datetime.date()`
- เพื่อจัดรูปแบบอ็อบเจกต์ Date ใช้ฟังก์ชัน `format()` เช่น `datetime.format("YYYY-MM-DD")`

### ตัวดำเนินการเปรียบเทียบ

ตัวดำเนินการเปรียบเทียบสามารถใช้เปรียบเทียบตัวเลขหรืออ็อบเจกต์ Date ได้ เท่ากับและไม่เท่ากับสามารถใช้กับค่าทุกชนิด ไม่ใช่แค่ตัวเลขและวันที่

| ตัวดำเนินการ | คำอธิบาย              |
| -------- | ------------------------ |
| `==`     | เท่ากับ                   |
| `!=`     | ไม่เท่ากับ                |
| `>`      | มากกว่า             |
| `<`      | น้อยกว่า                |
| `>=`     | มากกว่าหรือเท่ากับ |
| `<=`     | น้อยกว่าหรือเท่ากับ    |

### ตัวดำเนินการบูลีน

ตัวดำเนินการบูลีนสามารถใช้เพื่อรวมหรือกลับค่าตรรกะ ส่งผลเป็นค่าจริงหรือเท็จ

| ตัวดำเนินการ | คำอธิบาย |
| -------- | ----------- |
| `!`      | นิเสธตรรกะ |
| `&&`     | และตรรกะ |
| \|\|     | หรือตรรกะ  |

## ฟังก์ชัน

ดู[[ฟังก์ชัน|รายการฟังก์ชัน]]ที่สามารถใช้ได้ในสูตรและ[[มุมมอง|ตัวกรอง]]

## ชนิดข้อมูล

ฐานมีระบบชนิดข้อมูลที่ใช้โดยสูตรและตัวกรองเพื่อใช้ฟังก์ชันกับคุณสมบัติ

### สตริง ตัวเลข และบูลีน

สตริง ตัวเลข และบูลีนเป็นค่า "ดั้งเดิม" ที่ไม่ต้องใช้ฟังก์ชันในการสร้าง

- สตริงอยู่ในเครื่องหมายคำพูดเดี่ยวหรือคำพูดคู่ เช่น `"message"`
- ตัวเลขเขียนเป็นตัวเลข และอาจอยู่ในวงเล็บเพื่อความชัดเจน เช่น `1` หรือ `(2.5)`
- บูลีนเขียนเป็น `true` หรือ `false` โดยไม่มีเครื่องหมายคำพูด

### วันที่และช่วงเวลา

วันที่แทนวันที่เฉพาะ หรือวันที่และเวลา ขึ้นอยู่กับฟังก์ชันที่ใช้สร้าง หรือชนิดที่ถูกกำหนดให้กับ[[คุณสมบัติ|คุณสมบัติ]]

- เพื่อสร้างวันที่ ใช้ฟังก์ชัน `date` เช่น `date("2025-01-01 12:00:00")`
- เพื่อแก้ไขวันที่ ให้บวกหรือลบช่วงเวลา เช่น `now() + "1 hour"` หรือ `today() + "7d"`
- เปรียบเทียบวันที่โดยใช้ตัวดำเนินการเปรียบเทียบ (เช่น `>` หรือ `<`) และตัวดำเนินการทางคณิตศาสตร์ (เช่น `(now() + "1d") - now()` ส่งกลับ `86400000` มิลลิวินาที)
- เพื่อดึงส่วนต่างๆ ของวันที่ ใช้ฟิลด์ที่มีอยู่ (`now().hour`) หรือฟังก์ชันอำนวยความสะดวก (`now.time()`)
- มี[[ฟังก์ชัน|ฟิลด์และฟังก์ชัน]]อื่นๆ อีกมากมายที่ใช้ได้กับอ็อบเจกต์วันที่

### อ็อบเจกต์และรายการ

- แปลงองค์ประกอบเดียวเป็นรายการโดยใช้ฟังก์ชัน `list()` สิ่งนี้มีประโยชน์อย่างยิ่งสำหรับคุณสมบัติที่อาจมีทั้งรายการหรือค่าเดี่ยวปะปนกัน
- เข้าถึงองค์ประกอบรายการโดยใช้วงเล็บเหลี่ยมและดัชนีเริ่มจาก 0 เช่น `property[0]` ส่งกลับองค์ประกอบแรกจากรายการ
- เข้าถึงองค์ประกอบอ็อบเจกต์โดยใช้วงเล็บเหลี่ยมกับชื่อองค์ประกอบหรือใช้จุด เช่น `property.subprop` หรือ `property["subprop"]`

### ไฟล์และลิงก์

[[เชื่อมโยงโน้ต|ลิงก์วิกิ]]ใน[[คุณสมบัติ|คุณสมบัติเมทาดาทาตอนต้น]]จะถูกรู้จักโดยอัตโนมัติเป็นอ็อบเจกต์ Link ลิงก์จะแสดงเป็นลิงก์ที่คลิกได้ใน[[มุมมอง|มุมมอง]]

- เพื่อสร้างลิงก์ ใช้[[ฟังก์ชัน|ฟังก์ชัน]] `link` แบบทั่วไป เช่น `link("filename")` หรือ `link("https://obsidian.md")`
- คุณสามารถสร้างลิงก์จากสตริงใดก็ได้ เช่น `link(file.ctime.date().toString())`
- เพื่อตั้งข้อความที่แสดง ส่งสตริงหรือไอคอนเป็นพารามิเตอร์ตัวที่สอง เช่น `link("filename", "display")` หรือ `link("filename", icon("plus"))`

อ็อบเจกต์ File สามารถแปลงเป็นลิงก์โดยใช้ `file.asLink()` พร้อมข้อความที่แสดงเป็นตัวเลือก

ลิงก์สามารถเปรียบเทียบด้วย `==` และ `!=` ลิงก์จะเท่ากันตราบใดที่ชี้ไปยังไฟล์เดียวกัน หรือถ้าไฟล์ไม่มีอยู่เมื่อค้นหา ข้อความลิงก์ต้องเหมือนกันทุกประการ

ลิงก์สามารถเปรียบเทียบกับไฟล์ เช่น `file` หรือ `this` จะเท่ากันถ้าลิงก์ชี้ไปยังไฟล์นั้น เช่น `author == this`

ลิงก์ยังสามารถตรวจสอบในการมีอยู่ในรายการได้ เช่น `authors.contains(this)`
