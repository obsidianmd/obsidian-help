ใช้บล็อกเน้นเพื่อเพิ่มเนื้อหาเสริมโดยไม่ทำให้การไหลของโน้ตของคุณขาดตอน

ในการสร้างบล็อกเน้น ให้เพิ่ม `[!info]` ไปที่บรรทัดแรกของบล็อกคำคม โดยที่ `info` คือ _ตัวระบุประเภท_ ตัวระบุประเภทจะกำหนดรูปลักษณ์และสไตล์ของบล็อกเน้น หากต้องการดูประเภทที่ใช้ได้ทั้งหมด ให้ดูที่ [[#ประเภทที่รองรับ]] บล็อกเน้นยังรองรับโดยตรงบน [[แนะนำ Obsidian Publish|Obsidian Publish]]

```markdown
> [!info] นี่คือชื่อบล็อกเน้น
> นี่คือบล็อกเน้น
> รองรับ **Markdown**, [[ลิงค์ภายใน|ลิงก์วิกิ]] และ [[ฝังไฟล์|การฝังไฟล์]]!
> ![[Engelbart.jpg]]
```

> [!info] นี่คือชื่อบล็อกเน้น
> นี่คือบล็อกเน้น
> รองรับ **Markdown**, [[ลิงค์ภายใน|ลิงก์วิกิ]] และ [[ฝังไฟล์|การฝังไฟล์]]!
> ![[Engelbart.jpg]]

คุณสามารถแทรกบล็อกเน้น `[!note]` ค่าเริ่มต้นได้โดยใช้ [[กระดานคำสั่ง|คำสั่ง]] `Insert callout` เคอร์เซอร์จะถูกวางไว้ในช่องชื่อบล็อกเน้นโดยอัตโนมัติ ให้คุณลบชื่อค่าเริ่มต้นและพิมพ์ชื่อใหม่ก่อนแก้ไขเนื้อหา

หากต้องการครอบเนื้อหาที่มีอยู่แล้วด้วยบล็อกเน้น ให้เลือกข้อความ (รวมถึงรายการ บล็อกโค้ด ฯลฯ) แล้วรันคำสั่ง `Insert callout` เนื้อหาที่เลือกจะถูกล้อมในบล็อกเน้นโดยอัตโนมัติ

ใน [[มุมมองและโหมดแก้ไข#แสดงตัวอย่างแบบสด|แสดงตัวอย่างแบบสด]] คุณยังสามารถคลิกขวาที่ชื่อบล็อกเน้นเพื่อเปลี่ยนประเภทบล็อกเน้นได้


### เปลี่ยนชื่อเรื่อง

โดยค่าเริ่มต้น ชื่อเรื่องของบล็อกเน้นคือตัวระบุประเภทในรูปแบบตัวอักษรใหญ่นำหน้า คุณสามารถเปลี่ยนได้โดยเพิ่มข้อความหลังตัวระบุประเภท:

```markdown
> [!tip] บล็อกเน้นสามารถมีชื่อเรื่องที่กำหนดเองได้
> เช่นอันนี้
```

> [!tip] บล็อกเน้นสามารถมีชื่อเรื่องที่กำหนดเองได้
> เช่นอันนี้

คุณยังสามารถละเนื้อหาเพื่อสร้างบล็อกเน้นที่มีแค่ชื่อเรื่อง:

```markdown
> [!tip] บล็อกเน้นที่มีแค่ชื่อเรื่อง
```

> [!tip] บล็อกเน้นที่มีแค่ชื่อเรื่อง

### บล็อกเน้นแบบพับได้

คุณสามารถทำให้บล็อกเน้นพับได้โดยเพิ่มเครื่องหมายบวก (`+`) หรือเครื่องหมายลบ (`-`) ต่อท้ายตัวระบุประเภทโดยตรง

เครื่องหมายบวกจะขยายบล็อกเน้นโดยค่าเริ่มต้น และเครื่องหมายลบจะยุบมันแทน

```markdown
> [!faq]- บล็อกเน้นพับได้ไหม?
> ได้! ในบล็อกเน้นแบบพับได้ เนื้อหาจะถูกซ่อนเมื่อบล็อกเน้นถูกยุบ
```

> [!faq]- บล็อกเน้นพับได้ไหม?
> ได้! ในบล็อกเน้นแบบพับได้ เนื้อหาจะถูกซ่อนเมื่อถูกยุบ

### บล็อกเน้นซ้อนกัน

คุณสามารถซ้อนบล็อกเน้นได้หลายระดับ

```markdown
> [!question] บล็อกเน้นซ้อนกันได้ไหม?
> > [!todo] ได้! ซ้อนกันได้
> > > [!example] คุณยังสามารถใช้การซ้อนหลายระดับได้
```

> [!question] บล็อกเน้นซ้อนกันได้ไหม?
> > [!todo] ได้! ซ้อนกันได้
> > > [!example] คุณยังสามารถใช้การซ้อนหลายระดับได้

### ปรับแต่งบล็อกเน้น

[[snippet CSS]] และ [[ปลั๊กอินโดยชุมชน]] สามารถกำหนดบล็อกเน้นที่กำหนดเอง หรือแม้แต่เขียนทับการตั้งค่าเริ่มต้น

ในการกำหนดบล็อกเน้นที่กำหนดเอง ให้สร้างบล็อก CSS ต่อไปนี้:

```css
.callout[data-callout="custom-question-type"] {
    --callout-color: 0, 0, 0;
    --callout-icon: lucide-alert-circle;
}
```

ค่าของแอตทริบิวต์ `data-callout` คือตัวระบุประเภทที่คุณต้องการใช้ เช่น `[!custom-question-type]`

- `--callout-color` กำหนดสีพื้นหลังโดยใช้ตัวเลข (0–255) สำหรับแดง เขียว และน้ำเงิน
- `--callout-icon` สามารถเป็น ID ไอคอนจาก [lucide.dev](https://lucide.dev) หรือ SVG element

> [!warning] หมายเหตุเกี่ยวกับเวอร์ชันไอคอน lucide
> Obsidian จะอัปเดตไอคอน Lucide เป็นระยะ เวอร์ชันปัจจุบันที่รวมอยู่แสดงด้านล่าง; ใช้ไอคอนเหล่านี้หรือเวอร์ชันก่อนหน้าในบล็อกเน้นที่กำหนดเอง
> ![[เครดิต#^lucide]]

> [!tip] ไอคอน SVG
> แทนที่จะใช้ไอคอน Lucide คุณยังสามารถใช้ SVG element เป็นไอคอนบล็อกเน้นได้
>
> ```css
> --callout-icon: '<svg>...custom svg...</svg>';
> ```

### ประเภทที่รองรับ

คุณสามารถใช้ประเภทบล็อกเน้นและนามแฝงได้หลายแบบ แต่ละประเภทมีสีพื้นหลังและไอคอนที่แตกต่างกัน

ในการใช้สไตล์เริ่มต้นเหล่านี้ ให้แทนที่ `info` ในตัวอย่างด้วยประเภทใดก็ได้ เช่น `[!tip]` หรือ `[!warning]` ประเภทบล็อกเน้นยังสามารถเปลี่ยนได้โดยคลิกขวาที่บล็อกเน้นในโหมดแสดงตัวอย่างแบบสด

เว้นแต่คุณจะ [[#ปรับแต่งบล็อกเน้น]] ประเภทที่ไม่รองรับจะใช้ประเภท `note` เป็นค่าเริ่มต้น ตัวระบุประเภทไม่สนใจตัวพิมพ์ใหญ่-เล็ก

> [!note]
> ```md
> > [!note]
> > Lorem ipsum dolor sit amet
> ```

---

> [!abstract]-
> ```md
> > [!abstract]
> > Lorem ipsum dolor sit amet
> ```

> ```

---

> [!tip]-
> ```md
> > [!tip]
> > Lorem ipsum dolor sit amet
> ```
