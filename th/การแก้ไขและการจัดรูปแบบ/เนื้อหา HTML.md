Obsidian รองรับ HTML เพื่อให้คุณแสดงโน้ตในแบบที่ต้องการ หรือแม้แต่[[ฝังหน้าเว็บ|ฝังหน้าเว็บ]] การอนุญาตให้ใช้ HTML ภายในโน้ตของคุณมาพร้อมกับความเสี่ยง เพื่อป้องกันไม่ให้โค้ดที่เป็นอันตรายสร้างความเสียหาย Obsidian จะ _ล้างข้อมูล (sanitize)_ HTML ทุกตัวในโน้ตของคุณ

> [!example]
> โดยปกติแล้ว element `<script>` จะให้คุณรัน JavaScript ทุกครั้งที่โหลด หาก Obsidian ไม่ล้าง HTML ผู้โจมตีอาจหลอกให้คุณวางข้อความที่มี JavaScript ซึ่งดึงข้อมูลสำคัญจากคอมพิวเตอร์ของคุณและส่งกลับไปให้พวกเขา

อย่างไรก็ตาม เนื่องจากไวยากรณ์ Markdown ไม่รองรับการจัดรูปแบบทุกรูปแบบ การใช้ HTML ที่ผ่านการล้างแล้วจึงเป็นอีกวิธีหนึ่งในการเพิ่มคุณภาพให้กับโน้ตของคุณ เราได้รวมการใช้งาน HTML ที่พบบ่อยไว้ด้านล่าง

## ข้อจำกัดของ HTML

Obsidian มีข้อจำกัดเฉพาะเมื่อใช้ HTML ในโน้ตของคุณ:

### ไม่สามารถใช้ Markdown ภายใน HTML ได้

Obsidian ไม่แสดงผลไวยากรณ์ Markdown ภายใน element HTML นี่เป็นตัวเลือกการออกแบบโดยเจตนาเพื่อเพิ่มประสิทธิภาพและลดความซับซ้อนของ parser เมื่อจัดการเอกสารขนาดใหญ่

ตัวอย่างเช่น สิ่งนี้จะไม่ทำงานตามที่คาดหวัง:

```md
<div>
This **will not** be bold and this `will not` be code.
</div>
```

### บล็อก HTML ต้องสมบูรณ์ในตัวเอง

บล็อก HTML ต้องสมบูรณ์และไม่สามารถมีบรรทัดว่างอยู่ภายในได้ บรรทัดว่างจะทำให้บล็อก HTML เสียหาย

สิ่งนี้จะทำงานได้:

```md
<table>
<tr>
<td>Content here</td>
</tr>
</table>
```

สิ่งนี้จะไม่ทำงานอย่างถูกต้อง:

```md
<table>

<tr>

<td>Content here</td>

</tr>

</table>
```

### เมื่อ Markdown ดูเหมือนทำงานได้ภายใน HTML

แท็ก HTML แบบ inline บางตัว เช่น `<span>` หรือ `<a>` มีฟังก์ชันการทำงานที่จำกัดและอาจดูเหมือนแสดงผล Markdown ได้ แต่จริงๆ แล้วไม่ใช่สิ่งที่เกิดขึ้น Markdown ถูกประมวลผลนอกบริบทของ HTML

สำหรับรายละเอียดเพิ่มเติมเกี่ยวกับวิธีที่ Obsidian จัดการ Markdown ดูที่ [[Obsidian Flavored Markdown]]

## การใช้งาน HTML ที่พบบ่อย

> [!info] รายละเอียดเพิ่มเติมเกี่ยวกับการใช้ `<iframe>` สามารถดูได้ที่ [[ฝังหน้าเว็บ]]

### ความคิดเห็น

[[ไวยากรณ์การจัดรูปแบบพื้นฐาน#เชิงอรรถ|ความคิดเห็นแบบ Markdown]] เป็นวิธีที่แนะนำในการเพิ่มความคิดเห็นที่ซ่อนไว้ภายในโน้ตของคุณ อย่างไรก็ตาม บางวิธีในการแปลงโน้ต Markdown เช่น [Pandoc](https://pandoc.org) มีการรองรับความคิดเห็นแบบ Markdown ที่จำกัด ในกรณีเหล่านั้น คุณสามารถใช้ `<!-- HTML Comment -->` แทนได้!

### ขีดเส้นใต้

หากคุณต้องการขีดเส้นใต้รายการในโน้ตอย่างรวดเร็ว คุณสามารถใช้ `<u>Example</u>` เพื่อสร้าง <u>ข้อความที่ขีดเส้นใต้</u>

### Span/Div

แท็ก Span และ div สามารถใช้เพื่อใช้คลาสกำหนดเองจาก [[snippet CSS|snippet CSS]] หรือการจัดรูปแบบที่กำหนดเอง กับพื้นที่ข้อความที่เลือก ตัวอย่างเช่น การใช้ `<span style="font-family: cursive">your text</span>` สามารถให้คุณ <span style="font-family: cursive">เปลี่ยนฟอนต์</span> ได้อย่างรวดเร็ว

## ขีดทับ

ต้องการขีดทับ <s>ข้อความบางส่วน</s>? ใช้ `<s>this</s>` เพื่อขีดทับ
