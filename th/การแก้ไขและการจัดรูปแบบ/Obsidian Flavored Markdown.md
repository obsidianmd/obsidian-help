Obsidian มุ่งมั่นที่จะมีความสามารถสูงสุดโดยไม่ทำลายรูปแบบที่มีอยู่ ด้วยเหตุนี้ เราจึงใช้การผสมผสานรูปแบบต่างๆ ของ [[ไวยากรณ์การจัดรูปแบบพื้นฐาน|Markdown]]

Obsidian รองรับ [CommonMark](https://commonmark.org/), [GitHub Flavored Markdown](https://github.github.com/gfm/) และ [LaTeX](https://www.latex-project.org/)

> [!tip]- Markdown ภายใน HTML
> Obsidian ไม่แสดงผลไวยากรณ์ Markdown ภายในองค์ประกอบ HTML นี่เป็นการตัดสินใจด้านการออกแบบโดยเจตนาเพื่อเพิ่มประสิทธิภาพและลดความซับซ้อนของตัวแยกวิเคราะห์เมื่อจัดการเอกสารขนาดใหญ่
>
> ตัวอย่างเช่น การจัดรูปแบบ Markdown เช่น `**bold**` หรือ `` `code` `` จะไม่ถูกประมวลผลภายใน `<div>`, `<span>`, `<table>` หรือแท็ก HTML อื่นๆ
>
> ```md
> <div>
> This **will not** be bold.
> </div>
> ```

### ส่วนขยาย Markdown ที่รองรับ

| ไวยากรณ์          | คำอธิบาย                                                           |
| --------------- | --------------------------------------------------------------------- |
| `[[Link]]`      | [[ลิงค์ภายใน]]                                                    |
| `![[Link]]`     | [[ฝังไฟล์]]                                                       |
| `![[Link#^id]]` | [[ลิงค์ภายใน#ลิงค์ไปยังบล็อกในโน้ต\|การอ้างอิงบล็อก]]        |
| `^id`           | [[ลิงค์ภายใน#ลิงค์ไปยังบล็อกในโน้ต\|การกำหนดบล็อก]]        |
| `[^id]`         | [[ไวยากรณ์การจัดรูปแบบพื้นฐาน#เชิงอรรถ\|เชิงอรรถ]]                      |
| `%%Text%%`      | [[ไวยากรณ์การจัดรูปแบบพื้นฐาน#ความคิดเห็น\|ความคิดเห็น]]                        |
| `~~Text~~`      | [[ไวยากรณ์การจัดรูปแบบพื้นฐาน#ตัวหนา, ตัวเอียง, ไฮไลต์\|ขีดทับ]] |
| `==Text==`      | [[ไวยากรณ์การจัดรูปแบบพื้นฐาน#ตัวหนา, ตัวเอียง, ไฮไลต์\|ไฮไลต์]]     |

| `- [ ]`         | [[ไวยากรณ์การจัดรูปแบบพื้นฐาน#รายการงาน\|งานที่ยังไม่เสร็จ]]               |
| `- [x]`         | [[ไวยากรณ์การจัดรูปแบบพื้นฐาน#รายการงาน\|งานที่เสร็จแล้ว]]                |
