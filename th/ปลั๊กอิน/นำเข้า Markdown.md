นำเข้า Markdown เป็น[[ปลั๊กอินหลัก|ปลั๊กอินหลัก]]ที่ช่วยให้คุณแปลง Markdown จากแอปพลิเคชันอื่นเป็นรูปแบบ Obsidian นอกจากนี้ยังช่วยให้คุณแปลง[[คุณสมบัติ]]บางรายการเป็นรูปแบบใหม่ที่จำเป็น

> [!warning] คำเตือน
> นำเข้า Markdown จะแปลงห้องนิรภัยทั้งหมดของคุณตามการตั้งค่าของคุณ [[สำรองข้อมูลไฟล์ Obsidian ของคุณ]]ก่อนที่คุณจะทำการแปลง

ในการแปลงโน้ตทั้งหมดในห้องนิรภัยของคุณ:

1. ใน[[กระดานคำสั่ง]] เลือก **เปิดหน้าต่างนำเข้า Markdown** ซึ่งสามารถพบได้ที่[[ริบบอน]]ด้วยไอคอน **เปิดหน้าต่างนำเข้า Markdown** ![[lucide-binary.svg#icon]]
2. เปิดใช้งานรูปแบบที่คุณต้องการแปลง
3. คลิก **เริ่มแปลงข้อมูล**

สำหรับข้อมูลเพิ่มเติม โปรดดูที่ [[ไวยากรณ์การจัดรูปแบบพื้นฐาน]]

## รูปแบบที่รองรับ

### Roam Research

นำเข้า Markdown สามารถแปลงไวยากรณ์ Roam Research ต่อไปนี้:

- **แท็ก**: แปลง `#tag` และ `#[[tag]]` เป็น `[[tag]]`
- **ไฮไลต์**: แปลง `^^highlight^^` เป็น `==highlight==`
- **รายการ TODO**: แปลง `{{[[TODO]]}}` เป็น `[ ]`

### Bear

นำเข้า Markdown สามารถแปลงไวยากรณ์ Bear ต่อไปนี้:

- **ไฮไลต์**: แปลง `::highlight::` เป็น `==highlight==`

### Zettelkasten

นำเข้า Markdown สามารถแปลงไวยากรณ์ Zettelkasten ต่อไปนี้:

- **ลิงก์แบบเต็ม**: แปลง `[[UID]]` เป็น `[[UID File Name]]`
- **ลิงก์แบบสวย**: แปลง `[[UID]]` เป็น `[[UID File Name|File Name]]`

### [[คุณสมบัติ]]

ตั้งแต่ Obsidian `1.9.3` นำเข้า Markdown สามารถแปลงรูปแบบ[[คุณสมบัติ#คุณสมบัติที่เลิกใช้แล้ว|คุณสมบัติที่เลิกใช้แล้ว]]เป็นรูปแบบปัจจุบัน:

**นามแฝง**

```yaml
# ก่อน

alias: My Note Title

# หลัง

aliases:
  - My Note Title
```

**แท็ก**

```yaml
# ก่อน

tag: project, important

# หลัง

tags:
  - project
  - important
```

**คลาส CSS**

```yaml
# ก่อน

cssclass: custom-style

# หลัง

cssclasses:
  - custom-style
```
