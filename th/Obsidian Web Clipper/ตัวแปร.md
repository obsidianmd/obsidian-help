[[Obsidian Web Clipper/เทมเพลต|เทมเพลต Web Clipper]] สามารถใช้ตัวแปรเพื่อเติมข้อมูลจากหน้าเว็บลงในเทมเพลตโดยอัตโนมัติ ตัวแปรสามารถใช้ใน**ชื่อบันทึก** **ตำแหน่งบันทึก** **คุณสมบัติ** และ**เนื้อหาบันทึก** ตัวแปรยังสามารถปรับเปลี่ยนได้โดยใช้[[ตัวกรอง|ตัวกรอง]]

ใช้ไอคอน `...` ในส่วนขยาย [[แนะนำ Obsidian Web Clipper|Web Clipper]] เพื่อเข้าถึงตัวแปรหน้าปัจจุบันสำหรับใช้ในเทมเพลต มีตัวแปร 5 ประเภทที่คุณสามารถใช้ได้:

- [[ตัวแปร#ตัวแปรที่ตั้งไว้ล่วงหน้า|ตัวแปรที่ตั้งไว้ล่วงหน้า]]
- [[ตัวแปร#ตัวแปรพรอมต์|ตัวแปรพรอมต์]]
- [[ตัวแปร#ตัวแปรเมตา|ตัวแปรเมตา]]
- [[ตัวแปร#ตัวแปรตัวเลือก|ตัวแปรตัวเลือก]]
- [[ตัวแปร#ตัวแปร Schema.org|ตัวแปร Schema.org]]

## ตัวแปรที่ตั้งไว้ล่วงหน้า

ตัวแปรที่ตั้งไว้ล่วงหน้าจะถูกสร้างขึ้นโดยอัตโนมัติตามเนื้อหาของหน้า โดยทั่วไปจะใช้ได้กับเว็บไซต์ส่วนใหญ่

ตัวแปรเนื้อหาหลักคือ `{{content}}` ซึ่งประกอบด้วยเนื้อหาบทความ หรือ[[ไฮไลต์หน้าเว็บ|ไฮไลต์]] หรือส่วนที่เลือกหากมีข้อความที่ถูกเลือกบนหน้า โปรดทราบว่า `{{content}}` จะพยายามดึงเนื้อหาหลักของหน้า ซึ่งอาจไม่ใช่สิ่งที่คุณต้องการเสมอไป ในกรณีนั้น คุณสามารถใช้ตัวแปรที่ตั้งไว้ล่วงหน้าอื่นๆ หรือตัวแปรตัวเลือกเพื่อดึงเนื้อหาที่คุณต้องการ

| ตัวแปร              | คำอธิบาย                                                                               |
| ------------------- | -------------------------------------------------------------------------------------- |
| `{{author}}`        | ผู้เขียนของหน้า                                                                        |
| `{{content}}`       | เนื้อหาบทความ [[ไฮไลต์หน้าเว็บ\|ไฮไลต์]] หรือส่วนที่เลือก ในรูปแบบ Markdown          |
| `{{contentHtml}}`   | เนื้อหาบทความ [[ไฮไลต์หน้าเว็บ\|ไฮไลต์]] หรือส่วนที่เลือก ในรูปแบบ HTML              |
| `{{date}}`          | วันที่ปัจจุบัน สามารถจัดรูปแบบได้โดยใช้ตัวกรอง `date`                                  |
| `{{description}}`   | คำอธิบายหรือข้อความคัดย่อ                                                              |
| `{{domain}}`        | โดเมน                                                                                  |
| `{{favicon}}`       | URL ของ Favicon                                                                        |
| `{{fullHtml}}`      | HTML ที่ยังไม่ผ่านการประมวลผลของเนื้อหาหน้าทั้งหมด                                    |
| `{{highlights}}`    | [[ไฮไลต์หน้าเว็บ\|ไฮไลต์]] พร้อมข้อความและตราเวลา                                    |
| `{{image}}`         | URL ของรูปภาพแชร์สังคม                                                                 |
| `{{published}}`     | วันที่เผยแพร่ สามารถจัดรูปแบบได้โดยใช้ตัวกรอง `date`                                  |
| `{{selection}}`     | ส่วนที่เลือกในรูปแบบ Markdown                                                          |
| `{{selectionHtml}}` | ส่วนที่เลือกในรูปแบบ HTML                                                              |
| `{{site}}`          | ชื่อเว็บไซต์หรือผู้เผยแพร่                                                            |

| `{{time}}`          | วันที่และเวลาปัจจุบัน                                                                 |
| `{{url}}`           | URL ปัจจุบัน                                                                           |
| `{{words}}`         | การนับคำ                                                                               |

## ตัวแปรพรอมต์

ตัวแปรพรอมต์ใช้ประโยชน์จากโมเดลภาษาเพื่อดึงและปรับเปลี่ยนข้อมูลโดยใช้ภาษาธรรมชาติ ตัวแปรพรอมต์ต้องการให้[[แปลหน้าเว็บ|ตัวแปล]]เปิดใช้งานและตั้งค่าแล้ว

ตัวแปรพรอมต์ใช้ไวยากรณ์ `{{"สรุปของหน้า"}}` เครื่องหมายอัญประกาศคู่รอบพรอมต์มีความสำคัญและใช้แยกแยะพรอมต์จากตัวแปรที่ตั้งไว้ล่วงหน้า คำตอบของพรอมต์สามารถประมวลผลต่อด้วย[[ตัวกรอง|ตัวกรอง]] เช่น `{{"สรุปของหน้า"|blockquote}}`

### เมื่อใดควรใช้ตัวแปรพรอมต์

ตัวแปรพรอมต์มีข้อดีคือมีความยืดหยุ่นสูงและเขียนง่าย อย่างไรก็ตามมีข้อแลกเปลี่ยนหลายประการ: ทำงานช้ากว่า และอาจมีข้อพิจารณาเรื่องค่าใช้จ่ายและความเป็นส่วนตัวขึ้นอยู่กับ[[แปลหน้าเว็บ#โมเดล|ผู้ให้บริการ]]ที่คุณเลือก

ต่างจากตัวแปรประเภทอื่น ตัวแปรพรอมต์ต้องถูกประมวลผลโดยโมเดลภาษาภายนอก ดังนั้นจะถูกแทนที่เมื่อ[[แปลหน้าเว็บ|ตัวแปล]]ทำงานเสร็จแล้วเท่านั้น

ทางที่ดีไม่ควรใช้ตัวแปรพรอมต์หากข้อมูลที่คุณต้องการดึงอยู่ในรูปแบบที่สม่ำเสมอซึ่งสามารถดึงได้ด้วยตัวแปรประเภทอื่น

ในทางกลับกัน ตัวแปรพรอมต์อาจมีประโยชน์หากข้อมูลที่คุณต้องการดึงอยู่ในรูปแบบที่*ไม่สม่ำเสมอ*ข้ามเว็บไซต์ต่างๆ ตัวอย่างเช่น คุณสามารถสร้าง[[Obsidian Web Clipper/เทมเพลต|เทมเพลต]]เพื่อบันทึกหนังสือที่ไม่ขึ้นกับเว็บไซต์หนังสือใดเว็บไซต์หนึ่ง ตัวแปรพรอมต์อย่าง `{{"ผู้เขียนหนังสือ"}}` จะใช้ได้กับทุกเว็บไซต์หนังสือ ในขณะที่ตัวแปรตัวเลือกมักจะใช้ได้กับเว็บไซต์เดียวเท่านั้น

### ตัวอย่าง

พรอมต์สามารถใช้คิวรีภาษาธรรมชาติได้เกือบทุกแบบ ขึ้นอยู่กับโมเดลที่คุณใช้ พรอมต์สามารถค้นหาหรือแปลข้อมูลข้ามภาษาได้

- `{{"สรุปสามหัวข้อย่อย แปลเป็นภาษาฝรั่งเศส"}}` เพื่อดึงหัวข้อย่อยเกี่ยวกับหน้าและแปลเป็นภาษาฝรั่งเศส
- `{{"un resumé de la page en trois points"}}` เพื่อดึงสามหัวข้อย่อยโดยใช้พรอมต์เป็นภาษาฝรั่งเศส

พรอมต์สามารถแปลงเนื้อหาหน้าเป็น JSON ที่สามารถจัดการได้ด้วย[[ตัวกรอง|ตัวกรอง]] ตัวอย่างเช่น:

```
{{"return a JSON object for each tweet, that includes the author, tweet_text, date in YYYY-MM-DD format, and images array (if there are any)"|map:tweet => ({text: tweet.tweet_text, author: tweet.author, date: tweet.date})|template:"${text}\n— [[@${author}]], [[${date}]]\n"}}
```


## ตัวแปรเมตา

ตัวแปรเมตาช่วยให้คุณดึงข้อมูลจาก [อิลิเมนต์เมตา](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/meta) ในหน้า รวมถึงข้อมูล [Open Graph](https://ogp.me/) ที่ใช้เติมพรีวิวการแชร์สังคม

- `{{meta:name}}` ส่งคืนเนื้อหาของแท็กเมตาชื่อที่มีชื่อที่กำหนด เช่น `{{meta:name:description}}` สำหรับแท็กเมตา `description`
- `{{meta:property}}` ส่งคืนเนื้อหาของแท็กเมตาคุณสมบัติที่มีคุณสมบัติที่กำหนด เช่น `{{meta:property:og:title}}` สำหรับแท็กเมตา `og:title`

## ตัวแปรตัวเลือก

ตัวแปรตัวเลือกช่วยให้คุณดึงเนื้อหาข้อความจากอิลิเมนต์บนหน้าโดยใช้ [ตัวเลือก CSS](https://developer.mozilla.org/en-US/docs/Web/CSS/CSS_selectors/Selectors_and_combinators)

ไวยากรณ์คือ `{{selector:cssSelector?attribute}}` โดยที่ `?attribute` เป็นตัวเลือกที่ไม่จำเป็น หากไม่ได้ระบุ attribute จะส่งคืนเนื้อหาข้อความของอิลิเมนต์ คุณยังสามารถใช้ `{{selectorHtml:cssSelector}}` เพื่อรับเนื้อหา HTML ของอิลิเมนต์ ตัวแปรตัวเลือกมักจะทำงานได้ดีที่สุดบนเว็บไซต์เฉพาะหรือชุดเว็บไซต์ที่มีโครงสร้าง HTML ที่สม่ำเสมอ

- `{{selector:h1}}` ส่งคืนเนื้อหาข้อความของอิลิเมนต์ `h1` ใดๆ บนหน้า
- `{{selector:.author}}` ส่งคืนเนื้อหาข้อความของอิลิเมนต์ `.author` ใดๆ บนหน้า
- `{{selector:img.hero?src}}` ส่งคืน attribute `src` ของรูปภาพที่มีคลาส `hero`
- `{{selector:a.main-link?href}}` ส่งคืน attribute `href` ของแท็ก anchor ที่มีคลาส `main-link`
- `{{selectorHtml:body|markdown}}` ส่งคืน HTML ทั้งหมดของอิลิเมนต์ `body` แปลงเป็น Markdown โดยใช้[[ตัวกรอง#การประมวลผล HTML|ตัวกรอง]] `markdown`
- รองรับตัวเลือก CSS แบบซ้อนและตัวรวมหากคุณต้องการความเฉพาะเจาะจงมากขึ้น
- หากมีหลายอิลิเมนต์ตรงกับตัวเลือก จะส่งคืนอาร์เรย์ ซึ่งคุณสามารถประมวลผลด้วย[[ตัวกรอง#อาร์เรย์และอ็อบเจกต์|ตัวกรองอาร์เรย์และอ็อบเจกต์]] เช่น `join` หรือ `map`

ตัวแปรตัวเลือกยังสามารถใช้โดยตรงใน[[ลอจิก|ลอจิกเทมเพลต]]:

- ในลูป: `{% for comment in selector:.comment %}...{% endfor %}`
- ในเงื่อนไข: `{% if selector:.premium-badge %}...{% endif %}`
- ในการกำหนดค่าตัวแปร: `{% set items = selector:.list-item %}`

## ตัวแปร Schema.org

ตัวแปร schema ช่วยให้คุณดึงข้อมูลจาก [schema.org](https://schema.org/) JSON-LD บนหน้า ข้อมูล Schema.org ยังสามารถใช้เพื่อ[[Obsidian Web Clipper/เทมเพลต#การจับคู่ Schema.org|เรียกเทมเพลตโดยอัตโนมัติ]]

- `{{schema:@Type:key}}` ส่งคืนค่าของคีย์จาก schema
- `{{schema:@Type:parent.child}}` ส่งคืนค่าของคุณสมบัติที่ซ้อนกัน
- `{{schema:@Type:arrayKey}}` ส่งคืนรายการแรกในอาร์เรย์
- `{{schema:@Type:arrayKey[index].property}}` ส่งคืนรายการที่ดัชนีที่ระบุในอาร์เรย์
- `{{schema:@Type:arrayKey[*].property}}` ส่งคืนคุณสมบัติเฉพาะจากรายการทั้งหมดในอาร์เรย์

คุณยังสามารถใช้สัญกรณ์ย่อโดยไม่ต้องระบุประเภท schema:

- `{{schema:author}}` จะจับคู่คุณสมบัติ `author` แรกที่พบในประเภท schema ใดก็ได้
- `{{schema:name}}` จะจับคู่คุณสมบัติ `name` แรกที่พบในประเภท schema ใดก็ได้

สัญกรณ์ย่อนี้มีประโยชน์อย่างยิ่งเมื่อคุณไม่ทราบหรือไม่สนใจเกี่ยวกับประเภท schema เฉพาะ แต่คุณรู้ชื่อคุณสมบัติที่คุณกำลังมองหา

คุณสมบัติที่ซ้อนกันและการเข้าถึงอาร์เรย์ก็ใช้งานได้เช่นกัน ทั้งแบบมีและไม่มีการระบุ `@Type` ของ schema:

- `{{schema:author.name}}` จะค้นหาคุณสมบัติ `author` แรกแล้วเข้าถึงคุณสมบัติย่อย `name`
- `{{schema:author[0].name}}` จะเข้าถึง `name` ของผู้เขียนคนแรกในอาร์เรย์ของผู้เขียน
- `{{schema:author[*].name}}` จะส่งคืนอาร์เรย์ของชื่อผู้เขียนทั้งหมด
