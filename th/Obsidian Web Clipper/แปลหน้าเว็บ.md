ตัวแปลเป็นคุณสมบัติของ [[แนะนำ Obsidian Web Clipper|Web Clipper]] ที่ช่วยให้คุณโต้ตอบกับหน้าเว็บโดยใช้ภาษาธรรมชาติ ตัวแปลช่วยให้คุณจับและแก้ไขข้อมูลที่คุณต้องการบันทึกไปยัง Obsidian ตัวอย่างเช่น:

- ดึงชิ้นส่วนข้อความเฉพาะ
- สรุปหรืออธิบายข้อมูล
- แปลงข้อความจากรูปแบบหนึ่งไปเป็นอีกรูปแบบหนึ่ง
- แปลข้อความเป็นภาษาอื่น

ตัวแปลใช้โมเดลภาษาในการประมวลผลข้อมูลบนหน้าเว็บ และส่งคืนผลลัพธ์โดยใช้ [[ตัวแปร]] ที่คุณสามารถเพิ่มเข้าไปใน [[Obsidian Web Clipper/เทมเพลต|เทมเพลตของ Web Clipper]]

## ตัวอย่างพรอมต์

พรอมต์ใช้ไวยากรณ์ [[ตัวแปร|ตัวแปร]] `{{"your prompt"}}` คุณสามารถใช้ไวยากรณ์นี้กับคิวรีภาษาธรรมชาติใดก็ได้ เช่น

- `{{"a summary of the page"}}` เพื่อดึงสรุปของหน้า
- `{{"a three bullet point summary, translated to French"}}` เพื่อดึงหัวข้อย่อยเกี่ยวกับหน้า และแปลเป็นภาษาฝรั่งเศส
- `{{"un resumé de la page en trois points"}}` เพื่อดึงสามหัวข้อย่อยโดยใช้พรอมต์เป็นภาษาฝรั่งเศส

ผลลัพธ์ของพรอมต์สามารถปรับแต่งเพิ่มเติมได้โดยใช้ [[ตัวกรอง]] ตัวกรองจะถูกประมวลผลหลังจากได้รับการตอบกลับจากโมเดลแล้ว ตัวอย่างเช่น: `{{"a summary of the page"|blockquote}}` จะเปลี่ยนการตอบกลับเป็นบล็อกคำคม

## เริ่มต้นใช้งาน

ตัวแปลทำงานได้กับผู้ให้บริการโมเดลภาษาเกือบทุกราย รวมถึงตัวเลือกที่ทำงานแบบส่วนตัวบนอุปกรณ์ของคุณ ในการตั้งค่าตัวแปล:

1. ไปที่ส่วน **ตัวแปล** ในการตั้งค่า Web Clipper
2. เปิด **เปิดใช้งานตัวแปล**
3. ตั้งค่าผู้ให้บริการและโมเดลของคุณ ดูส่วน [[แปลหน้าเว็บ#โมเดล|โมเดล]] ด้านล่าง
4. เพิ่ม [[ตัวแปร|ตัวแปรพรอมต์]] เข้าไปใน [[Obsidian Web Clipper/เทมเพลต|เทมเพลต]] ของคุณ
5. หากเทมเพลตของคุณมีตัวแปรพรอมต์ ส่วนตัวแปลจะแสดงขึ้นเมื่อคุณ [[คลิปหน้าเว็บ|คลิปหน้าเว็บ]] คลิก **แปล** เพื่อประมวลผลตัวแปรพรอมต์

## วิธีการทำงาน

เมื่อเปิดใช้งานตัวแปล *และ* เทมเพลตของคุณมี [[ตัวแปร#ตัวแปรพรอมต์|ตัวแปรพรอมต์]] ส่วนตัวแปลใหม่จะแสดงในหน้าต่างส่วนขยาย เหนือปุ่ม **เพิ่มไปยัง Obsidian** ส่วนนี้ช่วยให้คุณเลือกโมเดลและเรียกใช้ตัวแปลสำหรับหน้าปัจจุบัน

เมื่อคุณคลิก **แปล** ตัวแปลจะส่งบริบทของหน้าไปยังโมเดลที่คุณเลือก พร้อมกับพรอมต์ *ทั้งหมด* ในเทมเพลตของคุณในคำขอเดียว ขึ้นอยู่กับผู้ให้บริการโมเดลที่คุณเลือก อาจเป็นการเรียกภายนอกหรือทำงานภายในอุปกรณ์ของคุณ โมเดลจะประเมินพรอมต์ของคุณเทียบกับบริบทของหน้า และส่งคืนการตอบกลับ จากนั้นตัวแปลจะแทนที่ตัวแปรพรอมต์ด้วยข้อมูลการตอบกลับ

กระบวนการทั้งหมดอาจใช้เวลาเพียงมิลลิวินาทีหรือมากกว่า 30 วินาที ขึ้นอยู่กับโมเดลที่คุณใช้และปริมาณข้อมูลที่คุณประมวลผล

## บริบท

คำว่า *บริบท* หมายถึงข้อมูลหน้าเว็บที่ตัวแปลใช้ในการประมวลผลพรอมต์ ยิ่งบริบทน้อยเท่าไร ตัวแปลก็ทำงานเร็วขึ้นเท่านั้น

โดยค่าเริ่มต้น ตัวแปลใช้ HTML ทั้งหน้าเป็นบริบท อย่างไรก็ตามสิ่งนี้อาจทำให้พรอมต์ช้าลงและมีค่าใช้จ่ายมากกว่าที่จำเป็น

คุณสามารถแทนที่บริบทเริ่มต้นได้ใน **การตั้งค่าขั้นสูง** ของตัวแปล และกำหนดบริบทต่อ [[Obsidian Web Clipper/เทมเพลต|เทมเพลต]]

ในการกำหนดบริบทที่ตรงเป้าหมายมากขึ้น ให้ใช้ [[ตัวแปร#ตัวแปร Selector|ตัวแปร selector]] (หรือตัวแปรประเภทอื่น) เพื่อแปลส่วนหนึ่งของหน้า ตัวอย่างเช่น คุณสามารถใช้ตัวแปร selector ต่อไปนี้ในบริบทตัวแปลของเทมเพลต:

```
{{selectorHtml:#main}}
```

สิ่งนี้จะเรียกใช้ตัวแปลเฉพาะบนองค์ประกอบ `#main` ของหน้าเว็บ ถ้ามีอยู่ [[ตัวกรอง#การประมวลผล HTML|ตัวกรองการประมวลผล HTML]] เช่น `remove_html`, `strip_tags` และ `strip_attr` สามารถเป็นประโยชน์ในการลดความยาวบริบทและเร่งความเร็วการประมวลผล

### การเลือกโมเดล

โดยทั่วไปเราแนะนำให้ใช้โมเดลขนาดเล็กกับ Web Clipper เพราะเร็วกว่าและมีความแม่นยำพอสมควรสำหรับงานนี้ ตัวอย่างของโมเดลขนาดเล็ก ได้แก่ **Claude Haiku ของ Anthropic**, **Google Gemini Flash**, **Llama** ที่มีพารามิเตอร์ 3B หรือ 8B หรือ **ซีรีส์ Mini ของ OpenAI**

### ผู้ให้บริการและโมเดลกำหนดเอง

ในการเพิ่มผู้ให้บริการและ/หรือโมเดลกำหนดเอง ไปที่ **[[การตั้งค่า]]** ของ Web Clipper → **ตัวแปล**:

- **เพิ่มผู้ให้บริการ** เพื่อตั้งค่าผู้ให้บริการที่ตั้งไว้ล่วงหน้าและกำหนดเอง
- **เพิ่มโมเดล** เพื่อตั้งค่าโมเดลที่ตั้งไว้ล่วงหน้าและกำหนดเอง

เมื่อเพิ่มผู้ให้บริการกำหนดเอง เราแนะนำให้คุณใช้ endpoint chat completions ของพวกเขาสำหรับ **URL พื้นฐาน** — โดยปกติจะลงท้ายด้วย `/chat/completions`

### โมเดลในเครื่อง

ตัวแปลสามารถใช้โมเดลในเครื่องซึ่งให้ความเป็นส่วนตัวมากขึ้นและใช้งานแบบออฟไลน์ได้ มีตัวเลือกหลายอย่างสำหรับการเรียกใช้โมเดลในเครื่อง หนึ่งในตัวเลือกที่ตั้งค่าง่ายที่สุดคือ Ollama

#### Ollama

[Ollama](https://ollama.com/) ช่วยให้คุณเรียกใช้โมเดลภาษาแบบในเครื่องและเป็นส่วนตัวบนอุปกรณ์ของคุณ

เมื่อคุณดาวน์โหลดและติดตั้ง Ollama แล้ว ให้เพิ่ม Ollama โดยใช้ **เพิ่มผู้ให้บริการ** ในการตั้งค่าตัวแปล Ollama ไม่ต้องใช้คีย์ API จากนั้นเลือกโมเดลจาก [รายการโมเดล](https://ollama.com/search) ตัวอย่างเช่น หากคุณต้องการใช้ [Llama 3.2](https://ollama.com/library/llama3.2) คลิก **เพิ่มโมเดล** จากนั้น:

- **ผู้ให้บริการ:** Ollama
- **ชื่อที่แสดง:** Llama 3.2 ค่านี้ปรับแต่งได้
- **ID โมเดล:** `llama3.2` ต้องตรงกับ ID โมเดลจาก Ollama อย่างแน่นอน

**เริ่มเซิร์ฟเวอร์ Ollama**

เพื่อให้ส่วนขยายเบราว์เซอร์สามารถโต้ตอบกับ Ollama ได้ คุณต้อง [ให้คำสั่งอย่างชัดเจน](https://github.com/ollama/ollama/issues/2308) เมื่อเรียกใช้เซิร์ฟเวอร์ มิฉะนั้นคุณจะเห็นข้อผิดพลาด `403`

ปิดแอป Ollama และเรียกใช้คำสั่งต่อไปนี้ในเทอร์มินัลของคุณ โปรโตคอลควรเปลี่ยนเป็นโปรโตคอลส่วนขยายของเบราว์เซอร์ของคุณ หากคุณไม่ได้ใช้ Chrome หรือ Firefox

```
OLLAMA_ORIGINS=moz-extension://*,chrome-extension://*,safari-web-extension://* ollama serve
```

จากนั้นเรียกใช้โมเดลของคุณกับ Ollama ตามปกติ เช่น

```
ollama run llama3.2
```

**ความยาวบริบท**

หน้าต่างบริบทของ Ollama เริ่มต้นที่ 2048 โทเค็น นี่คือจำนวนโทเค็นสูงสุดสำหรับข้อความและการตอบกลับ เมื่อคลิปหน้าเว็บยาวๆ คุณสามารถเกินขีดจำกัดนี้ได้อย่างง่ายดาย Ollama จะล้มเหลวอย่างเงียบๆ และส่งคืนผลลัพธ์ที่ไม่เกี่ยวข้อง ตัวเลือกบางอย่าง:

- เพิ่มพารามิเตอร์ `num_ctx` ของ Ollama โปรดทราบว่าบริบทที่ยาวขึ้นต้องใช้หน่วยความจำมากขึ้น
- ใช้ฟิลด์ [[แปลหน้าเว็บ#บริบท|บริบท]] ในเทมเพลตของคุณเพื่อระบุส่วนที่ตรงเป้าหมายมากขึ้นของหน้า หรือตัดบริบทโดยใช้ [[ตัวกรอง|ตัวกรอง]] เช่น `{{content|slice:0,1000}}`
