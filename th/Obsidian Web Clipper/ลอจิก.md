[[แนะนำ Obsidian Web Clipper|Web Clipper]] รองรับลอจิกเทมเพลตสำหรับเงื่อนไข ลูป และการกำหนดค่าตัวแปร ไวยากรณ์นี้ได้รับแรงบันดาลใจจากภาษาเทมเพลต [Twig](https://twig.symfony.com/) และ [Liquid](https://shopify.github.io/liquid/)


> [!warning] ต้องใช้เวอร์ชั่นล่าสุด
> ฟีเจอร์ลอจิกต้องใช้ Obsidian Web Clipper 1.0.0 ซึ่งยังไม่ได้รับการอนุมัติในทุกสโตร์ส่วนขยาย

## เงื่อนไข

ใช้ `{% if %}` เพื่อรวมเนื้อหาตามเงื่อนไขโดยอิงจากตัวแปรหรือนิพจน์

```twig
{% if author %}
Author: {{author}}
{% endif %}
```

ใช้ `{% else %}` เพื่อให้เนื้อหาสำรอง และ `{% elseif %}` เพื่อเชื่อมเงื่อนไขหลายรายการ:

```twig
{% if status == "published" %}
Live article
{% elseif status == "draft" %}
Draft article
{% else %}
Unknown status
{% endif %}
```

### ตัวดำเนินการเปรียบเทียบ

ตัวดำเนินการเปรียบเทียบที่รองรับมีดังนี้:

| ตัวดำเนินการ | คำอธิบาย |
|----------|-------------|
| `==` | เท่ากับ |
| `!=` | ไม่เท่ากับ |
| `>` | มากกว่า |
| `<` | น้อยกว่า |
| `>=` | มากกว่าหรือเท่ากับ |
| `<=` | น้อยกว่าหรือเท่ากับ |
| `contains` | ตรวจสอบว่าสตริงมีสตริงย่อย หรืออาร์เรย์มีค่าที่ระบุ |

ตัวอย่าง:
- `{% if title == "Home" %}` — เปรียบเทียบสตริง
- `{% if price >= 100 %}` — เปรียบเทียบตัวเลข
- `{% if title contains "Review" %}` — ตรวจสอบสตริงย่อย
- `{% if tags contains "important" %}` — ตรวจสอบสมาชิกในอาร์เรย์

### ตัวดำเนินการเชิงตรรกะ

รวมเงื่อนไขโดยใช้ตัวดำเนินการเชิงตรรกะ:

| ตัวดำเนินการ | ทางเลือก | คำอธิบาย                         |
| -------- | ----------- | ----------------------------------- |
| `and`    | `&&`        | ทั้งสองเงื่อนไขต้องเป็นจริง        |
| `or`     | \|\|        | อย่างน้อยหนึ่งเงื่อนไขต้องเป็นจริง |
| `not`    | `!`         | กลับค่าเงื่อนไข                 |

ตัวอย่าง:
- `{% if author and published %}` — ทั้งสองต้องมีอยู่
- `{% if draft or archived %}` — เงื่อนไขใดเงื่อนไขหนึ่ง
- `{% if not hidden %}` — การปฏิเสธ
- `{% if (premium or featured) and published %}` — เงื่อนไขแบบจัดกลุ่ม

### ความเป็นจริง (Truthiness)

เมื่อใช้ตัวแปรโดยไม่มีตัวดำเนินการเปรียบเทียบ ระบบจะประเมิน "ความเป็นจริง" ดังนี้:

- `false`, `null`, `undefined`, สตริงว่าง `""` และ `0` ถือว่าเป็น **เท็จ (falsy)**
- อาร์เรย์ว่าง `[]` ถือว่าเป็น **เท็จ (falsy)**
- ค่าอื่นทั้งหมดถือว่าเป็น **จริง (truthy)**

```twig
{% if content %}
Has content
{% endif %}
```

## กำหนดค่าตัวแปร

ใช้ `{% set %}` เพื่อสร้างหรือแก้ไขตัวแปรภายในเทมเพลตของคุณ:

```twig
{% set slug = title|lower|replace:" ":"-" %}
File: {{slug}}.md
```

ตัวแปรสามารถกำหนดค่าเป็น:
- ตัวแปรอื่น: `{% set name = author %}`
- ค่าตรง: `{% set count = 5 %}` หรือ `{% set label = "Draft" %}`
- นิพจน์พร้อมตัวกรอง: `{% set excerpt = content|truncate:100 %}`
- ผลลัพธ์จาก selector: `{% set comments = selector:.comment %}`

ตัวแปรที่ตั้งด้วย `{% set %}` สามารถใช้ในลอจิกเทมเพลตถัดไปและในเอาต์พุต `{{variable}}`

## ค่าสำรอง

ใช้ตัวดำเนินการ `??` เพื่อให้ค่าสำรองเมื่อตัวแปรว่างเปล่าหรือไม่ได้กำหนดค่า:

```twig
{{title ?? "Untitled"}}
```

หาก `title` ว่างเปล่า ไม่ได้กำหนดค่า หรือเป็นเท็จ ค่าสำรอง `"Untitled"` จะถูกใช้แทน

นี่เป็นรูปแบบย่อของคำสั่ง `if` ที่เทียบเท่า:

```twig
{% if title %}{{title}}{% else %}Untitled{% endif %}
```

### เชื่อมต่อค่าสำรอง

คุณสามารถเชื่อมต่อค่าสำรองหลายรายการ:

```twig
{{title ?? headline ?? "No title"}}
```

ระบบจะใช้ `title` ถ้ามี ไม่เช่นนั้นใช้ `headline` ไม่เช่นนั้นใช้สตริง `"No title"`

### กับตัวกรอง

ตัวกรองมีลำดับความสำคัญมากกว่า `??` ดังนั้นตัวกรองจะถูกใช้ก่อนการตรวจสอบค่าสำรอง:

```twig
{{title|upper ?? "UNTITLED"}}
```

คำสั่งนี้จะใช้ `upper` กับ `title` ก่อน จากนั้นจะใช้ค่าสำรอง `"UNTITLED"` หากผลลัพธ์ว่างเปล่า หากต้องการใช้ตัวกรองกับค่าสำรอง ให้ใช้วงเล็บหรือนิพจน์แยก:

```twig
{{title ?? "Untitled"|lower}}
```

คำสั่งนี้จะใช้ `title` ถ้ามี ไม่เช่นนั้นจะใช้ `lower` กับค่าสำรอง ซึ่งจะได้ผลลัพธ์เป็น `"untitled"`

## ลูป

ใช้ `{% for %}` เพื่อวนซ้ำผ่านอาร์เรย์:

```twig
{% for item in schema:author %}
- {{item.name}}
{% endfor %}
```

### แหล่งข้อมูลของลูป

คุณสามารถวนซ้ำผ่าน:
- อาร์เรย์ Schema: `{% for item in schema:author %}`
- ผลลัพธ์จาก Selector: `{% for comment in selector:.comment %}`
- ตัวแปรที่ตั้งไว้ก่อนหน้า: `{% set items = selector:.item %}{% for item in items %}`

### ตัวแปรในลูป

ภายในลูป คุณสามารถเข้าถึงอ็อบเจกต์ `loop` ที่มีคุณสมบัติดังนี้:

| ตัวแปร | คำอธิบาย |
|----------|-------------|
| `loop.index` | รอบการวนซ้ำปัจจุบัน (เริ่มที่ 1) |
| `loop.index0` | รอบการวนซ้ำปัจจุบัน (เริ่มที่ 0) |
| `loop.first` | `true` ถ้าเป็นรอบแรก |
| `loop.last` | `true` ถ้าเป็นรอบสุดท้าย |
| `loop.length` | จำนวนรายการทั้งหมด |

```twig
{% for tag in tags %}
{{loop.index}}. {{tag}}
{% if loop.last %} (end of list){% endif %}
{% endfor %}
```

เพื่อความเข้ากันได้ย้อนหลัง คุณยังสามารถใช้ `item_index` (โดย `item` คือชื่อตัวแปร iterator ของคุณ) เพื่อรับตำแหน่งแบบเริ่มที่ 0:

```twig
{% for tag in tags %}
{{tag_index}}. {{tag}}
{% endfor %}
```

### เข้าถึงรายการในอาร์เรย์ตามดัชนี

ใช้เครื่องหมายวงเล็บเหลี่ยมเพื่อเข้าถึงสมาชิกของอาร์เรย์ตามดัชนี:

```twig
{{items[0]}}
{{items[loop.index0]}}
```

สิ่งนี้มีประโยชน์เมื่อคุณต้องการเข้าถึงรายการจากหลายอาร์เรย์แบบขนาน:

```twig
{% set transcripts = selector:.transcript-text %}
{% set timestamps = selector:.timestamp %}

{% for line in transcripts %}
{{timestamps[loop.index0]}} - {{line}}
{% endfor %}
```

เครื่องหมายวงเล็บเหลี่ยมยังใช้ได้กับคุณสมบัติของอ็อบเจกต์:

```twig
{{user["name"]}}
{{data["my-key"]}}
```

### ลูปซ้อน

ลูปสามารถซ้อนกันได้สำหรับโครงสร้างข้อมูลที่ซับซ้อน:

```twig
{% for section in sections %}
## {{section.title}}
{% for item in section.items %}
- {{item}}
{% endfor %}
{% endfor %}
```

## รวมลอจิก

เงื่อนไขและลูปสามารถรวมกันได้:

```twig
{% for item in items %}
{% if item.active %}
- {{item.name}}
{% endif %}
{% endfor %}
```

## ลำดับการประเมินผล

ลอจิกเทมเพลตจะถูกประมวลผลตามลำดับดังนี้:

1. **ลอจิกเทมเพลต** — `{% if %}`, `{% for %}`, `{% set %}` และ `{{variables}}` จะถูกประเมินก่อน
2. **ตัวแปร Prompt** — [[ตัวแปร#ตัวแปร Prompt|ตัวแปร Prompt]] เช่น `{{"summarize this"|prompt}}` จะถูกส่งไปยังตัวแปลหลังจากลอจิกเทมเพลตเสร็จสมบูรณ์

ซึ่งหมายความว่าคุณสามารถใช้ลอจิกเทมเพลตเพื่อสร้าง prompt แบบไดนามิก แต่ผลลัพธ์จาก prompt จะไม่สามารถใช้ในเงื่อนไขหรือลูปได้
