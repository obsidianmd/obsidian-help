คุณสามารถตั้งค่าโดเมนกำหนดเองหรือซับโดเมนสำหรับเว็บไซต์ [[แนะนำ Obsidian Publish|Obsidian Publish]] ของคุณได้

> [!warning] คำเตือน
> ขณะนี้เรารองรับเฉพาะการตั้งค่าโดเมนกำหนดเองด้วยวิธีการต่อไปนี้เท่านั้น:
>
> - [[#ตั้งค่าโดยใช้ CloudFlare]] โดยใช้ [โหมด Full](https://developers.cloudflare.com/ssl/origin-configuration/ssl-modes/full/)
> - [[#ตั้งค่าโดยใช้ proxy]]
> 
> เรายังไม่มีวิธีการออกใบรับรอง SSL ในนามของคุณ

## ตั้งค่าโดยใช้ CloudFlare

วิธีที่ง่ายที่สุดในการตั้งค่าโดเมนกำหนดเองหรือซับโดเมนคือการสร้างบัญชี [CloudFlare](https://cloudflare.com) และให้ CloudFlare จัดการ DNS ของโดเมนคุณ

ขั้นตอนต่อไปนี้ใช้ CloudFlare เพื่อตั้งค่าโดเมนกำหนดเองสำหรับเว็บไซต์ Obsidian Publish ของคุณ ไม่ว่าจะใช้โดเมนหลัก (`mysite.com`) หรือซับโดเมน (`notes.mysite.com`)

> [!important] สำคัญ
> CloudFlare เป็น**ผู้ให้บริการเดียวที่รองรับอย่างเป็นทางการ**สำหรับการตั้งค่าโดเมนกำหนดเอง การใช้คำแนะนำต่อไปนี้กับผู้ให้บริการอื่นอาจไม่สามารถทำงานได้

**CloudFlare:**

1. เปิด Cloudflare ไปที่โดเมนที่คุณต้องการโฮสต์เว็บไซต์ Publish เช่น `mysite.com` แม้ว่าคุณต้องการซับโดเมนเช่น `notes.mysite.com`
2. ไปที่ **DNS** และคลิก **Add Record**
3. เลือก **CNAME**
4. ในช่อง **name** ให้ป้อนโดเมนหรือซับโดเมนของคุณ เช่น `notes.mysite.com`
5. ในช่อง **target** ให้ป้อน `publish-main.obsidian.md` อย่าใส่ sub-URL ส่วนตัวของคุณในค่านี้ Obsidian Publish จัดการสิ่งนี้จากการตั้งค่าของคุณ
6. ตรวจสอบให้แน่ใจว่า **proxy status** เปิดใช้งานอยู่ โดยค่าเริ่มต้นจะเปิดใช้งานอยู่แล้ว
7. ไปที่ **SSL/TLS** และตั้งค่าโหมดการเข้ารหัส SSL/TLS เป็น "Full" เพื่อตั้งค่าใบรับรอง SSL/TLS โดยอัตโนมัติ

> [!note] หมายเหตุ
> ในการเปลี่ยนเส้นทางทั้ง `mysite.com` และ `www.mysite.com` ไปยัง Obsidian Publish คุณต้องสร้าง [Page Rule](https://support.cloudflare.com/hc/en-us/articles/200172336-Creating-Page-Rules) ด้วยการตั้งค่าต่อไปนี้:
>
> - URL match: `www.mysite.com/*`
> - Forward URL - 301 Permanent Redirect
> - Redirect URL: `https://mysite.com/$1`
>
> หลังจากสร้าง page rule แล้ว ให้สร้างระเบียน CNAME สำหรับ `www.mysite.com` เช่นเดียวกับที่คุณทำสำหรับ `mysite.com`

**Obsidian:**

1. เปิด Obsidian บนคอมพิวเตอร์ของคุณ
2. ใน[[ริบบอน]]ทางด้านซ้าย คลิก **เผยแพร่การเปลี่ยนแปลง** ![[lucide-send.svg#icon]]
3. ภายใต้ **เผยแพร่การเปลี่ยนแปลง** เลือก **เปลี่ยนการตั้งค่าเว็บไซต์** ![[lucide-cog.svg#icon]]
4. ถัดจาก **Custom domain** เลือก **Configure**
5. ในช่อง **Custom URL** ให้ป้อน URL ของโดเมนหรือซับโดเมนของคุณ ตรวจสอบให้แน่ใจว่าไม่ได้ใส่ `www.` ในช่อง custom URL

> [!note] หมายเหตุ
> หากการตั้งค่าโดเมนกำหนดเองของคุณเกิดการวนลูปเปลี่ยนเส้นทาง อาจเป็นเพราะโหมดการเข้ารหัสใน CloudFlare ถูกตั้งค่าเป็น "Flexible" แทนที่จะเป็น "Full"

## ตั้งค่าโดยใช้ proxy

คุณยังสามารถตั้งค่า SSL/TLS สำหรับโดเมนกำหนดเองของคุณโดยใช้เว็บเซิร์ฟเวอร์ของคุณเอง

หากคุณกำลังโฮสต์เว็บไซต์อยู่แล้วภายใต้โดเมนหรือซับโดเมนของคุณ คุณยังสามารถใช้ตัวเลือกนี้และตั้งค่าเว็บไซต์ของคุณเพื่อโหลดเว็บไซต์ Obsidian Publish ภายใต้เส้นทาง URL เฉพาะ แทนที่จะโฮสต์เว็บไซต์ทั้งหมด

Proxy คำขอทั้งหมดภายใต้เส้นทาง URL นั้นไปที่ `https://publish.obsidian.md/serve?url=mysite.com/my-notes/...` และตั้งค่าตัวเลือกเว็บไซต์ใน Obsidian ให้เป็นเส้นทาง URL เดียวกัน โดยตั้งค่า **Custom URL** เป็น `mysite.com/my-notes`

คุณยังสามารถตั้งค่า Obsidian Publish เป็น sub-URL ของเว็บไซต์ที่คุณเป็นเจ้าของได้ ตัวอย่างเช่น `https://mysite.com/my-notes/` เพื่อทำเช่นนี้ คุณต้องโฮสต์เซิร์ฟเวอร์ของคุณเองและ proxy คำขอทั้งหมดไปยังเซิร์ฟเวอร์ของเราที่ `https://publish.obsidian.md/`

ตัวอย่างการตั้งค่า proxy ต่อไปนี้ไม่ได้ครอบคลุมทั้งหมด แต่ให้วิธีการทั่วไปสำหรับการใช้งานนี้

### NGINX

ในการตั้งค่า NGINX ของคุณ ให้เพิ่มสิ่งต่อไปนี้:

```nginx
location /my-notes {
  proxy_pass https://publish.obsidian.md/serve?url=mysite.com/my-notes/;
  proxy_ssl_server_name on;
  proxy_set_header Host publish.obsidian.md;
}
```


ผู้ใช้บางท่านรายงานว่าอาจต้องเพิ่ม `$request_uri` ใน proxy pass:

```nginx
location /my-notes {
  proxy_pass https://publish.obsidian.md/serve?url=mysite.com/my-notes$request_uri;
  proxy_ssl_server_name on;
  proxy_set_header Host publish.obsidian.md;
}
```

### Apache

ใน `.htaccess` ให้เพิ่มสิ่งต่อไปนี้:

```htaccess
RewriteEngine  on
RewriteRule    "^my-notes/(.*)$"  "https://publish.obsidian.md/serve?url=mysite.com/my-notes/$1"  [L,P]
```

> [!note] หมายเหตุ
> ต้องเปิดใช้งาน `mod_rewrite` และคุณอาจต้องตั้งค่า [SSLProxyEngine](https://stackoverflow.com/questions/40938148/reverse-proxy-for-external-url-apache) ด้วย

### Netlify

ใน `netlify.toml` [ตั้งค่า redirects](https://docs.netlify.com/routing/redirects/#syntax-for-the-netlify-configuration-file):

```plain
[[redirects]]
  from = "https://mysite.com/my-notes/*"
  to = "https://publish.obsidian.md/serve?url=mysite.com/my-notes/:splat"
  status = 200
  force = true
```

### Vercel

ใน `vercel.json` [ตั้งค่า rewrites](https://vercel.com/docs/configuration#project/rewrites):

```json
{
  ...

  "rewrites": [
    {
      "source": "/my-notes/",
      "destination": "https://publish.obsidian.md/serve?url=mysite.com/my-notes"
    },
    {
      "source": "/my-notes/:path*",
      "destination": "https://publish.obsidian.md/serve?url=mysite.com/my-notes/:path*"
    }
  ]
}
```

### Caddy

```plain
mysite.com {
  encode zstd gzip
  handle /my-notes* {
    reverse_proxy https://publish.obsidian.md {
      header_up Host {upstream_hostport}
    }
    rewrite * /serve?url=mysite.com{path}
  }
}
```

### Traefik

ตัวอย่างการตั้งค่าขั้นต่ำนี้เปลี่ยนเส้นทาง `mysite.com` ไปยัง Obsidian Publish
ดู [เอกสาร Traefik](https://doc.traefik.io/traefik/routing/overview/) สำหรับตัวอย่างที่สมบูรณ์

```yaml
http:
  routers:
    mysite:
      rule: Host(`mysite.com`)
      service: obsidian-publish
      middlewares:
        - "publish-headers"
  services:
    obsidian-publish:
      loadBalancer:
        servers:
          - url: https://publish.obsidian.md
  middlewares:
    publish-headers:
      headers:
        customRequestHeaders:
          Host: "publish.obsidian.md"
          x-obsidian-custom-domain: "mysite.com"
```

### HTTP X-Headers ที่รองรับ

หากบริการ proxy ของคุณไม่อนุญาต query paths คุณสามารถใช้ `https://publish.obsidian.md/` พร้อมกับ header กำหนดเอง `x-obsidian-custom-domain` ที่ตั้งค่าเป็น URL เว็บไซต์ของคุณ `mysite.com/my-subpath`

## เปลี่ยนเส้นทางเว็บไซต์เดิมไปยังโดเมนกำหนดเอง

หากคุณต้องการเปลี่ยนเส้นทางผู้เยี่ยมชมจากเว็บไซต์ `publish.obsidian.md` เดิมไปยังโดเมนกำหนดเองใหม่ของคุณ ให้เปิดใช้งานตัวเลือก **ชี้ทางไปยัง custom domain ของคุณ** เมื่อตั้งค่าโดเมนกำหนดเองของคุณ

## แก้ไขปัญหา

เมื่อคุณตั้งค่าโดเมนกำหนดเองแล้ว หากคุณเคยเยี่ยมชมเว็บไซต์ของคุณจากลิงก์ `https://publish.obsidian.md/slug` ก่อนหน้า คุณอาจต้องล้างแคชของเบราว์เซอร์เพื่อให้บางอย่าง (เช่น ฟอนต์ กราฟ หรือการเข้าถึงด้วยรหัสผ่าน) ทำงานได้อย่างถูกต้อง เนื่องจากข้อจำกัดด้านความปลอดภัยข้ามโดเมนที่เบราว์เซอร์สมัยใหม่บังคับใช้ ข่าวดีคือผู้อ่านเว็บไซต์ของคุณจะไม่ประสบปัญหานี้หากคุณให้ผู้เยี่ยมชมใช้โดเมนกำหนดเองของคุณเท่านั้น
