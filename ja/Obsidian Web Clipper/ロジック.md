[[Obsidian Web Clipperの紹介|Web Clipper]]はテンプレートロジックとして条件分岐、ループ、変数代入をサポートしています。この構文は[Twig](https://twig.symfony.com/)や[Liquid](https://shopify.github.io/liquid/)テンプレート言語にインスパイアされています。


> [!warning] 最新バージョンが必要です
> ロジック機能にはObsidian Web Clipper 1.0.0が必要ですが、まだすべての拡張機能ストアで承認されていません。

## 条件分岐

`{% if %}`を使用して、変数や式に基づいてコンテンツを条件付きで含めます。

```twig
{% if author %}
Author: {{author}}
{% endif %}
```

`{% else %}`を使用してフォールバックコンテンツを提供し、`{% elseif %}`を使用して複数の条件を連結します：

```twig
{% if status == "published" %}
Live article
{% elseif status == "draft" %}
Draft article
{% else %}
Unknown status
{% endif %}
```

### 比較演算子

以下の比較演算子がサポートされています：

| 演算子 | 説明 |
|----------|-------------|
| `==` | 等しい |
| `!=` | 等しくない |
| `>` | より大きい |
| `<` | より小さい |
| `>=` | 以上 |
| `<=` | 以下 |
| `contains` | 文字列が部分文字列を含むか、配列が値を含むかをチェック |

例：
- `{% if title == "Home" %}` — 文字列の等価比較
- `{% if price >= 100 %}` — 数値の比較
- `{% if title contains "Review" %}` — 部分文字列のチェック
- `{% if tags contains "important" %}` — 配列のメンバーシップ

### 論理演算子

論理演算子を使って条件を組み合わせます：

| 演算子 | 代替表記 | 説明                         |
| -------- | ----------- | ----------------------------------- |
| `and`    | `&&`        | 両方の条件が真である必要がある        |
| `or`     | \|\|        | 少なくとも一方の条件が真である必要がある |
| `not`    | `!`         | 条件を否定する                 |

例：
- `{% if author and published %}` — 両方が存在する必要がある
- `{% if draft or archived %}` — いずれかの条件
- `{% if not hidden %}` — 否定
- `{% if (premium or featured) and published %}` — グループ化された条件

### 真偽性

変数が比較演算子なしで使用された場合、「真偽性」が評価されます：

- `false`、`null`、`undefined`、空文字列`""`、`0`は**偽**と見なされます。
- 空の配列`[]`は**偽**と見なされます。
- それ以外はすべて**真**と見なされます。

```twig
{% if content %}
Has content
{% endif %}
```

## 変数の代入

`{% set %}`を使用して、テンプレート内で変数を作成または変更します：

```twig
{% set slug = title|lower|replace:" ":"-" %}
File: {{slug}}.md
```

変数には以下を設定できます：
- 他の変数：`{% set name = author %}`
- リテラル：`{% set count = 5 %}`または`{% set label = "Draft" %}`
- フィルター付きの式：`{% set excerpt = content|truncate:100 %}`
- セレクターの結果：`{% set comments = selector:.comment %}`

`{% set %}`で設定した変数は、後続のテンプレートロジックや`{{variable}}`の出力で使用できます。

## フォールバック

`??`演算子を使用して、変数が空またはundefinedの場合のフォールバック値を指定します：

```twig
{{title ?? "Untitled"}}
```

`title`が空、undefined、または偽の場合、フォールバック値`"Untitled"`が代わりに使用されます。

これは次の`if`文と同等の省略形です：

```twig
{% if title %}{{title}}{% else %}Untitled{% endif %}
```

### フォールバックの連結

複数のフォールバックを連結できます：

```twig
{{title ?? headline ?? "No title"}}
```

これは`title`が利用可能な場合は`title`を、そうでなければ`headline`を、それもなければ文字列`"No title"`を使用します。

### フィルターとの組み合わせ

フィルターは`??`よりも結合が強いため、フォールバックチェックの前にフィルターが適用されます：

```twig
{{title|upper ?? "UNTITLED"}}
```

これはまず`title`に`upper`を適用し、結果が空の場合は`"UNTITLED"`にフォールバックします。フォールバック値にフィルターを適用するには、括弧を使うか式を分けてください：

```twig
{{title ?? "Untitled"|lower}}
```

これは`title`が利用可能な場合は`title`を使用し、そうでなければフォールバックに`lower`を適用して`"untitled"`となります。

## ループ

`{% for %}`を使用して配列を反復処理します：

```twig
{% for item in schema:author %}
- {{item.name}}
{% endfor %}
```

### ループのソース

以下に対してループできます：
- スキーマ配列：`{% for item in schema:author %}`
- セレクターの結果：`{% for comment in selector:.comment %}`
- 事前に設定した変数：`{% set items = selector:.item %}{% for item in items %}`

### ループ変数

ループ内では、以下のプロパティを持つ`loop`オブジェクトにアクセスできます：

| 変数 | 説明 |
|----------|-------------|
| `loop.index` | 現在の反復回数（1始まり） |
| `loop.index0` | 現在の反復回数（0始まり） |
| `loop.first` | 最初の反復の場合`true` |
| `loop.last` | 最後の反復の場合`true` |
| `loop.length` | アイテムの総数 |

```twig
{% for tag in tags %}
{{loop.index}}. {{tag}}
{% if loop.last %} (end of list){% endif %}
{% endfor %}
```

後方互換性のために、`item_index`（`item`はイテレータ変数名）を使用して0始まりの位置を取得することもできます：

```twig
{% for tag in tags %}
{{tag_index}}. {{tag}}
{% endfor %}
```

### インデックスによる配列アイテムへのアクセス

ブラケット記法を使用して、インデックスで配列要素にアクセスします：

```twig
{{items[0]}}
{{items[loop.index0]}}
```

これは複数の配列から並列でアイテムにアクセスする必要がある場合に便利です：

```twig
{% set transcripts = selector:.transcript-text %}
{% set timestamps = selector:.timestamp %}

{% for line in transcripts %}
{{timestamps[loop.index0]}} - {{line}}
{% endfor %}
```

ブラケット記法はオブジェクトのプロパティでも動作します：

```twig
{{user["name"]}}
{{data["my-key"]}}
```

### ネストされたループ

複雑なデータ構造に対してループをネストできます：

```twig
{% for section in sections %}
## {{section.title}}
{% for item in section.items %}
- {{item}}
{% endfor %}
{% endfor %}
```

## ロジックの組み合わせ

条件分岐とループを組み合わせることができます：

```twig
{% for item in items %}
{% if item.active %}
- {{item.name}}
{% endif %}
{% endfor %}
```

## 評価順序

テンプレートロジックは以下の順序で処理されます：

1. **テンプレートロジック** — `{% if %}`、`{% for %}`、`{% set %}`、`{{variables}}`が最初に評価されます
2. **プロンプト変数** — `{{"summarize this"|prompt}}`のような[[変数#プロンプト変数|プロンプト変数]]は、テンプレートロジックの完了後にインタープリターに送信されます

つまり、テンプレートロジックを使用してプロンプトを動的に構築できますが、プロンプトの結果を条件分岐やループで使用することはできません。
