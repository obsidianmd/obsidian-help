[[Obsidian Publishの概要|Obsidian Publish]]サイトにカスタムドメインまたはサブドメインを設定できます。

> [!warning] 警告
> 現時点では、以下の方法でのみカスタムドメインの設定をサポートしています：
>
> - [[#CloudFlareを使用した設定]] - [Fullモード](https://developers.cloudflare.com/ssl/origin-configuration/ssl-modes/full/)を使用。
> - [[#プロキシを使用した設定]]
> 
> 現在、SSL証明書を代理でプロビジョニングする方法はまだありません。

## CloudFlareを使用した設定

カスタムドメインまたはサブドメインを設定する最も簡単な方法は、[CloudFlare](https://cloudflare.com)アカウントを作成し、CloudFlareにドメインのDNSを管理させることです。

以下の手順では、CloudFlareを使用してObsidian Publishサイトにカスタムドメインを設定します。ルートドメイン（`mysite.com`）またはサブドメイン（`notes.mysite.com`）のいずれにも対応しています。

> [!important] 重要
> CloudFlareは、カスタムドメイン設定における**唯一の公式サポートプロバイダー**です。以下の手順を他のプロバイダーで使用しても、おそらく動作しません。

**CloudFlare：**

1. Cloudflareを開き、Publishサイトをホストしたいドメイン（例：`mysite.com`）に移動します。`notes.mysite.com`のようなサブドメインを使いたい場合も同様です。
2. **DNS**に移動し、**Add Record**をクリックします。
3. **CNAME**を選択します。
4. **name**に、ドメインまたはサブドメインを入力します（例：`notes.mysite.com`）。
5. **target**に`publish-main.obsidian.md`と入力します。この値に個人のサブURLを含めないでください。Obsidian Publishが設定から自動的に処理します。
6. **proxy status**が有効になっていることを確認します。デフォルトで有効になっているはずです。
7. **SSL/TLS**に移動し、SSL/TLS暗号化モードを「Full」に設定して、SSL/TLS証明書を自動的に設定します。

> [!note] ノート
> `mysite.com`と`www.mysite.com`の両方をObsidian Publishにリダイレクトするには、以下の設定で[Page Rule](https://support.cloudflare.com/hc/en-us/articles/200172336-Creating-Page-Rules)を作成する必要があります：
>
> - URL match: `www.mysite.com/*`
> - Forward URL - 301 Permanent Redirect
> - Redirect URL: `https://mysite.com/$1`
>
> ページルールを作成した後、`mysite.com`と同様に`www.mysite.com`のCNAMEレコードも作成してください。

**Obsidian：**

1. コンピューターでObsidianを開きます。
2. 左側の[[リボン|リボンメニュー]]で、**変更をパブリッシュ**（ ![[lucide-send.svg#icon]] ）をクリックします。
3. **変更をパブリッシュ**の下にある**サイトオプションの変更**（ ![[lucide-cog.svg#icon]] ）を選択します。
4. **カスタムドメイン**の横にある**設定**を選択します。
5. **カスタムURL**に、ドメインまたはサブドメインのURLを入力します。カスタムURLボックスに`www.`を入れないようにしてください。

> [!note] ノート
> カスタムドメインの設定でリダイレクトループが発生する場合、CloudFlareの暗号化モードが「Full」ではなく「Flexible」に設定されている可能性があります。

## プロキシを使用した設定

独自のウェブサーバーを使用して、カスタムドメインのSSL/TLSを設定することもできます。

ドメインまたはサブドメインで既にウェブサイトをホストしている場合、このオプションを使用して、サイト全体をホストする代わりに、特定のURLパスでObsidian Publishサイトを読み込むようにウェブサイトを設定できます。

そのURLパス下のすべてのリクエストを`https://publish.obsidian.md/serve?url=mysite.com/my-notes/...`にプロキシし、Obsidianのサイトオプションで**カスタムURL**を`mysite.com/my-notes`に設定して同じURLパスを設定します。

Obsidian Publishを、自分が所有するサイトのサブURLとして設定することもできます。例えば、`https://mysite.com/my-notes/`のようにです。これを実現するには、独自のサーバーをホストし、すべてのリクエストを`https://publish.obsidian.md/`のサーバーにプロキシする必要があります。

以下のプロキシ設定例は網羅的ではありませんが、この実装の一般的な方法を示しています。

### NGINX

NGINXの設定に以下を追加します：

```nginx
location /my-notes {
  proxy_pass https://publish.obsidian.md/serve?url=mysite.com/my-notes/;
  proxy_ssl_server_name on;
  proxy_set_header Host publish.obsidian.md;
}
```


一部のユーザーから、プロキシパスに`$request_uri`を追加する必要がある場合があると報告されています：

```nginx
location /my-notes {
  proxy_pass https://publish.obsidian.md/serve?url=mysite.com/my-notes$request_uri;
  proxy_ssl_server_name on;
  proxy_set_header Host publish.obsidian.md;
}
```

### Apache

`.htaccess`に以下を追加します：

```htaccess
RewriteEngine  on
RewriteRule    "^my-notes/(.*)$"  "https://publish.obsidian.md/serve?url=mysite.com/my-notes/$1"  [L,P]
```

> [!note] ノート
> `mod_rewrite`を有効にする必要があり、[SSLProxyEngine](https://stackoverflow.com/questions/40938148/reverse-proxy-for-external-url-apache)の設定も必要になる場合があります。

### Netlify

`netlify.toml`で[リダイレクトを設定](https://docs.netlify.com/routing/redirects/#syntax-for-the-netlify-configuration-file)します：

```plain
[[redirects]]
  from = "https://mysite.com/my-notes/*"
  to = "https://publish.obsidian.md/serve?url=mysite.com/my-notes/:splat"
  status = 200
  force = true
```

### Vercel

`vercel.json`で[リライトを設定](https://vercel.com/docs/configuration#project/rewrites)します：

```json
{
  ...

  "rewrites": [
    {
      "source": "/my-notes/",
      "destination": "https://publish.obsidian.md/serve?url=mysite.com/my-notes"
    },
    {
      "source": "/my-notes/:path*",
      "destination": "https://publish.obsidian.md/serve?url=mysite.com/my-notes/:path*"
    }
  ]
}
```

### Caddy

```plain
mysite.com {
  encode zstd gzip
  handle /my-notes* {
    reverse_proxy https://publish.obsidian.md {
      header_up Host {upstream_hostport}
    }
    rewrite * /serve?url=mysite.com{path}
  }
}
```

### Traefik

この最小限の設定例は、`mysite.com`をObsidian Publishにリダイレクトします。
完全な例については[Traefikのドキュメント](https://doc.traefik.io/traefik/routing/overview/)を参照してください。

```yaml
http:
  routers:
    mysite:
      rule: Host(`mysite.com`)
      service: obsidian-publish
      middlewares:
        - "publish-headers"
  services:
    obsidian-publish:
      loadBalancer:
        servers:
          - url: https://publish.obsidian.md
  middlewares:
    publish-headers:
      headers:
        customRequestHeaders:
          Host: "publish.obsidian.md"
          x-obsidian-custom-domain: "mysite.com"
```

### サポートされるHTTP X-Headers

プロキシサービスがクエリパスを許可しない場合、`https://publish.obsidian.md/`にカスタムヘッダー`x-obsidian-custom-domain`をサイトURL `mysite.com/my-subpath`に設定して使用できます。

## 旧サイトからカスタムドメインへのリダイレクト

訪問者を以前の`publish.obsidian.md`サイトから新しいカスタムドメインにリダイレクトしたい場合は、カスタムドメインの設定時に**カスタムドメインへのリダイレクト**オプションを有効にしてください。

## トラブルシューティング

カスタムドメインを設定した後、以前の`https://publish.obsidian.md/slug`リンクからサイトを訪問したことがある場合、フォント、グラフ、パスワードアクセスなどが正しく動作するために、ブラウザのキャッシュをクリアする必要がある場合があります。これは、最新のブラウザが課すクロスドメインセキュリティ制限によるものです。良いニュースは、訪問者にカスタムドメインのみを使用させている場合、サイトの読者がこの問題に遭遇することはないということです。
