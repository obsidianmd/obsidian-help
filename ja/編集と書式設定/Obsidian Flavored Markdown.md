Obsidianは、既存のフォーマットを壊さずに最大限の機能を提供することを目指しています。そのため、複数のフレーバーの[[基本的な書式構文|Markdown]]を組み合わせて使用しています。

Obsidianは[CommonMark](https://commonmark.org/)、[GitHub Flavored Markdown](https://github.github.com/gfm/)、[LaTeX](https://www.latex-project.org/)をサポートしています。

> [!tip]- HTML内のMarkdown
> Obsidianは、HTML要素内のMarkdown構文をレンダリングしません。これは、パフォーマンスの最適化と、大規模なドキュメントを管理する際のパーサーの複雑さを低く保つための意図的な設計上の選択です。
>
> 例えば、`**bold**`や`` `code` ``のようなMarkdown書式は、`<div>`、`<span>`、`<table>`、またはその他のHTMLタグ内では処理されません。
>
> ```md
> <div>
> This **will not** be bold.
> </div>
> ```

### サポートされているMarkdown拡張機能

| 構文            | 説明                                                                  |
| --------------- | --------------------------------------------------------------------- |
| `[[Link]]`      | [[内部リンク]]                                                        |
| `![[Link]]`     | [[ファイルの埋め込み]]                                                |
| `![[Link#^id]]` | [[内部リンク#ノート内のブロックへのリンク\|ブロック参照]]              |
| `^id`           | [[内部リンク#ノート内のブロックへのリンク\|ブロックの定義]]            |
| `[^id]`         | [[基本的な書式構文#脚注\|脚注]]                                       |
| `%%Text%%`      | [[基本的な書式構文#コメント\|コメント]]                                |
| `~~Text~~`      | [[基本的な書式構文#ボールド、イタリック、ハイライト\|取り消し線]]      |
| `==Text==`      | [[基本的な書式構文#ボールド、イタリック、ハイライト\|ハイライト]]      |
| `` ``` ``       | [[基本的な書式構文#コードブロック\|コードブロック]]                    |
| `- [ ]`         | [[基本的な書式構文#タスクリスト\|未完了タスク]]                        |
| `- [x]`         | [[基本的な書式構文#タスクリスト\|完了タスク]]                          |
| `> [!note]`     | [[コールアウト]]                                                      |
| (リンク参照)    | [[高度な書式構文#表\|表]]                                             |
