يمكنك إعداد نطاق مخصص أو نطاق فرعي لموقع [[مقدمة عن Obsidian Publish|Obsidian Publish]] الخاص بك.

> [!warning] تحذير
> في الوقت الحالي، لا ندعم سوى تهيئة النطاقات المخصصة باستخدام الطرق التالية:
>
> - [[#الإعداد باستخدام CloudFlare]] باستخدام [الوضع الكامل (Full)](https://developers.cloudflare.com/ssl/origin-configuration/ssl-modes/full/).
> - [[#الإعداد باستخدام وكيل (Proxy)]]
> 
> لا تتوفر لدينا حتى الآن طريقة لتوفير شهادة SSL نيابةً عنك.

## الإعداد باستخدام CloudFlare

أسهل طريقة لإعداد نطاق مخصص أو نطاق فرعي هي إنشاء حساب على [CloudFlare](https://cloudflare.com) والسماح لـ CloudFlare بإدارة DNS الخاص بنطاقك.

توضح الخطوات التالية كيفية استخدام CloudFlare لتهيئة نطاق مخصص لموقع Obsidian Publish الخاص بك، سواء باستخدام نطاق جذر (`mysite.com`) أو نطاق فرعي (`notes.mysite.com`).

> [!important] هام
> CloudFlare هو **المزود الوحيد المدعوم رسميًا** لإعداد النطاقات المخصصة. من المرجح أن استخدام التعليمات التالية مع أي مزودين آخرين لن يعمل.

**CloudFlare:**

1. افتح Cloudflare على النطاق الذي تريد استضافة موقع Publish عليه، مثل `mysite.com`، حتى إذا كنت تريد نطاقًا فرعيًا مثل `notes.mysite.com`.
2. انتقل إلى **DNS** وانقر على **Add Record**.
3. اختر **CNAME**.
4. في حقل **name**، أدخل نطاقك أو نطاقك الفرعي، على سبيل المثال `notes.mysite.com`.
5. في حقل **target**، أدخل `publish-main.obsidian.md`. لا تقم بتضمين عنوان URL الفرعي الشخصي في هذه القيمة. يتعامل Obsidian Publish مع هذا من خلال التهيئة الخاصة بك.
6. تأكد من تمكين **proxy status**. يجب أن يكون مُمكّنًا بشكل افتراضي.
7. انتقل إلى **SSL/TLS** واضبط وضع تشفير SSL/TLS على "Full" لتهيئة شهادة SSL/TLS تلقائيًا.

> [!note] ملاحظة
> لإعادة توجيه كل من `mysite.com` و `www.mysite.com` إلى Obsidian Publish، تحتاج إلى إنشاء [قاعدة صفحة (Page Rule)](https://support.cloudflare.com/hc/en-us/articles/200172336-Creating-Page-Rules) بالإعدادات التالية:
>
> - URL match: `www.mysite.com/*`
> - Forward URL - 301 Permanent Redirect
> - Redirect URL: `https://mysite.com/$1`
>
> بعد إنشاء قاعدة الصفحة، أنشئ سجل CNAME لـ `www.mysite.com` تمامًا كما فعلت مع `mysite.com`.

**Obsidian:**

1. افتح Obsidian على حاسوبك.
2. في [[شريط|الشريط الجانبي]] على اليسار، انقر على **نشر التغييرات** ( ![[lucide-send.svg#icon]] ).
3. تحت **نشر التغييرات**، اختر **تغيير خيارات الموقع** ( ![[lucide-cog.svg#icon]] ).
4. بجوار **نطاق مخصص**، اختر **تهيئة**.
5. في حقل **رابط مخصص**، أدخل عنوان URL لنطاقك أو نطاقك الفرعي. تأكد من عدم وضع `www.` في مربع الرابط المخصص.

> [!note] ملاحظة
> إذا انتهى إعداد النطاق المخصص الخاص بك بحلقة إعادة توجيه، فمن المحتمل أن وضع التشفير في CloudFlare قد تم ضبطه على "Flexible" بدلاً من "Full".

## الإعداد باستخدام وكيل (Proxy)

يمكنك أيضًا إعداد SSL/TLS لنطاقك المخصص باستخدام خادم الويب الخاص بك.

إذا كنت تستضيف بالفعل موقعًا إلكترونيًا تحت نطاقك أو نطاقك الفرعي، يمكنك أيضًا استخدام هذا الخيار وإعداد موقعك لتحميل موقع Obsidian Publish الخاص بك تحت مسار URL محدد، بدلاً من استضافة الموقع بالكامل.

قم بتوجيه جميع الطلبات تحت مسار URL هذا إلى `https://publish.obsidian.md/serve?url=mysite.com/my-notes/...` وقم بتهيئة خيارات الموقع في Obsidian على نفس مسار URL، من خلال ضبط **رابط مخصص** على `mysite.com/my-notes`.

يمكنك أيضًا إعداد Obsidian Publish كعنوان URL فرعي لموقع تملكه. على سبيل المثال، `https://mysite.com/my-notes/`. لتحقيق ذلك، يجب عليك استضافة خادمك الخاص وتوجيه جميع الطلبات إلى خادمنا على `https://publish.obsidian.md/`.

أمثلة إعداد الوكيل التالية ليست شاملة، ولكنها توفر طرقًا شائعة لهذا التنفيذ.

### NGINX

في تهيئة NGINX الخاصة بك، أضف ما يلي:

```nginx
location /my-notes {
  proxy_pass https://publish.obsidian.md/serve?url=mysite.com/my-notes/;
  proxy_ssl_server_name on;
  proxy_set_header Host publish.obsidian.md;
}
```


أفاد بعض المستخدمين أن إضافة `$request_uri` إلى مسار الوكيل قد يكون مطلوبًا:

```nginx
location /my-notes {
  proxy_pass https://publish.obsidian.md/serve?url=mysite.com/my-notes$request_uri;
  proxy_ssl_server_name on;
  proxy_set_header Host publish.obsidian.md;
}
```

### Apache

في ملف `.htaccess`، أضف ما يلي:

```htaccess
RewriteEngine  on
RewriteRule    "^my-notes/(.*)$"  "https://publish.obsidian.md/serve?url=mysite.com/my-notes/$1"  [L,P]
```

> [!note] ملاحظة
> يجب تمكين `mod_rewrite`، وقد تحتاج أيضًا إلى تهيئة [SSLProxyEngine](https://stackoverflow.com/questions/40938148/reverse-proxy-for-external-url-apache)

### Netlify

في ملف `netlify.toml`، [قم بتهيئة إعادات التوجيه](https://docs.netlify.com/routing/redirects/#syntax-for-the-netlify-configuration-file):

```plain
[[redirects]]
  from = "https://mysite.com/my-notes/*"
  to = "https://publish.obsidian.md/serve?url=mysite.com/my-notes/:splat"
  status = 200
  force = true
```

### Vercel

في ملف `vercel.json`، [قم بتهيئة إعادات الكتابة](https://vercel.com/docs/configuration#project/rewrites):

```json
{
  ...

  "rewrites": [
    {
      "source": "/my-notes/",
      "destination": "https://publish.obsidian.md/serve?url=mysite.com/my-notes"
    },
    {
      "source": "/my-notes/:path*",
      "destination": "https://publish.obsidian.md/serve?url=mysite.com/my-notes/:path*"
    }
  ]
}
```

### Caddy

```plain
mysite.com {
  encode zstd gzip
  handle /my-notes* {
    reverse_proxy https://publish.obsidian.md {
      header_up Host {upstream_hostport}
    }
    rewrite * /serve?url=mysite.com{path}
  }
}
```

### Traefik

يقوم مقتطف التهيئة المصغر هذا بإعادة توجيه `mysite.com` إلى Obsidian Publish.
راجع [توثيق Traefik](https://doc.traefik.io/traefik/routing/overview/) للحصول على مثال كامل.

```yaml
http:
  routers:
    mysite:
      rule: Host(`mysite.com`)
      service: obsidian-publish
      middlewares:
        - "publish-headers"
  services:
    obsidian-publish:
      loadBalancer:
        servers:
          - url: https://publish.obsidian.md
  middlewares:
    publish-headers:
      headers:
        customRequestHeaders:
          Host: "publish.obsidian.md"
          x-obsidian-custom-domain: "mysite.com"
```

### ترويسات HTTP X المدعومة

إذا كانت خدمة الوكيل الخاصة بك لا تسمح بمسارات الاستعلام، يمكنك استخدام `https://publish.obsidian.md/` مع ترويسة مخصصة `x-obsidian-custom-domain` مضبوطة على عنوان URL لموقعك `mysite.com/my-subpath`.

## إعادة توجيه الموقع القديم إلى النطاق المخصص

إذا كنت تريد إعادة توجيه زوارك من موقع `publish.obsidian.md` القديم إلى نطاقك المخصص الجديد، قم بتمكين خيار **إعادة توجيه إلى النطاق المخصص الخاص بك** عند تهيئة نطاقك المخصص.

## استكشاف الأخطاء وإصلاحها

بمجرد إعداد نطاقك المخصص، إذا كنت قد زرت موقعك من رابط `https://publish.obsidian.md/slug` السابق، فقد تحتاج إلى مسح ذاكرة التخزين المؤقت للمتصفح حتى تعمل بعض الأشياء (مثل الخطوط والرسوم البيانية والوصول بكلمة المرور) بشكل صحيح. يرجع ذلك إلى قيود الأمان عبر النطاقات التي تفرضها المتصفحات الحديثة. الخبر الجيد هو أن قراء موقعك لن يواجهوا هذه المشكلة أبدًا إذا سمحت للزوار فقط باستخدام نطاقك المخصص.
