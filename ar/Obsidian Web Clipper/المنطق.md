يدعم [[مقدمة عن قاصّ أوبسيديان للويب|قاصّ أوبسيديان للويب]] منطق القوالب للشروط والحلقات وتعيين المتغيرات. هذه الصيغة مستوحاة من لغتي القوالب [Twig](https://twig.symfony.com/) و[Liquid](https://shopify.github.io/liquid/).

> [!warning] يتطلب أحدث إصدار
> تتطلب ميزات المنطق Obsidian Web Clipper 1.0.0 الذي لم تتم الموافقة عليه بعد في جميع متاجر الإضافات.

## الشروط

استخدم `{% if %}` لتضمين المحتوى بشكل مشروط بناءً على المتغيرات أو التعبيرات.

```twig
{% if author %}
Author: {{author}}
{% endif %}
```

استخدم `{% else %}` لتوفير محتوى بديل، و`{% elseif %}` لربط عدة شروط:

```twig
{% if status == "published" %}
Live article
{% elseif status == "draft" %}
Draft article
{% else %}
Unknown status
{% endif %}
```

### عوامل المقارنة

عوامل المقارنة التالية مدعومة:

| العامل | الوصف |
|----------|-------------|
| `==` | يساوي |
| `!=` | لا يساوي |
| `>` | أكبر من |
| `<` | أصغر من |
| `>=` | أكبر من أو يساوي |
| `<=` | أصغر من أو يساوي |
| `contains` | التحقق مما إذا كانت السلسلة النصية تحتوي على سلسلة فرعية، أو المصفوفة تحتوي على قيمة |

أمثلة:
- `{% if title == "Home" %}` — مساواة نصية
- `{% if price >= 100 %}` — مقارنة رقمية
- `{% if title contains "Review" %}` — التحقق من سلسلة فرعية
- `{% if tags contains "important" %}` — عضوية في مصفوفة

### العوامل المنطقية

اجمع بين الشروط باستخدام العوامل المنطقية:

| العامل | البديل | الوصف |
| -------- | ----------- | ----------------------------------- |
| `and`    | `&&`        | يجب أن يكون كلا الشرطين صحيحين |
| `or`     | \|\|        | يجب أن يكون شرط واحد على الأقل صحيحاً |
| `not`    | `!`         | ينفي الشرط |

أمثلة:
- `{% if author and published %}` — يجب وجود كليهما
- `{% if draft or archived %}` — أي من الشرطين
- `{% if not hidden %}` — نفي
- `{% if (premium or featured) and published %}` — شروط مُجمّعة

### القيم الصدقية

عند استخدام متغير بدون عامل مقارنة، يتم تقييمه من حيث "الصدقية":

- `false` و`null` و`undefined` والسلسلة النصية الفارغة `""` و`0` تُعتبر **خاطئة (falsy)**.
- المصفوفات الفارغة `[]` تُعتبر **خاطئة (falsy)**.
- كل شيء آخر يُعتبر **صادقاً (truthy)**.

```twig
{% if content %}
Has content
{% endif %}
```

## تعيين متغير

استخدم `{% set %}` لإنشاء أو تعديل المتغيرات داخل القالب الخاص بك:

```twig
{% set slug = title|lower|replace:" ":"-" %}
File: {{slug}}.md
```

يمكن تعيين المتغيرات إلى:
- متغيرات أخرى: `{% set name = author %}`
- قيم حرفية: `{% set count = 5 %}` أو `{% set label = "Draft" %}`
- تعبيرات مع مرشحات: `{% set excerpt = content|truncate:100 %}`
- نتائج المُحددات: `{% set comments = selector:.comment %}`

يمكن استخدام المتغيرات المُعيّنة بـ `{% set %}` في منطق القالب اللاحق وفي إخراج `{{variable}}`.

## القيم البديلة

استخدم العامل `??` لتوفير قيم بديلة عندما يكون المتغير فارغاً أو غير مُعرّف:

```twig
{{title ?? "Untitled"}}
```

إذا كان `title` فارغاً أو غير مُعرّف أو خاطئاً، سيتم استخدام القيمة البديلة `"Untitled"` بدلاً منه.

هذا اختصار لعبارة `if` المكافئة:

```twig
{% if title %}{{title}}{% else %}Untitled{% endif %}
```

### ربط القيم البديلة

يمكنك ربط عدة قيم بديلة:

```twig
{{title ?? headline ?? "No title"}}
```

سيستخدم هذا `title` إذا كان متاحاً، وإلا `headline`، وإلا السلسلة النصية `"No title"`.

### مع المرشحات

المرشحات لها أولوية أعلى من `??`، لذا تُطبّق المرشحات قبل التحقق من القيمة البديلة:

```twig
{{title|upper ?? "UNTITLED"}}
```

يطبّق هذا `upper` على `title` أولاً، ثم يعود إلى `"UNTITLED"` إذا كانت النتيجة فارغة. لتطبيق المرشحات على القيمة البديلة، استخدم الأقواس أو تعبيرات منفصلة:

```twig
{{title ?? "Untitled"|lower}}
```

سيستخدم هذا `title` إذا كان متاحاً، وإلا يطبّق `lower` على القيمة البديلة، مما ينتج عنه `"untitled"`.

## الحلقات

استخدم `{% for %}` للتكرار عبر المصفوفات:

```twig
{% for item in schema:author %}
- {{item.name}}
{% endfor %}
```

### مصادر الحلقات

يمكنك التكرار عبر:
- مصفوفات المخطط: `{% for item in schema:author %}`
- نتائج المُحددات: `{% for comment in selector:.comment %}`
- متغيرات مُعيّنة مسبقاً: `{% set items = selector:.item %}{% for item in items %}`

### متغيرات الحلقة

داخل الحلقة، يمكنك الوصول إلى كائن `loop` بالخصائص التالية:

| المتغير | الوصف |
|----------|-------------|
| `loop.index` | التكرار الحالي (يبدأ من 1) |
| `loop.index0` | التكرار الحالي (يبدأ من 0) |
| `loop.first` | `true` إذا كان التكرار الأول |
| `loop.last` | `true` إذا كان التكرار الأخير |
| `loop.length` | العدد الإجمالي للعناصر |

```twig
{% for tag in tags %}
{{loop.index}}. {{tag}}
{% if loop.last %} (end of list){% endif %}
{% endfor %}
```

للتوافق مع الإصدارات السابقة، يمكنك أيضاً استخدام `item_index` (حيث `item` هو اسم متغير المُكرّر الخاص بك) للحصول على الموضع المبدوء من الصفر:

```twig
{% for tag in tags %}
{{tag_index}}. {{tag}}
{% endfor %}
```

### الوصول إلى عناصر المصفوفة بالفهرس

استخدم تدوين الأقواس للوصول إلى عناصر المصفوفة بالفهرس:

```twig
{{items[0]}}
{{items[loop.index0]}}
```

هذا مفيد عندما تحتاج إلى الوصول إلى عناصر من عدة مصفوفات بشكل متوازٍ:

```twig
{% set transcripts = selector:.transcript-text %}
{% set timestamps = selector:.timestamp %}

{% for line in transcripts %}
{{timestamps[loop.index0]}} - {{line}}
{% endfor %}
```

يعمل تدوين الأقواس أيضاً مع خصائص الكائنات:

```twig
{{user["name"]}}
{{data["my-key"]}}
```

### الحلقات المتداخلة

يمكن تداخل الحلقات لهياكل البيانات المعقدة:

```twig
{% for section in sections %}
## {{section.title}}
{% for item in section.items %}
- {{item}}
{% endfor %}
{% endfor %}
```

## دمج المنطق

يمكن دمج الشروط والحلقات معاً:

```twig
{% for item in items %}
{% if item.active %}
- {{item.name}}
{% endif %}
{% endfor %}
```

## ترتيب التقييم

يُعالج منطق القالب بالترتيب التالي:

1. **منطق القالب** — يتم تقييم `{% if %}` و`{% for %}` و`{% set %}` و`{{variables}}` أولاً
2. **متغيرات الطلب** — [[المتغيرات#متغيرات الطلب|متغيرات الطلب]] مثل `{{"summarize this"|prompt}}` تُرسل إلى المفسر بعد اكتمال منطق القالب

هذا يعني أنه يمكنك استخدام منطق القالب لبناء الطلبات ديناميكياً، لكن نتائج الطلبات غير متاحة للاستخدام في الشروط أو الحلقات.
