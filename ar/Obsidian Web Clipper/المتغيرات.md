يمكن لـ [[Obsidian Web Clipper/القوالب|قوالب قاصّ الويب]] استخدام المتغيرات لملء البيانات تلقائيًا من الصفحة في القالب. يمكن استخدام المتغيرات في **اسم الملاحظة**، و**موقع الملاحظة**، و**الخصائص**، و**محتوى الملاحظة**. يمكن أيضًا تعديل المتغيرات باستخدام [[تصفية|التصفية]].

استخدم أيقونة `...` في إضافة [[مقدمة عن قاصّ أوبسيديان للويب|قاصّ الويب]] للوصول إلى متغيرات الصفحة الحالية لاستخدامها في القوالب. هناك خمسة أنواع من المتغيرات يمكنك استخدامها:

## متغيرات الإعداد المسبق

تُنشأ متغيرات الإعداد المسبق تلقائيًا بناءً على محتوى الصفحة. تعمل هذه المتغيرات عادةً مع معظم المواقع.

متغير المحتوى الرئيسي هو `{{content}}`، الذي يحتوي على محتوى المقال، أو [[تظليل صفحات الويب|التمييزات]]، أو التحديد إذا كان هناك نص محدد في الصفحة. لاحظ أن `{{content}}` يحاول استخراج المحتوى الرئيسي للصفحة، وقد لا يكون دائمًا ما تريده. في هذه الحالة، يمكنك استخدام متغيرات الإعداد المسبق الأخرى أو متغيرات المحدد لاستخراج المحتوى الذي تحتاجه.

| المتغير              | الوصف                                                                                   |
| ------------------- | -------------------------------------------------------------------------------------- |
| `{{author}}`        | مؤلف الصفحة                                                                            |
| `{{content}}`       | محتوى المقال، أو [[Highlight web pages\|التمييزات]]، أو التحديد، بتنسيق Markdown      |
| `{{contentHtml}}`   | محتوى المقال، أو [[Highlight web pages\|التمييزات]]، أو التحديد، بتنسيق HTML          |
| `{{date}}`          | التاريخ الحالي، يمكن تنسيقه باستخدام مرشح `date`                                       |
| `{{description}}`   | الوصف أو المقتطف                                                                       |
| `{{domain}}`        | النطاق                                                                                  |
| `{{favicon}}`       | عنوان URL للأيقونة المفضلة                                                              |
| `{{fullHtml}}`      | HTML غير معالج لمحتوى الصفحة الكامل                                                     |
| `{{highlights}}`    | [[Highlight web pages\|التمييزات]] مع النص والطوابع الزمنية                             |
| `{{image}}`         | عنوان URL لصورة المشاركة الاجتماعية                                                     |
| `{{published}}`     | تاريخ النشر، يمكن تنسيقه باستخدام مرشح `date`                                          |
| `{{selection}}`     | التحديد بتنسيق Markdown                                                                |
| `{{selectionHtml}}` | التحديد بتنسيق HTML                                                                    |
| `{{site}}`          | اسم الموقع أو الناشر                                                                   |
| `{{title}}`         | عنوان الصفحة                                                                            |
| `{{time}}`          | التاريخ والوقت الحاليان                                                                 |
| `{{url}}`           | عنوان URL الحالي                                                                        |
| `{{words}}`         | عدد الكلمات                                                                             |

## متغيرات الطلب

تستفيد متغيرات الطلب من النماذج اللغوية لاستخراج البيانات وتعديلها باستخدام اللغة الطبيعية. تتطلب متغيرات الطلب تمكين وتهيئة [[تفسير صفحات الويب|المفسر]].

تستخدم متغيرات الطلب الصيغة `{{"ملخص للصفحة"}}`. علامات الاقتباس المزدوجة حول الطلب مهمة وتميز الطلبات عن متغيرات الإعداد المسبق. يمكن معالجة استجابات الطلب لاحقًا باستخدام [[تصفية|المرشحات]]، مثل `{{"ملخص للصفحة"|blockquote}}`.

### متى تستخدم متغيرات الطلب

تتميز متغيرات الطلب بمرونة عالية وسهولة في الكتابة، إلا أنها تأتي مع عدة مقايضات: فهي أبطأ في التنفيذ، وقد تترتب عليها اعتبارات تتعلق بالتكلفة والخصوصية حسب [[تفسير صفحات الويب#النماذج|المزود]] الذي تختاره.

على عكس أنواع المتغيرات الأخرى، تحتاج متغيرات الطلب إلى معالجة بواسطة نموذج لغوي خارجي، لذا يتم استبدالها فقط بعد تشغيل [[تفسير صفحات الويب|المفسر]].

من الأفضل *عدم* استخدام متغيرات الطلب إذا كانت البيانات التي تريد استخراجها بتنسيق ثابت يمكن استخراجه بأنواع متغيرات أخرى.

من ناحية أخرى، يمكن أن تكون متغيرات الطلب مفيدة إذا كانت البيانات التي تريد استخراجها بتنسيق *غير ثابت* عبر المواقع المختلفة. على سبيل المثال، يمكنك إنشاء [[Obsidian Web Clipper/القوالب|قالب]] لحفظ الكتب يعمل بغض النظر عن موقع الكتب. ستعمل متغيرات الطلب مثل `{{"مؤلف الكتاب"}}` عبر أي موقع كتب، بينما تعمل متغيرات المحدد عادةً لموقع واحد فقط.

### أمثلة

يمكن أن تستخدم الطلبات أي استعلام باللغة الطبيعية تقريبًا. حسب النموذج الذي تستخدمه، يمكن للطلبات الاستعلام عن البيانات أو ترجمتها عبر اللغات.

- `{{"ملخص من ثلاث نقاط، مترجم إلى الفرنسية"}}` لاستخراج نقاط حول الصفحة وترجمتها إلى الفرنسية.
- `{{"un resumé de la page en trois points"}}` لاستخراج ثلاث نقاط باستخدام طلب بالفرنسية.

يمكن للطلبات تحويل محتوى الصفحة إلى JSON يمكن التعامل معه باستخدام [[تصفية|المرشحات]]. على سبيل المثال:

```
{{"return a JSON object for each tweet, that includes the author, tweet_text, date in YYYY-MM-DD format, and images array (if there are any)"|map:tweet => ({text: tweet.tweet_text, author: tweet.author, date: tweet.date})|template:"${text}\n— [[@${author}]], [[${date}]]\n"}}
```


## متغيرات Meta

تتيح لك متغيرات Meta استخراج البيانات من [عناصر meta](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/meta) في الصفحة، بما في ذلك بيانات [Open Graph](https://ogp.me/) المستخدمة لملء معاينات المشاركة الاجتماعية.

- `{{meta:name}}` تُرجع محتوى وسم meta name بالاسم المحدد، مثل `{{meta:name:description}}` لوسم meta الخاص بـ `description`.
- `{{meta:property}}` تُرجع محتوى وسم meta property بالخاصية المحددة، مثل `{{meta:property:og:title}}` لوسم meta الخاص بـ `og:title`.

## متغيرات المحدد

تتيح لك متغيرات المحدد استخراج المحتوى النصي من العناصر في الصفحة باستخدام [محددات CSS](https://developer.mozilla.org/en-US/docs/Web/CSS/CSS_selectors/Selectors_and_combinators).

الصيغة هي `{{selector:cssSelector?attribute}}`، حيث `?attribute` اختياري. إذا لم يتم تحديد سمة، يتم إرجاع المحتوى النصي للعنصر. يمكنك أيضًا استخدام `{{selectorHtml:cssSelector}}` للحصول على محتوى HTML للعنصر. تعمل متغيرات المحدد بشكل أفضل على موقع محدد أو مجموعة من المواقع التي لها بنية HTML ثابتة.

- `{{selector:h1}}` تُرجع المحتوى النصي لأي عناصر `h1` في الصفحة.
- `{{selector:.author}}` تُرجع المحتوى النصي لأي عناصر `.author` في الصفحة.
- `{{selector:img.hero?src}}` تُرجع سمة `src` للصورة ذات الفئة `hero`.
- `{{selector:a.main-link?href}}` تُرجع سمة `href` لوسم الرابط ذي الفئة `main-link`.
- `{{selectorHtml:body|markdown}}` تُرجع HTML الكامل لعنصر `body`، محولاً إلى Markdown باستخدام [[تصفية#معالجة HTML|مرشح]] `markdown`.
- محددات CSS المتداخلة والمجمعة مدعومة إذا كنت بحاجة إلى مزيد من التحديد.
- إذا تطابقت عدة عناصر مع المحدد، يتم إرجاع مصفوفة يمكنك معالجتها باستخدام [[تصفية#المصفوفات والكائنات|مرشحات المصفوفات والكائنات]] مثل `join` أو `map`.

يمكن أيضًا استخدام متغيرات المحدد مباشرة في [[المنطق|منطق القالب]]:

- في الحلقات: `{% for comment in selector:.comment %}...{% endfor %}`
- في الشروط: `{% if selector:.premium-badge %}...{% endif %}`
- في تعيين المتغيرات: `{% set items = selector:.list-item %}`

## متغيرات Schema.org

تتيح لك متغيرات Schema استخراج البيانات من [schema.org](https://schema.org/) JSON-LD في الصفحة. يمكن أيضًا استخدام بيانات Schema.org لـ[[Obsidian Web Clipper/القوالب#مطابقة Schema.org|تفعيل قالب]] تلقائيًا.

- `{{schema:@Type:key}}` تُرجع قيمة المفتاح من المخطط.
- `{{schema:@Type:parent.child}}` تُرجع قيمة خاصية متداخلة.
- `{{schema:@Type:arrayKey}}` تُرجع العنصر الأول في مصفوفة.
- `{{schema:@Type:arrayKey[index].property}}` تُرجع العنصر في الفهرس المحدد في مصفوفة.
- `{{schema:@Type:arrayKey[*].property}}` تُرجع خاصية محددة من جميع العناصر في مصفوفة.

يمكنك أيضًا استخدام صيغة مختصرة دون تحديد نوع المخطط:

- `{{schema:author}}` ستطابق أول خاصية `author` موجودة في أي نوع مخطط.
- `{{schema:name}}` ستطابق أول خاصية `name` موجودة في أي نوع مخطط.

هذه الصيغة المختصرة مفيدة بشكل خاص عندما لا تعرف أو لا تهتم بنوع المخطط المحدد، لكنك تعرف اسم الخاصية التي تبحث عنها.

تعمل الخصائص المتداخلة والوصول إلى المصفوفات أيضًا، سواء مع تحديد `@Type` للمخطط أو بدونه:

- `{{schema:author.name}}` ستجد أول خاصية `author` ثم تصل إلى خاصيتها الفرعية `name`.
- `{{schema:author[0].name}}` ستصل إلى `name` للمؤلف الأول في مصفوفة من المؤلفين.
- `{{schema:author[*].name}}` ستُرجع مصفوفة بجميع أسماء المؤلفين.
