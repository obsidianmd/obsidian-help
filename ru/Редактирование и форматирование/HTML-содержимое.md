Obsidian поддерживает HTML, чтобы вы могли отображать заметки так, как хотите, или даже [[Встраивание веб-страниц|встраивать веб-страницы]]. Разрешение HTML внутри заметок сопряжено с рисками. Чтобы предотвратить вред от вредоносного кода, Obsidian _очищает_ любой HTML в ваших заметках.

> [!example] 
> Элемент `<script>` обычно позволяет запускать JavaScript при загрузке. Если бы Obsidian не очищал HTML, злоумышленник мог бы убедить вас вставить текст, содержащий JavaScript, который извлекает конфиденциальную информацию с вашего компьютера и отправляет её обратно злоумышленнику.

Тем не менее, поскольку синтаксис Markdown не поддерживает все формы стилизации, использование очищенного HTML может стать ещё одним способом повышения качества ваших заметок. Мы включили некоторые наиболее распространённые варианты использования HTML.

## Ограничения HTML

Obsidian имеет определённые ограничения при использовании HTML в заметках:

### Markdown не работает внутри HTML

Obsidian не рендерит синтаксис Markdown внутри HTML-элементов. Это намеренное решение для оптимизации производительности и снижения сложности парсера при работе с большими документами.

Например, следующий код не сработает так, как ожидается:

```md
<div>
Это **не будет** жирным и это `не будет` кодом.
</div>
```

### HTML-блоки должны быть самодостаточными

HTML-блоки должны быть завершёнными и не могут содержать пустые строки внутри. Пустые строки разрывают HTML-блок.

Это будет работать:

```md
<table>
<tr>
<td>Содержимое здесь</td>
</tr>
</table>
```

Это не будет работать корректно:

```md
<table>

<tr>

<td>Содержимое здесь</td>

</tr>

</table>
```

### Когда Markdown кажется работающим внутри HTML

Некоторые строчные HTML-теги, такие как `<span>` или `<a>`, имеют ограниченную функциональность и могут казаться рендерящими Markdown, но на самом деле это не так. Markdown обрабатывается вне контекста HTML.

Для получения дополнительной информации о том, как Obsidian обрабатывает Markdown, см. [[Obsidian Flavored Markdown]].

## Распространённые варианты использования HTML

> [!info] Подробнее об использовании `<iframe>` можно узнать в разделе [[Встраивание веб-страниц]].

### Комментарии

[[Основной синтаксис форматирования#Комментарии|Markdown-комментарии]] — предпочтительный способ добавления скрытых комментариев в заметки. Однако некоторые методы конвертации Markdown-заметок, такие как [Pandoc](https://pandoc.org), имеют ограниченную поддержку Markdown-комментариев. В таких случаях вы можете использовать `<!-- HTML-комментарий -->` вместо них!

### Подчёркивание

Если вам нужно быстро подчеркнуть элемент в заметках, вы можете использовать `<u>Пример</u>`, чтобы создать <u>подчёркнутый текст</u>.

### Span/Div

Теги span и div можно использовать для применения пользовательских классов из [[CSS-сниппеты|CSS-сниппета]] или пользовательских стилей к выбранному фрагменту текста. Например, использование `<span style="font-family: cursive">ваш текст</span>` позволяет быстро <span style="font-family: cursive">изменить шрифт</span>.

## Зачёркивание

Нужно зачеркнуть <s>какой-то текст</s>? Используйте `<s>это</s>`, чтобы зачеркнуть его.
