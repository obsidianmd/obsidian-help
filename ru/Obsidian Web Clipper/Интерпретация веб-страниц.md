Интерпретатор — это функция [[Введение в Obsidian Web Clipper|Web Clipper]], которая позволяет взаимодействовать с веб-страницами с помощью естественного языка. Интерпретатор помогает захватывать и изменять данные, которые вы хотите сохранить в Obsidian. Например:

- Извлекать определённые фрагменты текста.
- Резюмировать или объяснять информацию.
- Преобразовывать текст из одного формата в другой.
- Переводить текст на другой язык.

Интерпретатор использует языковые модели для обработки информации на веб-странице и возвращает результаты с помощью [[Переменные|переменных]], которые можно добавить в ваши [[Obsidian Web Clipper/Шаблоны|шаблоны Web Clipper]].

![[web-clipper-interpreter-demo.mp4#interface]]

## Примеры промптов

Промпты используют синтаксис [[Переменные|переменных]] `{{"ваш промпт"}}`. Этот синтаксис можно использовать с любым запросом на естественном языке, например:

- `{{"a summary of the page"}}` — для извлечения краткого содержания страницы.
- `{{"a three bullet point summary, translated to French"}}` — для извлечения ключевых тезисов о странице и их перевода на французский.
- `{{"un resumé de la page en trois points"}}` — для извлечения трёх ключевых тезисов с использованием промпта на французском языке.

Результаты промптов можно дополнительно обработать с помощью [[Фильтры|фильтров]]. Фильтры применяются после получения ответа от модели. Например: `{{"a summary of the page"|blockquote}}` превратит ответ в блочную цитату.

## Начало работы

Интерпретатор работает практически с любым провайдером языковых моделей, включая варианты, работающие локально на вашем устройстве. Чтобы настроить Интерпретатор:

1. Перейдите в раздел **Интерпретатор** в настройках Web Clipper.
2. Включите **Включить интерпретатор**.
3. Настройте провайдера и модель — см. раздел [[Интерпретация веб-страниц#Модели|модели]] ниже.
4. Добавьте [[Переменные|переменные промптов]] в ваши [[Obsidian Web Clipper/Шаблоны|шаблоны]].
5. Если ваш шаблон содержит переменные промптов, раздел «Интерпретатор» станет видимым при [[Сохранение веб-страниц|сохранении страницы]]. Нажмите **интерпретировать**, чтобы обработать переменные промптов.

## Как это работает

Когда Интерпретатор включён *и* ваш шаблон содержит [[Переменные#Переменные промптов|переменные промптов]], в окне расширения над кнопкой **Добавить в Obsidian** появляется новый раздел «Интерпретатор». Этот раздел позволяет выбрать модель и запустить Интерпретатор для текущей страницы.

При нажатии **интерпретировать** Интерпретатор отправляет контекст страницы выбранной модели вместе со *всеми* промптами из вашего шаблона в одном запросе. В зависимости от выбранного провайдера модели это может быть внешний вызов или локальная обработка на вашем устройстве. Модель анализирует промпты в контексте страницы и возвращает ответы. Затем Интерпретатор заменяет переменные промптов полученными данными.

Весь процесс может занять от нескольких миллисекунд до более чем 30 секунд в зависимости от используемой модели и объёма обрабатываемых данных.

## Контекст

Термин *контекст* означает данные страницы, которые Интерпретатор использует для обработки промптов. Чем меньше контекст, тем быстрее работает Интерпретатор.

По умолчанию Интерпретатор использует весь HTML страницы в качестве контекста, однако это может делать промпты более медленными и дорогими, чем необходимо.

Вы можете переопределить контекст по умолчанию в **Расширенных настройках** Интерпретатора и задать контекст для каждого [[Obsidian Web Clipper/Шаблоны|шаблона]].

Чтобы задать более точный контекст, используйте [[Переменные#Переменные селекторов|переменные селекторов]] (или другие типы переменных) для интерпретации определённого раздела страницы. Например, вы можете использовать следующую переменную селектора в контексте Интерпретатора вашего шаблона:

```
{{selectorHtml:#main}}
```

Это запустит Интерпретатор только для элемента `#main` веб-страницы, если он существует. [[Фильтры#Обработка HTML|Фильтры обработки HTML]], такие как `remove_html`, `strip_tags` и `strip_attr`, могут быть полезны для дальнейшего сокращения длины контекста и ускорения обработки.

> Используя стороннего провайдера моделей, вы соглашаетесь с его условиями и политикой конфиденциальности. Запросы Интерпретатора отправляются напрямую выбранному провайдеру. Obsidian не собирает и не хранит никаких данных о ваших запросах.

### Выбор модели

В целом мы рекомендуем использовать малые модели с Web Clipper, поскольку они быстрее и достаточно точно справляются с этой задачей. Примеры малых моделей: **Anthropic Claude Haiku**, **Google Gemini Flash**, **Llama** с параметрами 3B или 8B, а также серия моделей **OpenAI Mini**.

### Пользовательские провайдеры и модели

Чтобы добавить пользовательского провайдера и/или модель, перейдите в **[[Настройки]]** Web Clipper → **Интерпретатор**:

- **Добавить провайдера** — для настройки предустановленных и пользовательских провайдеров.
- **Добавить модель** — для настройки предустановленных и пользовательских моделей.

При добавлении пользовательского провайдера мы рекомендуем использовать его эндпоинт chat completions в качестве **Базового URL** — обычно он заканчивается на `/chat/completions`.

### Локальные модели

Интерпретатор может использовать локальные модели, обеспечивающие большую конфиденциальность и возможность работы офлайн. Существует несколько вариантов запуска локальных моделей. Один из самых простых в настройке — Ollama.

#### Ollama

[Ollama](https://ollama.com/) позволяет запускать языковые модели локально и конфиденциально на вашем устройстве.

После загрузки и установки Ollama добавьте Ollama с помощью кнопки **Добавить провайдера** в настройках Интерпретатора. Ollama не требует API ключа. Затем выберите модель из [списка моделей](https://ollama.com/search). Например, если вы хотите использовать [Llama 3.2](https://ollama.com/library/llama3.2), нажмите **Добавить модель**, затем:

- **Провайдер:** Ollama
- **Отображаемое имя:** Llama 3.2 — это значение можно настроить.
- **ID модели:** `llama3.2` — должен точно совпадать с ID модели в Ollama.

**Запуск сервера Ollama**

Чтобы расширение браузера могло взаимодействовать с Ollama, необходимо [явно указать это](https://github.com/ollama/ollama/issues/2308) при запуске сервера, иначе вы увидите ошибку `403`.

Закройте приложение Ollama и выполните следующую команду в терминале. Измените протокол на протокол расширения вашего браузера, если вы не используете Chrome или Firefox.

```
OLLAMA_ORIGINS=moz-extension://*,chrome-extension://*,safari-web-extension://* ollama serve
```

Затем запустите модель в Ollama обычным способом, например:

```
ollama run llama3.2
```

**Длина контекста**

Окно контекста Ollama по умолчанию составляет 2048 токенов. Это максимальное количество токенов для сообщения и ответа. При сохранении длинной веб-страницы вы можете легко превысить этот лимит. Ollama молча завершит работу и вернёт нерелевантные результаты. Возможные решения:

- Увеличить параметр `num_ctx` в Ollama. Учтите, что больший контекст требует больше памяти.
- Используйте поле [[#Контекст]] в вашем шаблоне, чтобы указать более целевой раздел страницы, или сократите контекст с помощью [[Фильтры|фильтра]], например `{{content|slice:0,1000}}`.
