[[Obsidian Web Clipper/Шаблоны|Шаблоны Web Clipper]] могут использовать переменные для автоматического заполнения данных со страницы в шаблоне. Переменные можно использовать в **названии заметки**, **расположении заметки**, **свойствах** и **содержимом заметки**. Переменные также можно модифицировать с помощью [[Фильтры|фильтров]].

Используйте значок `...` в расширении [[Введение в Obsidian Web Clipper|Web Clipper]], чтобы получить доступ к переменным текущей страницы для использования в шаблонах. Существует пять типов переменных:

- [[Переменные#Предустановленные переменные|Предустановленные переменные]]
- [[Переменные#Переменные запросов|Переменные запросов]]
- [[Переменные#Мета-переменные|Мета-переменные]]
- [[Переменные#Переменные селекторов|Переменные селекторов]]
- [[Переменные#Переменные Schema.org|Переменные Schema.org]]

## Предустановленные переменные

Предустановленные переменные автоматически генерируются на основе содержимого страницы. Они обычно работают для большинства сайтов.

Основная переменная содержимого — `{{content}}`, которая содержит содержимое статьи, или [[Выделение на веб-страницах|выделения]], или выделенный фрагмент, если на странице есть выделенный текст. Обратите внимание, что `{{content}}` пытается извлечь основное содержимое страницы, которое не всегда может быть тем, что вам нужно. В таком случае вы можете использовать другие предустановленные переменные или переменные селекторов для извлечения нужного содержимого.

| Переменная          | Описание                                                                               |
| ------------------- | -------------------------------------------------------------------------------------- |
| `{{author}}`        | Автор страницы                                                                         |
| `{{content}}`       | Содержимое статьи, [[Выделение на веб-страницах\|выделения]] или выделенный фрагмент в формате Markdown |
| `{{contentHtml}}`   | Содержимое статьи, [[Выделение на веб-страницах\|выделения]] или выделенный фрагмент в формате HTML |
| `{{date}}`          | Текущая дата, может быть отформатирована с помощью фильтра `date`                      |
| `{{description}}`   | Описание или выдержка                                                                  |
| `{{domain}}`        | Домен                                                                                  |
| `{{favicon}}`       | URL фавикона                                                                           |
| `{{fullHtml}}`      | Необработанный HTML полного содержимого страницы                                       |
| `{{highlights}}`    | [[Выделение на веб-страницах\|Выделения]] с текстом и временными метками               |
| `{{image}}`         | URL изображения для социальных сетей                                                   |
| `{{published}}`     | Дата публикации, может быть отформатирована с помощью фильтра `date`                   |
| `{{selection}}`     | Выделенный фрагмент в формате Markdown                                                 |
| `{{selectionHtml}}` | Выделенный фрагмент в формате HTML                                                     |
| `{{site}}`          | Название сайта или издателя                                                            |
| `{{title}}`         | Заголовок страницы                                                                     |
| `{{time}}`          | Текущие дата и время                                                                   |
| `{{url}}`           | Текущий URL                                                                            |
| `{{words}}`         | Счётчик слов                                                                           |

## Переменные запросов

Переменные запросов используют языковые модели для извлечения и модификации данных с помощью естественного языка. Для переменных запросов необходимо, чтобы [[Интерпретация веб-страниц|Интерпретатор]] был включён и настроен.

Переменные запросов используют синтаксис `{{"краткое содержание страницы"}}`. Двойные кавычки вокруг запроса важны и отличают запросы от предустановленных переменных. Ответы на запросы можно дополнительно обработать с помощью [[Фильтры|фильтров]], например `{{"краткое содержание страницы"|blockquote}}`.

### Когда использовать переменные запросов

Переменные запросов обладают преимуществом чрезвычайной гибкости и простоты написания, однако имеют ряд компромиссов: они медленнее в выполнении и могут иметь ограничения по стоимости и конфиденциальности в зависимости от выбранного [[Интерпретация веб-страниц#Модели|провайдера]].

В отличие от других типов переменных, переменные запросов должны быть обработаны внешней языковой моделью, поэтому они заменяются только после выполнения [[Интерпретация веб-страниц|Интерпретатора]].

Лучше *не* использовать переменные запросов, если данные, которые вы хотите извлечь, имеют согласованный формат, который можно извлечь с помощью других типов переменных.

С другой стороны, переменные запросов могут быть полезны, если данные, которые вы хотите извлечь, имеют *несогласованный* формат на разных сайтах. Например, вы можете создать [[Obsidian Web Clipper/Шаблоны|шаблон]] для сохранения книг, который не зависит от конкретного сайта о книгах. Переменные запросов, такие как `{{"автор книги"}}`, будут работать на любом сайте о книгах, тогда как переменные селекторов обычно работают только для одного сайта.

### Примеры

Запросы могут использовать практически любой запрос на естественном языке. В зависимости от используемой модели запросы могут извлекать или переводить данные на разные языки.

- `{{"краткое содержание в трёх пунктах, переведённое на французский"}}` для извлечения пунктов о странице и их перевода на французский.
- `{{"un resumé de la page en trois points"}}` для извлечения трёх пунктов с помощью запроса на французском.

Запросы могут преобразовывать содержимое страницы в JSON, который можно обрабатывать с помощью [[Фильтры|фильтров]]. Например:

```
{{"return a JSON object for each tweet, that includes the author, tweet_text, date in YYYY-MM-DD format, and images array (if there are any)"|map:tweet => ({text: tweet.tweet_text, author: tweet.author, date: tweet.date})|template:"${text}\n— [[@${author}]], [[${date}]]\n"}}
```


## Мета-переменные

Мета-переменные позволяют извлекать данные из [мета-элементов](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/meta) на странице, включая данные [Open Graph](https://ogp.me/), используемые для заполнения превью при публикации в социальных сетях.

- `{{meta:name}}` возвращает содержимое мета-тега name с указанным именем, например `{{meta:name:description}}` для мета-тега `description`.
- `{{meta:property}}` возвращает содержимое мета-тега property с указанным свойством, например `{{meta:property:og:title}}` для мета-тега `og:title`.

## Переменные селекторов

Переменные селекторов позволяют извлекать текстовое содержимое из элементов на странице с помощью [CSS-селекторов](https://developer.mozilla.org/en-US/docs/Web/CSS/CSS_selectors/Selectors_and_combinators).

Синтаксис: `{{selector:cssSelector?attribute}}`, где `?attribute` является необязательным. Если атрибут не указан, возвращается текстовое содержимое элемента. Вы также можете использовать `{{selectorHtml:cssSelector}}` для получения HTML-содержимого элемента. Переменные селекторов лучше всего работают на конкретном сайте или наборе сайтов с согласованной HTML-структурой.

- `{{selector:h1}}` возвращает текстовое содержимое всех элементов `h1` на странице.
- `{{selector:.author}}` возвращает текстовое содержимое всех элементов `.author` на странице.
- `{{selector:img.hero?src}}` возвращает атрибут `src` изображения с классом `hero`.
- `{{selector:a.main-link?href}}` возвращает атрибут `href` тега ссылки с классом `main-link`.
- `{{selectorHtml:body|markdown}}` возвращает весь HTML элемента `body`, преобразованный в Markdown с помощью [[Фильтры#Обработка HTML|фильтра]] `markdown`.
- Поддерживаются вложенные CSS-селекторы и комбинаторы, если вам нужна большая специфичность.
- Если несколько элементов соответствуют селектору, возвращается массив, который можно обработать с помощью [[Фильтры#Массивы и объекты|фильтров массивов и объектов]], таких как `join` или `map`.

Переменные селекторов также можно использовать непосредственно в [[Шаблоны#Логика шаблонов|логике шаблонов]]:

- В циклах: `{% for comment in selector:.comment %}...{% endfor %}`
- В условиях: `{% if selector:.premium-badge %}...{% endif %}`
- В присвоении переменных: `{% set items = selector:.list-item %}`

## Переменные Schema.org

Переменные Schema позволяют извлекать данные из [schema.org](https://schema.org/) JSON-LD на странице. Данные Schema.org также можно использовать для автоматического [[Obsidian Web Clipper/Шаблоны#Сопоставление Schema.org|запуска шаблона]].

- `{{schema:@Type:key}}` возвращает значение ключа из схемы.
- `{{schema:@Type:parent.child}}` возвращает значение вложенного свойства.
- `{{schema:@Type:arrayKey}}` возвращает первый элемент массива.
- `{{schema:@Type:arrayKey[index].property}}` возвращает элемент по указанному индексу в массиве.
- `{{schema:@Type:arrayKey[*].property}}` возвращает определённое свойство из всех элементов массива.

Вы также можете использовать сокращённую запись без указания типа схемы:

- `{{schema:author}}` найдёт первое свойство `author` в любом типе схемы.
- `{{schema:name}}` найдёт первое свойство `name` в любом типе схемы.

Эта сокращённая запись особенно полезна, когда вы не знаете или не заботитесь о конкретном типе схемы, но знаете имя свойства, которое ищете.

Вложенные свойства и доступ к массивам работают как с указанием `@Type` схемы, так и без него:

- `{{schema:author.name}}` найдёт первое свойство `author` и затем обратится к его подсвойству `name`.
- `{{schema:author[0].name}}` обратится к `name` первого автора в массиве авторов.
- `{{schema:author[*].name}}` вернёт массив имён всех авторов.
