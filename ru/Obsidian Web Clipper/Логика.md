[[Введение в Obsidian Web Clipper|Web Clipper]] поддерживает логику шаблонов для условий, циклов и присвоения переменных. Этот синтаксис вдохновлён языками шаблонов [Twig](https://twig.symfony.com/) и [Liquid](https://shopify.github.io/liquid/).


> [!warning] Требуется последняя версия
> Функции логики требуют Obsidian Web Clipper 1.0.0, который ещё не одобрен во всех магазинах расширений.

## Условия

Используйте `{% if %}` для условного включения содержимого на основе переменных или выражений.

```twig
{% if author %}
Author: {{author}}
{% endif %}
```

Используйте `{% else %}` для резервного содержимого и `{% elseif %}` для цепочки нескольких условий:

```twig
{% if status == "published" %}
Live article
{% elseif status == "draft" %}
Draft article
{% else %}
Unknown status
{% endif %}
```

### Операторы сравнения

Поддерживаются следующие операторы сравнения:

| Оператор | Описание |
|----------|-------------|
| `==` | Равно |
| `!=` | Не равно |
| `>` | Больше |
| `<` | Меньше |
| `>=` | Больше или равно |
| `<=` | Меньше или равно |
| `contains` | Проверка, содержит ли строка подстроку или массив содержит значение |

Примеры:
- `{% if title == "Home" %}` — равенство строк
- `{% if price >= 100 %}` — числовое сравнение
- `{% if title contains "Review" %}` — проверка подстроки
- `{% if tags contains "important" %}` — принадлежность к массиву

### Логические операторы

Объединяйте условия с помощью логических операторов:

| Оператор | Альтернатива | Описание                         |
| -------- | ----------- | ----------------------------------- |
| `and`    | `&&`        | Оба условия должны быть истинными        |
| `or`     | \|\|        | Хотя бы одно условие должно быть истинным |
| `not`    | `!`         | Отрицание условия                 |

Примеры:
- `{% if author and published %}` — оба должны существовать
- `{% if draft or archived %}` — любое из условий
- `{% if not hidden %}` — отрицание
- `{% if (premium or featured) and published %}` — сгруппированные условия

### Истинность

Когда переменная используется без оператора сравнения, она проверяется на «истинность»:

- `false`, `null`, `undefined`, пустая строка `""` и `0` считаются **ложными**.
- Пустые массивы `[]` считаются **ложными**.
- Всё остальное считается **истинным**.

```twig
{% if content %}
Has content
{% endif %}
```

## Присвоение переменной

Используйте `{% set %}` для создания или изменения переменных в шаблоне:

```twig
{% set slug = title|lower|replace:" ":"-" %}
File: {{slug}}.md
```

Переменным можно присвоить:
- Другие переменные: `{% set name = author %}`
- Литералы: `{% set count = 5 %}` или `{% set label = "Draft" %}`
- Выражения с фильтрами: `{% set excerpt = content|truncate:100 %}`
- Результаты селекторов: `{% set comments = selector:.comment %}`

Переменные, заданные через `{% set %}`, можно использовать в последующей логике шаблона и в выводе `{{variable}}`.

## Резервные значения

Используйте оператор `??` для указания резервных значений, когда переменная пуста или не определена:

```twig
{{title ?? "Untitled"}}
```

Если `title` пуста, не определена или ложна, вместо неё будет использовано резервное значение `"Untitled"`.

Это сокращённая запись эквивалентного оператора `if`:

```twig
{% if title %}{{title}}{% else %}Untitled{% endif %}
```

### Цепочка резервных значений

Можно выстраивать цепочку из нескольких резервных значений:

```twig
{{title ?? headline ?? "No title"}}
```

Будет использовано `title`, если доступно, иначе `headline`, иначе строка `"No title"`.

### С фильтрами

Фильтры имеют более высокий приоритет, чем `??`, поэтому фильтры применяются до проверки резервного значения:

```twig
{{title|upper ?? "UNTITLED"}}
```

Здесь сначала применяется `upper` к `title`, затем, если результат пуст, используется `"UNTITLED"`. Чтобы применить фильтры к резервному значению, используйте скобки или отдельные выражения:

```twig
{{title ?? "Untitled"|lower}}
```

Будет использовано `title`, если доступно, иначе к резервному значению будет применён `lower`, что даст `"untitled"`.

## Циклы

Используйте `{% for %}` для итерации по массивам:

```twig
{% for item in schema:author %}
- {{item.name}}
{% endfor %}
```

### Источники циклов

Можно перебирать:
- Массивы схемы: `{% for item in schema:author %}`
- Результаты селекторов: `{% for comment in selector:.comment %}`
- Ранее заданные переменные: `{% set items = selector:.item %}{% for item in items %}`

### Переменные цикла

Внутри цикла доступен объект `loop` со следующими свойствами:

| Переменная | Описание |
|----------|-------------|
| `loop.index` | Текущая итерация (начиная с 1) |
| `loop.index0` | Текущая итерация (начиная с 0) |
| `loop.first` | `true` при первой итерации |
| `loop.last` | `true` при последней итерации |
| `loop.length` | Общее количество элементов |

```twig
{% for tag in tags %}
{{loop.index}}. {{tag}}
{% if loop.last %} (end of list){% endif %}
{% endfor %}
```

Для обратной совместимости можно также использовать `item_index` (где `item` — имя переменной итератора) для получения позиции с нуля:

```twig
{% for tag in tags %}
{{tag_index}}. {{tag}}
{% endfor %}
```

### Доступ к элементам массива по индексу

Используйте скобочную нотацию для доступа к элементам массива по индексу:

```twig
{{items[0]}}
{{items[loop.index0]}}
```

Это полезно, когда нужно обращаться к элементам нескольких массивов параллельно:

```twig
{% set transcripts = selector:.transcript-text %}
{% set timestamps = selector:.timestamp %}

{% for line in transcripts %}
{{timestamps[loop.index0]}} - {{line}}
{% endfor %}
```

Скобочная нотация также работает со свойствами объектов:

```twig
{{user["name"]}}
{{data["my-key"]}}
```

### Вложенные циклы

Циклы можно вкладывать для сложных структур данных:

```twig
{% for section in sections %}
## {{section.title}}
{% for item in section.items %}
- {{item}}
{% endfor %}
{% endfor %}
```

## Комбинирование логики

Условия и циклы можно комбинировать:

```twig
{% for item in items %}
{% if item.active %}
- {{item.name}}
{% endif %}
{% endfor %}
```

## Порядок вычисления

Логика шаблонов обрабатывается в следующем порядке:

1. **Логика шаблона** — `{% if %}`, `{% for %}`, `{% set %}` и `{{variables}}` вычисляются первыми
2. **Переменные запросов** — [[Переменные#Переменные запросов|Переменные запросов]] вроде `{{"summarize this"|prompt}}` отправляются в Интерпретатор после завершения обработки логики шаблона

Это означает, что вы можете использовать логику шаблонов для динамического построения запросов, но результаты запросов недоступны для использования в условиях или циклах.
