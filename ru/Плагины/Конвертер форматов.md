Конвертер форматов — это [[Основные плагины|встроенный плагин]], который позволяет конвертировать Markdown из других приложений в формат Obsidian. Он также позволяет конвертировать определённые [[Свойства]] в новые требуемые форматы.

> [!warning] Предупреждение
> Конвертер форматов конвертирует всё хранилище на основе заданных настроек. [[Резервное копирование файлов Obsidian|Сделайте резервную копию файлов Obsidian]] перед выполнением конвертации.

Чтобы конвертировать все заметки в хранилище:

1. В [[Палитра команд|палитре команд]] выберите **Импорт Markdown-файлов**. Эту команду также можно найти на [[Лента|вертикальной панели]] по значку **Импорт Markdown-файлов** ![[lucide-binary.svg#icon]].
2. Включите форматы, которые вы хотите конвертировать.
3. Нажмите **Начать конвертацию**.

Для получения дополнительной информации обратитесь к разделу [[Основной синтаксис форматирования]].

## Поддерживаемые форматы

### Roam Research

Конвертер форматов может конвертировать следующий синтаксис Roam Research:

- **Теги**: Конвертирует `#tag` и `#[[tag]]` в `[[tag]]`
- **Выделение**: Конвертирует `^^highlight^^` в `==highlight==`
- **Элементы TODO**: Конвертирует `{{[[TODO]]}}` в `[ ]`

### Bear

Конвертер форматов может конвертировать следующий синтаксис Bear:

- **Выделение**: Конвертирует `::highlight::` в `==highlight==`

### Zettelkasten

Конвертер форматов может конвертировать следующий синтаксис Zettelkasten:

- **Полные ссылки**: Конвертирует `[[UID]]` в `[[UID File Name]]`
- **Красивые ссылки**: Конвертирует `[[UID]]` в `[[UID File Name|File Name]]`

### [[Свойства]]

Начиная с Obsidian `1.9.3`, конвертер форматов может конвертировать [[Свойства#Устаревшие свойства|устаревшие форматы свойств]] в текущий формат:

**Псевдонимы**

```yaml
# До

alias: My Note Title

# После

aliases:
  - My Note Title
```

**Теги**

```yaml
# До

tag: project, important

# После

tags:
  - project
  - important
```

**CSS-классы**

```yaml
# До

cssclass: custom-style

# После

cssclasses:
  - custom-style
```
