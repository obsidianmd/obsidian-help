Вы можете настроить пользовательский домен или поддомен для вашего сайта [[Введение в Obsidian Publish|Obsidian Publish]].

> [!warning] Предупреждение
> На данный момент мы поддерживаем настройку пользовательских доменов только следующими способами:
>
> - [[#Настройка с помощью CloudFlare]] с использованием [режима Full](https://developers.cloudflare.com/ssl/origin-configuration/ssl-modes/full/).
> - [[#Настройка с помощью прокси]]
> 
> У нас пока нет возможности выпустить SSL-сертификат от вашего имени.

## Настройка с помощью CloudFlare

Самый простой способ настроить пользовательский домен или поддомен — создать аккаунт [CloudFlare](https://cloudflare.com) и позволить CloudFlare управлять DNS вашего домена.

Следующие шаги описывают использование CloudFlare для настройки пользовательского домена для вашего сайта Obsidian Publish, будь то корневой домен (`mysite.com`) или поддомен (`notes.mysite.com`).

> [!important] Важно
> CloudFlare — это **единственный официально поддерживаемый провайдер** для настройки пользовательских доменов. Использование следующих инструкций с любыми другими провайдерами, скорее всего, не сработает.

**CloudFlare:**

1. Откройте Cloudflare для домена, на котором вы хотите разместить сайт Publish, например `mysite.com`, даже если вам нужен поддомен вроде `notes.mysite.com`.
2. Перейдите в **DNS** и нажмите **Add Record**.
3. Выберите **CNAME**.
4. В поле **name** введите ваш домен или поддомен, например `notes.mysite.com`.
5. В поле **target** введите `publish-main.obsidian.md`. Не включайте в это значение ваш персональный под-URL. Obsidian Publish обрабатывает это из вашей конфигурации.
6. Убедитесь, что **proxy status** включён. По умолчанию он должен быть включён.
7. Перейдите в **SSL/TLS** и установите режим шифрования SSL/TLS на «Full» для автоматической настройки сертификата SSL/TLS.

> [!note] Примечание
> Чтобы перенаправить и `mysite.com`, и `www.mysite.com` на Obsidian Publish, вам нужно создать [Page Rule](https://support.cloudflare.com/hc/en-us/articles/200172336-Creating-Page-Rules) со следующими настройками:
>
> - URL match: `www.mysite.com/*`
> - Forward URL - 301 Permanent Redirect
> - Redirect URL: `https://mysite.com/$1`
>
> После создания правила страницы создайте запись CNAME для `www.mysite.com` так же, как вы сделали для `mysite.com`.

**Obsidian:**

1. Откройте Obsidian на вашем компьютере.
2. На [[Лента|вертикальной панели]] слева нажмите **Опубликовать изменения** ( ![[lucide-send.svg#icon]] ).
3. В разделе **Опубликовать изменения** выберите **Настройки сайта** ( ![[lucide-cog.svg#icon]] ).
4. Рядом с **Личный домен** выберите **Настроить**.
5. В поле **Личный URL** введите URL вашего домена или поддомена. Убедитесь, что в поле пользовательского URL не указано `www.`.

> [!note] Примечание
> Если настройка пользовательского домена приводит к циклу перенаправления, вероятно, режим шифрования в CloudFlare установлен на «Flexible» вместо «Full».

## Настройка с помощью прокси

Вы также можете настроить SSL/TLS для вашего пользовательского домена, используя собственный веб-сервер.

Если у вас уже размещён сайт на вашем домене или поддомене, вы также можете воспользоваться этим вариантом и настроить ваш сайт для загрузки Obsidian Publish по определённому URL-пути, вместо размещения полного сайта.

Проксируйте все запросы по этому URL-пути на `https://publish.obsidian.md/serve?url=mysite.com/my-notes/...` и настройте параметры сайта в Obsidian на тот же URL-путь, установив **Личный URL** на `mysite.com/my-notes`.

Вы также можете настроить Obsidian Publish как под-URL принадлежащего вам сайта. Например, `https://mysite.com/my-notes/`. Для этого вам необходимо разместить собственный сервер и проксировать все запросы на наш сервер по адресу `https://publish.obsidian.md/`.

Следующие примеры настройки прокси не являются исчерпывающими, но описывают распространённые методы реализации.

### NGINX

В конфигурации NGINX добавьте следующее:

```nginx
location /my-notes {
  proxy_pass https://publish.obsidian.md/serve?url=mysite.com/my-notes/;
  proxy_ssl_server_name on;
  proxy_set_header Host publish.obsidian.md;
}
```


Некоторые пользователи сообщают, что может потребоваться добавить `$request_uri` к proxy pass:

```nginx
location /my-notes {
  proxy_pass https://publish.obsidian.md/serve?url=mysite.com/my-notes$request_uri;
  proxy_ssl_server_name on;
  proxy_set_header Host publish.obsidian.md;
}
```

### Apache

В `.htaccess` добавьте следующее:

```htaccess
RewriteEngine  on
RewriteRule    "^my-notes/(.*)$"  "https://publish.obsidian.md/serve?url=mysite.com/my-notes/$1"  [L,P]
```

> [!note] Примечание
> Модуль `mod_rewrite` должен быть включён, и вам также может потребоваться настроить [SSLProxyEngine](https://stackoverflow.com/questions/40938148/reverse-proxy-for-external-url-apache).

### Netlify

В `netlify.toml` [настройте перенаправления](https://docs.netlify.com/routing/redirects/#syntax-for-the-netlify-configuration-file):

```plain
[[redirects]]
  from = "https://mysite.com/my-notes/*"
  to = "https://publish.obsidian.md/serve?url=mysite.com/my-notes/:splat"
  status = 200
  force = true
```

### Vercel

В `vercel.json` [настройте перезаписи](https://vercel.com/docs/configuration#project/rewrites):

```json
{
  ...

  "rewrites": [
    {
      "source": "/my-notes/",
      "destination": "https://publish.obsidian.md/serve?url=mysite.com/my-notes"
    },
    {
      "source": "/my-notes/:path*",
      "destination": "https://publish.obsidian.md/serve?url=mysite.com/my-notes/:path*"
    }
  ]
}
```

### Caddy

```plain
mysite.com {
  encode zstd gzip
  handle /my-notes* {
    reverse_proxy https://publish.obsidian.md {
      header_up Host {upstream_hostport}
    }
    rewrite * /serve?url=mysite.com{path}
  }
}
```

### Traefik

Этот минимальный фрагмент конфигурации перенаправляет `mysite.com` на Obsidian Publish.
Полный пример смотрите в [документации Traefik](https://doc.traefik.io/traefik/routing/overview/).

```yaml
http:
  routers:
    mysite:
      rule: Host(`mysite.com`)
      service: obsidian-publish
      middlewares:
        - "publish-headers"
  services:
    obsidian-publish:
      loadBalancer:
        servers:
          - url: https://publish.obsidian.md
  middlewares:
    publish-headers:
      headers:
        customRequestHeaders:
          Host: "publish.obsidian.md"
          x-obsidian-custom-domain: "mysite.com"
```

### Поддерживаемые HTTP X-заголовки

Если ваш прокси-сервис не позволяет использовать пути в запросах, вы можете использовать `https://publish.obsidian.md/` с пользовательским заголовком `x-obsidian-custom-domain`, установленным на URL вашего сайта `mysite.com/my-subpath`.

## Перенаправление старого сайта на пользовательский домен

Если вы хотите перенаправить посетителей со старого сайта `publish.obsidian.md` на ваш новый пользовательский домен, включите опцию **Перенаправлять на личный домен** при настройке пользовательского домена.

## Устранение неполадок

После настройки пользовательского домена, если вы ранее посещали свой сайт по старой ссылке `https://publish.obsidian.md/slug`, вам может потребоваться очистить кэш браузера, чтобы некоторые элементы (такие как шрифты, графы или доступ по паролю) работали корректно. Это связано с ограничениями междоменной безопасности, налагаемыми современными браузерами. Хорошая новость в том, что читатели вашего сайта никогда не столкнутся с этой проблемой, если вы предоставите посетителям только ваш пользовательский домен.
