Obsidian дозволяє легко перенести ваші нотатки з Notion за допомогою [[Імпортер|додатка Імпортер]]. Це перетворить ваші дані з Notion у довговічні файли Markdown, які ви можете використовувати офлайн з Obsidian та багатьма іншими застосунками.

Obsidian пропонує два способи імпорту даних з Notion:

1. **Імпорт через API** зберігає весь ваш робочий простір, включаючи бази даних та формули, які конвертуються в [[Вступ до Баз|Бази]], але потребує токен інтеграції Notion та підключення до інтернету.
2. **Імпорт файлів** не зберігає бази даних, але не потребує токена API або підключення до інтернету.

## Імпорт через API

### Створення токена інтеграції Notion API

Для доступу до ваших даних Notion через API вам потрібен токен інтеграції. Цей крок займає приблизно 2 хвилини.

Токен — це довгий рядок із цифр та літер, який зазвичай починається з `ntn_...` і дозволяє завантажити ваші дані з Notion.

1. Увійдіть до панелі [Notion Integrations](https://www.notion.so/profile/integrations/internal).
2. Виберіть **New integration**.

![[notion-integration.png#interface]]

2. Дайте інтеграції назву, наприклад "Personal". Можна використовувати будь-яку назву.
3. Виберіть робочий простір, який хочете експортувати.
4. Натисніть **Save** та перейдіть до **Configure integration settings**.
5. На вкладці **Configuration** ваш токен API доступний у полі **Internal Integration Secret**.
6. Виберіть **Show**, а потім **Copy**.
7. Збережіть токен у безпечному місці, наприклад у менеджері паролів.

![[notion-token.png#interface]]

Далі надайте вашій інтеграції доступ до сторінок та баз даних Notion, які ви хочете імпортувати.

1. Перейдіть на вкладку **Access** щойно створеної інтеграції.
2. Натисніть **Edit access**.
3. Додайте сторінки та бази даних, які ви хочете імпортувати.

Тепер ви можете конвертувати ваші дані за допомогою Obsidian Імпортер.

### Імпорт даних Notion через API

Вам знадобиться офіційний додаток Obsidian [[Імпортер]], який можна [встановити тут](obsidian://show-plugin?id=obsidian-importer).

1. Відкрийте **[[Налаштування]]**.
2. Перейдіть до **Додатки спільноти** та [встановіть Імпортер](obsidian://show-plugin?id=obsidian-importer).
3. Увімкніть додаток Імпортер.
4. Відкрийте додаток **Імпортер** за допомогою меню команд або значка на стрічковому меню.
5. У полі **File format** виберіть **Notion (API)**
6. У полі **API token** вставте ваш **Internal Integration Secret** з Notion.
7. Натисніть **Load**, щоб вибрати бази даних та сторінки для імпорту.
8. Перегляньте та відредагуйте параметри імпорту.
9. Натисніть **Import** та зачекайте завершення імпорту
10. Готово!

### Обмеження

> [!info] Імпорт через API є новою функцією
> Імпортер Notion API є новим. Через складність робочих просторів Notion деякі граничні випадки могли бути не враховані. Якщо ви зіткнетесь з проблемами при конвертації, [подайте звіт про помилку](https://github.com/obsidianmd/obsidian-importer/issues), щоб ми могли його покращити.

Через обмеження швидкості Notion API імпорт великих робочих просторів може зайняти значний час. Будь ласка, наберіться терпіння.

Через обмеження Notion API деякі дані недоступні або не можуть бути конвертовані:

- Імпортується лише основне подання для кожної бази даних.
- [Зв'язані джерела даних](https://developers.notion.com/docs/working-with-databases#additional-types-of-databases) не імпортуються: *"API Notion наразі не підтримує зв'язані джерела даних. При наданні доступу до бази даних вашій інтеграції переконайтеся, що вона містить оригінальне джерело даних!"*
- Функції `People`: `name()` та `email()`
- Функції `Text`: `style()` та `unstyle()`

Крім того, Імпортер внесе наступні зміни:

- Сторінки без дочірніх сторінок або баз даних будуть імпортовані як `[назва файлу].md` замість `[назва файлу]/[назва файлу].md`.
- Бази даних завжди представлені як теки з назвою `[назва бази даних]` з файлом `[назва бази даних].base` всередині.

## Імпорт файлів

Імпорт файлів — це альтернативний спосіб імпорту даних з Notion. Цей метод не зберігає бази даних, але не потребує токена API або підключення до інтернету.

### Експорт даних з Notion

Для підготовки даних до імпорту вам потрібно експортувати весь робочий простір за допомогою формату HTML-експорту Notion. Ми рекомендуємо не використовувати Markdown-експорт Notion, оскільки він пропускає важливі дані. Ви повинні мати права адміністратора робочого простору Notion, щоб експортувати весь вміст.

1. Перейдіть до **[[Налаштування]]** у верхній частині бічної панелі Notion.
2. У розділі **Workspace** виберіть **General**.
3. Знайдіть та виберіть **Export all workspace content**.
4. У полі **Export format** виберіть **HTML**.
5. Виберіть **Include everything**.
6. Увімкніть **Create folders for subpages**.
7. Ви отримаєте файл `.zip` електронною поштою або безпосередньо у браузері.

![[notion-export-2.png#interface]]

### Імпорт .zip файлу з Notion

Вам знадобиться офіційний додаток Obsidian [[Імпортер]], який можна [встановити тут](obsidian://show-plugin?id=obsidian-importer).

1. Відкрийте **[[Налаштування]]**.
2. Перейдіть до **Додатки спільноти** та [встановіть Імпортер](obsidian://show-plugin?id=obsidian-importer).
3. Увімкніть додаток Імпортер.
4. Відкрийте додаток **Імпортер** за допомогою меню команд або значка на стрічковому меню.
5. У полі **File format** виберіть **Notion (.zip)**
6. Виберіть файл `.zip` з файлами Notion, які ви хочете імпортувати. *Рекомендується імпортувати всі дані Notion одразу, щоб внутрішні посилання були правильно узгоджені.*
7. _За бажанням_ виберіть теку для імпорту. Ваші сторінки та бази даних Notion будуть вкладені всередину цієї теки.
8. Увімкніть **Save parent pages in subfolders**, щоб зберегти структуру Notion. *Зверніть увагу, що в Notion можна писати вміст у теках, в Obsidian це неможливо, і ці сторінки будуть додані як підсторінки всередині теки.*
9. Натисніть **Import** та зачекайте завершення імпорту
10. Готово!

### Вирішення проблем

Якщо ви зіткнулися з проблемами під час імпорту з Notion:

- Переконайтеся, що ви використовуєте формат експорту **HTML** у Notion, **а не Markdown**.
- Якщо Obsidian зависає під час імпорту, вимкніть додатки спільноти та спробуйте знову.

Зіткнулися з чимось іншим? Перевірте [репозиторій Імпортера](https://github.com/obsidianmd/obsidian-importer/issues), щоб дізнатися, чи стикалися з цим інші користувачі.

#### Імпорт великих робочих просторів

Якщо ви імпортуєте робочий простір з кількома гігабайтами даних, експорт з Notion може містити вкладені файли `.zip`. У цьому випадку ви можете побачити повідомлення про помилку імпорту, яке виглядає приблизно так:

```
Import failed {id}.zip/{id}-Part-1.zip undefined.
```

Якщо ви бачите цю помилку, ви можете розпакувати файл з Notion, а потім імпортувати вкладені файли `Export-{id}-Part-1.zip`.
