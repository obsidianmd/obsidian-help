Ви можете налаштувати особистий домен або піддомен для вашого сайту [[Вступ до Obsidian Publish|Obsidian Publish]].

> [!warning] Увага
> На даний момент ми підтримуємо налаштування особистих доменів лише наступними методами:
>
> - [[#Налаштування за допомогою CloudFlare]] з використанням [режиму Full](https://developers.cloudflare.com/ssl/origin-configuration/ssl-modes/full/).
> - [[#Налаштування за допомогою проксі]]
> 
> Ми ще не маємо можливості створити SSL-сертифікат від вашого імені.

## Налаштування за допомогою CloudFlare

Найпростіший спосіб налаштувати особистий домен або піддомен — створити обліковий запис [CloudFlare](https://cloudflare.com) і дозволити CloudFlare керувати DNS вашого домену.

Наступні кроки використовують CloudFlare для налаштування особистого домену для вашого сайту Obsidian Publish, як для кореневого домену (`mysite.com`), так і для піддомену (`notes.mysite.com`).

> [!important] Важливо
> CloudFlare є **єдиним офіційно підтримуваним провайдером** для налаштування особистих доменів. Використання наступних інструкцій з будь-якими іншими провайдерами, ймовірно, не працюватиме.

**CloudFlare:**

1. Відкрийте Cloudflare для домену, на якому ви хочете розмістити свій сайт Publish, наприклад `mysite.com`, навіть якщо ви хочете піддомен типу `notes.mysite.com`.
2. Перейдіть до **DNS** і натисніть **Add Record**.
3. Виберіть **CNAME**.
4. У полі **name** введіть ваш домен або піддомен, наприклад `notes.mysite.com`.
5. У полі **target** введіть `publish-main.obsidian.md`. Не включайте ваш особистий під-URL у це значення. Obsidian Publish обробляє це з вашої конфігурації.
6. Переконайтеся, що **proxy status** увімкнено. Він має бути увімкнений за замовчуванням.
7. Перейдіть до **SSL/TLS** і встановіть режим шифрування SSL/TLS на "Full", щоб налаштувати сертифікат SSL/TLS автоматично.

> [!note] Примітка
> Щоб перенаправити і `mysite.com`, і `www.mysite.com` на Obsidian Publish, вам потрібно створити [Page Rule](https://support.cloudflare.com/hc/en-us/articles/200172336-Creating-Page-Rules) з наступними налаштуваннями:
>
> - URL match: `www.mysite.com/*`
> - Forward URL - 301 Permanent Redirect
> - Redirect URL: `https://mysite.com/$1`
>
> Після створення правила сторінки, створіть CNAME-запис для `www.mysite.com` так само, як ви зробили для `mysite.com`.

**Obsidian:**

1. Відкрийте Obsidian на вашому комп'ютері.
2. У [[Стрічкове меню|стрічковому меню]] зліва натисніть **Опублікувати зміни** ![[lucide-send.svg#icon]].
3. У розділі **Опублікувати зміни** виберіть **Налаштування сайту** ![[lucide-cog.svg#icon]].
4. Поруч із **Custom domain** виберіть **Налаштувати**.
5. У полі **Custom URL** введіть URL вашого домену або піддомену. Переконайтеся, що не вводите `www.` у полі особистого URL.

> [!note] Примітка
> Якщо налаштування вашого особистого домену призводить до циклу перенаправлення, ймовірно, режим шифрування в CloudFlare було встановлено на "Flexible" замість "Full".

## Налаштування за допомогою проксі

Ви також можете налаштувати SSL/TLS для вашого особистого домену, використовуючи власний вебсервер.

Якщо ви вже розміщуєте вебсайт на вашому домені або піддомені, ви також можете скористатися цим варіантом і налаштувати ваш вебсайт для завантаження вашого сайту Obsidian Publish за певним URL-шляхом, замість розміщення повного сайту.

Проксіруйте всі запити за цим URL-шляхом до `https://publish.obsidian.md/serve?url=mysite.com/my-notes/...` і налаштуйте параметри сайту в Obsidian на той самий URL-шлях, встановивши **Custom URL** на `mysite.com/my-notes`.

Ви також можете налаштувати Obsidian Publish як під-URL сайту, який вам належить. Наприклад, `https://mysite.com/my-notes/`. Для цього ви повинні розмістити власний сервер і проксірувати всі запити до нашого сервера за адресою `https://publish.obsidian.md/`.

Наступні приклади налаштування проксі не є вичерпними, але демонструють поширені методи для цієї реалізації.

### NGINX

У вашій конфігурації NGINX додайте наступне:

```nginx
location /my-notes {
  proxy_pass https://publish.obsidian.md/serve?url=mysite.com/my-notes/;
  proxy_ssl_server_name on;
  proxy_set_header Host publish.obsidian.md;
}
```


Деякі користувачі повідомляли, що може знадобитися додати `$request_uri` до proxy pass:

```nginx
location /my-notes {
  proxy_pass https://publish.obsidian.md/serve?url=mysite.com/my-notes$request_uri;
  proxy_ssl_server_name on;
  proxy_set_header Host publish.obsidian.md;
}
```

### Apache

У `.htaccess` додайте наступне:

```htaccess
RewriteEngine  on
RewriteRule    "^my-notes/(.*)$"  "https://publish.obsidian.md/serve?url=mysite.com/my-notes/$1"  [L,P]
```

> [!note] Примітка
> `mod_rewrite` повинен бути увімкнений, і вам також може знадобитися налаштувати [SSLProxyEngine](https://stackoverflow.com/questions/40938148/reverse-proxy-for-external-url-apache)

### Netlify

У `netlify.toml` [налаштуйте перенаправлення](https://docs.netlify.com/routing/redirects/#syntax-for-the-netlify-configuration-file):

```plain
[[redirects]]
  from = "https://mysite.com/my-notes/*"
  to = "https://publish.obsidian.md/serve?url=mysite.com/my-notes/:splat"
  status = 200
  force = true
```

### Vercel

У `vercel.json` [налаштуйте перезаписи](https://vercel.com/docs/configuration#project/rewrites):

```json
{
  ...

  "rewrites": [
    {
      "source": "/my-notes/",
      "destination": "https://publish.obsidian.md/serve?url=mysite.com/my-notes"
    },
    {
      "source": "/my-notes/:path*",
      "destination": "https://publish.obsidian.md/serve?url=mysite.com/my-notes/:path*"
    }
  ]
}
```

### Caddy

```plain
mysite.com {
  encode zstd gzip
  handle /my-notes* {
    reverse_proxy https://publish.obsidian.md {
      header_up Host {upstream_hostport}
    }
    rewrite * /serve?url=mysite.com{path}
  }
}
```

### Traefik

Цей мінімальний фрагмент конфігурації перенаправляє `mysite.com` на Obsidian Publish.
Дивіться [документацію Traefik](https://doc.traefik.io/traefik/routing/overview/)
для повного прикладу.

```yaml
http:
  routers:
    mysite:
      rule: Host(`mysite.com`)
      service: obsidian-publish
      middlewares:
        - "publish-headers"
  services:
    obsidian-publish:
      loadBalancer:
        servers:
          - url: https://publish.obsidian.md
  middlewares:
    publish-headers:
      headers:
        customRequestHeaders:
          Host: "publish.obsidian.md"
          x-obsidian-custom-domain: "mysite.com"
```

### Підтримувані HTTP X-Headers

Якщо ваш проксі-сервіс не дозволяє шляхи запитів, ви можете використовувати `https://publish.obsidian.md/` з користувацьким заголовком `x-obsidian-custom-domain`, встановленим на URL вашого сайту `mysite.com/my-subpath`.

## Перенаправлення старого сайту на особистий домен

Якщо ви хочете перенаправити відвідувачів зі старого сайту `publish.obsidian.md` на ваш новий особистий домен, увімкніть опцію **Пересилати на користувацький домен** під час налаштування вашого особистого домену.

## Вирішення проблем

Після налаштування особистого домену, якщо ви раніше відвідували свій сайт за попереднім посиланням `https://publish.obsidian.md/slug`, вам може знадобитися очистити кеш браузера, щоб деякі функції (такі як шрифти, графи або доступ за паролем) працювали належним чином. Це пов'язано з обмеженнями безпеки між доменами, які накладаються сучасними браузерами. Хороша новина полягає в тому, що читачі вашого сайту ніколи не зіткнуться з цією проблемою, якщо ви дозволите відвідувачам використовувати лише ваш особистий домен.
