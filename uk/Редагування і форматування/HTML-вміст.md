Obsidian підтримує HTML, щоб ви могли відображати свої нотатки так, як вам зручно, або навіть [[Вбудовування вебсторінок|вбудовувати вебсторінки]]. Дозвіл HTML всередині ваших нотаток пов'язаний із ризиками. Щоб запобігти шкоді від зловмисного коду, Obsidian _очищує_ будь-який HTML у ваших нотатках.

> [!example] 
> Елемент `<script>` зазвичай дозволяє запускати JavaScript при завантаженні. Якби Obsidian не очищував HTML, зловмисник міг би переконати вас вставити текст, що містить JavaScript, який витягує конфіденційну інформацію з вашого комп'ютера та надсилає її назад зловмиснику.

Тим не менш, оскільки синтаксис Markdown не підтримує всі форми стилізації, використання очищеного HTML може бути ще одним способом покращити якість ваших нотаток. Нижче наведено деякі найпоширеніші випадки використання HTML.

## Обмеження HTML

Obsidian має певні обмеження при використанні HTML у ваших нотатках:

### Markdown всередині HTML не працює

Obsidian не відображає синтаксис Markdown всередині HTML-елементів. Це навмисне дизайнерське рішення для оптимізації продуктивності та зменшення складності парсера при роботі з великими документами.

Наприклад, це не працюватиме як очікується:

```md
<div>
This **will not** be bold and this `will not` be code.
</div>
```

### HTML-блоки повинні бути самодостатніми

HTML-блоки повинні бути повними та не можуть містити порожніх рядків всередині. Порожні рядки розіб'ють HTML-блок.

Це працюватиме:

```md
<table>
<tr>
<td>Content here</td>
</tr>
</table>
```

Це не працюватиме коректно:

```md
<table>

<tr>

<td>Content here</td>

</tr>

</table>
```

### Коли Markdown здається, що працює в HTML

Деякі вбудовані HTML-теги, як-от `<span>` або `<a>`, мають обмежену функціональність і можуть виглядати так, ніби відображають Markdown, але насправді це не так. Markdown обробляється поза контекстом HTML.

Для отримання додаткової інформації про те, як Obsidian обробляє Markdown, див. [[Obsidian Flavored Markdown]].

## Поширені випадки використання HTML

> [!info] Більше деталей про використання `<iframe>` можна знайти у [[Вбудовування вебсторінок]].

### Коментарі

[[Базовий синтаксис форматування#Коментарі|Коментарі Markdown]] є рекомендованим способом додавання прихованих коментарів у ваших нотатках. Однак деякі методи конвертації нотаток Markdown, такі як [Pandoc](https://pandoc.org), мають обмежену підтримку коментарів Markdown. У таких випадках ви можете використовувати `<!-- HTML Comment -->` натомість!

### Підкреслення

Якщо вам потрібно швидко підкреслити елемент у ваших нотатках, ви можете використати `<u>Example</u>`, щоб створити <u>підкреслений текст</u>.

### Span/Div

Теги span та div можна використовувати для застосування користувацьких класів зі [[Сніпети CSS|сніпету CSS]] або власного стилю до вибраної ділянки тексту. Наприклад, використання `<span style="font-family: cursive">your text</span>` дозволяє швидко <span style="font-family: cursive">змінити шрифт</span>.

## Закреслення

Потрібно закреслити <s>якийсь текст</s>? Використовуйте `<s>this</s>`, щоб його закреслити.
