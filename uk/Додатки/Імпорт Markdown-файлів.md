Імпорт Markdown-файлів — це [[Вбудовані додатки|вбудований додаток]], який дозволяє конвертувати Markdown з інших застосунків у формат Obsidian. Він також дозволяє конвертувати певні [[Властивості]] в нові необхідні формати.

> [!warning] Увага
> Імпорт Markdown-файлів конвертує все ваше сховище відповідно до ваших налаштувань. [[Резервне копіювання файлів Obsidian|Створіть резервну копію ваших файлів Obsidian]] перед виконанням конвертації.

Щоб конвертувати всі нотатки у вашому сховищі:

1. У [[Меню команд|меню команд]] виберіть **Імпорт Markdown-файлів**. Цю функцію також можна знайти у [[Стрічкове меню|стрічковому меню]] за допомогою значка **Імпорт Markdown-файлів** ![[lucide-binary.svg#icon]].
2. Увімкніть формати, які потрібно конвертувати.
3. Натисніть **Почати конвертацію**.

Для отримання додаткової інформації зверніться до [[Базовий синтаксис форматування]].

## Підтримувані формати

### Roam Research

Імпорт Markdown-файлів може конвертувати наступний синтаксис Roam Research:

- **Мітки**: Конвертує `#tag` та `#[[tag]]` у `[[tag]]`
- **Виділення**: Конвертує `^^highlight^^` у `==highlight==`
- **Елементи TODO**: Конвертує `{{[[TODO]]}}` у `[ ]`

### Bear

Імпорт Markdown-файлів може конвертувати наступний синтаксис Bear:

- **Виділення**: Конвертує `::highlight::` у `==highlight==`

### Zettelkasten

Імпорт Markdown-файлів може конвертувати наступний синтаксис Zettelkasten:

- **Повні посилання**: Конвертує `[[UID]]` у `[[UID File Name]]`
- **Гарні посилання**: Конвертує `[[UID]]` у `[[UID File Name|File Name]]`

### [[Властивості]]

Починаючи з Obsidian `1.9.3`, Імпорт Markdown-файлів може конвертувати [[Властивості#Застарілі властивості|застарілі формати властивостей]] у поточний формат:

**Псевдоніми**

```yaml
# До

alias: My Note Title

# Після

aliases:
  - My Note Title
```

**Мітки**

```yaml
# До

tag: project, important

# Після

tags:
  - project
  - important
```

**CSS-класи**

```yaml
# До

cssclass: custom-style

# Після

cssclasses:
  - custom-style
```
