Формули дозволяють створювати обчислювані властивості в [[Вступ до Баз|Базах]], використовуючи дані з інших [[Властивості|властивостей]]. Ви можете виконувати обчислення, маніпулювати текстом, працювати з датами та багато іншого.

## Що можуть формули

Формули допоможуть вам:

- **Обчислювати значення** — додавати ціни, розраховувати підсумки або виконувати математичні операції.
- **Маніпулювати текстом** — об'єднувати рядки, змінювати регістр або витягувати підрядки.
- **Працювати з датами** — обчислювати різницю в часі, форматувати дати або визначати дедлайни.
- **Застосовувати логіку** — використовувати умовні оператори для відображення різних значень.
- **Обробляти списки** — фільтрувати, сортувати, перетворювати або агрегувати дані списків.

## Створення властивості-формули

Щоб створити властивість-формулу:

1. У вашій базі натисніть **Властивості** на панелі інструментів.
2. Натисніть **Додати формулу** внизу меню.
3. Введіть назву для вашої властивості-формули.
4. Введіть формулу в полі **Формула**.
5. Закрийте діалогове вікно.

Редактор формул автоматично доповнюватиме назви [[Функції|функцій]] і властивостей під час введення та перевірятиме синтаксис вашої формули. Зелена позначка з'являється, коли формула є коректною.

Після створення ви можете використовувати властивість-формулу як будь-яку іншу властивість у вашій базі. Додавайте її до [[Подання|подань]], використовуйте у фільтрах, сортуйте за нею тощо.

## Написання формули

У редакторі формул введіть вираз, використовуючи властивості, оператори та функції.

### Посилання на властивості

У формулах можна посилатися на різні типи властивостей:

- **Властивості нотатки** — властивості з [[Властивості|початкових метаданих]] нотатки.
- **Властивості файлу** — вбудовані властивості, такі як `file.name`, `file.size` або `file.mtime`.
- **Властивості-формули** — інші формули через `formula.formula_name`.

**Приклади:**

- `price * quantity` — множення двох властивостей нотатки
- `file.name + " - " + description` — об'єднання імені файлу з властивістю нотатки
- `formula.price_per_unit * 1.1` — використання іншої властивості-формули

### Використання операторів

**Арифметичні оператори** виконують математичні операції з числами:

- `price + tax` — додавання
- `price - discount` — віднімання
- `price * quantity` — множення
- `price / quantity` — ділення
- `(part / whole) * 100` — використання дужок для порядку операцій

**Оператори порівняння** порівнюють значення:

- `price > 100` — більше ніж
- `age < 18` — менше ніж
- `status == "Done"` — дорівнює
- `status != "Done"` — не дорівнює
- `file.mtime > now() - '7d'` — порівняння дат

**Логічні оператори** об'єднують логічні умови:

- `!completed` — заперечення
- `price > 0 && quantity > 0` — і
- `urgent || important` — або

Дізнатися більше в [[Синтаксис Бази#Оператори|Синтаксис Бази]].

### Використання функцій

Функції виконують операції над значеннями. Доступні функції залежать від типу значення, з яким ви працюєте. Переглянути повний перелік [[Функції|функцій]].

**Основні категорії функцій:**

- **Глобальні функції** — `if()`, `now()`, `date()`, `link()`, `max()`, `min()`
- **Рядкові функції** — `contains()`, `replace()`, `split()`, `lower()`, `title()`
- **Числові функції** — `round()`, `ceil()`, `floor()`, `abs()`, `toFixed()`
- **Функції дат** — `format()`, `relative()`, `date()`, `time()`
- **Функції списків** — `filter()`, `map()`, `sort()`, `join()`, `unique()`

**Приклади:**

- `if(price, "$" + price.toFixed(2), "")` — визначення умови з форматуванням числа.
- `file.name.lower()` — перетворення на нижній регістр.
- `tags.contains("urgent")` — перевірка, чи містить список міток певне значення.
- `due_date.format("YYYY-MM-DD")` — форматування дати.

## Приклади формул

### Обчислення дедлайну

Встановлення дати завершення проєкту через 2 тижні після дати початку:

```js
start_date + "2w"
```

### Відображення статусу прострочення

Показати "Overdue", якщо дата завершення минула, а статус не "Done":

```js
if(due_date < now() && status != "Done", "Overdue", "")
```

### Форматування валюти

Відображення ціни з 2 десятковими знаками та символом валюти:

```js
if(price, "$" + price.toFixed(2), "")
```

### Підрахунок елементів списку

Підрахунок кількості елементів у властивості-списку:

```js
tasks.length
```

### Обчислення пріоритетного балу

Поєднання кількох факторів у пріоритетний бал:

```js
(impact * urgency) / effort
```

### Об'єднання текстових полів

Створення повного імені з імені та прізвища:

```js
first_name + " " + last_name
```

### Обчислення загальної вартості

Множення щомісячної вартості на кількість місяців володіння:

```js
monthlyUses * formula.Owned.round()
```

## Типи даних

Формули працюють з різними типами даних:

- **Рядки** — текст в лапках: `"hello"` або `'world'`
- **Числа** — числові значення: `42`, `3.14`, `(2 + 2)`
- **Булеві значення** — істина або хиба: `true`, `false`
- **Дати** — створюються за допомогою `date()`, `today()` або `now()`
- **Списки** — колекції значень: `[1, 2, 3]`
- **Об'єкти** — пари ключ-значення: `{"name": "value"}`

Тип результату формули визначається використаними даними та функціями.

## Посилання на інші формули

Формули можуть посилатися на інші формули, створюючи похідні обчислення. Наприклад, якщо у вас є формула з назвою `price_per_unit`:

```js
price / quantity
```

Ви можете посилатися на неї в іншій формулі:

```js
formula.price_per_unit * 1.1
```

> [!warning] Уникайте циклічних посилань
> Формула не може посилатися на саму себе прямо або опосередковано через інші формули.
