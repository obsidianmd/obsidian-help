Коли ви [[Створити базу|створюєте базу]] в Obsidian, вона зберігається як файл `.base`. Бази зазвичай редагуються за допомогою інтерфейсу застосунку, але синтаксис також можна редагувати вручну та вбудовувати в блок коду.

Синтаксис [[Вступ до Баз|Баз]] визначає [[Подання]], фільтри та [[Формули|формули]]. Бази повинні бути валідним YAML, що відповідає схемі, описаній нижче.

## Приклад

Ось приклад файлу бази. Ми детально розглянемо кожен розділ.

```yaml
filters:
  or:
    - file.hasTag("tag")
    - and:
        - file.hasTag("book")
        - file.hasLink("Textbook")
    - not:
        - file.hasTag("book")
        - file.inFolder("Required Reading")
formulas:
  formatted_price: 'if(price, price.toFixed(2) + " dollars")'
  ppu: "(price / age).toFixed(2)"
properties:
  status:
    displayName: Status
  formula.formatted_price:
    displayName: "Price"
  file.ext:
    displayName: Extension
summaries:
  customAverage: 'values.mean().round(3)'
views:
  - type: table
    name: "My table"
    limit: 10
    groupBy:
      property: note.age
      direction: DESC
    filters:
      and:
        - 'status != "done"'
        - or:
            - "formula.ppu > 5"
            - "price > 2.1"
    order:
      - file.name
      - file.ext
      - note.age
      - formula.ppu
      - formula.formatted_price
    summaries:
      formula.ppu: Average
```

### Фільтри

За замовчуванням база включає кожен файл у сховищі. Тут немає `from` або `source`, як у SQL або Dataview. Розділ `filters` дозволяє визначати умови для звуження набору даних.

```yaml
# Простий фільтр:
filters:
  and:
    - file.hasTag("tag")

# Складний фільтр:
filters:
  or:
    - file.hasTag("tag")
    - and:
        - file.hasTag("book")
        - file.hasLink("Textbook")
    - not:
        - file.hasTag("book")
        - file.inFolder("Required Reading")
```

Є два місця для застосування фільтрів:

1. На глобальному рівні `filters` (показано вище), де вони застосовуються до всіх подань у базі.
2. На рівні `view`, де вони застосовуються лише до конкретного подання.

Ці два розділи функціонально еквівалентні, і при обчисленні для подання вони об'єднуються за допомогою `AND`.

Розділ `filters` містить або один оператор фільтра як рядок, або рекурсивно визначений об'єкт фільтра. Об'єкти фільтрів можуть містити один із ключів `and`, `or` або `not`. Ці ключі є гетерогенним списком інших об'єктів фільтрів або операторів фільтрів у рядках. Оператор фільтра — це рядок, який повертає істинне або хибне значення при застосуванні до нотатки. Він може бути одним із наступних:

- Базове порівняння з використанням стандартних арифметичних операторів.
- Функція. Доступна низка вбудованих [[Функції|функцій]], а додатки можуть додавати додаткові функції.

Синтаксис та доступні функції для фільтрів і формул однакові.

### Формули

Розділ `formulas` визначає [[Формули|властивості-формули]], які можна відображати в усіх поданнях файлу бази.

```yaml
formulas:
  formatted_price: 'if(price, price.toFixed(2) + " dollars")'
  ppu: "(price / age).toFixed(2)"
```

Властивості-формули підтримують базові арифметичні оператори та низку вбудованих [[Функції|функцій]]. У майбутньому додатки зможуть додавати функції для використання у формулах.

Ви можете посилатися на властивості різними способами залежно від їхнього типу:

- **Властивості нотатки** — це властивості, визначені в початкових метаданих нотатки. Наприклад, `note.price` або `note["price"]`.
  Якщо префікс не вказано, властивість вважається властивістю `note`.
- **Властивості файлу** описують сам файл.
  Наприклад, `file.size` або `file.ext`. Ви також можете посилатися на об'єкт файлу безпосередньо, наприклад, `file.hasLink()`.
- **Властивості-формули** — це інші формули в базі.
  Наприклад, `formula.formatted_price`.

Формула може використовувати значення з інших властивостей-формул, якщо немає циклічного посилання.

Властивості-формули завжди зберігаються як рядки в YAML, але їхній фактичний **тип вихідних даних** визначається типом базових даних та значенням, яке повертають використані функції.

Зверніть увагу, що вкладені лапки необхідні для включення текстових літералів у поле YAML. Текстові літерали повинні бути укладені в одинарні або подвійні лапки.

### Властивості

Розділ `properties` дозволяє зберігати конфігураційну інформацію про кожну властивість. Як саме використовувати ці значення конфігурації, залежить від конкретного подання. Наприклад, у таблицях відображувана назва використовується для заголовків стовпців.

```yaml
properties:
  status:
    displayName: Status
  formula.formatted_price:
    displayName: "Price"
  file.ext:
    displayName: Extension
```

Відображувані назви не використовуються у фільтрах або формулах.

### Підсумки

Розділ `summaries` можна використовувати для визначення власних формул підсумків. Окрім визначення формул підсумків тут, доступні також кілька стандартних формул підсумків.

```yaml
summaries:
  customAverage: 'values.mean().round(3)'
```

У цьому прикладі формула `customAverage` аналогічна стандартному `Average`, за винятком того, що значення округлюється до іншої кількості знаків. У формулах підсумків ключове слово `values` — це список, що містить усі значення цієї властивості для кожної нотатки в результуючому наборі. Формула підсумку повинна повертати одне `Value`.

Зверніть увагу, що цей розділ `summaries` відрізняється від розділу `summaries` у конфігурації подання (пояснення нижче), де формули підсумків призначаються конкретним властивостям.

#### Стандартні формули підсумків

| Назва     | Тип вхідних даних | Опис                                                          |
| --------- | ------------------ | ------------------------------------------------------------- |
| Average   | Число              | Середнє арифметичне всіх чисел із вхідних значень.            |
| Min       | Число              | Найменше число з вхідних значень.                             |
| Max       | Число              | Найбільше число з вхідних значень.                            |
| Sum       | Число              | Сума всіх чисел у вхідних даних.                              |
| Range     | Число              | Різниця між `Max` і `Min`.                                    |
| Median    | Число              | Медіана всіх чисел із вхідних значень.                        |
| Stddev    | Число              | Стандартне відхилення всіх чисел із вхідних значень.           |
| Earliest  | Дата               | Найраніша дата з вхідних значень.                             |
| Latest    | Дата               | Найпізніша дата з вхідних значень.                            |
| Range     | Дата               | Різниця між `Latest` і `Earliest`.                            |
| Checked   | Логічний           | Кількість значень `true`.                                     |
| Unchecked | Логічний           | Кількість значень `false`.                                    |
| Empty     | Будь-який          | Кількість порожніх значень у вхідних даних.                   |
| Filled    | Будь-який          | Кількість непорожніх значень у вхідних даних.                 |
| Unique    | Будь-який          | Кількість унікальних значень у вхідних даних.                 |

### Подання

Розділ `views` визначає, як дані можуть бути відображені. Кожен запис у списку `views` визначає окреме подання тих самих даних, і різних подань може бути стільки, скільки потрібно.

```yaml
views:
  - type: table
    name: "My table"
    limit: 10
    groupBy:
      property: note.age
      direction: DESC
    filters:
      and:
        - 'status != "done"'
        - or:
            - "formula.ppu > 5"
            - "price > 2.1"
    order:
      - file.name
      - file.ext
      - note.age
      - formula.ppu
      - formula.formatted_price
    summaries:
      formula.ppu: Average
```

- `type` обирає серед вбудованих та доданих додатками типів подань.
- `name` — це відображувана назва, яка може використовуватися для визначення подання за замовчуванням.
- `filters` повністю аналогічні описаним вище, але застосовуються лише до подання.
- `groupBy` вказує властивість і напрямок сортування. Значення вказаної властивості для кожного рядка використовується для розподілу рядків по групах.
- `summaries` зіставляє назви властивостей із іменованим підсумком. Підсумки виконують агрегацію властивості по всіх рядках.

[[Подання]] можуть додавати додаткові дані для зберігання будь-якої інформації, необхідної для підтримки стану або правильного відображення, однак автори додатків повинні уникати використання ключів, які вже використовуються основним додатком Бази. Наприклад, подання таблиці може використовувати це для обмеження кількості рядків або для вибору стовпця сортування та його напрямку. Інший тип подання, наприклад карта, може використовувати це для зіставлення властивості нотатки з широтою і довготою та визначення властивості, яка відображається як назва маркера.

У майбутньому API дозволить поданням читати та записувати ці значення, що дасть змогу поданню створювати власний інтерфейс для конфігурації.

## Властивості

У базах використовуються три види властивостей:

1. **Властивості нотатки**, що зберігаються в початкових метаданих файлів Markdown.
2. **Властивості файлу**, доступні для всіх типів файлів.
3. **Властивості-формули**, визначені в самому файлі `.base` (див. вище).

### Властивості нотатки

[[Властивості|Властивості нотатки]] доступні лише для файлів Markdown і зберігаються в YAML-початкових метаданих кожної нотатки. Доступ до цих властивостей здійснюється у форматі `note.author` або просто `author` як скорочення.

### Властивості файлу

Властивості файлу стосуються файлу, який наразі перевіряється або обчислюється. Властивості файлу доступні для всіх [[Підтримувані формати файлів|типів файлів]], включаючи вкладення.

Наприклад, фільтр `file.ext == "md"` буде істинним для всіх файлів Markdown і хибним в інших випадках.

| Властивість   | Тип    | Опис                                                          |
| ------------- | ------ | ------------------------------------------------------------- |
| `file.backlinks`  | Список | Список файлів зворотних посилань. Примітка: ця властивість вимоглива до продуктивності. За можливості зверніть пошук і використовуйте `file.links`. Не оновлює результати автоматично при зміні сховища. |
| `file.ctime`  | Дата   | Час створення                                                 |
| `file.embeds` | Список | Список усіх вбудувань у нотатці                               |
| `file.ext`    | Рядок  | Розширення файлу                                              |
| `file.file`   | Файл   | Об'єкт файлу, використовується лише в певних функціях         |
| `file.folder` | Рядок  | Шлях до теки файлу                                            |
| `file.links`  | Список | Список усіх внутрішніх посилань у нотатці, включаючи початкові метадані |
| `file.mtime`  | Дата   | Час змінення                                                  |
| `file.name`   | Рядок  | Ім'я файлу                                                    |
| `file.path`   | Рядок  | Шлях до файлу                                                 |
| `file.properties`   | Об'єкт | Усі властивості файлу. Примітка: не оновлює результати автоматично при зміні сховища. |
| `file.size`   | Число  | Розмір файлу                                                  |
| `file.tags`   | Список | Список усіх міток у вмісті файлу та початкових метаданих      |

### Доступ до властивостей через `this`

Використовуйте об'єкт `this` для доступу до властивостей файлу. На що посилається `this`, залежить від місця відображення бази.

Коли база відкрита в основній області вмісту, `this` вказує на властивості самого файлу бази. Наприклад, використання `this.file.folder` повертає шлях до теки, де розташована база.

Коли база вбудована в інший файл, `this` вказує на властивості _файлу-вбудовувача_ (нотатки або полотна, що містить базу). Наприклад, використання `this.file.name` повертає ім'я файлу-вбудовувача, а не бази.

Коли база знаходиться на бічній панелі, `this` посилається на активний файл в основній області вмісту. Це дозволяє створювати запити на основі активного файлу. Наприклад, ви можете використовувати `file.hasLink(this.file)` для відтворення панелі зворотних посилань.

## Оператори

### Арифметичні оператори

Арифметичні оператори виконують арифметичні дії над числами. Наприклад, `radius * (2 * 3.14)`.

| Оператор | Опис        |
| -------- | ----------- |
| `+`      | додавання   |
| `-`      | віднімання  |
| `*`      | множення    |
| `/`      | ділення     |
| `%`      | остача      |
| `( )`    | дужки       |

### Арифметика дат

Дати можна модифікувати, додаючи та віднімаючи тривалості. Одиниці тривалості приймають кілька форматів:

| Одиниця                  | Тривалість |
| ------------------------ | ---------- |
| `y`, `year`, `years`     | рік        |
| `M`, `month`, `months`   | місяць     |
| `d`, `day`, `days`       | день       |
| `w`, `week`, `weeks`     | тиждень    |
| `h`, `hour`, `hours`     | година     |
| `m`, `minute`, `minutes` | хвилина    |
| `s`, `second`, `seconds` | секунда    |

Для модифікації або зміщення об'єктів Date використовуйте оператор `+` або `-` із рядком тривалості. Наприклад, `date + "1M"` додає 1 місяць до дати, а `date - "2h"` віднімає 2 години від дати.

Глобальна [[Функції|функція]] `today()` може використовуватися для отримання поточної дати, а `now()` — для отримання поточної дати з часом.

- `now() + "1 day"` повертає дату й час рівно через 24 години від моменту виконання.
- `file.mtime > now() - "1 week"` повертає `true`, якщо файл був змінений протягом останнього тижня.
- `date("2024-12-01") + "1M" + "4h" + "3m"` повертає об'єкт Date, що представляє `2025-01-01 04:03:00`.
- Відніміть дві дати, щоб отримати різницю в мілісекундах між ними, наприклад, `now() - file.ctime`.
- Щоб отримати лише дату з Date із часом, використовуйте `datetime.date()`.
- Щоб відформатувати об'єкт Date, використовуйте функцію `format()`, наприклад `datetime.format("YYYY-MM-DD")`.

### Оператори порівняння

Оператори порівняння можна використовувати для порівняння чисел або об'єктів Date. Оператори рівності та нерівності можна використовувати з будь-яким типом значень, не лише з числами та датами.

| Оператор | Опис                     |
| -------- | ------------------------ |
| `==`     | дорівнює                 |
| `!=`     | не дорівнює              |
| `>`      | більше ніж               |
| `<`      | менше ніж                |
| `>=`     | більше або дорівнює      |
| `<=`     | менше або дорівнює       |

### Логічні оператори

Логічні оператори можна використовувати для об'єднання або інвертування логічних значень, результатом чого є значення «істина» або «хибність».

| Оператор | Опис          |
| -------- | ------------- |
| `!`      | логічне НЕ    |
| `&&`     | логічне І     |
| \|\|     | логічне АБО   |

## Функції

## Типи

Бази мають систему типів, яку формули та фільтри використовують для застосування функцій до властивостей.

### Рядки, числа та логічні значення

Рядки, числа та логічні значення — це «примітивні» значення, які не потребують функції для створення.

- Рядки укладаються в одинарні або подвійні лапки, наприклад `"message"`.
- Числа записуються цифрами і за бажанням можуть бути укладені в дужки для наочності. Наприклад, `1` або `(2.5)`.
- Логічні значення записуються як `true` або `false` без лапок.

### Дати та тривалості

Дати представляють конкретну дату або дату з часом залежно від функції, використаної для їх створення, або типу, призначеного [[Властивості|властивості]].

- Щоб створити дату, використовуйте функцію `date`, наприклад `date("2025-01-01 12:00:00")`
- Щоб модифікувати дату, додайте або відніміть тривалість, наприклад `now() + "1 hour"` або `today() + "7d"`
- Порівнюйте дати за допомогою операторів порівняння (наприклад, `>` або `<`) та арифметичних операторів (наприклад, `(now() + "1d") - now()` повертає `86400000` мілісекунд.)
- Щоб витягти частини дати, використовуйте доступні поля (`now().hour`) або допоміжні функції (`now.time()`).
- Для об'єктів дат доступні багато інших [[Функції|полів та функцій]].

### Об'єкти та списки

- Перетворіть один елемент у список за допомогою функції `list()`. Це особливо корисно для властивостей, які можуть містити суміш списків або окремих значень.
- Доступ до елементів списку здійснюється за допомогою квадратних дужок і індексу, починаючи з 0. Наприклад, `property[0]` повертає перший елемент зі списку.
- Доступ до елементів об'єкта здійснюється за допомогою квадратних дужок з іменем елемента або крапкової нотації. Наприклад, `property.subprop` або `property["subprop"]`.

### Файли та посилання

[[Зв'язування нотаток|Wiki-посилання]] у [[Властивості|властивостях початкових метаданих]] автоматично розпізнаються як об'єкти Link. Посилання відображатимуться як клікабельні посилання в [[Подання|поданні]].

- Щоб створити посилання, використовуйте глобальну [[Функції|функцію]] `link`, наприклад `link("filename")` або `link("https://obsidian.md")`.
- Ви можете створити посилання з будь-якого рядка, наприклад `link(file.ctime.date().toString())`.
- Щоб задати текст відображення, передайте необов'язковий рядок або значок як другий параметр, наприклад `link("filename", "display")` або `link("filename", icon("plus"))`.

Об'єкт File можна перетворити на посилання за допомогою `file.asLink()` з необов'язковим текстом відображення.

Посилання можна порівнювати за допомогою `==` та `!=`. Вони еквівалентні, якщо вказують на один і той самий файл, або якщо файл не існує при перевірці — їхній текст посилання повинен бути ідентичним.

Посилання можна порівнювати з файлами, такими як `file` або `this`. Вони будуть рівними, якщо посилання вказує на цей файл. Наприклад, `author == this`.

Посилання також можна перевіряти у списках за допомогою contains, наприклад `authors.contains(this)`.
