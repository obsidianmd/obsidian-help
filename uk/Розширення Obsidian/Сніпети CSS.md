Дізнайтеся, як змінити аспекти зовнішнього вигляду застосунку Obsidian без необхідності [створювати тему](https://docs.obsidian.md/Themes/App+themes/Build+a+theme).

> [!tip] Якщо ви шукаєте інструкції щодо роботи з CSS для [[Вступ до Obsidian Publish|Obsidian Publish]], обов'язково перегляньте [[Налаштування вашого сайту]].

CSS — це мова, яка контролює зовнішній вигляд HTML. Додаючи сніпети CSS, ви можете змінювати частини інтерфейсу Obsidian, наприклад розмір і колір заголовків. Obsidian має [змінні CSS](https://docs.obsidian.md/Reference/CSS+variables/CSS+variables), які допомагають легко налаштувати інтерфейс.

Obsidian шукає сніпети CSS всередині [[Тека конфігурації|теки конфігурації]] сховища.

## Додавання сніпета

Щоб додати сніпет CSS на **десктопі** ![[lucide-monitor-check.svg#icon]], виконайте ці кроки:

1. Відкрийте **[[Налаштування]]** ![[lucide-settings.svg#icon]].
2. У розділі **Оформлення → Сніпети (користувацький CSS)** натисніть **Відкрити теку зі сніпетами** ![[lucide-folder-open.svg#icon]].
3. У теці сніпетів створіть CSS-файл, що містить ваш сніпет.
4. В Obsidian у розділі **Оформлення → Сніпети (користувацький CSS)** натисніть **Оновити список сніпетів** ![[lucide-refresh-cw.svg#icon]], щоб побачити сніпет у списку.
5. Увімкніть сніпет, натиснувши перемикач.

Щоб додати сніпет CSS на **мобільному пристрої/планшеті** ![[obsidian-icon-smartphone.svg#icon]], виконайте ці кроки:

1. Відкрийте файловий менеджер і знайдіть своє сховище. Ви можете перевірити розташування сховища в _Керувати сховищами…_, натиснувши на своє сховище та переглянувши шлях.
2. Відкрийте [[Тека конфігурації|теку конфігурації]] та створіть теку з назвою `snippets`, якщо вона не існує.
3. Додайте ваш сніпет CSS до цієї теки.
4. Відкрийте **[[Налаштування]]** ![[lucide-settings.svg#icon]] Obsidian.
5. Виберіть **Оформлення** зліва.
6. Прокрутіть вниз до розділу **Сніпети (користувацький CSS)**.
7. Натисніть **Оновити список сніпетів** ![[lucide-refresh-cw.svg#icon]], щоб оновити список.
8. Натисніть перемикач, щоб увімкнути сніпет.

Крім того, ви можете:
- [[Синхронізація нотаток між пристроями|Синхронізувати]] будь-які зміни за допомогою вашого сервісу синхронізації.
- Використати додаток спільноти для створення сніпета безпосередньо в Obsidian.

Після увімкнення Obsidian автоматично виявлятиме зміни в сніпетах CSS і застосовуватиме їх при збереженні файлу.

> [!tip] Вам не потрібно перезапускати Obsidian, щоб зміни набули чинності. Однак вам може знадобитися використати команду [[Меню команд|меню команд]] для перезавантаження Obsidian без збереження, щоб побачити зміни в поточній темі або нотатці.

## Написання CSS для Obsidian

Obsidian пропонує кілька методів, що спрощують і роблять потужнішим написання CSS.

Він має безліч [змінних CSS](https://docs.obsidian.md/Reference/CSS+variables/CSS+variables) для легкої зміни частин Obsidian та вбудований [[Властивості#Тип властивості|тип властивості]] для зміни зовнішнього вигляду однієї або кількох нотаток.

> [!example] Змінні
> Створіть файл з назвою `headers.css` із наступним вмістом, щоб змінити кольори шести [[Базовий синтаксис форматування#Заголовки|рівнів заголовків]] на веселку:
>
> ```css
> body {
>   --h1-color: red;
>   --h2-color: orange;
>   --h3-color: yellow;
>   --h4-color: green;
>   --h5-color: blue;
>   --h6-color: pink;
> }
> ```

> [!example] CSS-класи
> Призначте назву користувацького CSS-класу (або список CSS-класів) попередньо визначеній [[Властивості|властивості]] `cssclasses`, щоб одна або кілька нотаток виглядали інакше, ніж інші.
> 
> **CSS**:
> ```css
> .red-border img {
>    border-color: #ff0000;
>    border-style: solid;
> }
> ```
> 
> **YAML/Властивості**:
> ```yaml
> cssclasses:
>  - red-border
> ```
> 
> У кожній нотатці, що містить значення `red-border` у властивості `cssclasses`, зображення відображатимуться з червоною рамкою.

Щоб переконатися, що CSS-файл є валідним і правильно відформатованим, рекомендуємо перевірити його за допомогою такого інструменту, як [CSS Validation Service](https://jigsaw.w3.org/css-validator/), оскільки невалідний CSS не працюватиме.
