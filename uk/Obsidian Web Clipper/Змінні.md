[[Obsidian Web Clipper/Шаблони|Шаблони Web Clipper]] можуть використовувати змінні для автоматичного попереднього заповнення даних зі сторінки в шаблоні. Змінні можна використовувати в **назві нотатки**, **розташуванні нотатки**, **властивостях** і **вмісті нотатки**. Змінні також можна модифікувати за допомогою [[Фільтри|фільтрів]].

Скористайтеся значком `...` у розширенні [[Вступ до Obsidian Web Clipper|Web Clipper]], щоб переглянути змінні поточної сторінки для використання в шаблонах. Є п'ять типів змінних, які ви можете використовувати:

- [[Змінні#Пресетні змінні|Пресетні змінні]]
- [[Змінні#Змінні-промпти|Змінні-промпти]]
- [[Змінні#Мета-змінні|Мета-змінні]]
- [[Змінні#Селекторні змінні|Селекторні змінні]]
- [[Змінні#Змінні Schema.org|Змінні Schema.org]]

## Пресетні змінні

Пресетні змінні автоматично генеруються на основі вмісту сторінки. Зазвичай вони працюють для більшості вебсайтів.

Основна змінна вмісту — `{{content}}`, яка містить вміст статті, або [[Маркер|виділення]], або вибраний текст на сторінці, якщо такий є. Зверніть увагу, що `{{content}}` намагається витягнути основний вміст сторінки, що не завжди може бути тим, що вам потрібно. У такому випадку ви можете використовувати інші пресетні змінні або селекторні змінні для вилучення потрібного вмісту.

| Змінна              | Опис                                                                                   |
| ------------------- | -------------------------------------------------------------------------------------- |
| `{{author}}`        | Автор сторінки                                                                         |
| `{{content}}`       | Вміст статті, [[Маркер\|виділення]] або вибраний текст у форматі Markdown |
| `{{contentHtml}}`   | Вміст статті, [[Маркер\|виділення]] або вибраний текст у форматі HTML   |
| `{{date}}`          | Поточна дата, можна відформатувати за допомогою фільтра `date`                         |
| `{{description}}`   | Опис або уривок                                                                        |
| `{{domain}}`        | Домен                                                                                  |
| `{{favicon}}`       | URL фавікону                                                                           |
| `{{fullHtml}}`      | Необроблений HTML повного вмісту сторінки                                              |
| `{{highlights}}`    | [[Маркер\|Виділення]] з текстом і мітками часу                          |
| `{{image}}`         | URL зображення для соціальних мереж                                                    |
| `{{published}}`     | Дата публікації, можна відформатувати за допомогою фільтра `date`                      |
| `{{selection}}`     | Вибраний текст у форматі Markdown                                                     |
| `{{selectionHtml}}` | Вибраний текст у форматі HTML                                                         |
| `{{site}}`          | Назва сайту або видавець                                                               |
| `{{title}}`         | Назва сторінки                                                                         |
| `{{time}}`          | Поточна дата й час                                                                     |
| `{{url}}`           | Поточний URL                                                                           |
| `{{words}}`         | Лічильник слів                                                                         |

## Змінні-промпти

Змінні-промпти використовують мовні моделі для вилучення та модифікації даних за допомогою природної мови. Змінні-промпти вимагають увімкнення й налаштування [[Інтерпретатор|Інтерпретатора]].

Змінні-промпти використовують синтаксис `{{"підсумок сторінки"}}`. Подвійні лапки навколо промпту є важливими й відрізняють промпти від пресетних змінних. Відповіді промптів можна додатково обробити за допомогою [[Фільтри|фільтрів]], наприклад `{{"підсумок сторінки"|blockquote}}`.

### Коли використовувати змінні-промпти

Змінні-промпти мають перевагу надзвичайної гнучкості та простоти написання, однак вони мають кілька компромісів: вони повільніші у виконанні й можуть мати обмеження щодо вартості та конфіденційності залежно від обраного [[Інтерпретатор#Моделі|провайдера]].

На відміну від інших типів змінних, змінні-промпти потребують обробки зовнішньою мовною моделлю, тому вони замінюються лише після запуску [[Інтерпретатор|Інтерпретатора]].

Найкраще *не* використовувати змінні-промпти, якщо дані, які ви хочете витягнути, мають сталий формат, який можна витягнути іншими типами змінних.

З іншого боку, змінні-промпти можуть бути корисними, якщо дані, які ви хочете витягнути, мають *несталий* формат на різних вебсайтах. Наприклад, ви можете створити [[Obsidian Web Clipper/Шаблони|шаблон]] для збереження книг, який не залежить від конкретного книжкового сайту. Змінні-промпти на кшталт `{{"автор книги"}}` працюватимуть на будь-якому книжковому сайті, тоді як селекторні змінні зазвичай працюють лише для одного сайту.

### Приклади

Промпти можуть використовувати майже будь-який запит природною мовою. Залежно від моделі, яку ви використовуєте, промпти можуть запитувати або перекладати дані різними мовами.

- `{{"підсумок у трьох пунктах, перекладений французькою"}}` — для вилучення ключових пунктів про сторінку та перекладу їх французькою.
- `{{"un resumé de la page en trois points"}}` — для вилучення трьох ключових пунктів за допомогою промпту французькою.

Промпти можуть перетворювати вміст сторінки в JSON, який можна маніпулювати за допомогою [[Фільтри|фільтрів]]. Наприклад:

```
{{"return a JSON object for each tweet, that includes the author, tweet_text, date in YYYY-MM-DD format, and images array (if there are any)"|map:tweet => ({text: tweet.tweet_text, author: tweet.author, date: tweet.date})|template:"${text}\n— [[@${author}]], [[${date}]]\n"}}
```


## Мета-змінні

Мета-змінні дозволяють витягувати дані з [мета-елементів](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/meta) на сторінці, включаючи дані [Open Graph](https://ogp.me/), що використовуються для заповнення попереднього перегляду посилань у соціальних мережах.

- `{{meta:name}}` повертає вміст мета-тегу name із зазначеним ім'ям, наприклад `{{meta:name:description}}` для мета-тегу `description`.
- `{{meta:property}}` повертає вміст мета-тегу property із зазначеною властивістю, наприклад `{{meta:property:og:title}}` для мета-тегу `og:title`.

## Селекторні змінні

Селекторні змінні дозволяють витягувати текстовий вміст з елементів на сторінці за допомогою [CSS-селекторів](https://developer.mozilla.org/en-US/docs/Web/CSS/CSS_selectors/Selectors_and_combinators).

Синтаксис: `{{selector:cssSelector?attribute}}`, де `?attribute` є необов'язковим. Якщо атрибут не вказано, повертається текстовий вміст елемента. Ви також можете використовувати `{{selectorHtml:cssSelector}}`, щоб отримати HTML-вміст елемента. Селекторні змінні найкраще працюють на конкретному вебсайті або наборі вебсайтів зі сталою HTML-структурою.

- `{{selector:h1}}` повертає текстовий вміст усіх елементів `h1` на сторінці.
- `{{selector:.author}}` повертає текстовий вміст усіх елементів `.author` на сторінці.
- `{{selector:img.hero?src}}` повертає атрибут `src` зображення з класом `hero`.
- `{{selector:a.main-link?href}}` повертає атрибут `href` тегу посилання з класом `main-link`.
- `{{selectorHtml:body|markdown}}` повертає весь HTML елемента `body`, перетворений у Markdown за допомогою [[Фільтри#Обробка HTML|фільтра]] `markdown`.
- Вкладені CSS-селектори та комбінатори підтримуються, якщо вам потрібна більша специфічність.
- Якщо кільком елементам відповідає селектор, повертається масив, який можна обробити за допомогою [[Фільтри#Масиви та об'єкти|фільтрів масивів та об'єктів]], таких як `join` або `map`.

Селекторні змінні також можна використовувати безпосередньо в [[Логіка|логіці шаблонів]]:

- У циклах: `{% for comment in selector:.comment %}...{% endfor %}`
- В умовних виразах: `{% if selector:.premium-badge %}...{% endif %}`
- У присвоєнні змінних: `{% set items = selector:.list-item %}`

## Змінні Schema.org

Змінні schema дозволяють витягувати дані зі [schema.org](https://schema.org/) JSON-LD на сторінці. Дані schema.org також можуть використовуватися для автоматичного [[Obsidian Web Clipper/Шаблони#Відповідність Schema.org|запуску шаблону]].

- `{{schema:@Type:key}}` повертає значення ключа зі схеми.
- `{{schema:@Type:parent.child}}` повертає значення вкладеної властивості.
- `{{schema:@Type:arrayKey}}` повертає перший елемент масиву.
- `{{schema:@Type:arrayKey[index].property}}` повертає елемент за зазначеним індексом у масиві.
- `{{schema:@Type:arrayKey[*].property}}` повертає конкретну властивість з усіх елементів масиву.

Ви також можете використовувати скорочений запис без зазначення типу схеми:

- `{{schema:author}}` знайде першу властивість `author` у будь-якому типі схеми.
- `{{schema:name}}` знайде першу властивість `name` у будь-якому типі схеми.

Цей скорочений запис особливо корисний, коли ви не знаєте або не переймаєтесь конкретним типом схеми, але знаєте назву властивості, яку шукаєте.

Вкладені властивості та доступ до масивів також працюють як із зазначенням `@Type` схеми, так і без нього:

- `{{schema:author.name}}` знайде першу властивість `author` і потім отримає її підвластивість `name`.
- `{{schema:author[0].name}}` отримає `name` першого автора в масиві авторів.
- `{{schema:author[*].name}}` поверне масив усіх імен авторів.
