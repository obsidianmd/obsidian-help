[[Вступ до Obsidian Web Clipper|Web Clipper]] підтримує логіку шаблонів для умовних операторів, циклів та присвоєння змінних. Цей синтаксис натхненний мовами шаблонів [Twig](https://twig.symfony.com/) та [Liquid](https://shopify.github.io/liquid/).

> [!warning] Потрібна остання версія
> Функції логіки вимагають Obsidian Web Clipper 1.0.0, який ще не затверджений у всіх магазинах розширень.

## Умовні оператори

Використовуйте `{% if %}` для умовного включення вмісту на основі змінних або виразів.

```twig
{% if author %}
Author: {{author}}
{% endif %}
```

Використовуйте `{% else %}` для надання резервного вмісту, та `{% elseif %}` для створення ланцюжків кількох умов:

```twig
{% if status == "published" %}
Live article
{% elseif status == "draft" %}
Draft article
{% else %}
Unknown status
{% endif %}
```

### Оператори порівняння

Підтримуються наступні оператори порівняння:

| Оператор | Опис |
|----------|-------------|
| `==` | Дорівнює |
| `!=` | Не дорівнює |
| `>` | Більше ніж |
| `<` | Менше ніж |
| `>=` | Більше або дорівнює |
| `<=` | Менше або дорівнює |
| `contains` | Перевіряє, чи містить рядок підрядок, або масив містить значення |

Приклади:
- `{% if title == "Home" %}` — рівність рядків
- `{% if price >= 100 %}` — числове порівняння
- `{% if title contains "Review" %}` — перевірка підрядка
- `{% if tags contains "important" %}` — належність до масиву

### Логічні оператори

Поєднуйте умови за допомогою логічних операторів:

| Оператор | Альтернатива | Опис                                |
| -------- | ------------ | ----------------------------------- |
| `and`    | `&&`         | Обидві умови мають бути істинними   |
| `or`     | \|\|         | Хоча б одна умова має бути істинною |
| `not`    | `!`          | Заперечує умову                     |

Приклади:
- `{% if author and published %}` — обидва мають існувати
- `{% if draft or archived %}` — будь-яка умова
- `{% if not hidden %}` — заперечення
- `{% if (premium or featured) and published %}` — згруповані умови

### Істинність

Коли змінна використовується без оператора порівняння, вона оцінюється на «істинність»:

- `false`, `null`, `undefined`, порожній рядок `""` та `0` вважаються **хибними**.
- Порожні масиви `[]` вважаються **хибними**.
- Все інше є **істинним**.

```twig
{% if content %}
Has content
{% endif %}
```

## Присвоєння змінної

Використовуйте `{% set %}` для створення або зміни змінних у вашому шаблоні:

```twig
{% set slug = title|lower|replace:" ":"-" %}
File: {{slug}}.md
```

Змінним можна присвоювати:
- Інші змінні: `{% set name = author %}`
- Літерали: `{% set count = 5 %}` або `{% set label = "Draft" %}`
- Вирази з фільтрами: `{% set excerpt = content|truncate:100 %}`
- Результати селекторів: `{% set comments = selector:.comment %}`

Змінні, задані через `{% set %}`, можуть використовуватися в наступній логіці шаблону та у виведенні `{{variable}}`.

## Резервні значення

Використовуйте оператор `??` для надання резервних значень, коли змінна порожня або не визначена:

```twig
{{title ?? "Untitled"}}
```

Якщо `title` порожній, не визначений або хибний, замість нього буде використано резервне значення `"Untitled"`.

Це скорочена форма еквівалентного оператора `if`:

```twig
{% if title %}{{title}}{% else %}Untitled{% endif %}
```

### Ланцюжки резервних значень

Ви можете створювати ланцюжки з кількох резервних значень:

```twig
{{title ?? headline ?? "No title"}}
```

Це використає `title`, якщо доступний, інакше `headline`, інакше рядок `"No title"`.

### З фільтрами

Фільтри мають вищий пріоритет ніж `??`, тому фільтри застосовуються перед перевіркою резервного значення:

```twig
{{title|upper ?? "UNTITLED"}}
```

Це спочатку застосовує `upper` до `title`, потім повертає `"UNTITLED"`, якщо результат порожній. Щоб застосувати фільтри до резервного значення, використовуйте дужки або окремі вирази:

```twig
{{title ?? "Untitled"|lower}}
```

Це використає `title`, якщо доступний, інакше застосує `lower` до резервного значення, що дасть `"untitled"`.

## Цикли

Використовуйте `{% for %}` для ітерації по масивах:

```twig
{% for item in schema:author %}
- {{item.name}}
{% endfor %}
```

### Джерела циклів

Ви можете ітерувати по:
- Масивах схеми: `{% for item in schema:author %}`
- Результатах селекторів: `{% for comment in selector:.comment %}`
- Раніше заданих змінних: `{% set items = selector:.item %}{% for item in items %}`

### Змінні циклу

Усередині циклу ви маєте доступ до об'єкта `loop` з наступними властивостями:

| Змінна | Опис |
|----------|-------------|
| `loop.index` | Поточна ітерація (індексація з 1) |
| `loop.index0` | Поточна ітерація (індексація з 0) |
| `loop.first` | `true`, якщо перша ітерація |
| `loop.last` | `true`, якщо остання ітерація |
| `loop.length` | Загальна кількість елементів |

```twig
{% for tag in tags %}
{{loop.index}}. {{tag}}
{% if loop.last %} (end of list){% endif %}
{% endfor %}
```

Для зворотної сумісності ви також можете використовувати `item_index` (де `item` — назва вашої змінної ітератора) для отримання позиції з індексацією від 0:

```twig
{% for tag in tags %}
{{tag_index}}. {{tag}}
{% endfor %}
```

### Доступ до елементів масиву за індексом

Використовуйте дужкову нотацію для доступу до елементів масиву за індексом:

```twig
{{items[0]}}
{{items[loop.index0]}}
```

Це корисно, коли потрібно отримати доступ до елементів з кількох масивів паралельно:

```twig
{% set transcripts = selector:.transcript-text %}
{% set timestamps = selector:.timestamp %}

{% for line in transcripts %}
{{timestamps[loop.index0]}} - {{line}}
{% endfor %}
```

Дужкова нотація також працює з властивостями об'єктів:

```twig
{{user["name"]}}
{{data["my-key"]}}
```

### Вкладені цикли

Цикли можуть бути вкладеними для складних структур даних:

```twig
{% for section in sections %}
## {{section.title}}
{% for item in section.items %}
- {{item}}
{% endfor %}
{% endfor %}
```

## Поєднання логіки

Умовні оператори та цикли можна поєднувати:

```twig
{% for item in items %}
{% if item.active %}
- {{item.name}}
{% endif %}
{% endfor %}
```

## Порядок обчислення

Логіка шаблону обробляється в наступному порядку:

1. **Логіка шаблону** — `{% if %}`, `{% for %}`, `{% set %}` та `{{variables}}` обчислюються першими
2. **Змінні запитів** — [[Змінні#Змінні запитів|Змінні запитів]] на кшталт `{{"summarize this"|prompt}}` надсилаються до інтерпретатора після завершення обробки логіки шаблону

Це означає, що ви можете використовувати логіку шаблону для динамічного формування запитів, але результати запитів недоступні для використання в умовних операторах або циклах.
