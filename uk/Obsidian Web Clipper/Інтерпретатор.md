Інтерпретатор — це функція [[Вступ до Obsidian Web Clipper|Web Clipper]], яка дозволяє взаємодіяти з вебсторінками за допомогою природної мови. Інтерпретатор допомагає захоплювати та змінювати дані, які ви хочете зберегти в Obsidian. Наприклад:

- Витягувати конкретні фрагменти тексту.
- Підсумовувати або пояснювати інформацію.
- Конвертувати текст з одного формату в інший.
- Перекладати текст іншою мовою.

Інтерпретатор використовує мовні моделі для обробки інформації на вебсторінці та повертає результати за допомогою [[Змінні|змінних]], які можна додати до ваших [[Obsidian Web Clipper/Шаблони|шаблонів Web Clipper]].

## Приклади промптів

Промпти використовують [[Змінні|синтаксис змінних]] `{{"ваш промпт"}}`. Ви можете використовувати цей синтаксис з будь-яким запитом природною мовою, наприклад:

- `{{"a summary of the page"}}` — щоб отримати підсумок сторінки.
- `{{"a three bullet point summary, translated to French"}}` — щоб отримати ключові тези про сторінку та перекласти їх французькою.
- `{{"un resumé de la page en trois points"}}` — щоб отримати три ключові тези за допомогою промпту французькою мовою.

Результат ваших промптів можна додатково обробити за допомогою [[Фільтри|фільтрів]]. Фільтри обробляються після отримання відповіді від моделі. Наприклад: `{{"a summary of the page"|blockquote}}` перетворить відповідь на блок цитати.

## Початок роботи

Інтерпретатор працює практично з будь-яким провайдером мовних моделей, включно з варіантами, які працюють локально на вашому пристрої. Щоб налаштувати інтерпретатор:

1. Перейдіть до розділу **Інтерпретатор** у налаштуваннях Web Clipper.
2. Увімкніть перемикач **Увімкнути інтерпретатор**.
3. Налаштуйте провайдера та модель — див. розділ [[Інтерпретація вебсторінок#Моделі|моделі]] нижче.
4. Додайте [[Змінні|змінні промптів]] до ваших [[Obsidian Web Clipper/Шаблони|шаблонів]].
5. Якщо ваш шаблон містить змінні промптів, розділ Інтерпретатора буде видимий при [[Збереження вебсторінок|збереженні сторінки]]. Натисніть **Інтерпретувати**, щоб обробити змінні промптів.

## Як це працює

Коли Інтерпретатор увімкнено *і* ваш шаблон містить [[Змінні#Змінні промптів|змінні промптів]], у вікні розширення з'являється новий розділ Інтерпретатора над кнопкою **Додати в Obsidian**. Цей розділ дозволяє обрати модель та запустити Інтерпретатор для поточної сторінки.

Коли ви натискаєте **Інтерпретувати**, Інтерпретатор надсилає контекст сторінки до обраної моделі разом з *усіма* промптами вашого шаблону в одному запиті. Залежно від обраного провайдера моделі, це може бути зовнішній виклик або локальна обробка на вашому пристрої. Модель оцінює ваші промпти відносно контексту сторінки та повертає відповіді. Потім Інтерпретатор замінює змінні промптів даними відповідей.

Весь процес може зайняти від кількох мілісекунд до більш ніж 30 секунд залежно від моделі, яку ви використовуєте, та обсягу даних, які обробляються.

## Контекст

Термін *контекст* означає дані сторінки, які Інтерпретатор використовує для обробки промптів. Чим менший контекст, тим швидше працює Інтерпретатор.

За замовчуванням Інтерпретатор використовує весь HTML сторінки як контекст, проте це може робити промпти повільнішими та дорожчими, ніж необхідно.

Ви можете перевизначити контекст за замовчуванням у **Розширених налаштуваннях** Інтерпретатора та задати контекст для кожного [[Obsidian Web Clipper/Шаблони|шаблону]].

Щоб визначити більш цільовий контекст, використовуйте [[Змінні#Змінні селекторів|змінні селекторів]] (або інші типи змінних) для інтерпретації конкретного розділу сторінки. Наприклад, ви можете використати таку змінну селектора в контексті Інтерпретатора вашого шаблону:

```
{{selectorHtml:#main}}
```

Це запустить Інтерпретатор лише для елемента `#main` вебсторінки, якщо він існує. [[Фільтри#Обробка HTML|Фільтри обробки HTML]], такі як `remove_html`, `strip_tags` та `strip_attr`, можуть бути корисними для подальшого зменшення довжини контексту та прискорення обробки.

## Моделі

> [!warning] Конфіденційність
> Використовуючи стороннього провайдера моделей, ви погоджуєтеся з його умовами та політикою конфіденційності. Запити Інтерпретатора надсилаються безпосередньо до обраного вами провайдера. Obsidian не збирає та не зберігає жодних даних про ваші запити.

### Вибір моделі

Загалом ми рекомендуємо використовувати невеликі моделі з Web Clipper, оскільки вони швидші та досить точні для цього завдання. Приклади менших моделей: **Anthropic Claude Haiku**, **Google Gemini Flash**, **Llama** з 3B або 8B параметрами, або серія моделей **OpenAI Mini**.

### Користувацькі провайдери та моделі

Щоб додати користувацького провайдера та/або модель, перейдіть до **[[Налаштування]]** Web Clipper → **Інтерпретатор**:

- **Додати провайдера** — для налаштування пресетних та користувацьких провайдерів.
- **Додати модель** — для налаштування пресетних та користувацьких моделей.

При додаванні користувацького провайдера ми рекомендуємо використовувати їхній endpoint chat completions для **Базового URL** — зазвичай він закінчується на `/chat/completions`.

### Локальні моделі

Інтерпретатор може використовувати локальні моделі, які забезпечують більшу конфіденційність та сумісність з офлайн-режимом. Існує кілька варіантів запуску локальних моделей. Одним із найпростіших для налаштування є Ollama.

#### Ollama

[Ollama](https://ollama.com/) дозволяє запускати мовні моделі локально та приватно на вашому пристрої.

Після завантаження та встановлення Ollama додайте Ollama за допомогою **Додати провайдера** в налаштуваннях Інтерпретатора. Ollama не потребує API-ключа. Потім оберіть модель зі [списку моделей](https://ollama.com/search). Наприклад, якщо ви хочете використовувати [Llama 3.2](https://ollama.com/library/llama3.2), натисніть **Додати модель**, потім:

- **Провайдер:** Ollama
- **Відображувана назва:** Llama 3.2, це значення можна налаштувати.
- **ID моделі:** `llama3.2`, це повинно точно відповідати ID моделі з Ollama.

**Запуск сервера Ollama**

Щоб дозволити розширенню браузера взаємодіяти з Ollama, потрібно [надати явний дозвіл](https://github.com/ollama/ollama/issues/2308) при запуску сервера, інакше ви побачите помилку `403`.

Закрийте застосунок Ollama та виконайте наступну команду в терміналі. Протокол слід змінити на протокол розширення вашого браузера, якщо ви не використовуєте Chrome або Firefox.

```
OLLAMA_ORIGINS=moz-extension://*,chrome-extension://*,safari-web-extension://* ollama serve
```

Потім запустіть вашу модель з Ollama звичайним способом, наприклад:

```
ollama run llama3.2
```

**Довжина контексту**

Вікно контексту Ollama за замовчуванням становить 2048 токенів. Це максимальна кількість токенів для повідомлення та відповіді. При збереженні довгої вебсторінки ви можете легко перевищити цей ліміт. Ollama мовчки поверне нерелевантні результати. Деякі варіанти вирішення:

- Збільшити параметр `num_ctx` Ollama. Зверніть увагу, що довший контекст потребує більше пам'яті.
- Використовувати поле [[#Контекст]] у вашому шаблоні для надання більш цільового розділу сторінки, або обрізати контекст за допомогою [[Фільтри|фільтра]], наприклад `{{content|slice:0,1000}}`.
