[[Вступ до Obsidian Sync|Obsidian Sync]] пропонує автономний клієнт для синхронізації сховищ без використання настільного застосунку. Корисно для CI-пайплайнів, агентів та автоматизованих робочих процесів. Синхронізуйте останні зміни або підтримуйте файли постійно в актуальному стані.

Встановіть [[Автономна синхронізація|Obsidian Headless]] **(відкрита бета-версія)**, щоб взаємодіяти з [[Вступ до Obsidian Sync|Obsidian Sync]] з командного рядка без настільного застосунку Obsidian. Автономна синхронізація використовує ті самі [[Безпека та конфіденційність|засоби шифрування та захисту конфіденційності]], що й настільний застосунок, включаючи наскрізне шифрування.

## Швидкий старт

> [!error] Створіть резервну копію даних перед початком
> 1. Завжди створюйте резервну копію даних перед початком на випадок непередбачених ситуацій.
> 2. Не використовуйте *одночасно* синхронізацію настільного застосунку та автономну синхронізацію на одному пристрої, оскільки це може призвести до конфліктів даних. Використовуйте лише один метод синхронізації на пристрій.

Встановіть [[Автономна синхронізація|Obsidian Headless]] **(відкрита бета-версія)**:

```shell
npm install -g obsidian-headless
```

Необхідно мати активну [[Плани та обмеження сховища|підписку на Obsidian Sync]].

```shell
# Увійти
ob login

# Переглянути віддалені сховища
ob sync-list-remote

# Налаштувати сховище для синхронізації
cd ~/vaults/my-vault
ob sync-setup --vault "My Vault"

# Виконати одноразову синхронізацію
ob sync

# Запустити безперервну синхронізацію (відстежує зміни)
ob sync --continuous
```

## Команди

### `ob sync-list-remote`

Показати список усіх віддалених сховищ, доступних вашому обліковому запису, включаючи спільні сховища.

### `ob sync-list-local`

Показати список локально налаштованих сховищ та їхні шляхи.

### `ob sync-create-remote`

Створити нове віддалене сховище.

```
ob sync-create-remote --name "Vault Name" [--encryption <standard|e2ee>] [--password <password>] [--region <region>]
```

| Параметр | Опис |
| --- | --- |
| `--name` | Назва сховища (обов'язково) |
| `--encryption` | `standard` для керованого шифрування, `e2ee` для наскрізного шифрування |
| `--password` | Пароль наскрізного шифрування (буде запитано, якщо не вказано) |
| `--region` | [[Регіони Sync\|Регіон]] сервера (автоматично, якщо не вказано) |

### `ob sync-setup`

Налаштувати синхронізацію між локальним та віддаленим сховищем.

```
ob sync-setup --vault <id-or-name> [--path <local-path>] [--password <password>] [--device-name <name>] [--config-dir <name>]
```

| Параметр | Опис |
| --- | --- |
| `--vault` | Ідентифікатор або назва віддаленого сховища (обов'язково) |
| `--path` | Локальна тека (за замовчуванням: поточна тека) |
| `--password` | Пароль наскрізного шифрування (буде запитано, якщо не вказано) |
| `--device-name` | Назва пристрою, що відображається в [[Історія версій\|історії версій синхронізації]] |
| `--config-dir` | Назва [[Тека конфігурації\|теки конфігурації]] (за замовчуванням: `.obsidian`) |

### `ob sync`

Запустити синхронізацію для налаштованого сховища.

```
ob sync [--path <local-path>] [--continuous]
```

| Параметр | Опис |
| --- | --- |
| `--path` | Шлях до локального сховища (за замовчуванням: поточна тека) |
| `--continuous` | Безперервний режим, відстежує зміни |

### `ob sync-config`

Переглянути або змінити [[Налаштування Sync та вибіркова синхронізація|налаштування синхронізації]] для сховища. Запустіть без параметрів, щоб відобразити поточну конфігурацію.

```
ob sync-config [--path <local-path>] [options]
```

| Параметр              | Опис                                                                                                                                                                                                           |
| --------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `--path`              | Шлях до локального сховища (за замовчуванням: поточна тека)                                                                                                                                                    |
| `--mode`              | Режим синхронізації: `bidirectional` (за замовчуванням), `pull-only` (лише завантаження, ігнорування локальних змін) або `mirror-remote` (лише завантаження, відкат локальних змін)                               |
| `--conflict-strategy` | `merge` або `conflict`                                                                                                                                                                                         |
| `--file-types`        | Типи вкладень для синхронізації: `image`, `audio`, `video`, `pdf`, `unsupported` (через кому, порожнє значення для очищення)                                                                                    |
| `--configs`           | Категорії конфігурації для синхронізації: `app`, `appearance`, `appearance-data`, `hotkey`, `core-plugin`, `core-plugin-data`, `community-plugin`, `community-plugin-data` (через кому, порожнє значення для вимкнення синхронізації конфігурації) |
| `--excluded-folders`  | Теки для виключення (через кому, порожнє значення для очищення)                                                                                                                                                |
| `--device-name`       | Назва пристрою для ідентифікації цього клієнта в історії версій синхронізації                                                                                                                                   |
| `--config-dir`        | Назва теки конфігурації (за замовчуванням: `.obsidian`)                                                                                                                                                         |

### `ob sync-status`

Показати стан синхронізації та конфігурацію сховища.

```
ob sync-status [--path <local-path>]
```

### `ob sync-unlink`

Від'єднати сховище від синхронізації та видалити збережені облікові дані.

```
ob sync-unlink [--path <local-path>]
```

## Нативні модулі

Obsidian Headless включає попередньо зібраний нативний аддон для встановлення часу створення файлу (birthtime) на Windows та macOS. Це зберігає оригінальні мітки часу створення під час завантаження файлів з сервера.

Аддон орієнтований на N-API версії 3, тому скомпільовані бінарні файли є ABI-стабільними та працюють на різних версіях Node.js без перекомпіляції.

На Linux birthtime не підтримується — аддон не включено, і синхронізація працює нормально без нього.

Попередньо зібрані бінарні файли включено для:

- `win32-x64`
- `win32-arm64`
- `win32-ia32`
- `darwin-x64`
- `darwin-arm64`
