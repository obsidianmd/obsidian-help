Obsidian از HTML پشتیبانی می‌کند تا بتوانید یادداشت‌هایتان را به شکل دلخواه نمایش دهید، یا حتی [[جاسازی صفحات وب|صفحات وب جاسازی کنید]]. اجازه دادن به HTML درون یادداشت‌هایتان خطراتی به همراه دارد. برای جلوگیری از آسیب کدهای مخرب، Obsidian هر HTML موجود در یادداشت‌هایتان را _پاک‌سازی_ می‌کند.

> [!example] 
> عنصر `<script>` به طور معمول به شما اجازه می‌دهد هر زمان که بارگذاری می‌شود، JavaScript اجرا کنید. اگر Obsidian HTML را پاک‌سازی نمی‌کرد، یک مهاجم می‌توانست شما را متقاعد کند متنی حاوی JavaScript بچسبانید که اطلاعات حساس را از رایانه‌تان استخراج کرده و به او ارسال کند.

با این حال، از آنجا که سینتکس Markdown از همه اشکال سبک‌دهی پشتیبانی نمی‌کند، استفاده از HTML پاک‌سازی‌شده می‌تواند راه دیگری برای ارتقای کیفیت یادداشت‌هایتان باشد. ما برخی از رایج‌ترین کاربردهای HTML را در اینجا آورده‌ایم.

## محدودیت‌های HTML

Obsidian هنگام استفاده از HTML در یادداشت‌هایتان محدودیت‌های خاصی دارد:

### عدم پشتیبانی از Markdown درون HTML

Obsidian سینتکس Markdown را درون عناصر HTML پردازش نمی‌کند. این یک تصمیم طراحی عمدی برای بهینه‌سازی عملکرد و کاهش پیچیدگی تجزیه‌کننده هنگام مدیریت اسناد بزرگ است.

برای نمونه، این به شکل مورد انتظار کار نخواهد کرد:

```md
<div>
This **will not** be bold and this `will not` be code.
</div>
```

### بلوک‌های HTML باید مستقل باشند

بلوک‌های HTML باید کامل باشند و نمی‌توانند خطوط خالی درون خود داشته باشند. خطوط خالی بلوک HTML را می‌شکنند.

این کار خواهد کرد:

```md
<table>
<tr>
<td>Content here</td>
</tr>
</table>
```

این به درستی کار نخواهد کرد:

```md
<table>

<tr>

<td>Content here</td>

</tr>

</table>
```

### زمانی که Markdown به نظر می‌رسد درون HTML کار می‌کند

برخی تگ‌های درون‌خطی HTML مانند `<span>` یا `<a>` عملکرد محدودی دارند و ممکن است به نظر برسد Markdown را پردازش می‌کنند، اما در واقع این اتفاق نمی‌افتد. Markdown خارج از زمینه HTML پردازش می‌شود.

برای جزئیات بیشتر درباره نحوه مدیریت Markdown توسط Obsidian، به [[Obsidian Flavored Markdown]] مراجعه کنید.

## کاربردهای رایج HTML

> [!info] جزئیات بیشتر درباره استفاده از `<iframe>` در [[جاسازی صفحات وب]] موجود است.

### یادداشت‌ها

[[سینتکس قالب‌بندی پایه#یادداشت‌ها|یادداشت‌های Markdown]] روش ترجیحی برای افزودن یادداشت‌های پنهان درون یادداشت‌هایتان هستند. با این حال، برخی روش‌های تبدیل یادداشت‌های Markdown، مانند [Pandoc](https://pandoc.org)، پشتیبانی محدودی از یادداشت‌های Markdown دارند. در آن موارد، می‌توانید از `<!-- HTML Comment -->` به جای آن استفاده کنید!

### زیرخط

اگر نیاز دارید سریعاً یک مورد را در یادداشت‌هایتان زیرخط بکشید، می‌توانید از `<u>Example</u>` برای ایجاد <u>متن زیرخط‌دار</u> استفاده کنید.

### Span/Div

تگ‌های span و div می‌توانند برای اعمال کلاس‌های سفارشی از یک [[تکه‌کدهای CSS|تکه‌کد CSS]]، یا سبک‌دهی سفارشی تعریف‌شده، روی بخش انتخاب‌شده‌ای از متن استفاده شوند. برای نمونه، استفاده از `<span style="font-family: cursive">your text</span>` به شما اجازه می‌دهد سریعاً <span style="font-family: cursive">قلم خود را تغییر دهید</span>.

## خط‌خورده

نیاز دارید روی <s>متنی</s> خط بکشید؟ از `<s>this</s>` برای خط زدن آن استفاده کنید.
