شما می‌توانید یک دامنه‌ی دلخواه یا زیردامنه برای وبگاه [[مقدمه‌ای بر Obsidian Publish|Obsidian Publish]] خود تنظیم کنید.

> [!warning] هشدار
> در حال حاضر، ما فقط از پیکربندی دامنه‌های دلخواه با روش‌های زیر پشتیبانی می‌کنیم:
>
> - [[#راه‌اندازی با استفاده از CloudFlare]] با استفاده از [حالت Full](https://developers.cloudflare.com/ssl/origin-configuration/ssl-modes/full/).
> - [[#راه‌اندازی با استفاده از پراکسی]]
> 
> ما هنوز راهی برای صدور گواهی SSL از طرف شما نداریم.

## راه‌اندازی با استفاده از CloudFlare

ساده‌ترین راه برای تنظیم یک دامنه‌ی دلخواه یا زیردامنه، ایجاد یک حساب [CloudFlare](https://cloudflare.com) و واگذاری مدیریت DNS دامنه به CloudFlare است.

مراحل زیر از CloudFlare برای پیکربندی یک دامنه‌ی دلخواه برای وبگاه Obsidian Publish شما استفاده می‌کنند، چه با دامنه‌ی اصلی (`mysite.com`) و چه با زیردامنه (`notes.mysite.com`).

> [!important] مهم
> CloudFlare **تنها ارائه‌دهنده‌ی رسماً پشتیبانی‌شده** برای راه‌اندازی دامنه‌های دلخواه است. استفاده از دستورالعمل‌های زیر با هر ارائه‌دهنده‌ی دیگری احتمالاً کار نخواهد کرد.

**CloudFlare:**

1. CloudFlare را برای دامنه‌ای که می‌خواهید وبگاه Publish خود را روی آن میزبانی کنید باز کنید، مانند `mysite.com`، حتی اگر زیردامنه‌ای مثل `notes.mysite.com` می‌خواهید.
2. به **DNS** بروید و روی **Add Record** کلیک کنید.
3. **CNAME** را انتخاب کنید.
4. در **name**، دامنه یا زیردامنه خود را وارد کنید، مثلاً `notes.mysite.com`.
5. در **target**، مقدار `publish-main.obsidian.md` را وارد کنید. آدرس فرعی شخصی خود را در این مقدار قرار ندهید. Obsidian Publish این کار را از طریق پیکربندی شما انجام می‌دهد.
6. مطمئن شوید که **proxy status** فعال است. به‌صورت پیش‌فرض باید فعال باشد.
7. به **SSL/TLS** بروید و حالت رمزگذاری SSL/TLS را روی "Full" تنظیم کنید تا گواهی SSL/TLS به‌صورت خودکار پیکربندی شود.

> [!note] یادداشت
> برای هدایت هم `mysite.com` و هم `www.mysite.com` به Obsidian Publish، باید یک [Page Rule](https://support.cloudflare.com/hc/en-us/articles/200172336-Creating-Page-Rules) با تنظیمات زیر ایجاد کنید:
>
> - URL match: `www.mysite.com/*`
> - Forward URL - 301 Permanent Redirect
> - Redirect URL: `https://mysite.com/$1`
>
> پس از ایجاد قانون صفحه، یک رکورد CNAME برای `www.mysite.com` دقیقاً مانند کاری که برای `mysite.com` انجام دادید ایجاد کنید.

**Obsidian:**

1. Obsidian را روی رایانه خود باز کنید.
2. در [[روبان|نوار ابزار]] سمت چپ، روی **Publish changes** ![[lucide-send.svg#icon]] کلیک کنید.
3. در **Publish changes**، گزینه **Change site options** ![[lucide-cog.svg#icon]] را انتخاب کنید.
4. در کنار **Custom domain**، گزینه **Configure** را انتخاب کنید.
5. در **Custom URL**، آدرس دامنه یا زیردامنه خود را وارد کنید. مطمئن شوید که `www.` را در کادر آدرس اینترنتی دلخواه وارد نکنید.

> [!note] یادداشت
> اگر تنظیم دامنه‌ی دلخواه شما به یک حلقه‌ی بازهدایت منجر شد، احتمالاً حالت رمزگذاری در CloudFlare به جای "Full" روی "Flexible" تنظیم شده است.

## راه‌اندازی با استفاده از پراکسی

همچنین می‌توانید SSL/TLS را برای دامنه‌ی دلخواه خود با استفاده از وب‌سرور خودتان تنظیم کنید.

اگر از قبل وب‌سایتی تحت دامنه یا زیردامنه خود میزبانی می‌کنید، می‌توانید از این گزینه استفاده کنید و وب‌سایت خود را طوری تنظیم کنید که وبگاه Obsidian Publish شما را تحت یک مسیر URL خاص بارگیری کند، به جای اینکه کل وبگاه را میزبانی کنید.

تمام درخواست‌ها تحت آن مسیر URL را به `https://publish.obsidian.md/serve?url=mysite.com/my-notes/...` پراکسی کنید و تنظیمات وبگاه را در Obsidian به همان مسیر URL پیکربندی کنید، با تنظیم **Custom URL** به `mysite.com/my-notes`.

همچنین می‌توانید Obsidian Publish را به‌عنوان زیر-آدرس وبگاهی که مالک آن هستید تنظیم کنید. مثلاً `https://mysite.com/my-notes/`. برای این کار، باید سرور خود را میزبانی کنید و تمام درخواست‌ها را به سرور ما در `https://publish.obsidian.md/` پراکسی کنید.

نمونه‌های تنظیم پراکسی زیر جامع نیستند، اما روش‌های رایج برای این پیاده‌سازی را ارائه می‌دهند.

### NGINX

در پیکربندی NGINX خود، موارد زیر را اضافه کنید:

```nginx
location /my-notes {
  proxy_pass https://publish.obsidian.md/serve?url=mysite.com/my-notes/;
  proxy_ssl_server_name on;
  proxy_set_header Host publish.obsidian.md;
}
```


برخی کاربران گزارش داده‌اند که افزودن `$request_uri` به proxy pass ممکن است لازم باشد:

```nginx
location /my-notes {
  proxy_pass https://publish.obsidian.md/serve?url=mysite.com/my-notes$request_uri;
  proxy_ssl_server_name on;
  proxy_set_header Host publish.obsidian.md;
}
```

### Apache

در `.htaccess`، موارد زیر را اضافه کنید:

```htaccess
RewriteEngine  on
RewriteRule    "^my-notes/(.*)$"  "https://publish.obsidian.md/serve?url=mysite.com/my-notes/$1"  [L,P]
```

> [!note] یادداشت
> `mod_rewrite` باید فعال باشد و همچنین ممکن است نیاز به پیکربندی [SSLProxyEngine](https://stackoverflow.com/questions/40938148/reverse-proxy-for-external-url-apache) داشته باشید.

### Netlify

در `netlify.toml`، [بازهدایت‌ها را پیکربندی کنید](https://docs.netlify.com/routing/redirects/#syntax-for-the-netlify-configuration-file):

```plain
[[redirects]]
  from = "https://mysite.com/my-notes/*"
  to = "https://publish.obsidian.md/serve?url=mysite.com/my-notes/:splat"
  status = 200
  force = true
```

### Vercel

در `vercel.json`، [بازنویسی‌ها را پیکربندی کنید](https://vercel.com/docs/configuration#project/rewrites):

```json
{
  ...

  "rewrites": [
    {
      "source": "/my-notes/",
      "destination": "https://publish.obsidian.md/serve?url=mysite.com/my-notes"
    },
    {
      "source": "/my-notes/:path*",
      "destination": "https://publish.obsidian.md/serve?url=mysite.com/my-notes/:path*"
    }
  ]
}
```

### Caddy

```plain
mysite.com {
  encode zstd gzip
  handle /my-notes* {
    reverse_proxy https://publish.obsidian.md {
      header_up Host {upstream_hostport}
    }
    rewrite * /serve?url=mysite.com{path}
  }
}
```

### Traefik

این قطعه پیکربندی حداقلی، `mysite.com` را به Obsidian Publish هدایت می‌کند.
برای یک نمونه کامل، [مستندات Traefik](https://doc.traefik.io/traefik/routing/overview/) را ببینید.

```yaml
http:
  routers:
    mysite:
      rule: Host(`mysite.com`)
      service: obsidian-publish
      middlewares:
        - "publish-headers"
  services:
    obsidian-publish:
      loadBalancer:
        servers:
          - url: https://publish.obsidian.md
  middlewares:
    publish-headers:
      headers:
        customRequestHeaders:
          Host: "publish.obsidian.md"
          x-obsidian-custom-domain: "mysite.com"
```

### هدرهای HTTP X پشتیبانی‌شده

اگر سرویس پراکسی شما اجازه‌ی مسیرهای پرس‌وجو را نمی‌دهد، می‌توانید از `https://publish.obsidian.md/` با یک هدر سفارشی `x-obsidian-custom-domain` تنظیم‌شده روی آدرس وبگاه شما `mysite.com/my-subpath` استفاده کنید.

## بازهدایت وبگاه قدیمی به دامنه‌ی دلخواه

اگر می‌خواهید بازدیدکنندگان خود را از وبگاه قدیمی `publish.obsidian.md` به دامنه‌ی دلخواه جدید خود هدایت کنید، گزینه **Redirect to your custom domain** را هنگام پیکربندی دامنه‌ی دلخواه خود فعال کنید.

## عیب‌یابی

پس از تنظیم دامنه‌ی دلخواه، اگر قبلاً از پیوند `https://publish.obsidian.md/slug` قدیمی وبگاه خود بازدید کرده باشید، ممکن است نیاز باشد حافظه‌ی پنهان مرورگر خود را پاک کنید تا برخی موارد (مانند فونت‌ها، نمودارها یا دسترسی رمزی) به‌درستی کار کنند. این به دلیل محدودیت‌های امنیتی بین‌دامنه‌ای است که توسط مرورگرهای مدرن اعمال می‌شوند. خبر خوب این است که خوانندگان وبگاه شما هرگز با این مشکل مواجه نخواهند شد، مادامی که فقط اجازه دهید بازدیدکنندگان از دامنه‌ی دلخواه شما استفاده کنند.
