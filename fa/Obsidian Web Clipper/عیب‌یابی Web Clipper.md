اگر با [[معرفی Obsidian Web Clipper|Web Clipper]] مشکلی دارید، می‌توانید از طریق [کانال رسمی Discord](https://discord.com/channels/686053708261228577/1285652864089198672) کمک بگیرید. همچنین می‌توانید باگ‌ها را در [مخزن GitHub](https://github.com/obsidianmd/obsidian-clipper) گزارش کنید.

## عمومی

### برخی محتواها از دست رفته‌اند

به‌طور پیش‌فرض، Web Clipper تلاش می‌کند محتوای صفحه را به‌صورت هوشمندانه ذخیره کند. با این حال ممکن است در تمام وب‌سایت‌ها موفق نباشد.

Web Clipper از [Defuddle](https://github.com/kepano/defuddle) برای ذخیره فقط محتوای اصلی صفحه استفاده می‌کند. این کار سربرگ، پابرگ و سایر عناصر را حذف می‌کند، اما گاهی ممکن است بیش از حد محافظه‌کارانه عمل کند و محتوایی را که می‌خواهید نگه دارید حذف کند. می‌توانید [باگ‌ها را گزارش کنید](https://github.com/kepano/defuddle) به Defuddle.

برای دور زدن Defuddle در Web Clipper از روش‌های زیر استفاده کنید:

- متن را انتخاب کنید، یا از `Cmd/Ctrl+A` برای انتخاب همه متن استفاده کنید.
- [[هایلایتر|محتوا را هایلایت کنید]] تا دقیقاً آنچه می‌خواهید ذخیره شود را انتخاب کنید.
- از یک [[Obsidian Web Clipper/الگوها|قالب سفارشی]] برای وبگاه استفاده کنید.

### هیچ محتوایی در Obsidian ظاهر نمی‌شود

اگر هنگام کلیک بر **به ابسیدین اضافه کنید** هیچ محتوایی در Obsidian نمی‌بینید:

- خطاها را در [[راهنما و پشتیبانی#ضبط لاگ‌های کنسول|کنسول توسعه‌دهنده]] Obsidian بررسی کنید.
- بررسی کنید که نام خزانه در تنظیمات Web Clipper دقیقاً با *نام خزانه* شما در Obsidian مطابقت دارد، *نه مسیر خزانه*.
- بررسی کنید که نام پوشه به‌درستی قالب‌بندی شده باشد.

## لینوکس

#### Obsidian باز نمی‌شود

- مطمئن شوید که پروتکل [[Obsidian URI]] [[Obsidian URI#ثبت Obsidian URI|ثبت شده باشد]].
- اگر از Firefox استفاده می‌کنید ممکن است نیاز باشد [آن را در تنظیمات مرورگر ثبت کنید](https://kb.mozillazine.org/Register_protocol).

#### Obsidian باز می‌شود اما فقط نام فایل ذخیره می‌شود

احتمالاً Obsidian نمی‌تواند به کلیپ‌بورد شما دسترسی داشته باشد. دسترسی به کلیپ‌بورد برای انتقال داده‌ها از مرورگر به Obsidian ضروری است. پیکربندی شما می‌تواند بر نحوه سندباکس شدن اپلیکیشن‌ها و مجوزهای کلیپ‌بورد تأثیر بگذارد.

اگر از Wayland استفاده می‌کنید، مطمئن شوید که Obsidian مجوز خواندن کلیپ‌بورد را هنگامی که اپلیکیشن در تمرکز نیست دارد. این تنظیم ممکن است در مدیر پنجره کاشی‌ای شما باشد، مثلاً Hyprland یا Sway.

اگر از Hyprland استفاده می‌کنید:

```ini
# ~/.config/hypr/hyprland.conf
misc {
    focus_on_activate = true
}
```

اگر از Sway استفاده می‌کنید:

```ini
# ~/.config/sway/config
for_window [class="obsidian"] focus_on_window_activation focus
```

- اگر از Flatpak استفاده می‌کنید، [نسخه رسمی پشتیبانی‌شده Obsidian](https://obsidian.md/download) را امتحان کنید.
- اگر از KDE استفاده می‌کنید، به **System Settings** → **Window Management** → **Window Rules** بروید و اجازه دهید Obsidian تمرکز را بگیرد، [[web-clipper-kde.png|اسکرین‌شات را ببینید]].
- به‌عنوان راه‌حل جایگزین، در **تنظیمات Web Clipper** → **عمومی** به **حالت قدیمی** بروید. این کار کلیپ‌بورد را دور می‌زند و محتوا را مستقیماً از طریق URI ذخیره می‌کند. توجه داشته باشید که این کار تعداد کاراکترهایی که بسته به مرورگر و توزیع لینوکس شما قابل ذخیره‌سازی هستند را محدود می‌کند.

## iOS و iPadOS

برای فعال کردن افزونه Web Clipper برای Safari:

1. به Safari بروید، دکمه سمت چپ‌ترین نوار آدرس مرورگر را بزنید، شکل آن مانند مستطیلی با خطوط زیرش است.
2. **Manage Extensions** را بزنید.
3. **Obsidian Web Clipper** را در فهرست افزونه‌ها فعال کنید.
4. از منو خارج شوید.
5. برای استفاده از افزونه **آیکون پازل** را در نوار آدرس بزنید.

برای اجازه اجرای Web Clipper در تمام وب‌سایت‌ها:

1. به **[[تنظیمات]]** iOS → **Apps** → **Safari** → **Extensions** بروید.
2. در بخش **Permissions** اجازه اجرا در تمام وب‌سایت‌ها را بدهید.

برای اجازه دریافت همیشگی محتوای Web Clipper توسط Obsidian:

1. به **[[تنظیمات]]** iOS → **Apps** → **Obsidian** بروید.
2. **Paste from other apps** را روی **Allow** تنظیم کنید.
