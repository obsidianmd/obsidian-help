[[Obsidian Web Clipper/الگوها|قالب‌های Web Clipper]] می‌توانند از متغیرها برای پر کردن خودکار داده‌ها از صفحه در یک قالب استفاده کنند. متغیرها را می‌توان در **نام یادداشت**، **مکان یادداشت**، **ویژگی‌ها** و **محتوای یادداشت** به کار برد. همچنین متغیرها را می‌توان با استفاده از [[پالایه‌ها]] تغییر داد.

از آیکون `...` در افزونه [[معرفی Obsidian Web Clipper|Web Clipper]] برای دسترسی به متغیرهای صفحه فعلی جهت استفاده در قالب‌ها استفاده کنید. پنج نوع متغیر وجود دارد که می‌توانید استفاده کنید:

- [[متغیرها#متغیرهای پیش‌تنظیم‌شده|متغیرهای پیش‌تنظیم‌شده]]
- [[متغیرها#متغیرهای پرامپت|متغیرهای پرامپت]]
- [[متغیرها#متغیرهای متا|متغیرهای متا]]
- [[متغیرها#متغیرهای سلکتور|متغیرهای سلکتور]]
- [[متغیرها#متغیرهای Schema.org|متغیرهای Schema.org]]

## متغیرهای پیش‌تنظیم‌شده

متغیرهای پیش‌تنظیم‌شده بر اساس محتوای صفحه به صورت خودکار تولید می‌شوند. این متغیرها معمولاً برای اکثر وب‌سایت‌ها کار می‌کنند.

متغیر اصلی محتوا `{{content}}` است که شامل محتوای مقاله، یا [[هایلایت صفحات وب|هایلایت‌ها]]، یا متن انتخاب‌شده (در صورت وجود) در صفحه می‌باشد. توجه داشته باشید که `{{content}}` تلاش می‌کند محتوای اصلی صفحه را استخراج کند که ممکن است همیشه آنچه شما می‌خواهید نباشد. در آن صورت، می‌توانید از متغیرهای پیش‌تنظیم‌شده دیگر یا متغیرهای سلکتور برای استخراج محتوای مورد نیاز استفاده کنید.

| متغیر              | توضیح                                                                                   |
| ------------------- | -------------------------------------------------------------------------------------- |
| `{{author}}`        | نویسنده صفحه                                                                          |
| `{{content}}`       | محتوای مقاله، [[هایلایت صفحات وب\|هایلایت‌ها]]، یا متن انتخاب‌شده، در قالب Markdown |
| `{{contentHtml}}`   | محتوای مقاله، [[هایلایت صفحات وب\|هایلایت‌ها]]، یا متن انتخاب‌شده، در قالب HTML     |
| `{{date}}`          | تاریخ فعلی، قابل قالب‌بندی با فیلتر `date`                                            |
| `{{description}}`   | توضیح یا چکیده                                                                        |
| `{{domain}}`        | دامنه                                                                                  |
| `{{favicon}}`       | نشانی URL فاوآیکون                                                                    |
| `{{fullHtml}}`      | HTML پردازش‌نشده برای کل محتوای صفحه                                                  |
| `{{highlights}}`    | [[هایلایت صفحات وب\|هایلایت‌ها]] با متن و برچسب‌های زمانی                             |
| `{{image}}`         | نشانی URL تصویر اشتراک‌گذاری اجتماعی                                                  |
| `{{published}}`     | تاریخ انتشار، قابل قالب‌بندی با فیلتر `date`                                          |
| `{{selection}}`     | متن انتخاب‌شده در قالب Markdown                                                       |
| `{{selectionHtml}}` | متن انتخاب‌شده در قالب HTML                                                           |
| `{{site}}`          | نام وبگاه یا ناشر                                                                     |
| `{{title}}`         | عنوان صفحه                                                                             |
| `{{time}}`          | تاریخ و ساعت فعلی                                                                     |
| `{{url}}`           | نشانی URL فعلی                                                                         |
| `{{words}}`         | واژه‌شمار                                                                              |

## متغیرهای پرامپت

متغیرهای پرامپت از مدل‌های زبانی برای استخراج و تغییر داده‌ها با استفاده از زبان طبیعی بهره می‌برند. متغیرهای پرامپت نیاز دارند که [[تفسیر صفحات وب|تفسیرگر]] فعال و پیکربندی شده باشد.

متغیرهای پرامپت از سینتکس `{{"یک خلاصه از صفحه"}}` استفاده می‌کنند. گیومه‌های دوتایی اطراف پرامپت مهم هستند و پرامپت‌ها را از متغیرهای پیش‌تنظیم‌شده متمایز می‌کنند. پاسخ‌های پرامپت را می‌توان با [[پالایه‌ها]] پس‌پردازش کرد، مثلاً `{{"یک خلاصه از صفحه"|blockquote}}`.

### چه زمانی از متغیرهای پرامپت استفاده کنیم

متغیرهای پرامپت مزیت انعطاف‌پذیری بسیار بالا و سهولت نوشتن را دارند، اما با چندین مصالحه همراه هستند: اجرای آن‌ها کندتر است و بسته به [[تفسیر صفحات وب#مدل‌ها|ارائه‌دهنده]]‌ای که انتخاب می‌کنید، ممکن است ملاحظات هزینه‌ای و حریم خصوصی داشته باشند.

برخلاف سایر انواع متغیرها، متغیرهای پرامپت باید توسط یک مدل زبانی خارجی پردازش شوند، بنابراین تنها پس از اجرای [[تفسیر صفحات وب|تفسیرگر]] جایگزین می‌شوند.

بهتر است اگر داده‌ای که می‌خواهید استخراج کنید در قالبی ثابت است که می‌تواند با سایر انواع متغیرها استخراج شود، از متغیرهای پرامپت استفاده *نکنید*.

از سوی دیگر، متغیرهای پرامپت می‌توانند مفید باشند اگر داده‌ای که می‌خواهید استخراج کنید در قالبی *ناسازگار* در وب‌سایت‌های مختلف باشد. به عنوان نمونه، می‌توانید یک [[Obsidian Web Clipper/الگوها|قالب]] برای ذخیره کتاب‌ها بسازید که مستقل از وب‌سایت کتاب باشد. متغیرهای پرامپت مانند `{{"نویسنده کتاب"}}` در هر وب‌سایت کتابی کار می‌کنند، در حالی که متغیرهای سلکتور معمولاً فقط برای یک وب‌سایت کار می‌کنند.

### نمونه‌ها

پرامپت‌ها می‌توانند تقریباً از هر پرسش زبان طبیعی استفاده کنند. بسته به مدلی که استفاده می‌کنید، پرامپت‌ها می‌توانند داده‌ها را در زبان‌های مختلف جستجو یا ترجمه کنند.

- `{{"خلاصه‌ای در سه نقطه بولت، ترجمه شده به فرانسوی"}}` برای استخراج نکات کلیدی درباره صفحه و ترجمه آن‌ها به فرانسوی.
- `{{"un resumé de la page en trois points"}}` برای استخراج سه نکته با استفاده از پرامپت به زبان فرانسوی.

پرامپت‌ها می‌توانند محتوای صفحه را به JSON تبدیل کنند که با [[پالایه‌ها|فیلترها]] قابل دستکاری است. به عنوان نمونه:

```
{{"return a JSON object for each tweet, that includes the author, tweet_text, date in YYYY-MM-DD format, and images array (if there are any)"|map:tweet => ({text: tweet.tweet_text, author: tweet.author, date: tweet.date})|template:"${text}\n— [[@${author}]], [[${date}]]\n"}}
```


## متغیرهای متا

متغیرهای متا به شما امکان استخراج داده از [عناصر متا](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/meta) در صفحه را می‌دهند، از جمله داده‌های [Open Graph](https://ogp.me/) که برای پر کردن پیش‌نمایش‌های اشتراک‌گذاری اجتماعی استفاده می‌شوند.

- `{{meta:name}}` محتوای تگ متای name با نام مشخص‌شده را برمی‌گرداند، مثلاً `{{meta:name:description}}` برای تگ متای `description`.
- `{{meta:property}}` محتوای تگ متای property با ویژگی مشخص‌شده را برمی‌گرداند، مثلاً `{{meta:property:og:title}}` برای تگ متای `og:title`.

## متغیرهای سلکتور

متغیرهای سلکتور به شما امکان استخراج محتوای متنی از عناصر صفحه با استفاده از [سلکتورهای CSS](https://developer.mozilla.org/en-US/docs/Web/CSS/CSS_selectors/Selectors_and_combinators) را می‌دهند.

سینتکس آن `{{selector:cssSelector?attribute}}` است، که `?attribute` اختیاری است. اگر هیچ صفتی مشخص نشود، محتوای متنی عنصر برگردانده می‌شود. همچنین می‌توانید از `{{selectorHtml:cssSelector}}` برای دریافت محتوای HTML عنصر استفاده کنید. متغیرهای سلکتور معمولاً در یک وب‌سایت خاص یا مجموعه‌ای از وب‌سایت‌ها که ساختار HTML ثابتی دارند بهتر کار می‌کنند.

- `{{selector:h1}}` محتوای متنی هر عنصر `h1` در صفحه را برمی‌گرداند.
- `{{selector:.author}}` محتوای متنی هر عنصر `.author` در صفحه را برمی‌گرداند.
- `{{selector:img.hero?src}}` صفت `src` تصویر با کلاس `hero` را برمی‌گرداند.
- `{{selector:a.main-link?href}}` صفت `href` تگ لنگر با کلاس `main-link` را برمی‌گرداند.
- `{{selectorHtml:body|markdown}}` کل HTML عنصر `body` را برمی‌گرداند و با استفاده از [[پالایه‌ها#پردازش HTML|فیلتر]] `markdown` به Markdown تبدیل می‌کند.
- سلکتورهای CSS تودرتو و ترکیب‌کننده‌ها در صورت نیاز به دقت بیشتر پشتیبانی می‌شوند.
- اگر چندین عنصر با سلکتور مطابقت داشته باشند، یک آرایه برگردانده می‌شود که می‌توانید آن را با [[پالایه‌ها#آرایه‌ها و اشیاء|فیلترهای آرایه و شیء]] مانند `join` یا `map` پردازش کنید.

متغیرهای سلکتور همچنین می‌توانند مستقیماً در [[منطق|منطق قالب]] استفاده شوند:

- در حلقه‌ها: `{% for comment in selector:.comment %}...{% endfor %}`
- در شرط‌ها: `{% if selector:.premium-badge %}...{% endif %}`
- در تخصیص متغیر: `{% set items = selector:.list-item %}`

## متغیرهای Schema.org

متغیرهای Schema به شما امکان استخراج داده از [schema.org](https://schema.org/) JSON-LD در صفحه را می‌دهند. داده‌های Schema.org همچنین می‌توانند برای [[Obsidian Web Clipper/الگوها#تطبیق Schema.org|فعال‌سازی خودکار یک قالب]] استفاده شوند.

- `{{schema:@Type:key}}` مقدار کلید را از schema برمی‌گرداند.
- `{{schema:@Type:parent.child}}` مقدار یک ویژگی تودرتو را برمی‌گرداند.
- `{{schema:@Type:arrayKey}}` اولین آیتم در یک آرایه را برمی‌گرداند.
- `{{schema:@Type:arrayKey[index].property}}` آیتم در اندیس مشخص‌شده در یک آرایه را برمی‌گرداند.
- `{{schema:@Type:arrayKey[*].property}}` یک ویژگی خاص از تمام آیتم‌ها در یک آرایه را برمی‌گرداند.

همچنین می‌توانید از نماد مختصر بدون مشخص کردن نوع schema استفاده کنید:

- `{{schema:author}}` اولین ویژگی `author` یافت‌شده در هر نوع schema را مطابقت می‌دهد.
- `{{schema:name}}` اولین ویژگی `name` یافت‌شده در هر نوع schema را مطابقت می‌دهد.

این نماد مختصر به‌ویژه زمانی مفید است که نوع schema خاصی را نمی‌دانید یا برایتان مهم نیست، اما نام ویژگی مورد نظرتان را می‌دانید.

ویژگی‌های تودرتو و دسترسی به آرایه نیز کار می‌کنند، هم با مشخص کردن `@Type` schema و هم بدون آن:

- `{{schema:author.name}}` اولین ویژگی `author` را پیدا کرده و سپس به زیرویژگی `name` آن دسترسی پیدا می‌کند.
- `{{schema:author[0].name}}` به `name` اولین نویسنده در آرایه‌ای از نویسندگان دسترسی پیدا می‌کند.
- `{{schema:author[*].name}}` آرایه‌ای از تمام نام‌های نویسندگان را برمی‌گرداند.
