فرمول‌ها به شما امکان می‌دهند در [[معرفی پایگاه‌ها|پایگاه‌ها]] با استفاده از داده‌های سایر [[ویژگی‌ها|ویژگی‌ها]]، ویژگی‌های محاسباتی بسازید. می‌توانید محاسبات انجام دهید، متن را دستکاری کنید، با تاریخ‌ها کار کنید و بیشتر.

## فرمول‌ها چه کاری می‌توانند انجام دهند

فرمول‌ها می‌توانند به شما کمک کنند:

- **محاسبه مقادیر**، جمع قیمت‌ها، محاسبه مجموع‌ها، یا انجام عملیات ریاضی.
- **دستکاری متن**، ترکیب رشته‌ها، تغییر حالت حروف، یا استخراج زیررشته‌ها.
- **کار با تاریخ‌ها**، محاسبه اختلاف زمانی، قالب‌بندی تاریخ‌ها، یا تعیین مهلت‌ها.
- **اعمال منطق**، استفاده از عبارات شرطی برای نمایش مقادیر مختلف.
- **پردازش فهرست‌ها**، فیلتر، مرتب‌سازی، نگاشت، یا تجمیع داده‌های فهرست.

## ساختن یک ویژگی فرمول

برای ساختن یک ویژگی فرمول:

1. در پایگاه خود، روی **ویژگی‌ها** در نوار ابزار کلیک کنید.
2. روی **افزودن فرمول** در پایین منو کلیک کنید.
3. یک نام برای ویژگی فرمول خود وارد کنید.
4. فرمول خود را در فیلد **فرمول** تایپ کنید.
5. کادر گفتگو را ببندید.

ویرایشگر فرمول هنگام تایپ، نام [[توابع|توابع]] و ویژگی‌ها را به صورت خودکار تکمیل می‌کند تا سینتکس فرمول شما را اعتبارسنجی کند. وقتی فرمول شما معتبر باشد، یک علامت تیک سبز ظاهر می‌شود.

پس از ساختن، می‌توانید از ویژگی فرمول مانند هر ویژگی دیگری در پایگاه خود استفاده کنید. آن را به [[نماها|نماها]] اضافه کنید، در پالایه‌ها استفاده کنید، بر اساس آن مرتب‌سازی کنید و بیشتر.

## نوشتن یک فرمول

در ویرایشگر فرمول، یک عبارت با استفاده از ویژگی‌ها، عملگرها و توابع تایپ کنید.

### ارجاع به ویژگی‌ها

می‌توانید در فرمول‌های خود به انواع مختلف ویژگی‌ها ارجاع دهید:

- **ویژگی‌های یادداشت** — ویژگی‌ها از [[ویژگی‌ها|فرادادهٔ ابتدای فایل]] یادداشت.
- **ویژگی‌های پرونده** — ویژگی‌های داخلی مانند `file.name`، `file.size` یا `file.mtime`.
- **ویژگی‌های فرمول** — سایر فرمول‌ها با استفاده از `formula.formula_name`.

**نمونه‌ها:**

- `price * quantity` — ضرب دو ویژگی یادداشت
- `file.name + " - " + description` — ترکیب نام فایل با یک ویژگی یادداشت
- `formula.price_per_unit * 1.1` — استفاده از ویژگی فرمول دیگر

### استفاده از عملگرها

**عملگرهای حسابی** عملیات ریاضی روی اعداد انجام می‌دهند:

- `price + tax` — جمع
- `price - discount` — تفریق
- `price * quantity` — ضرب
- `price / quantity` — تقسیم
- `(part / whole) * 100` — استفاده از پرانتز برای ترتیب عملیات

**عملگرهای مقایسه‌ای** مقادیر را مقایسه می‌کنند:

- `price > 100` — بزرگ‌تر از
- `age < 18` — کوچک‌تر از
- `status == "Done"` — برابر با
- `status != "Done"` — نابرابر با
- `file.mtime > now() - '7d'` — مقایسه تاریخ‌ها

**عملگرهای بولی** شرایط منطقی را ترکیب می‌کنند:

- `!completed` — نقیض
- `price > 0 && quantity > 0` — و
- `urgent || important` — یا

اطلاعات بیشتر در [[سینتکس پایگاه‌ها#عملگرها|سینتکس پایگاه‌ها]].

### استفاده از توابع

توابع عملیاتی روی مقادیر انجام می‌دهند. توابع موجود به نوع مقداری که با آن کار می‌کنید بستگی دارد. فهرست کامل [[توابع]] را ببینید.

**دسته‌بندی‌های رایج توابع:**

- **توابع سراسری** — `if()`، `now()`، `date()`، `link()`، `max()`، `min()`
- **توابع رشته** — `contains()`، `replace()`، `split()`، `lower()`، `title()`
- **توابع عدد** — `round()`، `ceil()`، `floor()`، `abs()`، `toFixed()`
- **توابع تاریخ** — `format()`، `relative()`، `date()`، `time()`
- **توابع فهرست** — `filter()`، `map()`، `sort()`، `join()`، `unique()`

**نمونه‌ها:**

- `if(price, "$" + price.toFixed(2), "")` برای تعریف یک شرط با قالب‌بندی عدد.
- `file.name.lower()` برای تبدیل به حروف کوچک.
- `tags.contains("urgent")` برای بررسی اینکه آیا فهرست برچسب‌ها شامل یک مقدار است.
- `due_date.format("YYYY-MM-DD")` برای قالب‌بندی یک تاریخ.

## نمونه‌های فرمول

### محاسبه مهلت

تنظیم تاریخ سررسید پروژه به ۲ هفته پس از تاریخ شروع:

```js
start_date + "2w"
```

### نمایش وضعیت عقب‌افتادگی

نمایش «عقب‌افتاده» اگر تاریخ سررسید گذشته و وضعیت «انجام» نباشد:

```js
if(due_date < now() && status != "Done", "Overdue", "")
```

### قالب‌بندی ارز

نمایش قیمت با ۲ رقم اعشار و نماد ارز:

```js
if(price, "$" + price.toFixed(2), "")
```

### شمارش آیتم‌های فهرست

شمارش تعداد آیتم‌ها در یک ویژگی فهرست:

```js
tasks.length
```

### محاسبه امتیاز اولویت

ترکیب عوامل متعدد در یک امتیاز اولویت:

```js
(impact * urgency) / effort
```

### ترکیب فیلدهای متنی

ساختن یک نام کامل از نام و نام خانوادگی:

```js
first_name + " " + last_name
```

### محاسبه هزینه کل

ضرب هزینه ماهانه در تعداد ماه ها مالکیت:

```js
monthlyUses * formula.Owned.round()
```

## انواع داده

فرمول‌ها با انواع مختلف داده کار می‌کنند:

- **رشته‌ها** — متن محصور در گیومه: `"hello"` یا `'world'`
- **اعداد** — مقادیر عددی: `42`، `3.14`، `(2 + 2)`
- **بولی‌ها** — درست یا نادرست: `true`، `false`
- **تاریخ‌ها** — ایجاد شده با `date()`، `today()` یا `now()`
- **فهرست‌ها** — مجموعه‌ای از مقادیر: `[1, 2, 3]`
- **اشیا** — جفت‌های کلید-مقدار: `{"name": "value"}`

نوع خروجی یک فرمول توسط داده‌ها و توابع استفاده‌شده تعیین می‌شود.

## ارجاع به فرمول‌های دیگر

فرمول‌ها می‌توانند به فرمول‌های دیگر ارجاع دهند و محاسبات مشتق‌شده بسازند. برای مثال، اگر فرمولی به نام `price_per_unit` دارید:

```js
price / quantity
```

می‌توانید در فرمول دیگری به آن ارجاع دهید:

```js
formula.price_per_unit * 1.1
```

> [!warning] از ارجاعات دوری اجتناب کنید
> یک فرمول نمی‌تواند به صورت مستقیم یا غیرمستقیم از طریق فرمول‌های دیگر به خودش ارجاع دهد.
